# Z-알고리즘 사전 분류

## 1. 대상 정보

- 이름: Z-알고리즘 (Z-algorithm)
- 분야: 문자열 처리 / 패턴 매칭
- 본질: 문자열 S 의 각 위치 i 에 대해 "S 와 S[i..] 가 공유하는 최장 공통 접두사 길이" Z[i] 를 선형 시간(O(n))에 계산하는 알고리즘. 이 Z-배열을 이용해 패턴 검색을 O(n+m) 으로 수행.
- 핵심 메커니즘: Z-box (현재까지 발견한 접두사와 일치하는 최우측 구간 [l, r]) 를 유지하며, i 가 Z-box 안에 있을 때 이전에 계산한 Z[i-l] 을 재활용해 비교를 건너뛴다.

## 2. type 분류

**type = algorithm**

- 명확한 입력(문자열) → 출력(Z-배열) 매핑.
- 의사코드/실제 코드로 1:1 표현되는 절차.
- 단계별 비교 / 포인터 이동이 본질.
- IR / transpiler 적용 가능.

## 3. 전략 결정 (A/B)

**전략 = A타입 (공통 강화)**

근거:
- 핵심 학습 질문이 "어떻게 동작하나?" — Z-box 의 l, r 포인터가 어떻게 갱신되고, i 위치에서 비교를 어디서 건너뛰는지 단계별 추적이 학습의 본질.
- 정량 검증 (각 위치의 Z 값) 이 학습 가치를 가진다. 학습자가 Z[i] 결과를 직접 검증하고 싶어함.
- 코드/수식과 1:1 대응 가능 (l, r, i, Z[i] 모두 직접 표시 가능).
- 표준 시각 언어(문자 트랙 + Z-box 하이라이트 + Z 배열 막대) 가 이미 강력하게 작동. 디테일과 완성도로 차별화하는 편이 효율적.
- B타입의 일상 비유는 약함 — "거울 비교" 정도가 있으나 메커니즘 정확도가 더 중요한 영역.

가중치:
- 수식·코드 1:1 대응 가능 → A타입 가중 +1
- 정량 결과 검증 가치 → A타입 가중 +1
- 일상 비유 약함 → B타입 가중 없음

## 4. IR 보유 여부

**IR / transpiler 보유**

- type = algorithm 이므로 IR 적용 대상.
- 배열 인덱싱, while 비교 루프, 조건 분기로 구성되어 기존 IR 인터프리터와 transpiler 모두 호환.

## 5. 핵심 학습 질문

1. Z[i] 의 정의는 무엇이며, 패턴 매칭에 어떻게 쓰이나?
2. Z-box [l, r] 는 무엇을 기억하는 장치이며, 왜 유지하나?
3. i 가 Z-box 안에 있을 때 비교를 건너뛸 수 있는 이유는?
4. 각 단계에서 r 이 단조 증가하므로 전체가 O(n) 이 되는 이유는?

## 6. 후속 단계 가이드

- Stage 1 (레퍼런스 조사) 는 표준 시각 언어 — 문자 트랙, Z-box 하이라이트, Z 배열 막대, 포인터(l, r, i, k) 표시 — 의 베스트 프랙티스 수집에 집중.
- 디테일·완성도 차별화 포인트 후보: Z-box 의 l/r 갱신 애니메이션, "거울(mirror) 위치 k = i - l 의 재활용" 시각화, 비교 건너뛰기와 실제 비교의 색상 구분, r 의 단조성 강조.
- B타입 메타포 탐색은 비활성. 표준 표현을 더 잘 다듬는 방향.
- 도메인 슬러그: `string`, 대상 슬러그: `z-algorithm`.

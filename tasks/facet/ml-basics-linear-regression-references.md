## 2. 레퍼런스 요약

### R1. Setosa — Ordinary Least Squares Regression (Explained Visually)

- 출처: setosa.io / Victor Powell · Lewis Lehe (Bret Victor 의 Explorable Explanations 계보)
- URL: https://setosa.io/ev/ordinary-least-squares-regression/
- 한 줄 정체성: 잔차 제곱을 점이 아니라 "정사각형 면적" 으로 그려 RSS 의 "면적 합" 본질을 직관화한 표준 인터랙티브.
- 시각 객체: 까만 데이터 점, 파란 회귀 직선, 점과 직선을 잇는 연한 빨강 정사각형 (변 길이 = 잔차 절대값), 우측 상단 E² 카운터 (정사각형들의 총 면적).
- 시간 표현: 사용자가 직선을 드래그하면 실시간으로 사각형이 늘어나거나 줄어든다. 자동 재생 없음.
- 메타포: "넓이 줄이기 게임" — RSS 가 추상 수치가 아닌 눈으로 보이는 면적이라는 점.
- 강조: 잔차의 부호가 무의미하고 제곱(=면적) 만이 최소화 대상이라는 점. 한 점을 끌어 옮기면 그 점의 사각형만 즉시 변형.
- 인상적 디테일: 사각형이 직선과 점 사이의 수직 변에 정확히 외접하므로, 학습자가 "왜 수직 거리인가" 라는 질문을 묻기 전에 답을 받는다.
- 약점: 경사하강 운동은 부재. 정규방정식 결과(=최소 RSS 직선) 도 직접 보여주지 않고 사용자가 손으로 맞추는 게임으로만 다룸.

### R2. Seeing Theory — Regression Analysis (Brown Univ.)

- 출처: seeing-theory.brown.edu / Daniel Kunin et al. (Brown 통계학과 D3.js 기반)
- URL: https://seeing-theory.brown.edu/regression-analysis/index.html
- 한 줄 정체성: 슬라이더로 직선의 기울기·절편을 조작하며 잔차 그림이 즉시 갱신되는 통계 교과서식 인터랙티브.
- 시각 객체: 산점도 점, 사용자 직선, 점에서 직선으로 내려가는 수직 잔차 선분, 보조 패널의 R² · MSE 수치.
- 시간 표현: 슬라이더 조작 시간에 종속. "최적 직선" 토글로 정답 직선과 사용자 직선을 동시 비교.
- 메타포: "직선 두 손잡이 (기울기 손잡이 + 절편 손잡이) 로 점 무리에 끼워 맞추기".
- 강조: Anscombe's quartet 페이지를 같이 두어 "같은 회귀선이라도 잔차 모양이 전혀 다를 수 있다" 는 사실을 시각으로 못 박는다.
- 인상적 디테일: 점을 드래그해 추가/이동하면 OLS 선이 즉시 회전한다 — 한 점이 직선을 "끌어당기는" 영향력 (leverage) 의 직관.
- 약점: 경사하강·손실 표면은 다루지 않는다. 잔차 시각화도 선분뿐, 사각형 면적 표현은 없음.

### R3. Google Machine Learning Crash Course — Linear regression: Loss / Gradient descent

- 출처: developers.google.com (Google for Developers 공식 강의, 한국어 번역 동시 제공)
- URL: https://developers.google.com/machine-learning/crash-course/linear-regression/loss , https://developers.google.com/machine-learning/crash-course/linear-regression/gradient-descent
- 한 줄 정체성: "잔차 = 점에서 직선으로 내려가는 파란 선분" 메타포를 표준화한 입문 코스. 학습률 슬라이더 인터랙티브 포함.
- 시각 객체: 산점도 + 회귀 직선 + 점·직선을 잇는 파란 잔차 선분 (길이가 곧 손실 크기). 별도 위젯에서는 손실 곡선(반복 vs 손실) 이 시간축으로 펼쳐진다.
- 시간 표현: 반복(iteration) 카운터가 진행되며 직선이 점진적으로 회전하고 잔차 선분이 짧아진다. 손실 곡선은 초반 급락 → 후반 평탄화의 전형 패턴.
- 메타포: "선분 = 화살. 잘못된 예측만큼 화살이 길어진다."
- 강조: 손실은 부호가 아니라 "거리" 라는 점, 그리고 학습률이 너무 크면 손실 곡선이 진동·발산한다는 사실을 슬라이더로 직접 체험.
- 인상적 디테일: 한국어 페이지에서도 동일 위젯이 라벨 번역 없이 그대로 동작. 1000회 부근에서 곡선이 평평해지는 "수렴 이벤트" 가 뚜렷.
- 약점: 잔차 사각형(면적) 은 사용하지 않고 선분만. 매개변수 공간(w, b 평면) 의 운동은 코스 전반에서는 약하게 다뤄짐.

### R4. ml-visualized — Linear Regression / Gradient Descent

- 출처: ml-visualized.com (Python · NumPy · matplotlib 기반 정적 그림 + 코드 묶음 교과서)
- URL: https://ml-visualized.com/chapter1/linear_regression
- 한 줄 정체성: 데이터 평면과 매개변수 공간(3D 손실 표면 + 2D 등고선) 을 한 페이지에서 동시에 보여주는 두 화면 동기 도식의 정석.
- 시각 객체: 좌측은 산점도 + 직선 (반복마다 갱신), 우측은 (w, b) 평면 위의 손실 등고선 + 빨간 궤적 점선, 별도 3D 패널은 손실 표면 + "공처럼 굴러가는" 빨간 점.
- 시간 표현: 학습 초기 5회마다, 후반에는 1000회마다 스냅샷 — "초반 급변, 후반 미세조정" 의 비대칭 시간감을 명시적으로 설계.
- 메타포: "공이 골짜기를 굴러 내려간다 (ball rolling down a hill)". 등고선판은 그 공의 발자국 평면도.
- 강조: 같은 운동을 데이터 평면(직선의 회전) 과 매개변수 공간(점의 하강) 두 시점에서 동시에 봄으로써 "직선의 변화 = 점의 위치 변화" 라는 동치 관계.
- 인상적 디테일: 손실 표면이 항상 볼록(convex) 하다는 것을 3D 그림 한 장으로 못 박는다 — 비선형 모델과 다른 결정적 차이.
- 약점: 정적 PNG 위주라 운동감 자체는 GIF 시퀀스에 의존. 인터랙션 부재.

### R5. ConvNetJS Regression Demo (Andrej Karpathy)

- 출처: cs.stanford.edu/people/karpathy/convnetjs (스탠포드 시절 Karpathy, 브라우저 내 학습 시연의 고전)
- URL: https://cs.stanford.edu/people/karpathy/convnetjs/demo/regression.html , https://github.com/karpathy/convnetjs/blob/master/demo/regression.html
- 한 줄 정체성: 1차원 회귀를 SGD 로 실시간 학습하며 직선이 매 프레임 흔들리는 모습을 그대로 보여주는 라이브 데모.
- 시각 객체: 산점 데이터 + 매 0.1초마다 갱신되는 예측 곡선 (선형 + 작은 신경망 비교). 좌측 패널의 손실 수치가 라이브 카운터로 흐름.
- 시간 표현: 진정한 "시간 진행형" — 정지/재생 버튼만으로 SGD 가 1초에 수십 번 가중치를 갱신.
- 메타포: "직선이 데이터 위에서 떨고 있다" — 학습이 끝난 정적 결과가 아니라 살아 있는 운동.
- 강조: SGD 의 미니배치 잡음으로 직선이 미세하게 진동하는 모습이 그대로 노출됨. 학습률 등 하이퍼파라미터 실시간 변경 가능.
- 인상적 디테일: 같은 페이지에서 신경망(은닉층 추가) 과 선형 모델을 토글하며 "선형 = 신경망의 가장 단순한 특수 경우" 임을 시각으로 전달.
- 약점: UI 가 2014년대 그대로 노후. 잔차 그림이나 손실 표면은 없고 손실 수치 텍스트만 제공.

### R6. PhET — Least-Squares Regression (CU Boulder)

- 출처: phet.colorado.edu (Carl Wieman 설립, K-12·대학 교육 시뮬레이션의 사실상 표준)
- URL: https://phet.colorado.edu/en/simulations/least-squares-regression , https://phet.colorado.edu/sims/html/least-squares-regression/latest/least-squares-regression_en.html
- 한 줄 정체성: 사용자가 손으로 직선을 끼우고 "내 잔차 제곱합" 과 "OLS 잔차 제곱합" 을 동시에 비교하게 만든 교실용 시뮬.
- 시각 객체: 좌표평면, 드래그 가능한 데이터 점, 사용자 직선 (양 끝 손잡이), 토글 가능한 OLS 직선, 두 직선 각각의 잔차 사각형 + 합계 박스, r 카운터.
- 시간 표현: 직선 손잡이를 끌 때마다 사각형 합이 즉시 갱신. 자동 재생 없음.
- 메타포: "내가 만든 직선 vs 컴퓨터의 정답 — 누가 더 작은 사각형 합?" 하는 게임.
- 강조: 한 점을 추가/제거/이동했을 때 r 과 OLS 직선이 어떻게 바뀌는지 — 영향점(outlier) 과 적합도의 관계를 손으로 체감.
- 인상적 디테일: 실세계 데이터셋(키-몸무게, 자동차 연비 등) 을 내장해 "회귀가 어디에 쓰이나" 를 즉시 연결.
- 약점: 경사하강 학습 운동은 다루지 않음 (정규방정식의 결과만 토글로 노출). 매개변수 공간 시각도 없다.

### R8. TensorFlow Playground (선형 회귀 모드)

- 출처: playground.tensorflow.org (Google · D3.js)
- URL: https://playground.tensorflow.org/
- 한 줄 정체성: 은닉층을 0으로 두면 곧 선형 회귀가 되는 "신경망 놀이터" — 학습률·정규화의 효과를 같은 화면에서 즉시 비교.
- 시각 객체: 격자형 입력 평면 (배경 색이 예측값), 위에 흩뿌린 데이터 점, 우측의 손실 스파크라인 (학습 vs 검증).
- 시간 표현: 재생 버튼을 누르면 한 epoch 단위로 갱신, 손실 곡선이 시간축으로 흐른다. Step 버튼으로 한 발씩 진행 가능.
- 메타포: "학습이 색을 바꾼다" — 예측의 변화가 배경 색의 그라데이션 변화로 보임.
- 강조: 학습률을 너무 키우면 손실 스파크라인이 발산/진동하는 사건을 한 클릭으로 재현 가능.
- 인상적 디테일: 같은 도구로 "선형 = 은닉층 0" 과 "비선형 = 은닉층 ≥1" 을 토글하며 비교 — 모델의 표현력 한계가 직선의 평면 분할로 가시화.
- 약점: 분류 위주 데모라 회귀 시각화는 부수적. 잔차 사각형이나 매개변수 공간 (w, b) 그림은 없다.

- 약점: GIF 정도라 운동감이 약하고 매개변수 공간과 데이터 평면이 별개 그림으로 분리되어 있다.

## 3. 공통 요소

거의 모든 레퍼런스가 공유하는 시각 언어:

1. **산점도 + 회귀 직선** — 좌표평면 위에 까만 점과 한 줄의 직선. 가장 보편적이고 절대 빠지지 않는 기본형. (R1~R10 전부)
2. **잔차의 수직 선분** — 점에서 직선으로 내려가는 수직선이 곧 "오차". 부호 무관, 길이만이 정보. (R1~R7, R9, R10)
3. **두 손잡이(기울기·절편) 인터랙션** — 슬라이더 또는 직선 양끝 드래그로 (w, b) 두 매개변수를 직접 만짐. (R1, R2, R6, R7)
4. **손실의 시간 곡선** — x=반복, y=손실 의 단조 감소 곡선. 초반 급락 → 후반 평탄화의 전형. (R3, R4, R5, R8)
5. **매개변수 공간의 등고선/표면** — (w, b) 평면 위의 등고선 또는 3D 골짜기. 빨간 점/화살표가 최소점을 향해 이동. (R4, R9, R10)
6. **"공이 골짜기를 굴러 내려간다" 메타포** — 경사하강을 "지형 위 공" 으로 환원. 거의 모든 한국어·영어 강의에서 일관. (R3, R4, R8, R9, R10)
7. **잔차 제곱의 부호 무관성** — 선분이든 사각형이든 "부호가 사라지는 사건" 을 그림 자체가 보여줌. (R1, R3, R6, R7)
8. **선형 = 직선만 가능** 의 한계 시각화 — 비선형 데이터에 직선을 끼우면 잔차가 영원히 줄지 않는다는 사실을 산점도 형태(곡선 점 분포) 로 암시. (R2 의 Anscombe, R8)

학습자에게 형성된 기대:
- 선형 회귀는 "점 무리에 직선을 끼우는 게임" 이다.
- 잔차의 길이를 "전부 합쳐" 가장 작게 만들면 정답.
- 경사하강은 "골짜기를 굴러 내려가는 공" 이고, 학습률은 한 걸음의 크기다.

## 4. 공백

기존 레퍼런스에서 약하거나 비어 있는 영역:

1. **잔차 "사각형(면적)" 의 운동형 시각화** — Setosa(R1) 와 GeoGebra(R7) 가 정적으로 사각형을 보여줄 뿐, 경사하강이 진행되며 사각형들이 동시에 줄어드는 운동형 시각이 거의 없다. 학습 핵심 질문("제곱합이 어떻게 점진적으로 줄어드는가") 에 가장 직접적인 그림이 비어 있다.
2. **데이터 평면 ↔ 매개변수 공간의 동시 운동** — ml-visualized(R4) 가 정적으로 시도했을 뿐, 두 화면이 같은 시간축에서 1:1 동기화되어 살아 움직이는 인터랙티브가 부족하다. 한쪽에서 직선이 회전하면 반대쪽에서 점이 골짜기를 따라 내려가는 동치 운동의 인상이 약하다.
3. **학습률 변화의 진동·발산 사건** — TensorFlow Playground(R8) 와 Google MLCC(R3) 가 이 사건을 다루지만, 같은 데이터셋에 학습률만 다른 세 직선의 운동을 동시에 늘어놓는 비교형 시각은 거의 없다. "왜 학습률이 중요한가" 가 한 그림으로 전달되지 않는다.
4. **정규방정식 vs 경사하강의 동시 시각** — angeloyeo(R9) · ratsgo(R10) 가 텍스트로만 비교한다. "닫힌 해의 직선" 과 "반복 해의 직선이 다가가는 운동" 을 한 화면에서 동시에 그려, 두 길이 같은 답에 수렴함을 보여주는 도구가 없다.
5. **새 데이터 점이 직선을 끌어당기는 영향력** — Seeing Theory(R2) · PhET(R6) 가 입력 반응으로 다루지만, 시간 진행형 학습 도중에 점을 추가했을 때 "학습이 다시 시작되는" 운동을 보여주는 사례가 드물다. 외삽(extrapolation) 의 위험은 거의 시각화되지 않는다.
6. **잔차의 부호 차이 정렬** — 양의 잔차와 음의 잔차를 같은 색·같은 두께로 그리는 것이 관습이라, 직선이 데이터의 평균 방향에서 어느 쪽으로 치우쳐 있는지의 "정서적" 정보가 약하다. 상·하 색을 분리한 시각은 거의 없다.
7. **수렴 사건의 명시적 표시** — 손실 곡선이 평평해지는 시점이 언제인지 (예: 변화량 < ε) 를 그림 위에서 한 사건으로 깃발 꽂듯 표시한 사례가 부재. "학습이 끝났다" 가 운동의 자연 정지로만 처리된다.
8. **모델의 분포 가정과 잔차의 무작위성** — 선형 회귀가 가정하는 잔차의 정규성·등분산성을 잔차 산포(가로축=예측값, 세로축=잔차) 형태로 보여주는 자료(R-flavored 통계 교과서) 는 따로 있으나, 학습 운동과 같은 화면에 결합된 인터랙티브는 없다.

## 5. 대상 개념 요약

> 선형 회귀는 입력 x 와 정답 y 의 점 무리에 직선 y ≈ wx + b 를 끼우는 모델로, "잔차(점과 직선의 수직 거리) 의 제곱합" 을 최소화하는 (w, b) 를 정답으로 삼는다. 이 정답은 정규방정식이라는 닫힌 해로도 한 번에 구할 수 있고, 경사하강이라는 반복 갱신으로도 점진적으로 발견할 수 있다 — 학습 시각화의 본 무대는 후자의 시간 진행이다. 매 반복에서 (w, b) 는 손실의 기울기 반대 방향으로 한 걸음씩 이동하며, 같은 운동이 데이터 평면에서는 "직선의 회전·평행이동", 매개변수 공간에서는 "골짜기를 굴러 내려가는 공" 으로 이중 가시화된다.

학습자가 시각화와 함께 읽을 짧은 캡션:
- "직선 하나로 점들을 끼운다." (모델)
- "잔차의 제곱합 = 정사각형 면적의 합." (손실의 기하학적 정체)
- "경사하강은 골짜기를 굴러 내려가는 공이다." (학습 운동)
- "학습률은 한 걸음의 크기 — 너무 크면 굴러 넘어진다." (하이퍼파라미터)
- "선형 회귀의 골짜기는 항상 둥근 그릇이다." (볼록성)

용어 풀어쓰기:
- 잔차(residual) → "점이 직선에서 얼마나 떨어졌는가, 부호 포함"
- RSS / MSE → "잔차 제곱들의 합 / 평균. 작을수록 좋은 직선"
- 가중치 w · 편향 b → "직선의 기울기 · 직선이 y축과 만나는 높이"
- 기울기(gradient) → "손실이 가장 가파르게 커지는 방향"
- 학습률(learning rate) → "기울기 반대 방향으로 한 걸음에 얼마나 멀리 갈지"
- 수렴(convergence) → "더 이상 손실이 줄지 않을 만큼 (w, b) 가 자리잡음"
- 정규방정식(normal equation) → "반복 없이 한 번에 정답 (w, b) 를 계산하는 행렬 공식"

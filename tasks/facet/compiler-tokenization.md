# 토큰화 (어휘 분석)

### 0. 기본 정보
- 대상 이름: 토큰화 (tokenization, lexing/scanning)
- 도메인: compiler
- type: transformation
- 진행 모델: 입력 반응형
- 본질 한 문장: 원시 소스 문자열을 좌에서 우로 한 글자씩 응시하며 의미 있는 최소 단위로 끊어 라벨이 붙은 토큰의 줄로 바꾸는 첫 변환 단계.

### 1. 시그니처 행동
앞으로만 전진하는 응시가 같은 종류 글자들을 한 구간으로 점점 길게 묶다가 더는 못 묶는 순간 통째로 닫고, 그 구간을 종류 라벨과 원문 값이 함께 새겨진 한 장의 토큰으로 떨궈 출력열에 붙인다.

### 2. 시각적 은유
표준 메타포를 정교화한다. 위쪽에는 원시 소스 문자열이 가로 한 줄 띠로 펼쳐지고, 그 띠 위를 응시 표식이 좌에서 우로 한 칸씩 전진한다. 응시가 지난 자리에는 같은 토큰에 속한 글자들이 색이 일치하는 구간 박스로 누적되며 길어지고, 끊기는 순간 그 구간이 통째로 한 장의 토큰 카드로 떨어져 아래쪽 출력 행 끝에 가지런히 쌓인다. 공백·주석은 사라지지 않고 회색으로 가라앉아 띠 위에 흔적으로 남는다. NLP 형광펜식 인플레이스 색칠(L6, L7)과 컴파일러식 입력→출력 분리 표(L1, L3)를 한 화면에 결합한 형태다.

### 3. 결정적 순간
한 토큰이 닫히는 그 한 박자.
- 전: 응시가 어떤 글자 위에 있고, 그 직전 몇 글자가 같은 색의 구간으로 자라나 있다.
- 중: 다음 글자가 같은 종류로 못 묶인다고 판정되는 순간 — 자라던 구간의 테두리가 한 번 굳고, 색이 진해지며, 띠와 카드 사이에 한 줄기 낙하 궤적이 그어진다.
- 후: 그 구간이 통째로 한 장의 토큰 카드가 되어 출력 행 오른쪽 끝에 안착한다. 카드에는 종류 라벨(상단 띠 색)과 원문 값(중앙 글자) 두 가지가 함께 새겨진다. 동시에 응시는 새 글자 위에서 길이 0짜리 새 구간을 다시 키우기 시작한다.

강조 장치: 닫힘 순간의 짧은 펄스(테두리 굳기 + 색 진해짐), 띠→카드 낙하 궤적, 카드 안착 시의 가벼운 딸깍.

### 4. 다른 대상과의 구별점
- 구문 분석(파싱): 파싱은 토큰 줄을 받아 트리 구조를 세운다. 출력이 트리/계층이고 시각화는 이층 구조의 가지치기로 표현된다. 토큰화의 출력은 평평한 한 줄의 카드 시퀀스이며 트리 가지가 없다.
- 정규식 매칭(L7): 정규식은 하나의 패턴이 입력의 어디에 매치되는지를 색칠한다. 시각적 결과는 "패턴 ↔ 입력" 양방향 색상 매칭이며, 출력은 매치/캡처 그룹이다. 토큰화는 여러 종류의 패턴이 입력 전체를 빠짐없이 분배해 라벨 붙은 카드 줄로 떨어뜨린다 — 매치 영역이 연속적으로 잇닿아 입력을 완전히 덮는다.
- NLP 단어 분리(L6): 입력 띠 자체에 색만 입히고 끝낸다. 출력 행이 따로 있지 않다. 토큰화는 띠의 색칠과 별개로 아래쪽에 토큰 카드 행이 분리되어 쌓이며, 카드에는 원문 값에 더해 종류 라벨이 명시된다.
- 문자열 split: 한 가지 구분자만 보고 잘라 동질의 조각을 만든다. 시각적으로 모든 조각이 같은 색이다. 토큰화는 끊는 위치 자체가 종류마다 다른 패턴으로 결정되며, 카드의 색이 종류마다 달라진다.

### 5. 달성해야 할 시각 요소

1) 좌→우 전진 응시와 자라나는 구간 박스
   - 설명: 응시 표식이 띠 위를 한 글자씩 전진하면서, 같은 종류로 묶이는 글자들이 하나의 색 구간으로 점점 길어진다. 진행은 결코 뒤로 가지 않는다.
   - 시각 표현: 띠 위의 좁은 세로 막대(응시), 글자 아래의 색 띠가 한 글자씩 늘어남. 이미 본 영역은 옅은 톤, 안 본 영역은 무채색.

2) 닫힘과 토큰 카드 떨어짐
   - 설명: 더 못 묶는 순간 구간이 통째로 닫혀 출력 행 끝에 한 장의 카드로 안착한다. 카드에는 종류 라벨과 원문 값이 함께 새겨진다.
   - 시각 표현: 띠 → 출력 행으로 이어지는 짧은 낙하 궤적. 카드 상단 띠 색 = 종류, 카드 중앙 = 원문 글자열.

3) 공백·주석의 삼킴 흔적
   - 설명: 토큰이 되지 못한 글자도 사라지지 않고 띠 위에 회색으로 가라앉아 "지나쳐졌음"이 보인다.
   - 시각 표현: 응시가 지나면 글자가 회색·반투명으로 가라앉음. 어떤 카드도 그 구간을 가리키지 않는다.

4) 최장 일치의 망설임 한 박자
   - 설명: `>=`처럼 두 글자로 묶일 수 있는 자리에서, 한 글자에서 끊을 뻔하다 한 칸 더 들여다본 뒤 두 글자를 함께 묶는 결정 순간이 한 박자로 보인다.
   - 시각 표현: 구간 테두리가 잠시 점선으로 떠올랐다가, 다음 글자를 확인한 뒤 실선으로 굳으며 두 글자를 모두 감싼다.

5) 식별자 → 키워드 라벨 교체
   - 설명: 닫힌 카드가 우선 일반 분류로 표지되고, 그 직후 사전 대조를 거쳐 라벨이 더 강한 종류로 갈아끼워지는 두 단계 변신.
   - 시각 표현: 카드 상단 띠 색이 한 박자 뒤 다른 색으로 페이드 전환되며 라벨 텍스트도 함께 갱신.

6) 오류 토큰의 빨간 카드
   - 설명: 인식할 수 없는 글자도 사라지지 않고 빨간 카드로 출력열에 박힌다. 응시는 멈추지 않고 다음 가능한 시작점으로 건너간다.
   - 시각 표현: 카드 상단 띠는 빨강, 중앙은 문제 글자, 카드 외곽선이 가볍게 진동.

(요소 1·2·5·6이 핵심, 3·4가 강화 보조. 합계 6개지만 시각 평면 안에서는 항상 응시·구간·카드 한 묶음으로 보이므로 깨지지 않는다.)

### 6. 시각 구성
- 중심 시각: 위쪽 입력 띠 + 아래쪽 출력 토큰 카드 행. 띠 위의 응시·자라나는 구간과, 카드 행에 차례로 안착하는 카드들이 한 박자로 동기 진행한다.
- 보조 요소: 좌측 상단의 짧은 개념 설명문, 우측 하단의 작은 범례(종류별 색 ↔ 라벨), 띠 좌하단의 진행 미니 인디케이터(현재 위치 / 전체 길이).
- 요소 간 관계: 띠 위에서 한 구간이 닫히는 그 순간 = 카드 한 장이 떨어지는 순간. 두 평면은 같은 시각축을 공유한다. 카드 색은 띠 구간 색과 반드시 일치한다.

### 7. 레이아웃
임베드 폭 640~720px 기본, 확장 880px. 입력 띠가 가로 폭이 핵심이므로 상하 분할.

- 상단(높이 약 36%): 입력 띠 (한 줄 가로 폭 전체 사용). 띠 좌측에 작은 진행 인디케이터.
- 중단(높이 약 8%): 띠와 카드 행을 잇는 낙하 궤적이 지나가는 빈 가이드 영역.
- 하단(높이 약 40%): 출력 토큰 카드 행. 좌→우로 카드가 누적된다. 행 높이는 카드 1줄 + 줄바꿈 1줄까지.
- 좌상단 박스(약 폭 40%, 높이 16%): 개념 설명 2~4문장.
- 우하단 박스(약 폭 28%, 높이 16%): 종류↔색 범례.
- 카드 폭은 글자 수에 비례하되 최소·최대 폭을 두어 한 줄에 8~12장 노출. 880px 확장 시 12~16장.

### 8. 진행과 인터랙션
진행 모델은 입력 반응형. "다음 글자"라는 입력 이벤트마다 시각이 한 박자 갱신된다.

- 소스 텍스트 편집: 상단 띠의 글자가 바뀌면 응시는 처음으로 되돌아가고 분절이 다시 흐른다. 같은 입력은 늘 같은 분절을 낳는다(결정성)는 점이 반복 재생으로 자명해진다.
- 예제 전환: `var x = 42;`, `if x >= 10`, `// note\n  return n` 같은 예제를 1·2·3 칩으로 전환. 각 예제는 본질의 다른 면(키워드 승격, 최장 일치, 공백·주석 삼킴)을 강조.
- 스캔 속도 조절: 느리게 / 보통 / 빠르게 3단. 느리게에서는 망설임 박자(요소 4)가 또렷하다.
- 단계 이동: 한 박자씩 앞으로 / 시작으로 되돌리기. 결정적 순간에서 자연 정지하는 옵션.
- 매핑: 응시 위치 한 칸 전진 → 띠 색 구간 1칸 신장 또는 닫힘. 닫힘 → 카드 1장 안착. 인식 실패 → 빨간 카드 1장 + 응시 다음 시작점 점프. 사전 일치 → 직전 카드의 라벨 색·글씨 페이드 전환.

### 9. 초기 상태와 파라미터
- 초기 입력: `if x >= 10  // ok\n  return x;` (키워드 승격, 최장 일치, 공백, 주석, 식별자, 숫자, 연산자, 구분자, 줄바꿈을 한 예제 안에 모은 형태)
- 응시: 띠 좌측 끝, 길이 0 구간.
- 출력 행: 비어 있음.
- 속도: 보통 (한 박자 ≈ 280ms).
- 종류 색 팔레트(범례와 일치): 키워드(보라), 식별자(청록), 숫자(주황), 연산자(노랑), 구분자(회청), 문자열(연두), 오류(빨강), 삼킴(연회색).
- 폭: 720px 기본.

### 10. 시각화 안 텍스트
- 개념 설명(2~4문장):
  "토큰화는 컴파일러의 첫 단계입니다. 사람이 쓴 소스 문자열을 좌에서 우로 한 글자씩 읽어 의미 있는 최소 단위(토큰)로 끊어 냅니다. 끊는 기준은 종류별 패턴이며, 더 길게 묶을 수 있으면 더 길게 묶습니다. 공백과 주석은 토큰이 되지 못하고 흔적으로만 남습니다."
- 학습자 안내 한 줄: "위 띠에서 글자가 어떻게 묶여 아래의 토큰 카드로 떨어지는지 보세요."
- 결정적 순간 보조 캡션(닫힘 박자에만 잠깐 떠오름): "여기서 한 토큰이 끝났습니다."

### 11. 참고 레퍼런스
시각화 UI 우측 하단 또는 별도 패널에 노출.

- Crafting Interpreters — Scanning (Robert Nystrom): https://craftinginterpreters.com/scanning.html — `start`/`current` 두 인덱스로 lexeme을 묶고 토큰을 떨구는 루프의 정전.
- The Tokenizer Playground (Xenova): https://huggingface.co/spaces/Xenova/the-tokenizer-playground — 입력 텍스트를 인플레이스에서 색 띠로 끊어 보여주는 시각 어휘의 표준.
- Ruslan's Blog — Let's Build A Simple Interpreter Part 1: https://ruslanspivak.com/lsbasi-part1/ — `3+5`가 (type, value) 토큰 카드로 분해되는 입문 다이어그램.
- regex101: https://regex101.com/ — 패턴과 입력의 양방향 색상 매칭으로 "규칙 → 매치"를 시각화한 도구.

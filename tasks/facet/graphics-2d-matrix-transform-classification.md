# 2D 행렬 변환 분류

## 1. 대상 정보

- 이름: 2D 행렬 변환 (2D matrix transformation)
- 분야: 컴퓨터 그래픽 / 2D 변환
- 핵심 입력: 2×2 행렬 (또는 동차좌표 3×3) + 평면의 점·벡터·도형
- 핵심 출력: 변환 후 좌표 — 행렬의 두 열이 변환된 기저 벡터 (i-hat, j-hat) 위치
- 핵심 사건: 회전, 스케일, 전단, 반사 (이동은 동차좌표가 있어야 행렬로 표현)
- 핵심 통찰: 모든 점은 기저 벡터의 선형 결합으로 따라가며, 합성은 행렬 곱이고 순서가 비가환
- 도메인 prefix 근거: `graphics` 는 영문 소문자 kebab-case 단일 단어로 짧고, 기존 `database`/`network` 와 동급의 단순도를 유지한다 (`computer-graphics` 는 길고, `cg` 는 약어라 가독성이 떨어짐)
- 슬러그 근거: `2d-matrix-transform` — 2D 한정임을 슬러그에 명시해 추후 3D 변환 facet 과 충돌하지 않도록 분리

## 2. type

transformation

근거: 행렬은 평면의 점·도형을 새 위치로 매핑하는 행위 자체이며, 알고리즘적 단계 진행이나 단일 정적 개념 정의보다 "공간을 어떻게 바꾸는가" 가 시각화의 중심이다.

## 3. 진행 모델

입력 반응형

근거: 행렬 성분 (또는 슬라이더) 변경 → 격자·도형이 즉시 새 위치로 반영되는 매핑이 본질이다. 항등→대상으로의 보간 애니메이션은 표현 수단이지 시각화 본질의 시간 진행이 아니다.

## 4. 핵심 학습 질문

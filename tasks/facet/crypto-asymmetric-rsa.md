# RSA 시각화 기획서

### 0. 기본 정보

- 대상 이름: RSA (비대칭 암호 / 공개키 암호)
- 도메인: 보안·암호
- type: algorithm
- 진행 모델: 시간 진행형
- 본질 한 문장: RSA 는 두 큰 소수가 합성수 n 을 낳는 가벼운 곱셈 위에 한 짝의 비대칭 키를 세워, 누구나 가진 공개키로는 잠그기만 하고 주인만 가진 비밀키로만 풀리도록, 그리고 같은 n 을 거꾸로 쪼개는 길은 사실상 막아 두는 알고리즘이다.

### 1. 시그니처 행동

두 소수에서 태어난 한 짝의 키 중 한쪽으로만 잠기고 다른 쪽으로만 풀리며, 잠근 쪽으로는 결코 되돌릴 수 없다.

### 2. 시각적 은유

표준 메타포인 Alice·Bob 두 인물, 공개 자물쇠와 개인 열쇠 짝, 그리고 잠긴 봉투가 채널을 건너는 우편함 그림을 채택해 정교화한다. 화면 좌측에는 편지를 보내는 Alice 가, 우측에는 받는 Bob 이 자기 책상 위에 자물쇠와 열쇠 한 짝을 올려 두고 앉는다. 두 키는 거울 대칭이 아니라 형태가 다르다 — 자물쇠는 아래쪽이 둥글게 닫힌 빗장, 열쇠는 톱니가 한쪽으로만 깊게 파인 막대로, 모양만 봐도 "잠그는 손" 과 "푸는 손" 임이 분리된다. 채널 중앙에는 잠긴 봉투(암호문 c) 가 떠가는 회색 통로가 있고, 그 통로 위쪽에는 외부 관찰자의 시점 — 공개된 합성수 n 만이 떠 있는 어두운 영역 — 이 얇은 띠로 분리되어, 같은 n 을 거꾸로 쪼개려는 화살표가 무거운 운동으로 막혀 있는 모습을 시각의 한 자리로 가진다.

### 3. 결정적 순간

학습자가 "이게 RSA 구나" 알아차리는 한 호흡은, Bob 의 책상 위에서 두 소수 카드 p 와 q 가 합쳐져 합성수 n 을 낳고, 그 n 위에 자물쇠와 열쇠 한 짝이 동시에 출생해 자물쇠 한 벌이 채널을 건너 Alice 의 책상 위로 사본되는 출생의 장면, 곧이어 Alice 가 평문 m 을 봉투에 넣고 그 공개 자물쇠로 봉투를 채워 채널로 흘려보내는 잠금의 장면, Bob 의 책상에 도착한 봉투가 Bob 의 비밀 열쇠 앞에서만 열려 평문 m 이 그대로 꺼내지는 풀림의 장면, 그리고 누군가 같은 공개 자물쇠로 채널 위 봉투를 다시 풀려고 시도하는 순간 자물쇠가 빨갛게 흔들리며 거부되는 막힘의 장면 — 이 네 장면이 한 시간축 위에서 끊김 없이 이어진다.

- 전: 빈 책상 위에 두 소수 카드 p, q 만 놓여 있다. 채널은 비어 있고 외부 관찰자 영역도 어둡다. Alice 의 책상에는 평문 m 만 떠 있다.
- 중: p·q → n 의 가벼운 화살표가 흘러 합성수 n 카드가 책상 중앙에 안착하면서, 그 위에 자물쇠와 열쇠 한 짝이 동시에 출생한다. 자물쇠 한 벌이 채널을 건너 Alice 책상 위로 사본되며 같은 자물쇠가 외부 관찰자 영역에도 옅게 비친다. Alice 가 평문 m 을 봉투에 넣고 자물쇠로 채우면 봉투가 잠긴 봉투 c 로 변해 채널 위를 건너간다. Bob 의 비밀 열쇠가 봉투 위에 닿는 순간 자물쇠가 부드럽게 풀리고 평문 m 이 다시 떠오른다.
- 후: 출력 자리에는 Bob 의 책상에서 복원된 m 이, 채널에는 사용된 봉투의 잔상이, 외부 관찰자 영역에는 같은 공개 자물쇠로 봉투를 다시 풀려고 시도했다가 거부된 빨간 흔들림의 잔상이 동시에 한 화면에 남는다.
- 강조 장치: 자물쇠와 열쇠의 형태 비대칭, 두 키가 사는 영역(공개 마당 / 비밀 방) 의 색·배경 분리, p·q→n 화살표의 가벼움과 n→(p,q) 화살표의 무거움 차이, 거꾸로 시도 시 자물쇠의 빨간 흔들림.

### 4. 다른 대상과의 구별점

- 대칭 암호 (AES 등): 대칭 암호는 한 개의 같은 키로 잠그고 같은 키로 푸는 거울 운동이다. Alice 와 Bob 양쪽 책상에 같은 모양의 키 한 자루씩이 놓이고, 잠금과 풀림이 정확히 대칭의 두 운동으로 그려진다. RSA 는 키가 두 개이고 서로 형태가 다르며, 두 책상에 다른 키가 놓인다. 같은 키로는 절대 풀리지 않는다는 사건이 거꾸로 시도 토글에서 빨간 흔들림으로 강제된다.
- 다른 비대칭 암호 (ECC): ECC 도 공개키·비밀키 짝을 가지지만 그 출생이 타원 곡선 위 점의 운동에서 비롯된다. RSA 는 출생 메타포가 정수론에 박혀 있다 — 두 소수 카드가 곱해져 합성수 n 을 낳는 산수 그림이 키 생성 시퀀스의 첫 자리를 차지한다. ECC 시각화에서는 곡선 위 점이 그 자리를 차지할 것이다. 키 생성의 첫 그림이 무엇이냐에서 두 알고리즘의 정체성이 갈린다.
- 해시 함수: 해시는 일방향이지만 키 짝이 없고, 입력으로 되돌아오는 운동 자체가 정의되지 않는다. 화면에 키도 없고 채널도 없으며, 임의 길이 입력이 고정 길이 지문으로 바뀌어 그대로 머무는 한 방향 운동만 있다. RSA 는 일방향처럼 보이지만 비밀 열쇠 한 짝으로만 평문이 정확히 귀환한다는 닫힌 두 사건의 짝이 화면에 살아 있다.

### 5. 달성해야 할 시각 요소

1. **두 소수에서의 출생 시퀀스 — 키 짝의 인과 사슬**
   - 설명: 키 짝이 갑자기 두 박스로 등장하는 것이 아니라, 두 소수 p, q 가 먼저 자리에 앉고 그것이 곱해져 n 을 낳고 n 위에 자물쇠와 열쇠가 차례로 출생하는 운동이 시간차로 보여야 한다.
   - 시각 표현: 화면 상단 키 생성 시퀀스 영역에서 좌측 끝에 소수 카드 p, q 두 장이 먼저 슬라이드인 한 뒤, 가벼운 곱셈 화살표(얇은 직선) 가 두 카드를 합쳐 가운데에 합성수 카드 n 을 낳는다. n 카드 위로 자물쇠 모양과 열쇠 모양의 윤곽이 동시에 떠오르며 분리되어, 자물쇠는 아래 채널 쪽으로 사본되고 열쇠는 Bob 의 비밀 방으로 미끄러져 들어간다. 각 단계 사이에 짧은 정지가 있어 인과의 차례가 운동의 차례로 강제된다.

2. **공개 마당과 비밀 방 — 권한 영역의 공간 분리**
   - 설명: 같은 부모에서 났음에도 두 키가 다른 권한 영역으로 흩어지는 사실이 텍스트 라벨이 아닌 공간 위치로 강제되어야 한다.
   - 시각 표현: 화면 우측 Bob 영역이 두 층으로 나뉜다. 위쪽 층은 옅은 색 배경의 "공개 마당" — 누구나 볼 수 있는 공간이고 자물쇠 한 벌과 합성수 n 이 거기에 놓인다. 아래쪽 층은 어두운 색 배경의 "비밀 방" — Bob 만 접근하는 공간이고 비밀 열쇠와 두 소수 p, q 가 거기에 머문다. 채널을 건너 Alice 책상 위로 가는 사본은 공개 마당의 자물쇠뿐이고, 비밀 방의 객체들은 채널에 결코 닿지 않는다.

3. **두 키의 형태 비대칭 — 모양만으로도 다른 일을 한다**
   - 설명: 표준 자료가 좌우 대칭으로 그려 잃었던 비대칭을, 키 두 개의 형태·운동·역할이 모두 다르게 보이도록 강화한다.
   - 시각 표현: 자물쇠는 사각 몸체에 위쪽으로 ㄷ 자 빗장이 솟은 둥근 형상으로, 봉투 위에 얹히면 빗장이 닫히는 운동을 한다. 열쇠는 가는 막대에 한쪽으로만 톱니가 깊게 파인 형상으로, 자물쇠 옆구리 구멍에 미끄러져 들어가 시계 방향으로 한 번 회전하는 운동을 한다. 색도 다르다 — 자물쇠는 밝은 청록 계열, 열쇠는 깊은 자주 계열로, 모양과 색 모두에서 두 키가 절대 같은 객체가 아님이 즉시 인지된다.

4. **잠금과 풀림의 한 프레임 사건성 — 두 다른 키의 동시 가시화**
   - 설명: 평문 m → 암호문 c → 평문 m 의 흐름 안에서, 첫 변환과 두 번째 변환에 사용되는 키가 서로 다른 객체임이 같은 화면에 동시에 보여야 한다.
   - 시각 표현: 채널을 가로지르는 봉투의 양 끝에 잠금 사건과 풀림 사건이 정지된 잔상으로 동시에 살아남는다. 좌측 끝에는 청록 자물쇠가 봉투를 닫는 순간이, 우측 끝에는 자주 열쇠가 같은 자물쇠를 회전시켜 여는 순간이 작은 아이콘 스냅샷으로 채널 양쪽에 박힌다. 사용된 두 키가 같은 화면에 동시에 보이며 형태·색이 다른 두 객체임이 한눈에 확인된다.

5. **인수분해 어려움의 막힘 화살표 — 외부 관찰자 영역**
   - 설명: 일방향성이 텍스트 주석이 아니라 시각적 막힘으로 보여야 한다. 같은 두 점을 잇는 두 방향이 형식적 역방향이 아니라 비용이 비대칭인 두 사건임이 운동의 무게로 드러난다.
   - 시각 표현: 채널 위쪽의 어두운 외부 관찰자 영역에 공개된 합성수 n 카드만이 옅게 떠 있고, 그 카드에서 두 소수 p, q 를 향해 뻗는 화살표가 굵고 무거운 점선으로 표현된다. 화살표는 도중에 빨간 격벽에 부딪혀 끊어진 채로 멈춰 있다. 반대로 키 생성 시퀀스에서 p·q → n 으로 흐른 화살표는 얇은 실선이고 가볍게 그어진다. 같은 두 점, 같은 운동의 방향만 다른 두 화살표가 무게 차이로 일방향성을 신체화한다.

6. **거꾸로 시도의 거부 — 비대칭의 시각적 강제**
   - 설명: 잠그는 데 쓴 그 공개키로는 절대 풀리지 않는다는 비대칭이 한 운동으로 강제되어야 한다. 텍스트로 "되지 않는다" 가 아니라 자물쇠 자체가 거부하는 사건.
   - 시각 표현: 거꾸로 시도 토글이 켜진 상태에서 채널 위 잠긴 봉투에 같은 공개 자물쇠를 다시 들이대면, 자물쇠 빗장이 위로 들어 올려지지 않고 빨갛게 흔들리며 짧은 좌우 진동을 일으킨다. 봉투 위쪽에 빨간 X 표시가 잠시 떠올랐다 사라진다. 이어 비밀 열쇠로 시도하면 같은 자물쇠가 부드럽게 풀린다 — 같은 자물쇠 앞에서 두 키의 운명이 정반대로 갈리는 모습이 한 자리에서 두 운동으로 비교된다.

### 6. 시각 구성

- **중심 시각**: 좌우로 분리된 두 인물 영역 (Alice / Bob) 과 그 사이를 가로지르는 채널, 그리고 Bob 영역 위에 자리 잡은 키 생성 시퀀스 영역. 이 셋이 화면의 척추를 이룬다.
- **보조 요소**:
  - 외부 관찰자 영역: 채널 위쪽 어두운 띠. 공개된 n 카드와 막힌 인수분해 화살표가 거기 산다.
  - 비밀 방: Bob 영역 아래쪽 어두운 박스. 비밀 열쇠와 두 소수 p, q 가 거기 머문다.
  - 상단 캡션: 짧은 개념 문장 한 줄 + 현재 사건 메시지.
  - 하단 컨트롤: 두 소수 슬라이더, 평문 입력, 재생/일시정지/속도, 거꾸로 시도 토글.
- **요소 간 관계**: 키 생성 시퀀스 영역에서 태어난 자물쇠는 채널을 건너 Alice 영역으로 사본되어 평문을 잠그는 도구가 되고, 같은 시퀀스에서 함께 태어난 열쇠는 Bob 의 비밀 방으로 들어가 봉투를 푸는 도구가 된다. 채널은 잠긴 봉투 c 의 이동 통로이며, 그 위쪽 외부 관찰자 영역은 공개된 n 만 보고도 비밀 방에 닿지 못하는 막힘을 시각으로 가진다. 비밀 방의 모든 객체는 채널·외부 관찰자 영역과 단절되어 있다 — 어떤 화살표도 비밀 방을 가로지르지 않는다.

```
│ [p ▽] [q ▽]  [평문 ▣]  [▶/⏸] [속도] [거꾸로 시도 ☐]   (60)  │
└───────────────────────────────────────────────────────────────┘
```

- 키 생성 시퀀스 영역: 상단에 폭 전체로 70px. 좌→우로 p, q → n → 자물쇠+열쇠가 시간차로 등장.
- 외부 관찰자 영역: 키 생성 아래 폭 전체로 40px. 어두운 배경에 공개된 n 카드와 막힌 화살표.
- 본체: 좌측 Alice 영역 폭 180px, 중앙 채널 폭 260px, 우측 Bob 영역 폭 220px. Bob 영역 안에서 위 100px 가 공개 마당, 아래 100px 가 비밀 방.
- 확장 폭 880px 일 때는 채널이 360px 로 늘어나 봉투 이동 운동의 호흡이 길어진다. 다른 영역은 그대로.
- 상단 캡션 40px, 키 생성 70px, 외부 관찰자 40px, 본체 200px, 하단 컨트롤 60px, 영역 사이 여백 70px 로 총 약 480px.

### 8. 진행과 인터랙션

시간 진행형. 단계 (A) 키 생성, (B) 암호화, (C) 복호화 가 시간순으로 펼쳐지며 결정적 순간 직후 거꾸로 시도 시연이 한 호흡 더 붙는다.

- **(A) 키 생성**: 두 소수 카드 p, q 가 키 생성 시퀀스 영역 좌측에 슬라이드인 한다. 가벼운 곱셈 화살표가 두 카드를 합쳐 합성수 n 카드를 영역 중앙에 안착시킨다. n 카드 위로 자물쇠와 열쇠 윤곽이 동시에 떠오르며 분리된다 — 자물쇠 한 벌은 채널을 건너 Alice 영역 위쪽에 사본으로 안착하고 다른 한 벌은 Bob 공개 마당에 머문다. 비밀 열쇠는 Bob 비밀 방으로 미끄러져 들어간다. 동시에 외부 관찰자 영역에 공개된 n 카드가 옅게 떠오른다.
- **(B) 암호화**: Alice 영역의 평문 m 카드가 봉투 안으로 들어가고, 봉투 위에 Alice 가 가진 공개 자물쇠가 얹혀 빗장이 닫힌다. 잠긴 봉투 c 가 채널을 따라 좌→우로 부드럽게 흘러간다. 채널 위쪽에 잠금 사건의 작은 스냅샷이 잔상으로 박힌다.
- **(C) 복호화**: 봉투가 Bob 공개 마당에 도착한다. Bob 비밀 방에서 비밀 열쇠가 떠올라 자물쇠 옆구리에 미끄러져 들어가고 시계 방향으로 한 번 회전한다. 자물쇠가 풀리며 봉투에서 평문 m 이 다시 꺼내져 Bob 영역의 출력 자리에 안착한다. 채널 위쪽에 풀림 사건의 작은 스냅샷이 잔상으로 박혀 잠금 스냅샷과 한 화면에 같이 남는다.
- **거꾸로 시도 (토글)**: 거꾸로 시도 토글이 켜져 있으면 (C) 직전에 짧은 시연이 끼어든다. 누군가의 손이 채널 위 봉투에 같은 공개 자물쇠를 다시 들이대면, 자물쇠가 빨갛게 흔들리며 빗장이 들리지 않고 봉투 위에 빨간 X 가 잠시 떠올랐다 사라진다. 이어 정상 흐름으로 비밀 열쇠가 들어와 자물쇠를 풀어낸다. 토글이 꺼져 있으면 이 시연은 생략되고 (B) → (C) 가 곧장 이어진다.
- **외부 관찰자 영역의 막힘 운동**: 키 생성 직후, 외부 관찰자 영역의 공개된 n 카드에서 p, q 를 향한 점선 화살표가 짧게 그어졌다가 빨간 격벽에 부딪혀 멈추는 운동이 한 번 일어난다. 이 막힘 운동은 거꾸로 시도 시연이 일어날 때마다 한 번씩 다시 짧게 반복된다.
- **제어 지점**:
  - 두 소수 슬라이더: p, q 를 작은 학습용 짝 (예: 5, 11, 13, 17, 19) 사이에서 선택. 슬라이더를 움직이면 n 카드와 자물쇠·열쇠가 즉시 다시 출생.
  - 평문 입력: 한 글자 또는 작은 정수. 입력이 바뀌면 봉투 안 평문 m 카드가 갱신.
  - 재생 / 일시정지 / 속도: 시퀀스 자동 재생. 속도는 0.5×, 1×, 2× 세 단.
  - 거꾸로 시도 토글: 켜고 끄기로 거부 시연의 표시 여부 결정.
- 모든 단일 운동은 약 400~700ms. 단계 사이 정지 200~300ms 로 인과의 차례가 운동의 차례로 강제된다.

### 9. 초기 상태와 파라미터

- **초기 상태**: 키 생성 시퀀스 영역에 두 소수 카드 p=5, q=11 이 놓인 직후 정지 상태. n=55 가 막 안착했고 자물쇠·열쇠 한 짝이 출생을 마쳐 자물쇠 사본이 Alice 영역과 Bob 공개 마당에 한 벌씩 놓여 있다. 비밀 열쇠는 Bob 비밀 방에 있다. Alice 영역에 평문 m=2 카드가 떠 있고, 채널은 비어 있다. 외부 관찰자 영역에는 공개된 n=55 카드만 옅게 떠 있다. 거꾸로 시도 토글은 꺼짐.
- **권장 시연 시퀀스**: 진입 직후 자동 재생이 한 번 흐른다 — Alice 가 평문 m 을 봉투에 넣고 자물쇠로 잠가 채널로 보내고, Bob 의 비밀 열쇠가 봉투를 풀어 m 을 복원한다. 학습자는 그 뒤 슬라이더로 p, q 를 바꾸어 키 짝의 재출생을 보거나, 거꾸로 시도 토글을 켜 같은 자물쇠로 풀려는 시도가 거부되는 장면을 직접 호출한다.
- **파라미터**:
  - p, q 후보: {5, 7, 11, 13, 17, 19, 23} 중 두 개 선택 (학습자가 손으로 검산 가능한 규모).
  - 평문 m 범위: 1 ≤ m < min(p, q) 사이 작은 정수 또는 한 글자 (코드포인트 매핑 후 같은 범위로 클램프).
  - 재생 속도: 0.5×, 1×, 2× 세 단.
  - 운동 지속 시간: 키 생성 단계 600ms, 잠금 500ms, 채널 이동 700ms, 풀림 500ms, 거부 시연 400ms.
  - 색 팔레트: 자물쇠는 밝은 청록, 열쇠는 깊은 자주, 평문 m 은 따뜻한 노랑, 봉투는 옅은 회백, 비밀 방 배경은 짙은 남색, 공개 마당 배경은 옅은 베이지, 외부 관찰자 영역 배경은 어두운 회색.

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> RSA 는 두 큰 소수에서 태어난 한 짝의 키로 메시지를 잠그고 푼다. 누구나 가진 공개 자물쇠로는 잠그기만 할 수 있고, 주인만 가진 비밀 열쇠로만 풀 수 있다. 두 소수를 곱하는 길은 가볍지만 그 결과만 보고 두 소수를 되찾는 길은 사실상 막혀 있어, 같은 자물쇠로 봉투를 거꾸로 풀려는 시도는 닿지 않는다.

**학습자 안내 (보조 캡션 회전 문구)**

- "키 한 짝이 두 소수에서 태어난다 — 그 출생을 따라가 보자."
- "공개 자물쇠는 누구나, 비밀 열쇠는 주인만."
- "잠근 그 자물쇠로는 절대 풀리지 않는다."
- "공개된 합성수만 보고는 비밀 방에 닿지 못한다."

**사건별 짧은 메시지**

- 소수 선택: "두 소수 p, q 가 자리에 앉았다."
- n 합성: "두 소수가 곱해져 합성수 n 을 낳았다."
- 키 짝 출생: "n 위에 자물쇠와 열쇠 한 짝이 태어났다."
- 공개키 배포: "자물쇠 한 벌이 채널을 건너 모두에게 사본되었다."
- 잠금: "Alice 가 평문을 봉투에 넣고 공개 자물쇠로 잠갔다."
- 채널 통과: "잠긴 봉투가 채널을 건넌다 — 안의 평문은 보이지 않는다."
- 풀림: "Bob 의 비밀 열쇠가 봉투를 열고 평문이 돌아왔다."
- 거꾸로 시도 거부: "같은 자물쇠로는 풀리지 않는다 — 비대칭의 벽."
- 인수분해 막힘: "공개된 n 만으로는 두 소수에 닿지 못한다."

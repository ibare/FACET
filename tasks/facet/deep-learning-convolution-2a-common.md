# 딥러닝 합성곱 시각화 — 공통 강화 기획 (2A)

## 0. 기본 정보

- 대상 이름: 딥러닝 합성곱 연산 (2D Convolution)
- 도메인 / 카테고리: 딥러닝 / CNN 핵심 연산
- 대상의 본질: 작은 가중치 격자(커널)가 큰 입력 격자 위를 칸 단위로 미끄러지면서, 매 위치마다 같은 가중치로 원소별 곱과 합을 수행해 출력 격자를 한 칸씩 채우는 연산. 동일 가중치가 모든 위치에서 재사용되며 (가중치 공유), 다중 입력 채널은 더해지고 다중 필터는 출력 채널을 늘린다.
- 이 기획의 접근: **공통 강화** — 레퍼런스 평균이 공유하는 시각 언어(2D 격자, 슬라이딩 사각형, 색 매칭, 수식 사이드 패널, 호버 시간 통제)를 채택하고, 디테일과 완성도로 표준 도구들을 능가하는 후보를 만든다.

---

## 1. 시그니처 행동

**"색이 동기화된 작은 사각형 한 개가 입력 격자 위를 칸 단위로 점프하며, 점프할 때마다 출력 격자에 정확히 한 칸이 같은 색으로 떠오르고, 옆 패널에는 그 한 칸을 만든 곱·합 식이 풀어쓰여 동시에 나타난다."**

- 점프 단위: 격자 칸 (스트라이드만큼).
- 사각형의 색: 매 시점마다 고유한 색. 그 색이 receptive field, 커널 셀, 출력 셀, 수식 라인을 한 묶음으로 묶는다.
- 다른 슬라이딩 윈도우 알고리즘(이미지 블러 단독, 풀링)과 구별되는 핵심: 곱셈 항이 **고정된 가중치**(커널)와의 dot product 라는 사실이 매 위치에서 시각적으로 동일하다는 점이다.

## 2. 시각적 은유

레퍼런스 공통 메타포 두 갈래(**도장 찍기** + **스프레드시트 형광펜**)를 합쳐 정교화한다.

- 입력 격자 = 도장 찍을 종이.
- 커널 = 도장 (반투명, 진한 테두리, 항상 동일한 글리프).
- 출력 격자 = 도장 자국이 누적되는 결과지.
- 형광펜 트레이스 = 한 출력 셀이 어떤 입력 9 칸과 어떤 커널 9 칸에서 왔는지 같은 색으로 묶어 보여 주는 양방향 하이라이트.

이 메타포가 가지는 보존도:
- 슬라이딩 → 도장이 자리를 옮긴다.
- 가중치 공유 → **도장은 한 자루다**. 위치마다 새로 새기지 않는다.
- 곱·합 → 도장이 찍힐 때 9 개 잉크 점이 합쳐져 한 자국이 된다.
- receptive field → 한 도장 자국 아래에 있던 종이 영역.

---

## 3. 결정적 순간

합성곱 시각화의 결정적 순간은 **"한 번의 도장"** 이다 — 커널이 한 위치에 멈추고, 그 위치의 곱·합이 계산되어 출력 셀 하나가 빈칸에서 색칠된 칸으로 변하는 그 한 박자.

전 / 중 / 후 시각 상태:

- **전(찍기 직전)**: 커널 사각형은 다음 칸으로 점프해 멈춰 있다. 입력에서 그 9 칸이 옅은 외곽선으로 강조된다. 출력의 해당 칸은 점선 테두리만 있는 빈칸이다. 수식 패널에는 식 한 줄이 *문법은 갖춰져 있고 값은 비어 있는* 상태로 미리 적힌다 ("(?·w11)+(?·w12)+...").
- **중(도장이 내려앉는 한 박자)**: 9 개 곱셈 항이 동시에 점등된다 (병렬 시간). 입력 9 칸 ↔ 커널 9 칸 ↔ 수식 9 항이 각각 짝 색으로 짧게 펄스. 그 다음 합산 표시(시그마 또는 + 줄긋기)가 좌→우로 흐르며 부분합이 누적된다. bias 더하기, ReLU 적용이 마지막 두 줄에 추가된다.
- **후(자국이 남음)**: 출력 셀이 활성값 색으로 채워진다. 점선 테두리는 실선이 된다. 입력의 외곽선과 커널의 펄스는 사라지지만, **그 자국 색은 출력 셀에 영구히 남아** 호버 시 다시 트레이스를 깨울 수 있는 키가 된다.

강조 장치:
- 한 박자 안의 **병렬 점등** (9 항 동시) — 직렬 슬라이드만 보여 주는 평균을 넘어선다.
- 한 박자의 끝에 작은 **딸깍 모션** (커널 사각형이 살짝 눌렸다 떠오르는 1 px 의 위아래 움직임) — 도장 메타포의 햅틱적 강화.
- **카운트 업 부분합**: 수식 라인 우측에 부분합이 0 → 최종값으로 카운트 업되며, 같은 숫자가 출력 셀의 호버 툴팁에서 동일 폰트로 등장.

## 4. 다른 대상과의 구별점

가장 가까운 두 가지 — **2D 평균 풀링**, **이미지 블러(고정 커널 신호 처리)** — 와의 시각 차이.

- vs **2D 평균 풀링**:
  - 풀링은 일반적으로 stride = kernel size 라 출력이 커널을 따라 *겹치지 않는 타일*로 분해된다. 합성곱은 stride < kernel size 일 때 **타일이 겹친다** — 이 기획은 같은 입력 픽셀이 여러 출력에 빨려 들어가는 사실을 입력 호버 시 *여러 출력이 동시에 점등*하는 식으로 가시화한다 (양방향 트레이스).
  - 풀링은 곱셈 항이 없거나 (max) 모두 동일 가중 (avg) 이라 수식 패널의 계수가 비어 있거나 1/N 로 동일하다. 합성곱은 **9 개 계수가 서로 다르고 학습된 값**이라 수식 패널의 계수가 의미 있는 가중치로 표시된다. 커널 사각형 안 9 칸에 들어 있는 숫자/색이 풀링과의 결정적 시각 차이.

- vs **이미지 블러 (Setosa 류 신호 처리)**:
  - 이미지 블러는 단일 채널, 단일 커널, 학습 없음 — 커널 값은 사전에 손으로 정해진다. 합성곱은 **다중 입력 채널 합산**과 **다중 출력 채널 (필터 여러 개)** 이 본질이다. 이 기획은 첫 화면부터 RGB 3 채널 입력이 동시에 슬라이드하고 합쳐지는 모습을 보여 준다 — 단일 채널만 보여 주는 신호 처리 시각화와 정면에서 갈린다.
  - 블러의 출력은 항상 같은 채널 수. 합성곱은 출력 채널이 입력과 다를 수 있다 — 화면 우측에 **여러 출력 채널이 세로로 나열**되어 동시에 채워지는 것이 블러와 결정적으로 다르다.

- vs **풀-커넥티드 레이어**:
  - FC 는 위치 개념이 없고 모든 입력이 모든 출력에 연결된다. 합성곱은 **국소성**과 **가중치 공유**가 핵심 — 이 기획은 한 출력 셀이 입력의 정확히 k×k 영역만 보고, 그 가중치는 위치를 옮겨도 변하지 않는다는 사실을 두 가지 인터랙션으로 증명한다 (한 칸 호버 = 좁은 receptive field 가시화 / 커널 셀 값 변경 = 모든 출력이 동시 갱신).

## 5. 달성해야 할 시각 요소

다섯 항목. 서로 다른 측면을 다룬다.

### 5.1 색 동기화 트레이스 (양방향)

- 설명: 한 도장 행동에 고유 색을 할당. 그 색으로 입력 9 칸 / 커널 9 칸 / 수식 9 항 / 출력 1 칸을 묶는다. 출력 셀에 호버하면 그 셀의 색을 가진 receptive field 와 수식이 부활하고, 입력 셀에 호버하면 그 입력이 기여한 모든 출력 셀이 같은 점등 색으로 떠오른다.
- 구현 힌트: 색은 출력 셀의 (row, col) 인덱스를 HSL 의 hue 로 매핑(hue = (r·cols + c) / total · 360°). 채도/명도는 활성값에 비례. 호버 매칭은 출력 셀의 "기여 입력 좌표" 와 "기여 커널 좌표" 를 사전 계산해 둔 룩업 테이블로 즉시 점등.

### 5.2 한 박자 = 9 곱셈의 병렬 점등 + 수식 동기화

- 설명: 한 출력 셀이 만들어지는 한 박자 안에서, 9 곱셈 항이 *동시에* 점등되고 그 후 부분합이 좌→우로 누적된다. 옆 수식 패널에 식이 풀어쓰여 같은 박자에 펜 라이트가 항을 따라 흐른다. + bias, ReLU 두 줄도 같은 박자의 마지막에 추가된다.
- 구현 힌트: 박자 = 200~400 ms. 0~50 % 에 9 항 펄스, 50~85 % 에 부분합 누적 라인, 85~100 % 에 bias + ReLU. 사용자는 박자 길이 슬라이더(0.5x~3x) 와 "박자 단위 일시정지" 버튼으로 시간을 통제. 호버는 정지 상태에서도 트레이스만 다시 부른다.

### 5.3 다중 채널 동시 슬라이드 (입력 합산 + 출력 다중 필터)

- 설명: RGB 3 채널 입력을 첫 화면부터 *겹쳐진 3 장의 격자*로 보여 준다. 한 박자에 3 채널의 9 곱셈이 모두 점등되고, 채널별 부분합이 합쳐져 한 출력 셀로 떨어진다. 출력 측에는 N 개 필터가 세로로 줄지어 *동시에* 자기 출력 채널을 채워 간다 (폴리포니).
- 구현 힌트: 입력은 R/G/B 격자를 z-stacking 으로 약간 어긋나게 그려 깊이감 표현 (CSS transform: translateZ 또는 단순 오프셋). 한 박자의 수식 패널은 채널별 줄이 3 줄로 분리되고, 마지막에 sigma_c 합산 줄이 + 로 합친다. 필터 폴리포니는 우측에 N 개 출력 격자를 세로 stack — 한 박자에 N 개가 동시에 자기 색의 도장을 찍는다 (각 필터는 hue 가 다르다).

### 5.4 인터랙티브 파라미터 슬라이더 (스트라이드 / 패딩 / dilation / 커널 크기)

- 설명: 사용자가 stride, padding, dilation, kernel size 슬라이더를 움직이면 출력 격자 크기가 즉시 재계산되어 화면이 부드럽게 재배치된다. 출력 크기 산식 W' = (W − F + 2P)/S + 1 이 별도 줄에 표시되고 변수에 현재 값이 라이브로 들어간다. dilation 이 늘면 커널 사각형이 *구멍 뚫린 격자* 로 변형된다.
- 구현 힌트: 파라미터 변경 시 출력 격자 좌표 트랜지션을 200 ms 로 부드럽게. 산식의 각 변수는 별개 span 으로 두고 값 바뀔 때 짧은 글로우. dilation 의 구멍은 커널 사각형 내부에 비어 있는 격자 셀로 표현 — 그 셀은 입력의 "건너뛰는" 칸을 가리키며 입력 측에서도 흐릿하게 처리.

### 5.5 가중치 공유의 라이브 증명 (디테일 보너스)

- 설명: 사용자가 커널 격자의 한 셀을 클릭해 값을 +/- 0.1 로 바꾸면, **출력 격자의 모든 칸**이 같은 박자에 동시에 갱신되어 색이 바뀐다. 이는 "도장은 한 자루다 — 위치마다 새로 만들지 않는다" 의 인터랙션 증명이다.
- 구현 힌트: 커널 셀에 인라인 +/- 버튼 또는 스크롤 휠로 값 조절. 변경 직후 모든 출력 셀에 동시에 짧은 펄스 글로우(공유 색)를 깔고, 그 다음 새 활성값으로 색 보간. 풀-커넥티드 레이어와의 결정적 차이가 이 한 인터랙션에서 발생한다.

## 6. 시각 구성

중심 시각은 **세 격자의 트라이앵글**:

```
[입력 격자 (RGB 3 채널 z-stack)]   →   [출력 격자 N 채널 세로 stack]
                  \                 /
                   [커널 사각형 (현재 위치) + 작은 커널 보드 (원본)]
                                ↓
                         [수식 사이드 패널]
```

- **중심**: 입력 격자(왼쪽 큰 면적), 출력 격자들(오른쪽 세로 stack), 그 사이의 슬라이딩 커널 사각형. 커널 사각형은 입력 위에 떠 있고, 동시에 우측에 *별도의 작은 커널 보드*가 항상 보이며 9 개 가중치 값이 숫자로 적혀 있다 (가중치 공유의 시각적 닻).
- **보조 요소**:
  - 수식 사이드 패널 (오른쪽 또는 아래) — 9 곱셈 + 채널 합산 + bias + ReLU 의 5 줄짜리 식.
  - 출력 크기 산식 줄 — 파라미터 슬라이더 옆에 라이브 표시.
  - 파라미터 컨트롤 — stride, padding, dilation, kernel size 슬라이더 + 박자 속도 슬라이더 + 일시정지 / 재생 / 한 박자 전진.
  - 트레이스 컬러 레전드 — 호버 중인 출력 셀의 색이 현재 묶인 색이라는 표시.
- **요소 간 관계**:
  - 커널 사각형(입력 위 떠다님) ↔ 작은 커널 보드(원본): 항상 동일한 9 개 값. 어느 한쪽을 편집하면 양쪽 모두 즉시 반영 (가중치 공유의 라이브 증명).
  - 입력의 receptive field ↔ 커널 9 칸 ↔ 수식 9 항 ↔ 출력 셀: 색 동기화. 한쪽 호버가 나머지 모두를 점등.
  - 파라미터 슬라이더 ↔ 격자 크기: 즉시 재배치 + 산식 라이브.

```
│                              │  (세로 스크롤로 N 개)               │
├──────────────────────────────┴─────────────────────────────────────┤
│ [수식 사이드 패널 — 5 줄 식, 색 펜라이트가 항을 따라 흐름]        │
│   y[r,c] = sum_c sum_i sum_j x[c, r·s+i, c·s+j] · w[c, i, j] + b   │
│   (현재 박자의 풀어쓴 곱·합 / 채널 합 / + b / ReLU(·) / 결과)     │
├────────────────────────────────────────────────────────────────────┤
│ [파라미터 컨트롤 행]                                               │
│   stride [─●─]  padding [●──]  dilation [─●─]  kernel [3 / 5]      │
│   출력 크기 산식: W' = (W − F + 2P)/S + 1 = (7−3+2·1)/1+1 = 7      │
└────────────────────────────────────────────────────────────────────┘
```

- 좁은 화면 (모바일/태블릿) 에서는 입력→커널→출력→수식 순으로 세로 적층. 파라미터 컨트롤은 접이식 패널.
- 줌 / 패닝은 입력 격자 위에서 가능. 텐서 크기가 작으므로 (≤ 9×9) 기본 줌으로 충분.

## 8. 진행과 이벤트

### 8.1 단계

1. **초기 (정지)** — 입력 격자에 픽셀 값이 채색되어 있고, 출력 격자는 모든 칸이 점선 테두리. 커널 사각형은 좌상단 첫 위치에 멈춰 있다. 수식 패널은 첫 위치의 빈 식 (값 미정).
2. **첫 박자 시작** — 사용자가 "재생" 또는 "한 박자 전진" 을 누름. 커널 사각형이 좌상단에서 9 항 병렬 점등. 부분합 누적. + bias, ReLU. 출력 (0,0) 셀이 활성값 색으로 채워진다.
3. **연속 박자** — 커널이 stride 만큼 우측으로 점프. 행 끝에 닿으면 다음 행 좌측으로. 한 박자 = 한 출력 셀.
4. **종료** — 모든 출력 셀이 채워지면 커널 사각형은 마지막 위치에 머문다. 사용자는 임의 출력 셀에 호버해 트레이스 부활 가능.

다중 필터 시 단계 3 에서 모든 N 개 필터의 출력이 같은 박자에 동시에 한 칸씩 채워진다 (필터별 색 hue 다름).

### 8.2 단계별 시각 변화

- 박자 시작 → 9 항 병렬 펄스 (50 ms 안에).
- 박자 중간 → 부분합 누적 라인이 좌→우로 흐름 (150 ms).
- 박자 끝 → 출력 셀 색 채움 + 1 px 도장 모션.
- 박자 사이 → 커널 사각형이 다음 위치로 부드럽게 이동 (50~100 ms 트랜지션).

### 8.3 사용자 제어 지점

- **재생 / 일시정지** — 자동 재생 토글.
- **한 박자 전진 / 후퇴** — 스페이스 / 화살표 키.
- **박자 속도 슬라이더** — 0.25x ~ 4x.
- **출력 셀 호버** — 그 셀이 만들어진 박자의 트레이스 부활 (정지 상태에서도).
- **입력 셀 호버** — 그 입력이 기여한 모든 출력 셀 동시 점등 (양방향 트레이스).
- **커널 셀 클릭/스크롤** — 가중치 변경 → 모든 출력 동시 갱신 (라이브 증명).
- **파라미터 슬라이더** — stride / padding / dilation / kernel size 변경 → 출력 크기 재배치 + 산식 라이브.
- **필터 추가/제거** — 출력 채널 수 변경.

## 9. 초기 상태와 파라미터

- **입력**: 7×7 RGB 이미지 (3 채널). 기본 콘텐츠는 단순한 사선 / 모서리 패턴 — 학습된 엣지 감지 커널이 의미 있는 활성을 만들도록.
- **커널 (필터)**: 기본 3×3, 3 채널 (입력 채널 수와 일치). 기본 가중치는 학습된 엣지 감지 커널 (예: 수직 엣지). 사용자 편집 가능.
- **출력**: stride=1, padding=1 일 때 7×7 (same conv). 기본 출력 채널 수 N = 3 (수직 엣지 / 수평 엣지 / 대각 엣지).
- **bias**: 기본 0. 슬라이더로 조절 가능.
- **활성 함수**: ReLU 기본 적용. 토글로 끌 수 있음.
- **박자 속도**: 1x = 250 ms / 박자.
- **파라미터 기본값**: stride=1, padding=1, dilation=1, kernel size=3.

## 10. 레퍼런스 대비 차별점

표준 레퍼런스(CNN Explainer, ezyang, vdumoulin, Setosa, Deeplizard, CS231n 등) 대비 이 기획이 *공통 요소를 어떻게 더 잘 다루는지* 와 *표준이 놓치는 어떤 측면을 추가하는지*.

### 10.1 공통 요소를 더 잘 다룬다

- **색 매칭**: 표준은 호버 시점에만 한두 색이 잠깐 묶인다. 이 기획은 **출력 셀마다 영구 고유색** 을 부여해 출력 격자 자체가 색의 분포로 receptive field 의 공간 패턴을 한눈에 보여 준다 (출력 (0,0) 의 도장 자국은 나중에 호버해도 그 색이 다시 살아난다).
- **수식 사이드 패널**: 표준은 수식 한 줄을 보여 주거나 (CNN Explainer) 풀어쓰지 않는다 (ezyang). 이 기획은 **5 줄 풀어쓰기** — 9 항 곱·합 / 채널 합 / + bias / ReLU / 결과 — 를 박자에 동기화해 표시한다. + bias 와 ReLU 는 평균이 누락한다.
- **이산 스텝**: 평균은 이산 점프를 보여 주지만 한 박자 안의 9 곱셈이 *직렬*로 흐르거나 (3Blue1Brown) 아예 압축된다 (vdumoulin). 이 기획은 한 박자 안에서 **9 항 병렬 점등 + 부분합 누적** 으로 미시 시간을 분해한다.
- **호버 시간 통제**: 표준은 호버를 "그 위치에서 멈춤" 정도로 쓴다 (Setosa, ezyang). 이 기획은 호버를 **양방향**으로 — 출력→입력 및 입력→출력 — 사용하고, 자동 재생과 호버 트레이스를 *공존*시킨다 (재생 중에도 호버하면 트레이스가 부활).

### 10.2 표준이 놓치는 측면을 추가한다

- **다중 채널을 첫 화면에 일급 시민으로**: CNN Explainer 만이 다중 입력 채널을 정면으로 보여 주지만 출력 채널 다수성은 약하다. 이 기획은 RGB 입력 z-stack 과 N 개 출력 채널 폴리포니를 **기본 화면**에 둔다.
- **bias 와 ReLU 가시화**: 평균이 모두 누락한 + b 와 ReLU(·) 줄을 수식 패널 마지막에 항상 보여 준다.
- **파라미터의 라이브 산식**: vdumoulin 은 16 종 GIF 정적 모음, ezyang 만이 인터랙티브. 이 기획은 W' = (W − F + 2P)/S + 1 을 **변수 단위로 라이브 갱신** 하는 한 줄로 통합한다.
- **가중치 공유의 라이브 증명**: 평균이 거의 누락한 본질을 인터랙션으로 증명한다 (커널 셀 값 변경 → 모든 출력 동시 갱신). 디테일 한 점이지만 합성곱과 풀-커넥티드의 결정적 차이를 가르치는 가장 짧은 시연이다.
- **양방향 트레이스의 기본화**: ezyang 만 시도한 양방향을 기본 인터랙션으로 격상.

### 10.3 학습자가 "표준 도구의 그 시각화보다 낫다" 고 느낄 부분

- 한 박자 안에서 9 곱셈이 *동시에* 점등되는 시각 — "직렬로 하나씩 더해지는 줄 알았는데 사실 한 번에 일어나는구나" 의 깨달음.
- 출력 격자의 색 분포가 곧 receptive field 위치 분포라는 한눈 인식 — 호버 없이도 공간 구조가 읽힌다.
- 커널 셀 하나의 값을 바꿨더니 모든 출력이 동시에 흔들리는 인터랙션 — "가중치 공유" 라는 단어가 머리가 아니라 손끝으로 들어온다.
- 출력 크기 산식이 슬라이더 옆에서 라이브로 갱신되는 한 줄 — vdumoulin 의 16 종 GIF 가 한 인터랙션으로 압축된다.
- 다중 필터가 폴리포니로 동시에 도는 우측 stack — "필터 하나당 출력 채널 하나" 가 시간으로 겹쳐 들린다.

## 부록: 기획 메모

### 왜 도장 + 형광펜의 결합인가

도장만으로는 양방향 트레이스의 "역방향 추적" 이 자연스럽지 않다 (도장은 누른 쪽에서 종이로의 일방향). 형광펜만으로는 가중치 공유가 약하다 (형광펜은 아무 곳이나 칠할 수 있어 보임). 둘을 합치면 *행위*는 도장이 (한 자루로 여러 자국), *추적*은 형광펜이 (출력↔입력 양방향) 담당해 본질 7 개 (슬라이딩 / 곱·합 / 가중치 공유 / 다중 채널 합산 / 다중 필터 / stride·padding / receptive field) 가 모두 보존된다.

### 왜 출력 셀 영구 고유색인가

평균이 호버 시 일시 색을 쓰는 이유는 색 충돌을 피하기 위해서다. 7×7 출력이면 49 색이 필요한데, hue 만으로 49 단계는 너무 좁다. 그러나 receptive field 는 공간적으로 *연속*하므로 인접 출력 셀의 색이 비슷해도 무방하고, 오히려 그것이 공간 연속성을 가르친다. HSL hue 를 좌→우로 360° 회전시키면 49 칸이 무지개로 펼쳐져 "출력 격자 = 색의 격자" 라는 별도 정보를 얹는다.

### 가중치 공유 라이브 증명을 5 번째 요소로 둔 이유

"공통 강화" 의 본분은 표준 디테일이지만, 가중치 공유 증명은 *표준 도구 모두가 누락한 본질 증명* 이다. 인터랙션이 한 줄짜리라 (커널 셀 값 변경 → 모든 출력 갱신) 디테일 가산점으로 추가했다. 이게 메인 메타포가 되면 2B (새로운 방식) 영역으로 넘어가지만, 보조 인터랙션으로는 "공통 강화" 안에서 추가 가능하다.

### 무엇을 일부러 빼는가

- **3D 카드 더미** (Adam Harley, TensorSpace) — 한 합성곱의 미시 단계를 흐리므로 첫 화면에서는 뺀다. 다층 시각화는 별도 facet 의 영역.
- **im2col 행렬곱 토글** — 매력적이지만 새로운 표현을 격자와 *동시* 에 보여 주려면 화면 분할이 추가로 필요하다. 2B 영역에 양보.
- **forward / backward 토글** — 학습 직관에 결정적이지만 합성곱 *연산* 시각화의 본분과 어긋난다. 별도 facet (역전파) 의 영역.
- **feature visualization (학습된 환각 이미지)** — 결과물 미학이라 메커니즘 시각화와 별개.

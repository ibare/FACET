## 1. 시그니처 행동

**같은 연료 한 통을 일곱 클래스 우주선에 똑같이 나눠 주었을 때, 각각이 어디까지 가서 멈추는지를 한 줄 위에 줄 세워 보여 주는 것** — Big-O 는 "시간이 얼마나 걸리는가" 가 아니라 "주어진 자원으로 어디까지 갈 수 있는가" 의 운명이다.

## 2. 시각적 은유

### 메타포

**연료 한 통과 일곱 척의 우주선**.

- 시간을 가로축으로 두는 표준 곡선 그래프와 정반대 방향. **시간(연료)을 자원으로 고정**하고, 각 클래스가 그 연료로 처리해 낸 n 을 **우주선이 도달한 거리** 로 본다.
- 일곱 클래스(O(1), O(log n), O(n), O(n log n), O(n^2), O(2^n), O(n!)) 는 각각 별 모양의 우주선 한 척. 같은 연료를 받고 출발한다.
- O(1) 우주선은 연료를 거의 안 쓰고 즉시 도착해 화면 끝(또는 "도달 한계 표시기" 끝)에 박힌다 — "n 이 아무리 커도 같은 자리".
- O(log n) 우주선은 연료를 거의 안 쓰지만 도달은 천천히 — 같은 연료로 매우 멀리 간다.
- O(n^2) 우주선은 중간에서 헐떡이며 멈춘다 — "여기까지가 한계".
- O(2^n) / O(n!) 우주선은 출발하자마자 멈춘다 — 연료 한 통으로 갈 수 있는 거리가 화면 1픽셀 미만. 우주선 옆에 작은 **체스판 아이콘** 이 떠 있어 "이 우주선의 연료는 체스판 후반부 칸 한 칸도 못 건넌다" 라는 보조 메타포를 띄운다.

### 출처와 본질 적합성

- 출처: 우주여행 + 연료 소진 메타포(레퍼런스 4.2 의 "마라톤/우주여행", "연료 소진" 공백을 결합).
- 본질 적합성:
  - "운명" — 같은 연료 → 도달 거리의 차이 = 성장률의 차이.
  - "점근 상한" — "더 멀리 가지는 못한다" 의 한계점이 우주선의 정지 위치로 자연 표현.
  - "상수항 무시" — 연료 한 통에서 시작 시 잠깐 더 쓴다고 도달 거리가 의미 있게 변하지 않음을 시각적으로 보임 (시작점 옆 작은 흔들림은 무시되고, 멀리서 본 위치만 살아남음).
  - "같은 클래스 = 같은 부류" — 같은 별 모양 = 같은 색 = 같은 정류장.

### 임베드 폭(640~720px) 검증

- 일곱 우주선이 한 가로 줄 위에 있는 1D 트랙. 좌→우 한 줄이라 폭만 있으면 충분, 좌우 분할 4~6 패널이 아님.
- 각 우주선의 라벨(O(1), O(log n) 등)은 우주선 바로 옆 또는 아래 작은 칩으로 — 가로 폭 압박 없음.
- 체스판 보조 아이콘은 우주선 옆 16px 정도의 작은 배지. 폭 비용 거의 없음.
- 도달 거리가 너무 짧은 우주선(O(2^n), O(n!))은 시작점 부근에 밀집 — 이 부분만 우측 상단 "확대경" 패널(120~150px)로 확대해 보여 주는 보조 영역 마련. 이 보조 영역까지 해서 가로 720px 안에 충분히 수용.

## 3. 결정적 순간

**"n 슬라이더를 키우는 도중, 우주선들이 차례로 떨어져 나가는 순간"**.

- 슬라이더 초기 위치(n 작음): 일곱 우주선이 모두 화면 안 어딘가에 있다. O(n^2)·O(2^n)·O(n!) 도 "조금 짧지만 살아 있다". 순서가 점근과 어긋날 수도 있다(상수의 영향).
- n 을 키우는 중: O(n!) → O(2^n) → O(n^2) 순서로 우주선이 **시작선 근처로 빨려 들어가** 더 이상 의미 있는 거리를 못 간다. 화면 좌측 끝에 "탈락" 표시(우주선이 회색 처리되고 옆에 작은 십자가 또는 "포기" 라벨)가 뜬다.
- n 이 충분히 커진 끝: O(1) / O(log n) / O(n) / O(n log n) 만 살아남아 분명히 다른 위치에 있다. O(1) 과 O(log n) 은 거의 붙어 있고, O(n) 과 O(n log n) 사이가 살짝 벌어지고, O(n^2) 이상은 시작선에 박혀 있다.

### 강조 장치

- **탈락 애니메이션**: 우주선 별이 회색이 되며 살짝 떨고, 옆에 "이 연료로는 더 이상 의미 있는 n 을 못 간다" 라는 마이크로 툴팁이 1~2초 떠올랐다 사라진다.
- **트랙 위 점근 라벨**: 트랙 우측 끝에 "더 멀리 → 더 큰 입력 처리" 라는 단조성 라벨이 항상 붙어 있어 "위/오른쪽 = 더 좋다" 가 시각적으로 박힌다.
- **컷오프 점선**: 화면 안에서 의미 있게 살아남는 한계선(예: 트랙의 1/8 지점)을 점선으로 표시. 이 선보다 왼쪽으로 떨어진 우주선은 "사실상 풀 수 없는 영역" 이라는 영역으로 들어가고, 영역 자체가 옅은 붉은 톤으로 깔린다.

## 4. 다른 대상과의 구별점

가장 가까운 비교 대상:

### A. 일반 자료구조 / 알고리즘 시각화 (예: 정렬 경주, 트리 탐색)

- 정렬 경주(R10)는 "같은 데이터 위에서 다른 알고리즘이 달린다" — 본 기획은 **알고리즘이 아니라 클래스 자체** 가 우주선이다. 알고리즘 동작은 보이지 않고 운명만 본다.
- 본 기획의 우주선에는 비교/스왑 같은 단계 카운터가 없다. 단계 자체를 추상화한 "연료 소비량" 만 본다.
- 정렬 경주는 결승선이 같고 시간이 다르다. 본 기획은 **연료(시간)가 같고 도달점이 다르다**. 좌표축이 정확히 뒤집혀 있다.

### B. 표준 Big-O 곡선 그래프 (R2/R3/R4)

- 표준은 x=n / y=연산수의 2D 평면 위 곡선. 본 기획은 **1D 트랙 + 우주선 위치**.
- 표준은 "y축이 화면 밖으로 튀어 나가는" 장면이 폭발의 표현. 본 기획은 **"우주선이 시작선에 박혀 더 안 움직이는" 장면** 이 폭발의 표현. 같은 사실의 좌표 변환.
- 표준은 수학적 정량을 본다. 본 기획은 **"같은 연료로 어디까지 갈 수 있는가" 의 운명** 을 본다.

### 새 메타포가 다른 대상에 적용되었을 때

- 만약 같은 메타포를 "정렬 알고리즘 비교" 에 쓴다면? 정렬 알고리즘마다 우주선이 하나씩 있고, 각자 같은 연료를 받아 정렬을 끝낸 데이터 양이 도달 거리가 된다. 이 경우 quicksort/mergesort 의 우주선이 거의 같은 위치에 박히고, bubble sort 우주선만 한참 뒤에 있게 된다 — **클래스가 같으면 같은 별, 다른 클래스면 명확히 다른 별** 의 시각이 자연 발생한다. 본 기획은 알고리즘 단위가 아니라 클래스 단위라는 점에서 이보다 추상의 한 층 위에 있다.

## 5. 달성해야 할 시각 요소

### 요소 1. 가로 트랙과 일곱 우주선

- **설명**: 화면 가로로 1D 트랙 한 줄. 트랙 위 일곱 별 모양 우주선이 각자 클래스를 대표.
- **구현 힌트**: SVG `<g>` 로 트랙 라인 한 줄 + 별 SVG 7개. 별 색은 클래스별 고정(O(1) 시안, O(log n) 청록, O(n) 녹, O(n log n) 라임, O(n^2) 주황, O(2^n) 빨강, O(n!) 자주). 별 위치는 단일 수식: `x = trackLeft + (trackWidth) * (1 - normalize(cost(n)))` 로 잡되, 비용이 너무 커서 거의 0 에 수렴하는 클래스는 좌측 끝 클램프 + 살짝 흔들림.

### 요소 2. n 슬라이더와 도달 거리 변환

- **설명**: 트랙 아래 n 슬라이더(2 ~ 10^6, 로그 스케일). 슬라이더 변경 시 우주선이 각자의 식에 따라 위치 갱신.
- **구현 힌트**: 슬라이더 값 n 에 대해 클래스별 비용 함수 `cost_c(n)` 을 평가, 같은 연료 예산 B(예: 10^9) 를 두고 `reach_c(n) = B / cost_c(n)` 을 도달 거리로 본다. 도달 거리는 다시 트랙 폭에 매핑하기 위해 `log(reach_c)` 로 정규화 (도달 거리 자체가 천문학적이므로 로그 보정 필수). 정규화는 매 프레임 7개 클래스의 max/min 으로 동적 정규화하면 슬라이더 끝에서도 모든 별이 살아 있어 보이는 함정에 빠지므로, **고정 기준** (예: O(1) 의 도달 거리 = 트랙 우측 끝, O(n!) 의 도달 거리 = 트랙 좌측 끝) 을 쓴다.

### 요소 3. 탈락(컷오프) 시각화

- **설명**: n 이 커지면서 의미 있는 도달을 못 하는 우주선이 회색으로 변하고 시작선에 박히며, 트랙 좌측에 "탈락 영역" 이 옅은 붉은 톤으로 칠해진다.
- **구현 힌트**: 도달 거리가 트랙 폭의 1/(8 또는 16) 미만으로 떨어지면 탈락 상태 진입. 별의 fill 을 채도 0(회색)으로 보간, opacity 0.5, 옆에 작은 십자가 SVG. 탈락은 영구가 아니라 슬라이더가 작아지면 부활 — 학습자가 양방향으로 운명을 만질 수 있게.

### 요소 4. 시각 옆 짧은 설명 영역

- **설명**: 트랙 위쪽에 2~4문장 짜리 개념 카드. 학습자가 Big-O 를 모를 수 있다는 가정 하에, 시각화 자체가 자기소개를 한다.
- **구현 힌트**: HTML `<aside>` 또는 SVG `<foreignObject>` 로 트랙 위 60~80px 영역. 텍스트는 보고서 6절의 "대상 개념 요약" 에서 발췌. 폰트는 본문보다 한 단계 작게(13~14px) 하여 시각이 주인공임을 유지.

### 요소 5. 점근 vs 작은 n 토글 (선택적이나 강력)

- **설명**: 슬라이더 옆 토글로 "상수 포함 모드" / "점근 모드" 전환. 상수 포함 모드에서는 작은 n 영역에서 O(n^2) 우주선이 일시적으로 O(n log n) 보다 더 멀리 가 있는 모습을 보여 주고, n 을 키우면 두 우주선의 위치가 교차하며 운명이 뒤집히는 순간을 강조.
- **구현 힌트**: 상수 포함 모드의 비용 함수는 `100 * n log n` vs `1 * n^2` 같은 상수 차이 큰 식. 토글 전환 시 우주선 위치 부드럽게 보간(0.4s ease). 교차 시점의 n 값 부근에서 두 우주선 사이에 점선 + "이 지점부터 운명이 갈린다" 캡션 표시.

## 6. 시각 구성

- **중심 시각**: 가로 트랙 1줄과 그 위의 별 7개 (트랙 폭 580~640px, 높이 80~100px).
- **보조 요소**:
  - 좌측 트랙 끝 부근의 "사실상 풀 수 없음" 영역(옅은 붉은 그라디언트).
  - 우측 트랙 끝 부근의 "한 줌의 연료로 도달" 영역(옅은 녹색).
  - 트랙 아래 n 슬라이더 + 현재 n 값 텍스트.
  - 슬라이더 우측 작은 토글: "상수 포함" / "점근만".
  - 트랙 위쪽 개념 설명 카드.
  - 트랙 좌측 끝(O(2^n)/O(n!) 밀집 영역) 위에 떠 있는 작은 확대경 패널(120~150px) — 거기서만 두 별이 분리되어 보인다.
  - 우측 하단에 "[보조] 곡선 그래프 보기" 토글 — 클릭 시에만 표준 곡선이 작은 인셋(180~220px)으로 펼쳐진다. 기본 닫힘.
- **요소 간 관계**:
  - 슬라이더는 일곱 별의 위치를 동시에 결정한다.
  - 토글은 별의 비용 함수만 바꾼다 (트랙 자체는 안 바뀜).
  - 확대경은 트랙 좌측 끝의 부분을 거울처럼 비춘다.
  - 곡선 인셋은 어디까지나 검증용 보조. 닫혀 있을 때가 기본.

## 8. 진행과 이벤트

이 시각화는 algorithm 재생이 본질이 아니다(분류 보고서 6절 명시). algorithm.ts 의 emit 흐름은 단순하고, 사용자 입력 중심.

### 주요 진행 단계

1. **초기 마운트**: n=8(또는 작은 값)로 일곱 별이 트랙에 분포. 모든 별이 살아 있다(탈락 없음). 컷오프 영역 비활성.
2. **슬라이더 드래그(키 이벤트)**: n 변경 → 일곱 별 위치 보간 갱신 → 도달 거리가 임계 이하인 별 탈락 처리.
3. **상수 토글**: 비용 함수 교체 → 별 위치 보간 갱신 → 교차 발생 시 캡션 1.5초 노출.
4. **곡선 보조 토글**: 인셋 펼침. 표준 라인 차트가 작게 표시되어 학습자가 본 메타포의 결과를 검증하도록 한다.

### 단계별 시각 변화

- 슬라이더 드래그 시 우주선 별의 이동은 0.25s ease — 너무 빠르면 운명의 느낌이 없고, 너무 느리면 인터랙션이 답답.
- 탈락 진입 시 별 흔들림(2px amplitude, 0.4s)이 1회 발생하며 회색으로 페이드.
- 부활(슬라이더 작아질 때)은 회색 → 본래 색으로 0.4s 페이드.

### 사용자 제어 지점

- **n 슬라이더** (2 ~ 10^6, 로그 눈금).
- **상수 포함 / 점근만 토글**.
- **곡선 보조 토글**.
- **별 클릭** — 클릭 시 그 클래스의 비용식·연료 소비량·현재 n 에서의 도달 거리(천문학적 수치는 "우주 나이의 X배" 같은 인간 단위로도 환산) 가 트랙 위에 작은 카드로 1회 표시.

## 9. 초기 상태와 파라미터

- **초기 n**: 16 — 작은 n 영역에서 운명이 아직 뚜렷하지 않은 구간.
- **연료 예산 B**: 10^9 (개념적 상수, 화면 노출은 "연료 한 통").
- **트랙 정규화 기준**: O(1) 의 도달 거리 = 트랙 우측 끝의 95%, O(n!) 의 도달 거리(at n=20 기준) = 트랙 좌측 끝의 0% — 고정 기준.
- **클래스 색**:
  - O(1): 시안 (#22d3ee)
  - O(log n): 청록 (#14b8a6)
  - O(n): 녹 (#22c55e)
  - O(n log n): 라임 (#84cc16)
  - O(n^2): 주황 (#f97316)
  - O(2^n): 빨강 (#ef4444)
  - O(n!): 자주 (#a855f7)
- **컷오프 임계**: 도달 거리 < 트랙 폭의 1/16 → 탈락.
- **상수 토글 기본값**: "점근만".
- **곡선 인셋 기본값**: 닫힘.

## 10. 레퍼런스 대비 가치 (B타입)

표준 곡선 그래프(R2/R3/R4)가 주지 못하는 통찰:

1. **"운명" 의 직관**: 표준 곡선은 "y축이 위로 가는 것" 을 보여 준다. 이는 학습자에게 "결과의 정량 비교" 만 남기고 "왜 어떤 알고리즘은 사실상 풀 수 없는지" 의 정서를 주지 않는다. 본 기획은 "같은 연료로 어디까지 갈 수 있는가" 의 형태로 학습자에게 **유한한 자원의 무게** 를 체감시킨다.
2. **컷오프의 시각화**: 표준 곡선에서 O(2^n) 곡선은 "위로 튀어 나가서 화면 밖" 이지만, 학습자에게 그건 그냥 "큰 수" 다. 본 기획에서 O(2^n) 우주선은 **시작선에 박혀 회색이 되어 사라진다** — 큰 수가 아니라 "갈 수 없음" 으로 표현된다. 사실상 풀 수 없음의 실감이 다르다.
3. **상수 무시의 살아 있는 데모**: 표준 곡선은 상수항을 정의로만 다룬다. 본 기획의 토글은 "작은 n 에서는 O(n^2) 가 O(n log n) 보다 빨랐다" 를 직접 보여 주고, n 을 키우면 운명이 뒤집히는 순간까지 학습자 손에 쥐여 준다. 분류 보고서 학습 질문 4 에 직격.
4. **클래스의 카탈로그성**: 표준 색띠 표(R2)가 주는 "줄 세우기" 는 본 기획에서 **트랙 위 별의 위치** 자체로 자연 발생한다. 표를 외울 필요 없이 시각이 그것이다.
5. **"보지 말 것" 의 의도된 부재**: 곡선은 옵션이고 기본은 닫혀 있다. 학습자가 표준 시각 언어로 "검증" 하고 싶을 때만 펼쳐 본다. 학습자의 머리에 "곡선이 아닌 무대" 가 먼저 박힌다.

## 11. 본질 보존 검증 (B타입 전용)

분류 보고서 5.2 의 "절대 빼면 안 될 본질" 별 검증:

### 본질 a. 격차가 폭발하는 순간의 체감 (수가 아니라 시각 변화로)

- **새 표현에서의 보존**: n 슬라이더를 키우는 도중 좌측에서 우주선들이 차례로 회색이 되며 떨어지는 시각. 위치 차이가 아니라 "탈락" 자체가 격차의 표현.
- **위험**: 별 색이 너무 비슷하면 어느 별이 떨어졌는지 식별 어려움. **방지**: 클래스별 색을 7색 명도/색상 모두 충분히 분리하고, 탈락 시 라벨을 강제 노출.

### 본질 b. 상수·저차항 무시

- **새 표현에서의 보존**: "상수 포함 / 점근만" 토글로 직접 토대를 흔들어 본다. 토글 시 별 위치가 부드럽게 이동하지만, 큰 n 으로 가면 두 모드의 위치가 거의 같아진다 → "멀리서 보면 상수는 사라진다".
- **위험**: 토글이 본 기획의 보조이므로 학습자가 못 보고 지나칠 수 있음. **방지**: 토글 옆에 작은 안내 텍스트 "작은 n 에서는 운명이 뒤집힐 수도" 상시 표시.

### 본질 c. 클래스 카탈로그 (7개 모두 노출)

- **새 표현에서의 보존**: 일곱 별 모두 트랙 위에 항상 존재. 탈락한 별도 시작선에 회색으로 남는다.
- **위험**: 좌측 끝에 O(2^n) 과 O(n!) 이 완전히 겹쳐 두 별이 하나로 보임. **방지**: 확대경 인셋(120~150px)으로 좌측 끝을 별도 확대해 두 별의 분리 식별 보장.

### 본질 d. n 을 키우는 인터랙션

- **새 표현에서의 보존**: 슬라이더가 시각의 중심 컨트롤. 슬라이더 없이는 운명이 정적인 별 7개일 뿐.
- **위험**: 슬라이더 범위가 너무 좁으면 격차가 안 살아남. **방지**: 로그 스케일로 2 ~ 10^6 까지 6 자릿수 범위 보장.

### 본질 e. 점근 상한의 단조성 ("위 = 더 나쁨")

- **새 표현에서의 보존**: 트랙 위 위치가 "오른쪽 = 더 멀리 = 더 좋음 = 더 작은 클래스" 로 단조 정렬. 트랙 우측 끝에 "더 멀리 → 더 큰 입력 처리" 라벨 상시 노출. 표준 곡선의 "위쪽 = 나쁨" 단조성을 좌우 단조성으로 변환.
- **위험**: 학습자가 트랙을 "y축 없는 그래프" 로 오해할 수 있음. **방지**: 트랙은 의도적으로 1D 트랙(연료 게이지처럼 보이게)으로 디자인. 그래프와 시각 언어 분리.

### 메타포가 본질을 왜곡하지 않는가

- "연료 = 시간" 의 환원이 정확한가? — 시간복잡도의 정의는 "단계 수" 이고, 본 기획은 "연료 단위 = 한 단계" 로 본다. R1 samwho.dev 의 "터치 횟수" 와 같은 추상화 — 학문적으로 정합.
- "도달 거리 = 처리한 n" 의 환원이 정확한가? — `reach = B / cost(n)` 이 아니라, "고정 시간 예산 B 안에서 처리 가능한 최대 n" 으로 보면 `cost(n_max) = B → n_max = cost^{-1}(B)`. 이 변환은 클래스의 성장률 단조성과 정확히 1:1 — 빠른 성장률 = 작은 n_max = 짧은 도달 거리. 정합.
- 표면적 다름이 아닌가? — 좌표축 변환 + 메타포 환원 + 컷오프라는 새 시각 언어가 동시에 들어가 있다. "곡선의 모양만 바꾼 것" 이 아님.

## 12. 글 흐름 안에서의 위치

이 시각화는 Big-O 를 처음 만나는 학습자에게 **"왜 이걸 알아야 하는가"** 를 한 번에 가르치는 머리말 위치에 들어간다. 정의(점근 표기, 상한)나 수식이 등장하기 전에, 시각이 먼저 학습자의 직관에 충격을 준다. 글의 다른 부분은 이 시각화가 만든 직관 위에 정의를 쌓는다.

### 시각화에 포함될 짧은 설명 텍스트

> Big-O 는 시계가 아니라 운명을 말한다. 입력이 커질 때 너의 알고리즘이 어디로 가는지를 본다.
>
> 같은 연료 한 통을 일곱 척의 우주선에 똑같이 나눠 주었다. 별 모양 우주선마다 다른 클래스 — O(1), O(log n), O(n), O(n log n), O(n^2), O(2^n), O(n!). 같은 연료로 어디까지 가는지가 그 클래스의 운명이다.
>
> n 슬라이더를 키워 보라. 어떤 우주선이 먼저 떨어지는가.

### 새 메타포 안내

> 표준 Big-O 그래프는 "시간을 가로축, 연산수를 세로축" 으로 둔다. 여기서는 그 좌표를 뒤집었다. **시간(연료)을 똑같이 나눠 주고, 도달 거리(처리한 n)를 본다.** 곡선 그래프가 보고 싶다면 우측 하단 토글로 펼칠 수 있다. 단, 그건 검증용이고 무대는 트랙이다.

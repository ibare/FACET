# 2D 행렬 변환 시각화 기획서

### 0. 기본 정보

- 대상 이름: 2D 행렬 변환 (2D matrix transformation)
- 도메인: 컴퓨터 그래픽 (graphics)
- type: transformation
- 진행 모델: 입력 반응형
- 본질 한 문장: 2D 행렬 변환은 평면을 평면으로 보내는 선형 사상이며, 그 정체는 두 기저 벡터 i-hat·j-hat 의 도착지로 완전히 결정되어 평면의 모든 점이 같은 (u, v) 계수로 새 기저의 결합을 따라가는 운동이다.

### 1. 시그니처 행동

행렬의 한 셀을 잡고 흔들면 정확히 한 기저 벡터의 한 좌표만 변하고, 격자 전체와 그 위의 한 점이 같은 운동 법칙으로 즉시 따라 휜다.

### 2. 시각적 은유

표준 메타포인 "고무판 격자 + 두 색의 기저 화살표 + 단위 정사각형 → 평행사변형" 을 채택하고, 그 위에 두 가지 정교화를 더한다. 첫째, 격자는 한 겹이 아니라 두 겹으로 그려진다 — 항등 상태의 옅은 회색 격자가 배경에 항상 깔리고, 그 위에 변환된 진한 격자가 겹쳐 보여 "어디에서 어디로" 가 한 컷에 동시에 읽힌다. 둘째, 행렬 셀과 기저 벡터의 색이 1:1 로 묶인다 — 첫째 열 [a; c] 의 두 셀은 i-hat 화살표 색(청록), 둘째 열 [b; d] 의 두 셀은 j-hat 화살표 색(주황) 으로 칠해져, 셀 한 칸을 만지면 그 색의 화살표 끝이 정확히 한 좌표만 변한다는 미시적 동기가 색의 동일성으로 강제된다. 화면 중앙에는 큰 정사각형 격자 평면이 놓이고, 평면 위에는 i-hat·j-hat 두 화살표·단위 평행사변형(두 화살표가 만드는 면)·사용자가 찍어 둔 보조 점 한 개가 함께 떠 있다. 평면 우측에는 2×2 행렬 셀 패널이, 그 아래에는 표준 4종 프리셋 토글(회전·스케일·전단·반사) 과 |det| 게이지가 배치된다. 행렬 셀은 추상 숫자가 아니라 두 기저 벡터의 좌표를 그대로 적은 두 열로 직접 발화한다.

### 3. 결정적 순간

학습자가 셀 a 한 칸을 잡고 슬라이더로 흔드는 그 한 컷이다. 셀 a 의 숫자가 실시간으로 변하는 동안, 평면 위 i-hat 화살표의 끝점이 가로 방향으로만 미끄러진다 — j-hat 은 그 자리에 그대로 있고, 격자는 i-hat 의 운동에 맞춰 가로로 늘어나거나 비스듬해지면서 직선성과 등간격을 유지한 채 휜다. 보조 점 (u=1.5, v=0.7) 은 같은 (u, v) 계수로 새 기저의 결합 위치 u·i-hat' + v·j-hat' 으로 즉시 옮겨 가고, 점 옆의 라벨 "(1.5, 0.7)" 은 변하지 않는다 — 좌표는 같고 자리만 따라간다. 단위 평행사변형의 면적이 동시에 갱신되며 |det| 게이지가 새 값으로 미끄러진다. 학습자는 이 한 컷에서 "한 셀 = 한 좌표", "행렬의 두 열 = 두 기저의 도착지", "평면의 모든 점은 같은 계수로 따라간다" 세 사실을 동시에 받아들인다.

- 전: 평면이 항등 상태. 옅은 격자 한 겹과 진한 격자 한 겹이 완전히 겹쳐 한 겹처럼 보인다. i-hat 은 (1, 0), j-hat 은 (0, 1) 자리에 떠 있다. 단위 평행사변형은 한 변 1 의 정사각형. 보조 점은 (1.5, 0.7) 자리에 있고 라벨이 그대로. 행렬 셀은 [[1, 0], [0, 1]] 항등. |det| 게이지는 1.00.
- 중: 사용자가 셀 a 를 1 → 2 로 슬라이더로 끄는 동안, i-hat 화살표의 끝이 (1, 0) → (2, 0) 으로 가로로만 미끄러지고 j-hat 은 그대로다. 진한 격자가 가로 방향으로 늘어나며 옅은 항등 격자 위에 겹쳐 두 겹의 차이가 시각화된다. 보조 점은 가로 방향으로 따라가며, 라벨 "(1.5, 0.7)" 은 변하지 않고 위치만 새 기저 결합 자리로 이동한다. 단위 평행사변형이 가로로 늘어나 면적이 1.0 → 2.0 으로 차오르고 |det| 게이지가 1.00 → 2.00 으로 동기. 직선은 직선, 원점은 그 자리.
- 후: 평면은 새 변환 상태에서 정지. 두 격자(옅은 항등, 진한 변환) 의 차이가 한 컷의 정지 화면으로 보존된다. 단위 평행사변형은 가로 2 × 세로 1 의 직사각형. 보조 점은 새 자리에 있고 라벨은 그대로. 행렬 셀 [[2, 0], [0, 1]], |det| = 2.00. 사용자가 같은 a 셀을 2 → -1 로 더 끌면 |det| 가 0 을 통과하는 순간 평행사변형이 한 직선으로 무너졌다가 음수 영역에 진입하면서 색이 반전(반사) 되고, |det| 게이지가 청록 → 주황 색으로 전환된다.
- 강조 장치: 셀 a / b / c / d 의 색과 화살표 색의 1:1 동기·옅은 항등 격자와 진한 변환 격자의 두 겹·보조 점 라벨 "(u, v)" 의 불변과 위치의 변·단위 평행사변형의 면적 차오름·|det| 게이지 부호 색 전환·격자 직선의 직선성 유지·원점 고정점의 강조 표지.

### 4. 다른 대상과의 구별점

- **CSS 2D transform (`matrix(a, b, c, d, e, f)`)**: 같은 가설이 같은 격자 운동을 그리지만 결정의 단위가 다르다. CSS 의 `matrix()` 는 마지막 두 파라미터 e, f 로 이동(translation) 까지 포함하는 어파인 변환이며, 6 개 파라미터로 정의된다. 시각적으로는 변환 후에도 원점이 자기 자리에 있을 필요가 없어 격자 전체가 좌·우 또는 상·하로 통째로 미끄러지는 운동이 함께 발화한다. 본 시각화는 2×2 의 4 개 파라미터만 다루며 원점은 고정된 점으로 항상 그 자리에 박혀 있다 — "원점이 움직인다" 와 "원점이 박혀 있다" 의 차이가 평면 중앙의 빨간 점 표지의 운동/정지로 즉시 구별된다. 또한 CSS `matrix()` 는 행 우선/열 우선 혼동이 흔하지만 본 시각화는 셀과 화살표의 색을 직접 묶어 "첫째 열 = i-hat, 둘째 열 = j-hat" 의 약속을 위치만으로 강제한다.
- **비선형 변환 (자유 워프 / 곡선 변환)**: 같은 평면을 같은 평면으로 보내지만 격자선이 곡선이 된다. 시각적으로는 변환 후 가로·세로선이 휘어지고 등간격이 깨진다. 본 시각화는 어떤 셀 조합을 흔들어도 격자선이 직선을 유지하며 등간격이 깨지지 않는다 — "직선은 직선, 간격은 균등" 이라는 선형성의 시각 조건이 평면 운동의 한계로 강제된다. "곡선이 가능하다" 와 "곡선이 불가능하다" 의 차이가 격자선의 모양으로 즉시 구별된다.
- **어파인 변환 (이동 포함, 3×3 동차좌표)**: 회전·스케일·전단·반사에 더해 평행이동까지 한 행렬로 표현하는 일반화. 시각적으로는 격자 전체가 변환 후에도 직선·등간격을 유지하지만 원점이 다른 자리로 옮겨 갈 수 있다. 본 시각화의 4 종 변환은 모두 원점을 고정시키므로, "원점은 그 자리" 가 어파인과 본 변환을 가르는 시각 사실이다. 어파인은 본 시각화의 외연이며, 별도 사이클(공백 활용 후보) 의 주제다.

### 5. 달성해야 할 시각 요소

1. **셀과 화살표의 1:1 색 동기 — 한 칸 = 한 좌표의 미시 동기**
   - 설명: 행렬의 a / b / c / d 한 칸을 만지면 정확히 i-hat 또는 j-hat 의 한 좌표(x 또는 y) 만 변하고 나머지는 그대로다. "한 칸 = 한 좌표" 의 1:1 대응이 셀 색·화살표 색의 동기와 함께 시각 사건으로 발화된다. essence 본질 1(두 기저의 도착지 = 행렬 두 열) 과 강화 요소 1(셀 한 칸 ↔ 한 좌표의 미시 동기) 을 한 항목으로 묶어 운반한다.
   - 시각 표현: 행렬 셀 패널은 2×2 입력 박스로 우측에 배치된다. 첫째 열 셀 a (좌상)·c (좌하) 두 칸은 i-hat 색(청록) 의 옅은 배경 + 진한 테두리, 둘째 열 셀 b (우상)·d (우하) 두 칸은 j-hat 색(주황) 의 옅은 배경 + 진한 테두리로 칠해진다. 평면 위 i-hat 화살표는 같은 청록, j-hat 화살표는 같은 주황. 셀 옆에는 작은 라벨이 붙어 a 옆에 "i-hat.x", c 옆에 "i-hat.y", b 옆에 "j-hat.x", d 옆에 "j-hat.y" 로 그 셀이 어느 화살표의 어느 좌표를 잡고 있는지가 항상 명시된다. 셀 한 칸을 슬라이더 또는 직접 입력으로 흔드는 동안, 그 셀의 테두리가 강조 펄스로 한 번 빛나고 평면 위 해당 화살표 끝이 그 좌표 방향으로만 미끄러진다 — 다른 셀과 화살표는 정지. 사용자가 화살표 끝 자체를 평면 위에서 드래그할 수도 있으며, 이 경우 두 셀(예: i-hat 끝 드래그 시 a 와 c) 이 동시에 새 값으로 미끄러진다. 셀 → 평면 / 평면 → 셀 두 방향의 매핑이 모두 색과 위치로 즉시 동기된다.

2. **두 겹 격자 — 항등 잔상과 변환 결과의 동시 겹침**
   - 설명: 평면 위에 항등 상태의 옅은 회색 격자가 배경에 항상 깔리고, 그 위에 현재 행렬에 의해 변환된 진한 격자가 겹쳐 그려진다. "어디에서 어디로" 가 한 컷에 동시에 읽혀, 보간 애니메이션 없이도 변환의 운동 방향이 즉시 인지된다. essence 본질 2(선형성의 시각 조건 — 직선은 직선, 원점은 고정), 강화 요소 2(변환 전·후 격자의 동시 겹침) 를 정면으로 운반한다.
   - 시각 표현: 평면 영역 폭 약 360px·높이 약 360px 의 정사각형. 좌표축은 가로·세로 모두 -3 ~ 3 의 범위에 1 단위 간격의 격자선이 깔린다. 항등 격자는 옅은 회색(투명도 25%) 의 1px 점선, 가로·세로선 21 개. 변환 격자는 진한 색(중성 회청색) 1px 실선, 변환 행렬에 의해 휘어진 21 개의 가로·세로선. 격자선이 휘는 동안에도 모두 직선이며 등간격을 유지한다 — 곡선이 되는 일은 없다. 원점 (0, 0) 자리에는 작은 빨간 점 표지가 박혀 있고, 어떤 행렬 변환에서도 이 점은 그 자리를 떠나지 않는다 — 원점 고정의 시각적 강제. |det| 가 0 에 닿는 순간(예: a=0, b=0 의 한 행 0 행렬) 변환 격자의 모든 가로·세로선이 한 직선 위로 무너지는 사건이 발화하고, 이 직선이 옅은 항등 격자 위에 명확히 떠올라 "면적 0 = 평면이 직선으로 무너졌다" 가 한 컷의 모양 차이로 보존된다.

3. **표준 4종 프리셋 토글 — 셀 패턴의 학습**
   - 설명: 회전·스케일·전단·반사 네 버튼이 항상 노출되어, 누르는 순간 네 셀이 해당 패턴으로 채워지고 격자가 즉시 그 변환의 결과로 바뀐다. "각 변환은 셀 어디에 어떤 모양의 숫자가 들어가는가" 의 패턴이 토글 한 번으로 외워진다. essence 본질 5(4 종이 모두 같은 행렬 형식의 특수 사례) 와 강화 요소 4(프리셋 토글로 셀 패턴 학습) 를 정면으로 운반한다.
   - 시각 표현: 행렬 셀 패널 아래에 가로 한 줄로 4 개의 토글 버튼이 배치된다 — [회전 θ] [스케일 s,t] [전단 k] [반사 축] 의 네 라벨. 각 버튼은 둥근 모서리 직사각형, 폭 약 70px·세로 30px. 클릭 시 그 변환 모드가 활성화되고, 버튼 아래에 그 모드의 보조 슬라이더(회전 = θ ∈ [-π, π], 스케일 = s ∈ [-2, 2] / t ∈ [-2, 2] 두 줄, 전단 = k ∈ [-2, 2], 반사 = x축 / y축 / 원점 세 토글) 가 펼쳐진다. 슬라이더를 흔드는 동안 행렬 셀의 네 칸이 해당 패턴 — 회전 [[cosθ, -sinθ], [sinθ, cosθ]], 스케일 [[s, 0], [0, t]], 전단 [[1, k], [0, 1]] (가로 전단의 경우), 반사 (x축) [[1, 0], [0, -1]] — 으로 자동 채워지며, 채워지는 셀의 숫자가 슬라이더 운동에 동기되어 실시간으로 갱신된다. 학습자는 슬라이더 운동에 따라 셀의 어느 칸이 어떤 모양으로 변하는지를 직접 보며 패턴을 외운다. 자유 모드 토글 [자유] 버튼을 누르면 슬라이더가 사라지고 네 셀이 모두 직접 입력 가능 상태로 복귀한다.

4. **|det| 게이지와 부호 사건성 — 면적과 방향의 동시 운동**
   - 설명: 단위 평행사변형이 슬라이더를 흔드는 동안 면적이 변하고, 그 면적의 숫자가 도형 옆에 항상 따라다니며 변한다. 부호가 음수로 진입하는 순간 평행사변형이 색·채움 패턴으로 시각적으로 뒤집혀 (반사) "방향이 뒤집혔다" 가 사건으로 발화된다. 면적이 0 으로 무너지는 순간도 같은 어휘 안에서 평행사변형이 직선으로 압축되는 사건으로 보인다. essence 본질 4(단위 정사각형 → 평행사변형, 면적 = |det|) 와 강화 요소 5(|det| 의 면적·부호 동시 사건성) 를 정면으로 운반한다.
   - 시각 표현: 평면 위 단위 정사각형 (i-hat 과 j-hat 으로 이루어진 한 변 1 의 정사각형) 이 변환 후 평행사변형으로 바뀐다. 평행사변형 내부는 항등 상태에서 옅은 보라 채움(투명도 25%), det 가 양수일 때 같은 보라 톤 유지, det 가 음수로 진입하는 순간 채움 색이 보라 → 진한 청록으로 전환되고 빗금 방향이 반대로 뒤집힌다(/// → \\\). 평행사변형 옆에는 작은 라벨 "면적 = 1.45" 가 따라다니며 매 셀 갱신마다 동기 갱신. 평면 우측 행렬 셀 패널 아래에는 |det| 게이지 — 폭 약 180px·높이 12px 의 가로 막대 + 부호 표시 — 가 배치된다. 막대는 0 을 중심으로 좌·우로 음수·양수가 펼쳐지는 양방향 형태이며, 양수 영역은 청록, 음수 영역은 주황. 막대 끝 위에 숫자 라벨 "det = +1.45" 또는 "det = -0.72" 가 표시되어 부호와 크기가 동시에 보인다. det 가 0 에 닿는 순간 막대가 중심에 멈추고 평행사변형이 한 직선으로 압축되며 게이지 색이 회색으로 변한다 — "면적 0 = 변환의 평면이 무너졌다" 가 한 사건으로 발화한다. 캡션에 "면적이 0 이다 — 평면이 직선으로 무너졌다" 가 잠시 뜬다.

5. **보조 점의 동기 운동 — 격자와 한 점이 같은 법칙**
   - 설명: 사용자가 평면 위에 한 점을 찍어 두면, 행렬을 흔드는 동안 그 점이 새 좌표 (u, v) 의 새 기저 결합으로 즉시 이동한다. 점의 (u, v) 좌표 라벨이 변환 동안에도 유지되고, 위치만이 새 기저에 맞춰 바뀐다는 사실이 라벨과 함께 보인다. 격자 전체의 휨과 단일 점의 운동이 같은 운동 법칙임이 시각으로 증명된다. essence 본질 3(평면의 모든 점이 같은 (u, v) 계수로 따라간다) 와 강화 요소 3(보조 점의 동기 운동) 을 정면으로 운반한다.
   - 시각 표현: 평면 초기 상태에서 (u=1.5, v=0.7) 자리에 작은 보라색 점(지름 8px) 이 박혀 있다. 점 옆에는 두 줄의 라벨이 붙는다 — 위 줄 "(u, v) = (1.5, 0.7)" 은 기저 결합 계수, 아래 줄 "위치 = (1.5, 0.7)" 은 평면상 좌표. 사용자가 점을 평면 위에서 직접 드래그하면 두 라벨이 동시에 새 값으로 갱신되며 항등 상태에서는 두 라벨이 항상 같다. 행렬을 흔드는 동안에는 (u, v) 라벨은 변하지 않고 "위치" 라벨만 새 기저 결합 자리 u·i-hat' + v·j-hat' 의 좌표로 갱신되어, 두 라벨이 다르게 표시된다 — "계수는 같고 자리만 따라간다" 가 두 라벨의 비대칭으로 운반된다. 점 자체는 변환 격자 위의 정확한 (u, v) 격자 칸 위에 머무르며, 그 격자 칸이 휘는 운동을 따라 점이 이동한다. 격자 칸 안의 한 점이 격자가 휠 때 격자와 함께 따라간다는 시각이, 점과 격자의 위치 동기로 한 컷에 보존된다. 보조 점은 한 개로 시작하지만 사용자가 평면을 클릭해 추가 점을 1~2 개 더 찍을 수 있으며, 모든 점은 같은 법칙으로 동시 따라간다.

### 6. 시각 구성

- **중심 시각**: 화면 좌측의 큰 정사각형 격자 평면(폭 360px·높이 360px). 평면 위에는 옅은 항등 격자(점선 21 줄) + 진한 변환 격자(실선 21 줄) + 원점 빨간 점 + i-hat 청록 화살표 + j-hat 주황 화살표 + 단위 평행사변형(보라 또는 청록 채움) + 보조 점 1~3 개 + 점 옆 두 줄 라벨. 격자선은 어떤 행렬 운동에서도 직선을 유지하며, 원점은 어떤 운동에서도 그 자리를 떠나지 않는다.
- **보조 요소**:
  - 우측 행렬 셀 패널: 2×2 입력 박스. 셀 a·c 청록 배경 + i-hat.x / i-hat.y 라벨, 셀 b·d 주황 배경 + j-hat.x / j-hat.y 라벨. 직접 숫자 입력 또는 그 옆에 붙은 작은 슬라이더로 흔들 수 있다.
  - 셀 패널 아래 4종 프리셋 토글: [회전 θ] [스케일 s,t] [전단 k] [반사 축] [자유] 다섯 버튼 + 활성화된 모드의 보조 슬라이더 패널.
  - 프리셋 패널 아래 |det| 게이지: 양방향 가로 막대 + 부호 색 + 숫자 라벨.
  - 평면 좌상단 모서리의 변환 모드 표지: 현재 활성 모드 ("자유 모드", "회전 모드 θ = 0.7 rad", "스케일 모드 s = 1.5, t = 0.5") 한 줄.
  - 평면 옆 단위 평행사변형의 면적 라벨: 평행사변형 내부 또는 옆에 "면적 = 1.45" 한 줄.
  - 보조 점 옆 두 줄 라벨: "(u, v) = (1.5, 0.7)" + "위치 = (3.0, 0.7)".
  - 상단 캡션 영역: 짧은 개념 문장 한 줄 + 현재 사건 메시지.
  - 하단 컨트롤 영역: [점 추가] [점 제거] [항등으로 리셋] [초기화] 네 버튼.
- **요소 간 관계**: 행렬 셀 한 칸의 변경 → 해당 색의 기저 화살표 한 좌표 갱신 → 변환 격자 21 줄 동시 재계산 → 단위 평행사변형 면적 / 부호 갱신 → |det| 게이지 갱신 → 보조 점 위치 라벨 갱신 + 점 위치 새 기저 결합 자리로 이동 의 6 단 동시 운동이 1:1 대응된다. 셀 운동과 평면 운동은 한 시간축 위에서 즉시(보간 없음) 동기되어, 슬라이더 = 결과의 입력 반응형 약속이 모든 요소에 일괄 적용된다. 프리셋 토글의 보조 슬라이더 운동도 같은 6 단 시퀀스를 거쳐 셀 → 평면으로 흐른다. 화살표 끝의 직접 드래그는 역방향 — 평면 → 셀 → 격자/도형 — 의 운동을 발화하며, 두 방향이 모두 같은 6 단 동기 위에서 작동한다.

### 8. 진행과 인터랙션

입력 반응형. 사용자의 한 번의 입력이 6 단 동시 운동(셀 → 화살표 → 격자 → 평행사변형 → 게이지 → 보조 점) 으로 1:1 대응되며, 보간 애니메이션 없이 즉시(또는 매우 짧은 200ms 의 동기 미끄러짐) 반영된다.

- **입력 → 시각 갱신 매핑**:

  1. **행렬 셀 직접 입력 또는 셀 옆 슬라이더 운동**:
     - 셀 a (i-hat.x) 변경 → i-hat 화살표 끝의 x 좌표만 변함 (y 그대로). 진한 변환 격자의 가로선들이 새 i-hat 방향으로 휘어지고 세로선들도 새 격자 노드에 맞춰 재배치. 단위 평행사변형의 i-hat 변 길이가 갱신되어 면적·부호 재계산. |det| 게이지 갱신. 보조 점이 새 기저 결합 자리로 즉시 이동, "위치" 라벨만 갱신.
     - 셀 c (i-hat.y) 변경 → i-hat 화살표 끝의 y 좌표만 변함. 같은 6 단 운동.
     - 셀 b (j-hat.x) 변경 → j-hat 화살표 끝의 x 좌표만 변함. 같은 6 단 운동.
     - 셀 d (j-hat.y) 변경 → j-hat 화살표 끝의 y 좌표만 변함. 같은 6 단 운동.
     - 운동 길이: 셀 입력 즉시(0ms) 결과 반영, 시각적 부드러움을 위해 화살표·격자·평행사변형이 약 200ms 의 미소 미끄러짐으로 새 자리에 안착. 슬라이더는 드래그 중 실시간 매 프레임 갱신.

  2. **i-hat / j-hat 화살표 끝의 직접 드래그**:
     - i-hat 끝(청록 원) 드래그 → 끝의 (x, y) 가 마우스 위치를 따라가고, 셀 a·c 두 칸의 숫자가 동시에 새 값으로 갱신됨. 같은 6 단 운동.
     - j-hat 끝(주황 원) 드래그 → 끝의 (x, y) 가 마우스 위치를 따라가고, 셀 b·d 두 칸의 숫자가 동시에 새 값으로 갱신. 같은 6 단 운동.
     - 평면 → 셀 의 역방향 매핑. 사용자는 셀 숫자를 보지 않고 화살표 끝만 만져도 행렬을 편집할 수 있다.

  3. **4종 프리셋 토글 + 보조 슬라이더**:
     - [회전 θ] 클릭 → 회전 모드 활성. 보조 슬라이더 θ ∈ [-π, π] 등장. 슬라이더 운동 시 셀 a, b, c, d 가 각각 cosθ, -sinθ, sinθ, cosθ 로 자동 갱신. 같은 6 단 운동.
     - [스케일 s, t] 클릭 → 스케일 모드. 슬라이더 두 줄 s ∈ [-2, 2], t ∈ [-2, 2]. 셀 a = s, d = t, b = c = 0 자동 갱신.
     - [전단 k] 클릭 → 전단 모드(가로 전단 기본). 슬라이더 k ∈ [-2, 2]. 셀 a = 1, b = k, c = 0, d = 1.
     - [반사 축] 클릭 → 반사 모드. 세 토글 (x축 / y축 / 원점) 등장. x축 반사: 셀 [[1, 0], [0, -1]]. y축 반사: 셀 [[-1, 0], [0, 1]]. 원점 반사: 셀 [[-1, 0], [0, -1]].
     - [자유] 클릭 → 자유 모드. 보조 슬라이더 사라지고 네 셀 모두 직접 입력 가능 상태.
     - 모드 전환 시 보조 슬라이더 펼침 운동 약 200ms.

  4. **보조 점의 평면 위 드래그**:
     - 점을 평면 위에서 드래그 → (u, v) 라벨이 마우스 위치를 따라 갱신 (항등 상태에서는 (u, v) = 위치). 변환 행렬이 항등이 아닌 상태에서 점을 드래그하면 점은 항등 격자가 아닌 변환 격자 위의 격자 칸에 안착하고 (u, v) 는 그 격자 칸의 인덱스로 갱신된다 — 점은 변환 후 평면의 (u, v) 자리.
     - [점 추가] 버튼: 평면 중앙에 새 점이 생성. [점 제거] 버튼: 마지막 점 또는 선택된 점 제거. 최대 3 개까지.

  5. **[항등으로 리셋] 버튼**: 셀이 [[1, 0], [0, 1]] 로 즉시 복귀. 평면이 항등 상태로 약 300ms 동안 부드럽게 복귀(보간). 보조 점은 자기 (u, v) 자리에 그대로 머문다. 프리셋 모드는 자유 모드로 복귀.

  6. **[초기화] 버튼**: 셀 항등 복귀 + 보조 점 모두 제거 + 보조 점 (1.5, 0.7) 한 개 재생성 + 프리셋 모드 자유로 복귀. 초기 상태 t=0 으로 완전히 돌아간다.

- **사건성 발화 (시각의 한계 사건)**:
  - **det 가 0 에 닿는 순간**: 변환 격자의 모든 가로·세로선이 한 직선 위로 무너지고, 단위 평행사변형이 한 직선으로 압축됨. |det| 게이지가 중심에 멈추고 게이지 색이 회색으로. 캡션에 "면적이 0 이다 — 평면이 직선으로 무너졌다" 잠시.
  - **det 가 음수로 진입**: 평행사변형 채움 색 보라 → 진한 청록, 빗금 방향 반전. 게이지 막대가 0 좌측(주황 영역) 으로 이동. 캡션에 "방향이 뒤집혔다 — 평면이 거울에 비쳤다" 잠시.
  - **반사 프리셋 활성**: x축 / y축 / 원점 반사 토글 즉시 평행사변형이 뒤집히는 사건이 강조 펄스로 발화.

- 모든 운동의 기본 길이: 셀 입력 → 평면 갱신 200ms 의 부드러운 미끄러짐 (또는 슬라이더 드래그 중 실시간 매 프레임), 모드 전환 보조 슬라이더 펼침 200ms, 항등 복귀 300ms, det 0 / 부호 사건 200ms 강조 펄스. 보간 애니메이션은 베이스에 두지 않고, 입력 = 결과의 즉시 매핑이 약속.

### 9. 초기 상태와 파라미터

- **초기 상태 (t = 0)**: 격자 평면이 항등 상태. 항등 격자(옅은 점선) 와 변환 격자(진한 실선) 가 완전히 겹쳐 한 겹처럼 보인다. i-hat 은 (1, 0), j-hat 은 (0, 1). 행렬 셀 [[1, 0], [0, 1]]. 단위 평행사변형은 한 변 1 의 정사각형, 면적 = 1.00, 보라 채움. |det| 게이지 +1.00, 청록. 보조 점 한 개가 (u, v) = (1.5, 0.7) 자리에 박혀 있고 두 라벨 모두 "(1.5, 0.7)". 프리셋 모드 [자유]. 변환 모드 표지 "자유 모드". 원점 빨간 점이 (0, 0) 자리에 박혀 있다.
- **권장 시연 시퀀스**: 처음 진입 시 약 1.5 초간 자동 시연 — 셀 a 가 1 → 1.5 로 슬라이더 운동, 그 후 셀 c 가 0 → 0.3 으로, 그 후 셀 b 가 0 → -0.5 로 차례로 흔들리며 평면이 비스듬히 휘어지고 보조 점이 따라 이동하는 운동을 보여준 뒤 정지. 학습자는 그 상태에서 직접 셀을 흔들거나 화살표 끝을 드래그하거나 프리셋 토글을 눌러 추가 운동을 발화시킨다. 그다음 [회전 θ] 토글을 눌러 θ 슬라이더를 흔들면 격자가 부드럽게 회전하는 모양 차이를 직접 본다.
- **파라미터**:
  - 격자 평면 좌표 범위: 기준 폭 -3 ~ 3 (가로·세로 각 6 단위), 격자 간격 60px = 1 단위. 확장 폭 -4 ~ 4.
  - 격자선 수: 가로·세로 각 7 줄(±3 까지). 확장 폭 9 줄.
  - 항등 격자 색·투명도: 중성 회색, 투명도 25%, 1px 점선.
  - 변환 격자 색: 중성 회청색, 1px 실선.
  - i-hat 색: 청록 (예: #2E9E9E). j-hat 색: 주황 (예: #E89A3C). 색은 design-token 의 facet 자체 hex 미사용 원칙에 따라 표준 토큰으로 매핑.
  - 단위 평행사변형 채움: 양수 det 옅은 보라(투명도 25%), 음수 det 진한 청록 + 빗금.
  - 보조 점 색: 보라 (예: #8B5CF6). 점 직경 8px, 라벨 8pt 두 줄.
  - 행렬 셀 입력 범위: a, b, c, d 각각 [-3, 3] 의 자유 입력 (직접 입력 시 평면 가장자리를 벗어나는 화살표는 가장자리에서 잘림 표지).
  - 회전 슬라이더 θ: [-π, π], 단계 0.05 rad.
  - 스케일 슬라이더 s, t: [-2, 2], 단계 0.05.
  - 전단 슬라이더 k: [-2, 2], 단계 0.05.
  - 반사 축: x축 / y축 / 원점 세 토글.
  - |det| 게이지 폭: 220px (양방향, 0 중심), 높이 12px. 표시 범위 [-4, +4].
  - 보조 점 최대 개수: 3 개.
  - 운동 길이: 셀 입력 → 평면 갱신 200ms, 모드 전환 200ms, 항등 복귀 300ms, 사건 강조 펄스 200ms.

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> 2D 행렬 변환은 평면의 모든 점을 새 자리로 보내는 선형 사상이며, 그 정체는 두 기저 벡터 i-hat·j-hat 의 도착지로 완전히 결정된다. 2×2 행렬의 첫 열은 i-hat 이 어디로 가는지, 둘째 열은 j-hat 이 어디로 가는지를 그대로 적은 좌표이고, 평면의 모든 점은 자신을 두 기저의 선형 결합으로 적은 뒤 그 결합을 도착지에서 반복하는 것만으로 따라간다. 변환은 직선을 직선으로 보내고 원점을 그 자리에 박아 두며, 단위 정사각형이 변환 후 갖는 면적이 곧 행렬식의 절댓값이고, 부호가 음수면 평면이 거울에 비친 것처럼 뒤집힌다. 회전·스케일·전단·반사 네 가지 표준 변환은 모두 같은 행렬 형식의 특수 사례이며, 셀 네 칸의 패턴 차이일 뿐이다.

**학습자 안내 (보조 캡션 회전 문구)**

- "두 열은 두 기저 벡터의 도착지 — 셀 한 칸은 한 좌표를 잡는다."
- "직선은 직선으로, 원점은 그 자리에 — 격자가 휘어도 곡선은 없다."
- "평면의 모든 점은 같은 (u, v) 계수로 새 기저를 따라간다."
- "단위 정사각형의 새 면적이 곧 |det| — 부호가 음수면 평면이 뒤집힌다."
- "회전·스케일·전단·반사는 모두 같은 행렬 형식 — 셀 패턴의 차이뿐이다."
- "셀 a 는 i-hat.x, 셀 c 는 i-hat.y, 셀 b 는 j-hat.x, 셀 d 는 j-hat.y 다."

**사건별 짧은 메시지**

- 셀 변경 (i-hat 한 좌표): "i-hat 의 한 좌표만 변했다 — j-hat 은 그대로다."
- 셀 변경 (j-hat 한 좌표): "j-hat 의 한 좌표만 변했다 — i-hat 은 그대로다."
- 화살표 끝 드래그: "두 기저의 도착지를 직접 잡았다."
- 회전 프리셋: "회전 — 두 기저가 같이 돈다."
- 스케일 프리셋: "스케일 — 두 기저가 각자 늘거나 줄어든다."
- 전단 프리셋: "전단 — 한 기저가 비스듬해진다."
- 반사 프리셋: "반사 — 평면이 거울에 비친다."
- det 가 0 에 닿음: "면적이 0 이다 — 평면이 직선으로 무너졌다."
- det 가 음수로 진입: "방향이 뒤집혔다 — 평면이 거울에 비쳤다."
- 보조 점 추가: "한 점이 격자와 같은 법칙으로 따라간다."
- 항등으로 리셋: "처음의 격자로 돌아갔다 — 두 기저가 (1, 0) 과 (0, 1)."

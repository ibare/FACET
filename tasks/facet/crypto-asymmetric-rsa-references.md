## 2. 레퍼런스 요약

### 2.1 CrypTool — RSA visual / step-by-step
- 출처: CrypTool Project (오픈소스 암호 학습 도구).
- 한 줄 정체성: 작은 소수로 RSA 전체 파이프라인을 직접 굴려 보는 단계 슬라이더형 시뮬레이터.
- 시각 객체: p, q 입력 박스 → φ(n)·n 자동 채움 박스 → e, d 박스. 평문/암호문 숫자 매핑 그래프, 거듭제곱·모듈러 연산이 순서대로 나타나는 산식 패널, 단계 진행/되감기 버튼.
- 시간 표현: 명시적 step 슬라이더와 자동 재생 속도 조절. "지금 어느 줄을 계산 중" 인지를 동기화 강조.
- 메타포: 없음. 메타포 대신 "수가 어떻게 굴러가는가"를 그대로 노출.
- 강조점: 키 생성과 암호화/복호화의 산식적 사건을 시간축 위에 흩뿌린다.
- 인상 디테일: 작은 (p,q)=(5,11) 같은 학습용 짝을 권장 — 학습자가 손으로 검산 가능한 규모 유지.
- 약점: 시각 언어가 거의 전부 숫자·수식. Alice/Bob 같은 인간 행위자나 비대칭 사건의 극적 표현이 없음.

### 2.2 위키백과 RSA cryptosystem
- 출처: Wikipedia 영문 RSA 항목.
- 한 줄 정체성: 공식 절차서. 키 생성 → 암호화 → 복호화 → 서명까지 수식 중심으로 나열.
- 시각 객체: Adi Shamir 사진, 수식 블록 (n=pq, λ(n), c≡m^e mod n, m≡c^d mod n), 단계 번호 리스트.
- 시간 표현: 번호 매겨진 절차 리스트로 단계가 직선화.
- 메타포: 본문에 우편함/자물쇠 비유 언급은 있으나 시각화는 거의 없음.
- 강조점: 표준성. 어떤 수가 공개되고 어떤 수가 비밀인지 표 형태로 분리.
- 인상 디테일: 공개키 (n,e) 와 비밀키 (n,d) 또는 (p,q,d) 의 분리 표기.
- 약점: 정적 텍스트·수식 위주. 시간 진행과 사건성이 거의 없고, 비대칭의 시각적 충격이 없음.

### 2.3 Khan Academy — RSA encryption Step 1~4
- 출처: Khan Academy "Journey into Cryptography".
- 한 줄 정체성: 트랩도어 일방향 함수 직관을 4편의 강의 영상으로 천천히 쌓아 올리는 내러티브 시각화.
- 시각 객체: 손글씨 다이어그램, 색이 다른 두 개의 키 아이콘, 큰 수의 인수분해를 시도하는 컴퓨터 일러스트, 시계·시간 비교 그래프 (인수분해 시간 vs. 곱셈 시간).
- 시간 표현: 영상 자체의 선형 시간. 트랩도어 발견까지 "왜 우리는 막혔는가"를 회상하며 쌓음.
- 메타포: 시계의 모듈러 산술 (시계바늘이 원을 도는 그림), 자물쇠/열쇠 짝.
- 강조점: 일방향성 (한쪽으로 쉽고 반대쪽으로 어렵다) 그 자체.
- 인상 디테일: 큰 수 곱셈은 빠른 컴퓨터 그림, 인수분해는 같은 컴퓨터가 절망하는 그림으로 비대칭을 신체화.
- 약점: 손글씨 강의체라 데이터 객체가 화면에 영속적으로 살지 않음 (지나가면 사라짐). 키 생성의 운동을 "동시에 살아 있는 객체들" 로 보여 주지 못함.

### 2.4 Computerphile — Public Key Cryptography / Prime Numbers & RSA
- 출처: 노팅엄대 Computerphile 채널 (Robert Miles, Tim Muller, Mike Pound).
- 한 줄 정체성: 종이와 펜으로 큰 수의 곱셈/인수분해 비대칭을 손으로 시연하는 영상 해설.
- 시각 객체: 손이 종이에 적는 소수 두 개, 그것이 곱해지는 화살표, 인수분해를 시도하다가 멈추는 손, 자물쇠가 잠긴 박스 그림, 두 개의 다른 모양 키.
- 시간 표현: 발표자의 손동작 시간 = 알고리즘 시간. 펜이 움직이는 속도가 곧 단계 진행.
- 메타포: 자물쇠와 열쇠 짝, 공개된 박스에 누구나 메시지를 넣고 잠가서 돌려 보내는 그림.
- 강조점: 왜 어렵냐 — "큰 수를 인수분해해 보세요" 하고 발표자가 직접 쩔쩔매는 장면.
- 인상 디테일: 카메라가 손과 종이만 비추기 때문에 "두 객체가 살아 있다" 는 감각이 강함.
- 약점: 영상 의존. 동시에 살아 있는 다중 객체 (n, e, d, p, q) 의 상태판이 없음.

### 2.5 Stanford / Dan Boneh — RSA Cryptosystem 슬라이드
- 출처: Stanford CS255 / Online Cryptography Course.
- 한 줄 정체성: 트랩도어 순열로서의 RSA 를 정식으로 정의하는 학술 슬라이드.
- 시각 객체: 함수 박스 F: X → Y 와 역함수 박스 F^-1: Y → X, 트랩도어 t 가 별도 칸으로 매달린 다이어그램, 수식 블록.
- 시간 표현: 슬라이드 번호 = 단계. 입력→출력 화살표가 시간을 대신.
- 메타포: 트랩도어 (한 방향만 열리는 문) 자체가 메타포.
- 강조점: 비대칭의 형식적 정의 — "F 는 모두에게 쉽고 F^-1 은 t 를 가진 자에게만 쉬움".
- 인상 디테일: 키 짝을 함수와 그 역함수로 쌍으로 그리는 도식. Alice/Bob 인물보다는 박스와 화살표.
- 약점: 추상적이라 키 생성의 구체적 운동(소수 두 개가 무엇을 낳는가) 이 그림으로는 가려짐.

### 2.6 Brilliant.org — RSA Encryption
- 출처: Brilliant Math & Science Wiki.
- 한 줄 정체성: 작은 수치 예제를 단계별로 따라 풀게 하는 인터랙티브 위키.
- 시각 객체: 수식 박스, 단계별 계산표 (p, q, n, φ, e, d 칸), 색이 다른 두 키 아이콘, 작은 자물쇠/열쇠.
- 시간 표현: 스크롤 = 시간. 위에서 아래로 단계가 쌓임.
- 메타포: 자물쇠와 열쇠.
- 강조점: 학습자가 손으로 따라 풀게 하는 검산 친화 표.
- 인상 디테일: e 의 선택과 d 의 계산 (확장 유클리드) 을 별도 표로 노출.
- 약점: 한 페이지의 정적 표라 객체들이 한꺼번에 살아 있는 느낌이 약함.

### 2.7 Simulations4All / RSALearner / MiniWebTool — 인터랙티브 시뮬레이터
- 출처: 교육용 웹앱 (Eva Maria Kiss, Simulations4All, MiniWebTool).
- 한 줄 정체성: 두 소수를 직접 입력하면 키 짝과 암복호 결과까지 한 화면에서 펼쳐 보여 주는 도구 모음.
- 시각 객체: p, q 입력 → n, φ(n), e, d 가 채워지는 카드 패널, "Encrypt"/"Decrypt" 버튼, 모듈러 거듭제곱 진행 바, 확장 유클리드 단계 표, (RSALearner) Jupyter 위젯 형식의 두 탭 (Key generation / Encryption).
- 시간 표현: 사용자 클릭 = 시간. 단계 버튼이 직접적 시간축.
- 메타포: 거의 없음. 숫자가 흐른다.
- 강조점: 키 생성과 암호화·복호화가 같은 화면에서 "같은 키 짝" 으로 묶이는 일관성.
- 인상 디테일: Simulations4All 은 인수분해 공격 데모와 PKCS#1/OAEP 패딩 설명까지 같은 페이지에 묶음.
- 약점: 두 키가 시각적으로 거울처럼 그려져 비대칭이 데이터로만 드러나고 그림으로는 약함.

### 2.8 Wikibooks Public Key Overview — 자물쇠·우편함·이중자물쇠 메타포
- 출처: Wikibooks 영문.
- 한 줄 정체성: 공개키 암호의 직관을 자물쇠·우편함·이중자물쇠 비유로 제시하는 메타포 카탈로그.
- 시각 객체: 열린 자물쇠를 우편으로 보내는 그림, 두 개의 문이 달린 우편함 (위쪽 투입구는 누구나, 본체는 주인만), 박스에 두 개의 자물쇠가 동시에 채워졌다가 하나씩 풀리는 시퀀스.
- 시간 표현: 우편이 오가는 단계 = 시간.
- 메타포: 자물쇠와 박스, 우편함, 이중 자물쇠 (commutative encryption).
- 강조점: "공개된 도구로 잠그고, 비밀 도구로만 풀 수 있다" 는 사건성.
- 인상 디테일: 이중자물쇠 비유는 키 교환 (Diffie-Hellman 류) 직관과 RSA 직관을 분리할 때 유용.
- 약점: 메타포가 강해 두 키가 형태상 "다르다" 는 비대칭은 잘 살리지만, 실제 RSA 의 수론적 비대칭 (n=pq) 은 비추지 못함.

### 2.9 나무위키 / 한국어 위키백과 — RSA 암호
- 출처: namu.wiki, ko.wikipedia.org.
- 한 줄 정체성: 한국어 학습자에게 가장 많이 노출되는 RSA 절차서.
- 시각 객체: 단계 번호 리스트 (소수 선택 → n → φ(n) → e → d), 공개키 {e,n}/비밀키 {d, p, q} 표, 자물쇠·우편함 비유 문장.
- 시간 표현: 번호 매겨진 절차.
- 메타포: 좁은 투입구가 있는 잠긴 편지함.
- 강조점: 공개키/비밀키 분리와 절차 순서.
- 인상 디테일: "투입구는 누구나, 열쇠는 주인만" 이라는 한국어 문구가 정착되어 있음.
- 약점: 시각 자료보다 텍스트 절차서 — 사건성·시간 진행·비대칭의 운동성은 약함.

### 2.10 서울대학교 자연과학대학 뉴스룸 — RSA 수학적 원리
- 출처: 서울대 자연과학대학 뉴스룸 칼럼.
- 한 줄 정체성: RSA 의 수학적 핵심 (오일러 정리·페르마 소정리) 을 한국어로 풀어 쓴 해설.
- 시각 객체: 수식 중심. 공개키·비밀키 한 쌍이 왜 한 메시지를 동일 평문으로 되돌리는가의 증명 과정.
- 시간 표현: 증명의 줄 = 시간.
- 메타포: 거의 없음. 수학적 사건만.
- 강조점: 비대칭이 "왜 작동하는가" 를 정수론으로 보임.
- 인상 디테일: 인수분해 어려움이라는 가정과 알고리즘 정확성 (correctness) 을 분리해 다룸.
- 약점: 시각 객체 거의 없음. 동적 시각화로 옮길 때 참고용.

## 3. 공통 요소

거의 모든 레퍼런스가 공유하는 시각 언어:

- **두 인물 Alice/Bob** — 메시지 송수신의 주체. 한국어 자료에서는 A/B 또는 갑/을로 치환되기도 함.
- **자물쇠와 열쇠 짝** — 공개된 자물쇠와 주인만 가진 열쇠. 우편함 비유로 확장.
- **공개키 ↔ 비밀키 짝** — (n,e) 와 (n,d), 또는 (p,q,d). 한 화면에 함께 띄워 짝을 강조.
- **평문 m → 암호문 c → 평문 m 의 일방향 흐름** — 화살표·상자 구성.
- **거듭제곱과 모듈러 연산** — c ≡ m^e mod n, m ≡ c^d mod n 이 페어로 등장.
- **큰 수 인수분해의 어려움** — n=pq 라는 명제와 "p,q 를 모르면 막힌다" 는 서술.
- **단계 번호** — 키 생성 → 키 배포 → 암호화 → 복호화. 시간 진행을 절차 번호로 직선화.

## 4. 공백

레퍼런스 군이 잘 다루지 않는 측면:

- **키 짝의 "비대칭" 이 시각적으로 거울처럼 보임.** 대부분 공개키/비밀키를 좌우 대칭으로 그려 형식적 짝만 강조하고, 둘이 진짜로 "다른 일을 한다" 는 비대칭은 그림에 없음. 둘의 모양·운동·접근 권한이 시각적으로 달라야 한다.
- **키 생성 단계의 운동이 빈약함.** 두 소수 p, q 가 → n 을 낳고 → φ(n) 또는 λ(n) 을 낳고 → e 를 고르고 → d 가 e 의 모듈러 역원으로 떨어지는 일련의 출생 사건이 정적인 표로만 나열됨. 시간축 위에 객체가 차례로 태어나는 느낌이 거의 없다.
- **일방향성의 시각적 강조 부재.** 곱셈은 가벼운 화살표, 인수분해는 무거운 (또는 막힌) 화살표 같은 비대칭의 신체적 표현이 Computerphile/Khan 정도를 빼면 거의 없음.
- **키 보유자의 배포 모델이 모호.** 누가 (n,e) 를 가지고 누가 (p,q,d) 를 가지는가, 누가 무엇을 보낼 수 있는가의 권한 분포가 그림에 잘 안 나타남. 자물쇠 비유가 이 부분을 떠맡지만 실제 데이터 객체와 분리되어 있음.
- **"암호화에 쓴 키와 복호화에 쓴 키가 다른 키" 라는 사건성.** 평문→암호문→평문 흐름에서 두 변환이 서로 다른 키를 사용한다는 사실이 한 프레임 안에 강조되는 사례가 드물다 (대개 두 줄 수식으로만).
- **공개키 암호화 vs 비밀키 서명 두 모드의 구분** (베이스 범위 밖이지만 메모) — 같은 짝을 거꾸로 쓰면 서명이 된다는 대칭적 사건은 거의 모든 입문 시각화에서 누락됨.
- **"왜 되돌아오는가" 의 시각화 부재.** m^(ed) ≡ m mod n 이 성립하는 이유 (오일러 정리) 가 그림으로 와닿게 표현되는 사례가 드물고, 대부분 수식 증명에 위탁된다.

## 5. 대상 개념 요약

RSA 는 두 개의 큰 소수 p, q 를 곱해 만든 모듈러스 n 위에서, 공개 지수 e 와 그 모듈러 역원인 비밀 지수 d 라는 한 짝의 비대칭 키를 만들어 낸다. 누구나 가진 (n,e) 로 평문을 거듭제곱·모듈러 연산하면 암호문이 되고, 같은 사건을 비밀 지수 d 로 다시 거듭제곱하면 원래 평문으로 돌아오지만, n 을 가지고 있을 뿐 p, q 를 모르는 자에게는 d 를 구하는 길이 사실상 닫혀 있다. 이 비대칭의 정체는 "큰 수의 곱셈은 가볍지만 역방향의 인수분해는 무겁다" 는 일방향성이며, 학습자는 키 짝의 출생 → 공개·비밀 분리 → 한쪽으로만 잠그고 다른 쪽으로만 풀리는 사건의 차례를 시간축 위에서 따라가야 한다.

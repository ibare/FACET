# Z-알고리즘 시각화 기획서

## 0. 기본 정보

- 대상 이름: Z-알고리즘 (Z-algorithm)
- 도메인 / 카테고리: 문자열 처리 / 패턴 매칭
- 대상의 본질: 문자열 S 의 모든 위치 i 에 대해 "S 와 S[i..] 가 공유하는 최장 공통 접두사 길이" Z[i] 를 단 한 번의 좌→우 스캔(O(n))으로 계산한다. 직전까지 발견한 "전체 접두사와 일치하는 가장 오른쪽 구간" [l, r] (Z-box) 를 유지하며, i 가 박스 안일 때 거울 위치 k = i - l 의 결과 Z[k] 를 빌려와 비교를 건너뛴다. r 이 단조 증가하므로 전체 비교 횟수가 선형이다.
- type: algorithm (Stage 0)
- 적용 전략: A타입 (공통 강화)
- IR 보유 여부: 보유 — 배열 인덱싱 / while 비교 루프 / 조건 분기로 구성되어 IR 인터프리터 및 transpiler 와 직접 호환
- 참조한 레퍼런스 (영향 큰 순):
  1. **R2. CMU/Gusfield 강의노트** — 곡선 호(arc)로 prefix↔Z-box 동일성을 보여주는 정식 도식. 거울 위치 k 라벨링.
  2. **R3. asherLZR 인터랙티브 시각화** — Z-box 박스 테두리, 케이스별 색 분기, 의사코드 라인 동기화.
  3. **R6. ialy 한국어 블로그** — `i > r` (재출발) vs `i ≤ r` (재활용) 두 케이스를 같은 색 매핑으로 깔끔히 분리.

## 1. 시그니처 행동

**문자 트랙을 좌→우로 훑으며, "이미 본 가장 오른쪽 매치 구간" 을 사각형 박스로 끌고 다니다가, 새 위치가 박스 안에 들어오는 순간 박스 시작점을 축으로 접두사 쪽 거울 위치의 답을 빌려와 비교를 건너뛴다.**

이 한 문장에서 다른 패턴 매칭(KMP / Boyer-Moore / Naive)과 구별되는 것:
- "박스" 라는 명시적 기억 장치를 끌고 다닌다 (KMP 의 실패 함수는 점프 테이블, Z 는 구간).
- "거울 위치" 라는 공간적 대칭이 본질이다 (KMP 의 prefix function 도 유사하지만 박스 메타포는 Z 고유).
- r 의 단조성 — 박스의 오른쪽 끝이 절대 후퇴하지 않는다.

## 2. 시각적 은유

**"좌→우로 미는 사각형 슬라이더 박스 + 박스 시작점을 거울 축으로 한 접두사 호(arc) 비추기"**

A타입이므로 R2/R3/R6 의 공통 메타포를 그대로 채택하되 정교화한다:

- 문자 트랙(가로 모노스페이스 셀)
- Z-box 는 셀들 위를 덮는 반투명 사각형 (단순 라벨이 아니라 *명시적 면적*)
- prefix 영역(S[0..r-l])과 Z-box 영역(S[l..r])은 같은 색으로 채색 — "이 둘은 같다" 는 invariant 의 직접 표현
- i 가 박스 안에 들어오면 박스 시작점 l 을 *거울 축*으로 i 를 접두사 쪽 k = i - l 위치로 *비추는 곡선 호*가 한 박자 동안 그려진다

**임베드 폭 검증** (640~720px 가용):
- 가로 트랙 길이 한도: 셀 폭 28~32px 기준 약 20~22 글자가 한 줄에 들어감. 예시 문자열 길이를 18 이내로 잡으면 폭 여유 있음.
- 박스/호/포인터 라벨이 같은 트랙 위에 겹치므로 *세로 분할은 최소화*. 코드 동기화 패널은 트랙 아래에 배치.
- 좌우 분할 4~6개 패널 금지 원칙 — 본 기획은 *상하 적층(stack)* 구조로 폭 안전.

## 3. 결정적 순간

"이게 Z-알고리즘이구나" 를 만드는 순간은 **i 가 처음으로 r 안에 들어오는 step** 이다. 정확히는 다음 3-비트 모션:

1. **전 (前)**: i 가 박스 바깥 (i > r). 박스가 좌측 어딘가에 정지. i 포인터가 우측으로 한 칸 이동하다 박스 우측 경계 r 안쪽으로 미끄러지듯 들어간다.
2. **중 (中)**: i 가 박스 안에 도착하는 순간 — *거울 호(arc)* 가 한 박자 동안 그려진다. l 을 축으로 i → k = i - l 로 곡선이 휘어진다. 동시에 접두사 쪽 k 셀 위로 Z[k] 만큼의 영역이 *복사 채색* 으로 i 위치에 붙여 넣어진다 (= "이만큼은 비교 안 해도 같다").
3. **후 (後)**: 복사 영역의 *오른쪽 끝* 에서만 실제 비교가 시작된다. 이미 안 부분(연두색)과 새로 비교하는 부분(빨간 깜빡임)이 색으로 분리된다. 비교가 r 을 넘어 확장되면 박스가 우측으로 *늘어나며* l, r 이 갱신된다 (박스의 좌측 경계는 절대 후퇴 안 함을 강조하기 위해 좌측 경계는 굵은 고정 막대로).

강조 장치:
- **거울 호의 잔상** — 호가 그려지고 200ms 잔상 후 사라짐. 이 잔상이 *왜 비교를 건너뛸 수 있는지* 의 시각적 근거.
- **r 의 트레일** — 박스 우측 경계가 갱신될 때 이전 r 위치에 옅은 점선 자국을 남겨, "절대 줄지 않음" 을 우측 누적 자국으로 보여준다.
- **건너뛴 비교 vs 실제 비교의 색 분리** — 연두(이미 안다) vs 빨강(지금 비교 중) vs 회색(아직 모름).

## 4. 다른 대상과의 구별점

가장 가까운 친척 2종과의 시각 차이:

### Naive 패턴 매칭과의 차이
- Naive: 패턴이 텍스트를 따라 *통째로 미끄러지는* 슬라이더. 매번 처음부터 비교, 박스/메모리 없음.
- Z: 박스라는 *기억 영역* 이 화면에 항상 떠 있다. 거울 호는 Naive 에는 절대 등장하지 않는다.

### KMP 와의 차이
- KMP: 패턴 옆에 별도의 *실패 함수 테이블* 이 그려지고, 미스매치 시 점프 화살표가 패턴 안에서 좌측으로 점프.
- Z: 점프 화살표가 아니라 *거울 호* 가 단일 트랙 안에서 좌우 대칭으로 그려진다. Z 배열은 *결과* 로 트랙 아래에 누적되지, 점프 표로 쓰이지 않는다.

핵심 시각 차별 포인트:
- **박스의 면적성** — Z 만 사각형 면적으로 메모리를 표현. KMP 는 표, Naive 는 없음.
- **거울 호의 곡선** — Z 고유. 좌우 대칭 곡선이 트랙 위에 떠오른다.
- **r 의 단조 트레일** — Z 만의 시각적 약속.

## 5. 달성해야 할 시각 요소

총 5개. 각 항목은 (이름 / 설명 / 구현 힌트).

### 5.1 Z-box 사각형 (반투명 면적)
- **설명**: 셀 트랙 위를 덮는 반투명 사각형. 좌측 경계 l 은 굵은 세로 막대(고정성 강조), 우측 경계 r 은 화살촉이 달린 세로 막대(이동성 강조). 박스 내부는 prefix 영역과 같은 채도의 옅은 채색.
- **구현 힌트**:
  - 박스는 `<g>` 그룹으로 처리, l/r 갱신 시 `transform` + `width` 트윈(200ms ease-out).
  - 좌측 경계는 한 번 정해지면 *그 step 안에서는* 고정. 새 박스로 갈아탈 때만 점프.
  - prefix 영역 (트랙 좌측 0..r-l) 과 동일 색상 토큰 사용 — 색상 이름은 `--z-mirror-fill`.

### 5.2 거울 호 (mirror arc)
- **설명**: i 가 박스 안일 때, l 을 축으로 한 베지어 곡선. i 셀 중심에서 시작해 위로 휘어 k = i - l 셀 중심에 도착. 곡선의 양 끝점에 작은 점, 곡선 자체에 화살표 머리.
- **구현 힌트**:
  - SVG `<path>` quadratic Bezier. 제어점은 (l 의 x좌표, 트랙 위쪽 -32px).
  - 그려지는 시간 300ms (stroke-dasharray 애니메이션), 잔상 200ms 후 fade.
  - 호 색상은 거울 영역 채색과 일치시켜 "같은 부분" 신호 강화.

### 5.3 비교 상태 색 분리 (3색 시스템)
- **설명**: 셀의 배경색이 비교 진행 상태를 보여준다. 연두(skipped — 이미 같다고 안다) / 빨강 깜빡(comparing — 지금 비교 중) / 회색(unknown) / 짙은 녹(matched and committed). 깜빡임은 1회 200ms.
- **구현 힌트**:
  - 셀당 `data-state` 속성 + CSS transition. 상태 머신: `unknown → comparing → matched | mismatched`.
  - skipped 영역은 거울 호가 도착하는 순간 *일괄* 연두로 칠해진다. 그 후 r 너머 첫 셀이 comparing 상태로 진입.

### 5.4 Z 배열 동행 막대
- **설명**: 문자 트랙 바로 아래, 같은 셀 폭으로 정렬된 *세로 막대* + 막대 위 숫자. Z[i] 가 확정되는 step 에 막대가 0 에서 위로 자란다.
- **구현 힌트**:
  - 막대 최대 높이는 트랙 높이의 60% 로 제한, 길이가 큰 Z 값은 같은 비율로 스케일링.
  - 자라는 모션 250ms ease-out + 숫자 카운트업 (0 → 최종값).
  - 확정된 Z[i] 셀은 *짙은 녹* 으로 잠금 — 이후 step 에서 변하지 않음을 시각적으로 약속.

### 5.5 r-단조 트레일과 누적 비교 카운터
- **설명**: 트랙 우측 위에 "r 의 최댓값" 라벨이 박스 우측 끝을 따라 이동. 박스 우측이 갱신될 때마다 *이전 r 위치* 에 옅은 점선 자국을 남긴다. 화면 우상단에 "총 비교 횟수: N (Naive 예상: n×m)" 카운터 — 비교 1회마다 +1.
- **구현 힌트**:
  - 트레일은 점선 `<line>` 들의 누적. 각 트레일 자국 opacity 0.3, 색상 박스 채색의 흐릿한 버전.
  - 카운터는 화면 우상단 작은 칩(chip). Naive 비교는 정적 라벨 (계산식 표시), 실제 비교는 동적 정수.

## 6. 시각 구성

**중심 시각**: 가로 문자 트랙 (한 줄). 박스, 호, 거울 셀 채색이 모두 이 트랙 위에서 일어난다.

**보조 요소** (위→아래):
1. (상단) 짧은 캡션 영역: 현재 step 의 한 문장 설명 ("i=5 가 박스 [3,7] 안에 들어왔습니다. 거울 위치 k=2 의 답 Z[2]=3 을 빌려옵니다.").
2. (좌상) "Z-알고리즘이란?" 2~3문장 카드 (학습자 진입용 — Stage 1 보고서의 캡션 활용).
3. (우상) 누적 비교 카운터 칩.
4. (중앙) **문자 트랙 + Z-box 사각형 + 거울 호 + i/l/r/k 포인터 라벨** (시각화 본체).
5. (트랙 바로 아래) **Z 배열 동행 막대**.
6. (하단) 의사코드 6~8줄 + 현재 라인 하이라이트.
7. (최하단) 단계 진행 컨트롤: ⏮ ◀ ▶ ⏭ + 자동 재생 토글 + 속도 슬라이더 + "패턴 매칭 모드" 토글.

**요소 간 관계**:
- 박스 ↔ prefix 영역: 같은 색 (invariant).
- 거울 호 ↔ k 위치: 호의 끝점이 k 셀에 정확히 닿는다.
- Z 배열 막대 ↔ 거울 호: 호가 가리키는 k 의 막대 높이가 *그대로 i 위치로 복사* 되며 자라는 모션.
- 의사코드 ↔ 트랙 셀 색: 코드 라인의 변수명(i, l, r, k)이 트랙 라벨과 동일 색상 토큰.

## 8. 진행과 이벤트

### 주요 step 단위
한 step = "i 가 한 칸 이동하고 그 위치의 Z[i] 가 확정될 때까지의 미니 시퀀스".

### 단계별 시각 변화

각 step 안에서 다음 마이크로 모션이 일어난다:

1. **i 이동**: i 포인터가 한 칸 우측으로 이동 (100ms).
2. **케이스 분기 판정**:
   - **i > r 인 경우**: 캡션 "박스 바깥. 처음부터 비교 시작." → i 위치부터 회색 셀이 빨강 깜빡임으로 진입하며 한 칸씩 비교 진행 → 매치 길이만큼 박스가 *새로 그려짐* (좌측 경계는 i, 우측 경계는 i+Z[i]-1).
   - **i ≤ r 인 경우**: 캡션 "박스 안. 거울 위치 k=i-l 의 답 빌려보기." → **거울 호 등장** (300ms) → 접두사 쪽 k 셀의 Z[k] 만큼 영역이 *연두로 복사* 되어 i 위치에 붙음 → Z[k] vs r-i+1 비교 위젯 (캡션 옆에 작은 부등호 그래픽) → 결과에 따라:
     - Z[k] < r-i+1: Z[i] = Z[k] 확정, 비교 없이 다음 step.
     - Z[k] > r-i+1: Z[i] = r-i+1 확정, 비교 없이 다음 step.
     - Z[k] = r-i+1: r 너머에서 *추가 실제 비교* (빨강 깜빡임) → 박스 *우측 확장*.
3. **Z 배열 막대 갱신**: 막대가 0 에서 Z[i] 값까지 자라며 숫자 카운트업.
4. **카운터 갱신**: 실제 비교가 일어났으면 카운터 +n.

### 사용자 제어 지점

- ⏮ 처음으로 / ⏭ 끝으로 / ◀ 이전 step / ▶ 다음 step
- ▶ 자동 재생 토글 (1배~4배 속도 슬라이더)
- **단계 내 미세 분기 일시정지 옵션** — Case 2a/2b/2c 가 발생하는 step 에서 부등호 비교 위젯이 떴을 때 자동으로 0.5초 추가 정지 (학습자가 분기를 음미할 시간).
- **"패턴 매칭 모드" 토글** — 켜면 입력이 두 줄 (패턴 P / 텍스트 T) 로 분리되어 받아지고, 내부적으로 `P + $ + T` 결합 문자열의 Z 배열을 계산. Z[i] = |P| 가 되는 위치를 *깜빡이는 별표* 로 표시.
- **입력 문자열 편집** (트랙 위 셀 클릭으로 글자 변경) — 즉시 처음으로 되감기.

## 10. 레퍼런스 대비 가치

A타입 — 공통 요소를 어떻게 더 잘 다루는가:

### 공통 요소 정교화

1. **Z-box 의 면적성** — R3 는 박스를 그리지만 면적 채색이 약하다. R5 는 셀 색만 쓴다. 본 기획은 *반투명 사각형 + prefix 영역 동일 채색* 으로 invariant 를 면적으로 직접 보여준다.
2. **거울 호의 운동화** — R2 의 정적 호를 *애니메이션* 으로. R3 의 색 분기를 *공간적 호의 모션* 으로 끌어올린다. 거울이 비추는 행위가 시간축에 펼쳐진다.
3. **케이스 분기의 단계적 노출** — R3 는 케이스를 색만 다르게 하지만, 본 기획은 부등호 비교 위젯(Z[k] vs r-i+1)을 캡션 옆에 띄워 2a/2b/2c *세 분기를 명시적으로* 학습자에게 보여준다.

### 표준이 놓치는 추가 요소

1. **r 의 단조 트레일** — 어떤 레퍼런스도 "박스 우측이 절대 줄지 않음" 을 시각적 누적 자국으로 보여주지 않는다. 본 기획의 점선 트레일은 O(n) 직관의 결정적 단서.
2. **건너뛴 비교 영역의 연두색 일괄 채색** — 거울 호가 도착하는 순간 Z[k] 만큼 영역이 *한 번에* 연두로 칠해지는 모션. R3 가 색만 분기하지만 본 기획은 "이만큼은 비교 안 했다" 를 *면적의 즉시 점등* 으로 보여준다.
3. **누적 비교 카운터 + Naive 대비** — R4 가 일부 시도했으나 표준화 안 됨. 본 기획은 우상단 칩으로 상시 표시 + Naive 예상치 병기로 O(n) 우월성을 *수치로* 체감.
4. **패턴 매칭 응용 모드 통합** — 대부분 자료가 Z 배열 자체에 그치나, 본 기획은 한 화면 안에서 토글로 응용까지 잇는다. 학습 동선의 단절 제거.

### 종합

표준 시각 언어를 보존하면서 (트랙/박스/호/막대) **거울 비추기를 운동으로**, **건너뛴 비교를 면적으로**, **단조성을 누적 자국으로** 표현해 정적 강의 노트와 단편적 데모 사이의 빈틈을 메운다.

### 시각화 안에 포함되는 짧은 설명 텍스트 (개념 카드)

> **Z-알고리즘이란?**
> 문자열의 각 위치에서 "처음과 얼마나 겹치는지" 를 한 번의 좌→우 스캔으로 모두 구합니다. 지금까지 발견한 가장 오른쪽 매치 구간 [l, r] 을 박스로 기억하고, 새 위치가 박스 안이면 거울 위치의 답을 빌려 비교를 건너뜁니다. r 이 줄지 않아 전체가 O(n) 입니다.

### 단계별 캡션 예시 (step 진행 시 상단에 갱신)

- "i=1: 박스 바깥, 처음부터 직접 비교."
- "i=4: 박스 [3,6] 안. 거울 위치 k=1 의 답 Z[1]=2 를 빌립니다."
- "i=5: 거울이 박스 끝을 넘겠다 — 실제 비교로 박스를 우측 확장."
- "i=7: 박스 우측 r 이 갱신. 점선 자국이 r 의 단조 증가를 기록."

# 테이블과 키 시각화 기획서

### 0. 기본 정보

- 대상 이름: 테이블과 키 (Tables and Keys)
- 도메인: 데이터베이스 / 관계모델
- type: concept
- 진행 모델: 정적
- 본질 한 문장: 테이블은 같은 형태의 행을 모은 이름 붙은 격자이고, 그 안의 기본키가 한 행을 유일하게 식별하며 다른 테이블의 외래키가 그 기본키 값을 가리켜 두 격자를 일관된 한 구조로 엮는다.

### 1. 시그니처 행동

한 격자의 기본키 컬럼이 모든 행의 정체를 굳건히 잠그는 동안, 다른 격자의 외래키 컬럼이 그 잠긴 값을 가리키며 두 격자가 같은 값으로 짝을 맺는다.

### 2. 시각적 은유

표준 메타포 두 갈래 — "이름 붙은 사각 격자 테이블" 과 "두 컬럼을 잇는 곡선 + 카디널리티 기호" — 를 한 화면에서 합쳐 정교화한다. 화면에는 두 개의 카드형 격자가 좌우로 놓인다. 각 격자 상단에는 진한 띠로 테이블 이름이 박히고, 그 아래로 컬럼 헤더 줄과 본문 행들이 균일한 그리드 라인으로 정돈된다. 키 컬럼은 좌측 행간 슬롯에 작은 열쇠 아이콘을 두르고, 컬럼명에는 굵기 + 밑줄을 입혀 다른 컬럼과 시각적으로 한 단 떠 있다. 기본키는 진한 색의 채워진 열쇠, 외래키는 같은 형태이되 윤곽선만 그어진 열쇠 — 같은 어휘 안에서 정의하는 자리와 참조하는 자리의 비대칭이 채움 vs 윤곽으로 즉시 갈린다. 두 격자 사이에는 부드러운 베지어 곡선이 한 줄 그어져, 왼쪽 테이블의 기본키 컬럼 끝점에서 출발해 오른쪽 테이블의 외래키 컬럼 끝점에 닿는다. 곡선 양 끝에는 Crow's Foot 의 카디널리티 기호(한 자리는 짧은 두 줄, 다른 자리는 까마귀 발 세 줄)가 붙어 1:N 관계가 위치만으로 강제된다. 셀 안의 값은 순수 텍스트지만, 외래키 셀과 그 참조 대상 기본키 셀은 같은 옅은 배경 강조색으로 짝지어져 "이 값과 저 값은 같은 값이다" 가 색의 일치로 보존된다.

### 3. 결정적 순간

한 컷에서 학습자가 두 격자 사이에 그어진 곡선을 따라 시선을 옮기다, 오른쪽 테이블의 어느 외래키 셀 값과 왼쪽 테이블의 어느 기본키 셀 값이 정확히 같은 글자임을 확인하고, 그 한 쌍의 셀이 같은 옅은 배경으로 동시에 빛나는 장면이다. 그 옆에는 같은 격자 안에 들이밀린 후보 행 한 줄이 기본키 컬럼에서 기존 행과 같은 값을 가진 채 빨간 충돌 표식으로 막혀 있는 작은 인서트가 함께 떠 있어, "키가 행을 유일하게 만든다" 와 "외래키 값은 어딘가에 실재해야 한다" 두 제약이 한 화면 한 시점에 공존한다.

- 중심 한 컷: 좌 회원 테이블 + 우 주문 테이블 + 곡선 한 줄 + 짝지어진 두 셀의 같은 배경 강조 + 곡선 양 끝 카디널리티 기호.
- 결정적 인서트: 회원 테이블 아래에 옅은 회색으로 들이밀린 후보 행 한 줄. 기본키 컬럼 셀이 빨간 테두리로 굵어지며 빨간 X 표식이 떠 있고, 그 옆 작은 글씨로 "이 값은 이미 있다" 가 한 줄.
- 결정적 인서트: 주문 테이블 아래에 옅은 회색으로 들이밀린 후보 행 한 줄. 외래키 컬럼 셀이 빨간 테두리로 굵어지며 같은 빨간 X 와 "가리킬 회원이 없다" 가 한 줄.
- 강조 장치: 짝지어진 두 셀의 옅은 배경 강조 / 채움 vs 윤곽 열쇠 아이콘의 비대칭 / 곡선 끝 카디널리티 기호의 비대칭 / 충돌·거부 인서트의 빨간 어휘 통일.

### 4. 다른 대상과의 구별점

- **단순 표 / 스프레드시트**: 화면 위에 단 하나의 격자만 있고, 어떤 컬럼도 다른 컬럼보다 시각적으로 떠 있지 않으며 셀 간 어떤 짝짓기도 없다. 행은 단순한 누적이고, 같은 값이 두 행에 들어와도 충돌 표식이 일어나지 않는다. 테이블과 키는 두 격자가 동시에 존재하고 키 컬럼이 굵기 + 열쇠로 떠 있으며 두 셀이 곡선과 같은 배경으로 짝지어져 있다 — "단일 격자 vs 짝지어진 격자" 가 첫 화면에서 갈린다.
- **JSON · 문서형 자료구조 도식**: 중첩된 중괄호 트리와 들여쓰기로 부모-자식이 위계적으로 표현되고, 같은 값이 여러 자리에 등장해도 그 자체가 곧 사본이지 참조가 아니다. 테이블과 키는 같은 격자 형태가 옆으로 두 개 놓이고 한 값이 두 자리에 동시에 살되, 한쪽이 정의·다른 쪽이 참조라는 비대칭이 채움 vs 윤곽 열쇠로 명시된다. "위계적 중첩" vs "수평적 짝짓기" 가 형태로 나뉜다.
- **ERD 의 엔티티-관계 도식 자체 (Chen 표기)**: 사각형은 엔티티의 이름만, 마름모는 관계의 이름만, 타원은 속성의 이름만 담고 행 자체는 그리지 않는다 — 스키마의 추상 골격이다. 테이블과 키는 같은 사각의 격자 안에 컬럼 헤더 + 실제 행이 살아 있고, 키 표기가 컬럼 단위에서 인스턴스 단위까지 일관되게 보인다. "스키마 골격" vs "스키마 + 인스턴스" 가 갈린다.
- **일반 그래프 (노드와 엣지)**: 노드는 단순 원이고 엣지는 노드끼리 잇는 선이며, 노드 안에 컬럼이나 행이 없다. 테이블과 키는 노드 자체가 다열다행 격자이고 엣지가 노드의 한 컬럼에서 다른 노드의 한 컬럼으로 정확히 닿으며 끝점에 카디널리티 기호가 붙는다 — 엣지의 출발지와 도착지가 노드의 어느 부분인지가 본질이다.

### 5. 달성해야 할 시각 요소

1. **이름 붙은 격자 — 컬럼 헤더와 행이 살아 있는 카드형 테이블 두 개**
   - 설명: 테이블이 같은 형태의 행을 모은 이름 붙은 격자라는 사실이 두 격자의 동시 존재로 즉시 보인다. 한 격자만 그리면 "두 테이블 사이의 참조" 라는 본질이 사라진다.
   - 시각 표현: 두 카드형 테이블이 좌우로 놓인다. 상단에는 진한 띠 위 흰 글씨로 테이블 이름(예: 회원 / 주문). 그 아래 컬럼 헤더 줄(옅은 띠 + 굵은 글씨) 과 본문 4~5 행. 모든 셀은 균일한 그리드 라인으로 정돈되고 행 간 옅은 줄무늬로 가독성을 보강한다. 카드 테두리는 1px, 모서리는 살짝 둥글다.

2. **기본키 컬럼의 시각적 무게 — 채워진 열쇠와 굵은 밑줄**
   - 설명: 어느 컬럼이 그 격자의 기본키인지가 다른 컬럼과 즉시 구분되는 무게로 떠 있어야 한다. 기본키가 한 행을 유일하게 만든다는 사실이 컬럼 단위 표기로 굳건히 보존된다.
   - 시각 표현: 기본키 컬럼의 헤더 글자에 굵기 + 밑줄을 입히고, 컬럼 좌측 행간 슬롯마다 진한 색의 채워진 열쇠 아이콘이 작게 박힌다. 기본키 컬럼 본문 셀들은 다른 컬럼보다 한 단 진한 글자 무게로 표시되어 "이 컬럼의 값들이 행을 정의한다" 가 시각 무게로 즉각 식별된다.

3. **외래키 컬럼의 비대칭 시각 — 윤곽 열쇠와 짝지어진 곡선**
   - 설명: 외래키가 단독 컬럼이 아니라 "어딘가의 기본키" 를 향한 컬럼이라는 사실이 형상으로 강제된다. 정의하는 자리와 참조하는 자리가 같은 어휘 안에서 다른 무게로 보여야 PK ↔ FK 의 쌍대성이 보존된다.
   - 시각 표현: 외래키 컬럼은 헤더에 굵기 + 밑줄(같은 키 어휘 유지)을 입히되, 좌측 슬롯의 열쇠 아이콘은 윤곽선만 그어진 비어 있는 열쇠로 표시된다. 두 테이블 사이에는 부드러운 베지어 곡선 한 줄이 그어져, 왼쪽 테이블의 기본키 컬럼 끝점에서 시작해 오른쪽 테이블의 외래키 컬럼 끝점에 닿는다. 곡선 양 끝에는 Crow's Foot 의 카디널리티 기호 — 기본키 쪽에 짧은 두 줄(정확히 하나), 외래키 쪽에 까마귀 발 세 줄(여럿) — 가 붙어 1:N 관계의 비대칭이 위치 어휘로 박힌다.

4. **참조 무결성의 셀 단위 짝짓기 — 같은 값을 같은 배경으로 묶기**
   - 설명: 외래키 셀의 값과 참조 대상 기본키 셀의 값이 같은 값이라는 제약이 텍스트 정의가 아니라 두 셀의 시각적 짝짓기로 보여야 한다. 곡선 한 줄만으로는 "어떤 행과 어떤 행이 짝인가" 가 흐릿하다.
   - 시각 표현: 마우스가 외래키 셀 위에 얹히면 그 셀과 참조 대상 기본키 셀이 동시에 같은 옅은 배경 강조색으로 빛난다. 두 셀 사이에는 곡선과 별개로 가는 점선 연결선이 잠시 떠올라, "이 값과 저 값은 같은 값" 이 시선의 한 호로 묶인다. 기본 상태에서도 같은 값을 가진 두 셀의 글자에는 옅은 색 마커가 깔려 있어 색을 따라 짝을 추적할 수 있다.

5. **충돌과 거부의 인서트 — 키 제약을 사건으로 보기**
   - 설명: 같은 기본키 값으로 들이밀린 후보 행과 참조 대상 없는 외래키 값으로 들이밀린 후보 행이 두 격자의 빈 자리 아래에 빨간 거부 표식으로 떠 있어, "유일성 충돌" 과 "참조 거부" 두 제약이 표준 자료가 그리지 않은 사건으로 보존된다.
   - 시각 표현: 회원 테이블 마지막 행 아래에 옅은 회색 후보 행이 한 줄 들이밀려 있고, 기본키 컬럼 셀이 빨간 테두리 + 빨간 X 아이콘으로 거부 상태다. 그 옆 짧은 텍스트 — "이 값은 이미 있다 — 새 행은 들어올 수 없다". 주문 테이블 마지막 행 아래에도 같은 형식의 후보 행이 들이밀려 있고, 외래키 컬럼 셀이 빨간 테두리 + 빨간 X 와 "가리킬 회원이 없다 — 이 값은 거부된다". 두 인서트는 회색 톤으로 본 격자보다 한 단 가라앉아, 본 격자의 정상 행들과 거부 후보 행이 시각적으로 분리된다.

6. **후보키 둘과 기본키 선택의 분기**
   - 설명: 한 테이블에 행을 유일하게 식별할 수 있는 컬럼이 둘 이상 있을 때, 그중 하나가 기본키로 선택되고 나머지가 대체키로 물러난다는 분기가 시각적으로 보여야 한다. 표준 자료가 결과만 강조해 사라뜨린 선택의 사실을 살린다.
   - 시각 표현: 회원 테이블에서 두 컬럼(예: 학번·이메일)이 후보키로 떠 있다. 기본키로 선택된 컬럼은 채워진 열쇠 + 굵은 밑줄, 대체키로 물러난 컬럼은 같은 형태의 채워진 열쇠 + 굵은 밑줄에 작은 "alt" 라벨이 헤더 우측에 붙어 동등한 자격이지만 한 단 가라앉은 배경색으로 표시된다. 헤더 위쪽에 작은 토글 칩 — "기본키: [학번 ▼]" 가 떠 있고, 토글하면 두 컬럼의 강조가 서로 자리바꿈한다.

### 6. 시각 구성

- **중심 시각**: 좌우로 놓인 두 카드형 테이블. 좌측은 회원 테이블(기본키 컬럼 + 후보키 + 일반 컬럼 두엇), 우측은 주문 테이블(기본키 컬럼 + 외래키 컬럼 + 일반 컬럼 한엇). 두 테이블 사이를 잇는 베지어 곡선 한 줄과 양 끝 Crow's Foot 카디널리티 기호.
- **보조 요소**:
  - 상단 캡션 영역: 짧은 개념 문장 한 줄과 현재 강조 메시지 (예: "외래키 값은 회원 테이블의 학번을 가리킨다").
  - 두 테이블 아래 거부 인서트: 회원 측 유일성 충돌 후보 행 + 주문 측 참조 거부 후보 행. 옅은 회색 톤으로 본 격자보다 한 단 가라앉음.
  - 좌측 회원 테이블 헤더 위 토글 칩: "기본키: [학번 ▼]" — 후보키 ↔ 기본키 자리바꿈.
  - 우측 하단 키 범례: 채워진 열쇠 = 기본키 / 채워진 열쇠 + alt = 대체키 / 윤곽 열쇠 = 외래키 / 짧은 두 줄 = 정확히 하나 / 까마귀 발 = 여럿.
- **요소 간 관계**: 두 테이블의 키 컬럼은 곡선 한 줄로 짝지어지며, 곡선의 출발지와 도착지가 컬럼 끝점에 정확히 닿는다. 셀 단위 짝짓기는 같은 옅은 배경 강조색으로 두 셀을 묶고, 호버 시 점선 연결선이 그 위에 잠시 더해진다. 거부 인서트는 본 격자의 마지막 행 아래에 들이밀려, 본 행들의 정상성과 거부 후보 행의 빨간 어휘가 같은 컬럼 정렬 위에서 대조된다.

- 두 카드형 테이블 폭 각 280px, 중앙 곡선 영역 약 100px (양 테이블 사이 마진 포함). 카드 모서리 6px 둥글림.
- 컬럼 헤더 줄 높이 28px, 본문 행 24px. 본문 4행 + 거부 인서트 1행 = 약 144px. 카드 헤더 띠 32px 포함.
- 곡선은 기본키 컬럼 우측 끝점에서 외래키 컬럼 좌측 끝점까지 베지어. 호 굵기 1.5px, 색은 본 텍스트보다 한 단 옅음. 양 끝 카디널리티 기호 크기 12px.
- 열쇠 아이콘 12×12px. 채움 열쇠는 진한 단색, 윤곽 열쇠는 1.5px 윤곽선만.
- 짝지어진 두 셀의 옅은 배경 강조색은 셀 폭에 꽉 차되 글자 가독성을 해치지 않는 밝은 톤. 호버 시 점선 연결선 0.75px.
- 거부 인서트 행은 본 행과 같은 컬럼 정렬, 행 배경은 옅은 회색 줄무늬 + 빨간 좌측 보더 2px. 빨간 X 아이콘 10×10px, 안내 글자 10px.
- 확장 폭 880px 일 때는 두 카드 폭 각 360px, 중앙 곡선 영역 약 120px 로 늘어 컬럼 간 여백과 카디널리티 기호의 가독성이 한 단 또렷해진다.
- 상단 캡션 40px, 본체 약 320px, 하단 안내 60px, 여백 합쳐 총 약 460px.

### 8. 진행과 인터랙션

정적 중심. 한 컷에 모든 본질이 박혀 있고, 사용자의 가벼운 입력은 이미 떠 있는 시각의 부분을 강조해 보는 정도다. 상태가 진행되는 시뮬레이션이 아니다.

- **외래키 셀 호버**: 그 셀과 참조 대상 기본키 셀이 동시에 같은 옅은 배경 강조색으로 빛나고, 두 셀 사이에 가는 점선 연결선이 잠시 떠오른다. 곡선 자체도 한 단 진해진다. 캡션에 "외래키 값 1001 은 회원 테이블의 학번 1001 을 가리킨다" 가 잠시 뜬다.
- **기본키 셀 호버**: 그 셀을 가리키는 모든 외래키 셀들이 같은 옅은 배경 강조색으로 동시에 빛난다. 한 행이 여러 행에 의해 참조되는 1:N 의 분포가 셀 단위로 즉시 보인다. 캡션에 "이 회원을 가리키는 주문이 N 건" 이 잠시 뜬다.
- **기본키 토글 칩 [기본키: 학번 ▼]**: 학번과 이메일 두 후보키 중 어느 쪽을 기본키로 둘지 토글한다. 토글하면 두 컬럼의 채움 열쇠 + alt 라벨이 자리를 바꾸고, 두 테이블 사이의 곡선 출발점도 새 기본키 컬럼으로 미끄러지듯 옮겨 붙는다. 이 운동은 "후보키는 동등하지만 그중 하나만 대표가 된다" 의 분기를 한 컷의 자리바꿈으로 보여준다. 운동 시간 약 300ms.
- **거부 인서트 보기 토글**: 두 격자 아래의 회색 후보 행 두 줄을 켜고 끌 수 있다. 켜진 상태에서는 빨간 X 와 안내 글이 함께 떠 있고, 끄면 두 격자만 깔끔하게 남는다. 학습자가 "정상" 과 "거부" 를 비교 보기 위한 시각 토글일 뿐 시간 진행이 아니다.
- **셀에서 시선이 벗어나면**: 강조 배경과 점선 연결선이 부드럽게 사라지고 기본 상태(같은 값 글자에 옅은 색 마커만 깔린 상태) 로 돌아간다. 약 200ms 페이드.
- 모든 운동은 250~300ms. 사건이 아니라 강조의 잠깐 변화로만 작동한다.

### 9. 초기 상태와 파라미터

- **초기 상태**: 좌측 회원 테이블에 기본 4행, 우측 주문 테이블에 기본 4행이 채워져 있다. 회원의 학번이 기본키(채움 열쇠 + 굵은 밑줄), 이메일이 대체키(채움 열쇠 + alt 라벨). 주문의 주문번호가 기본키, 학번이 외래키(윤곽 열쇠 + 굵은 밑줄). 두 테이블 사이 곡선이 학번 → 학번(FK) 으로 그어져 있고, 양 끝에 1 자리 짧은 두 줄과 N 자리 까마귀 발 기호가 붙어 있다. 두 격자 아래 거부 인서트가 기본 켜진 상태로 떠 있어 충돌·거부 사실이 첫 화면에 보인다. 캡션은 기본 개념 문장.
- **권장 시연 시퀀스**: 진입 후 약 1초 뒤 외래키 셀 한 줄에 자동 호버 강조가 한 번 일어나, 외래키 1001 → 기본키 1001 의 짝짓기를 첫 시선으로 보여준다. 그 후 사용자가 자유롭게 셀을 호버하거나 토글한다.
- **파라미터**:
  - 회원 테이블 행 수: 기본 4행, 확장 폭 5행. 컬럼 4개 (학번 PK / 이메일 alt / 이름 / 가입일).
  - 주문 테이블 행 수: 기본 4행, 확장 폭 5행. 컬럼 4개 (주문번호 PK / 학번 FK / 금액 / 일자).
  - 거부 인서트: 회원 측 1행(중복 학번), 주문 측 1행(미존재 학번 1099). 토글로 on/off.
  - 키 어휘 색: 기본키 채움 열쇠 진한 톤 1, 대체키 채움 열쇠 + alt 한 단 가라앉음, 외래키 윤곽 열쇠 같은 톤 윤곽.
  - 카디널리티 기호: 1 자리는 짧은 평행 두 줄, N 자리는 까마귀 발 세 줄. 크기 12px.
  - 호버 강조 운동 시간: 진입 200ms / 이탈 200ms. 토글 자리바꿈 300ms.

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> 테이블은 같은 형태의 행을 모은 이름 붙은 격자이고, 각 행은 기본키 컬럼의 값으로 유일하게 식별된다. 다른 테이블의 외래키 컬럼은 그 기본키 값을 가리켜 두 테이블을 잇고, 같은 키 값을 가진 두 행은 공존할 수 없으며 참조 대상 없는 외래키 값도 받아들여지지 않는다. 키와 참조는 표기가 아니라 제약이며, 두 제약이 동시에 살아있을 때 격자들이 한 구조로 묶인다.

**학습자 안내 (보조 캡션 회전 문구)**

- "채워진 열쇠는 정의하는 자리, 윤곽 열쇠는 참조하는 자리."
- "외래키 값은 어딘가의 기본키 값과 같은 값이다."
- "곡선의 한 끝은 정확히 하나, 다른 끝은 여럿."
- "후보키는 동등하지만 그중 하나만 대표가 된다."
- "키 없이는 같은 값을 가진 두 행을 구분할 길이 없다."

**사건별 짧은 메시지**

- 외래키 셀 호버: "외래키 값 1001 은 회원의 학번 1001 을 가리킨다."
- 기본키 셀 호버: "이 회원을 가리키는 주문이 N 건이다."
- 기본키 토글: "이메일도 후보키였다 — 이번엔 학번 대신 이메일을 기본키로 둔다."
- 회원 측 거부 인서트: "이 값은 이미 있다 — 새 행은 들어올 수 없다."
- 주문 측 거부 인서트: "가리킬 회원이 없다 — 이 외래키 값은 거부된다."
- 곡선 호버: "회원 학번 → 주문 학번. 한 회원이 여러 주문을 가질 수 있다."

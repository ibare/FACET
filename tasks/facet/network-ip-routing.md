# IP 라우팅 시각화 기획서

### 0. 기본 정보

- 대상 이름: IP 라우팅 (IP routing)
- 도메인: 컴퓨터 네트워크 (network)
- type: systemBehavior
- 진행 모델: 메시지 시퀀스형 (한 패킷의 hop-by-hop 운동이 시간 진행으로 펼쳐지는 혼합)
- 본질 한 문장: IP 라우팅은 한 패킷이 출발지 호스트에서 목적지 호스트에 닿기까지, 매 hop 마다 그 라우터의 표 한 장만으로 next-hop 을 정하고 TTL 을 한 칸 줄여 한 걸음씩만 전진하는 hop-by-hop 분산 결정의 시스템 행동이다.

### 1. 시그니처 행동

패킷은 직행하지 않고 라우터마다 한 번씩 멈춰, 그 자리에서 자기 표만 펼쳐 가장 긴 일치 prefix 가 가리키는 next-hop 으로 한 걸음만 떠나며, 그때마다 TTL 이 한 칸 깎인다.

### 2. 시각적 은유

표준 메타포인 "노드 + 링크 그래프 + 봉투처럼 미끄러지는 패킷" 을 채택하고, 그 위에 두 가지 정교화를 더한다. 첫째, 라우터 박스마다 라우팅 테이블이 그 박스 옆에 펼침 패널로 붙어 있되 **현재 패킷이 도착한 라우터의 표만 펼쳐 보이고 나머지 라우터의 표는 박스 안에 접혀 있다** — 라우터의 시야 한계가 위치만으로 강제된다. 둘째, 패킷은 단순한 색 봉투가 아니라 **헤더 펼침 카드** 로 그려져 src·dst·TTL 세 필드가 항상 카드 위에 노출되며, 매 hop 마다 TTL 칸의 숫자가 카운트다운한다. 화면 중앙에는 호스트 A — R1 — R2 — R3 — 호스트 B 의 4 노드 사슬과 R1 옆 분기 라우터 R4 한 대로 이루어진 작은 망형 토폴로지가 놓이고, 호스트는 라운드형 모니터 아이콘, 라우터는 둥근 모서리의 박스 + 화살표 마크로 위상이 구분된다. 패킷 카드가 링크 위를 미끄러질 때 링크 자체가 짧게 빛나고, 라우터에 도착한 순간 그 라우터의 표가 커튼처럼 펼쳐져 longest prefix match 의 비트 비교가 그 자리에서 발화한다. 다른 라우터의 표 영역은 옅은 회색 안개 레이어로 덮여 "지금 그 자리는 보이지 않는다" 가 시각으로 운반된다.

### 3. 결정적 순간

호스트 A 가 만든 패킷 한 개가 R1 에 도착하는 그 한 컷이다. R1 옆에 그동안 접혀 있던 라우팅 테이블이 펼쳐지고, 패킷 카드의 dst 필드(예: 203.0.113.42) 의 비트 표시가 표의 여러 prefix 행과 동시에 비트 단위로 비교된다. 두 prefix 가 매칭되지만(예: 203.0.0.0/16 과 203.0.113.0/24) 한 행이 더 길게 일치하며 그 행만 점진적으로 진해지고, 다른 행들은 흐려진다. 채택된 행이 가리키는 next-hop(R2) 으로 카드가 떠나는 직전, TTL 칸의 숫자가 64 에서 63 으로 한 칸 카운트다운한다. 학습자는 이 한 컷에서 "이 라우터는 자기 표만 본다", "여러 일치 중 가장 긴 것이 이긴다", "한 hop 마다 수명이 한 칸 줄어든다" 세 사실을 동시에 받아들인다.

- 전: 호스트 A 의 책상 위에 패킷 카드가 막 만들어져 있다. src=A, dst=203.0.113.42, TTL=64. R1·R2·R3·R4 의 라우팅 테이블은 모두 박스 안에 접혀 있고, 옅은 회색 안개로 덮여 있다. 링크들은 차분한 회색 굵기 1px.
- 중: 카드가 A→R1 링크를 미끄러져 R1 에 도착한다. R1 옆의 표가 커튼처럼 좌→우로 펼쳐지고 안개가 R1 영역에서만 걷힌다. 표의 prefix 컬럼 옆에 패킷의 dst 비트 표시줄이 위에 떠올라 한 비트씩 좌→우로 일치 검사가 흐른다. 일치 행 두 개의 길이가 차오르고, 더 긴 일치를 가진 행만 굵은 강조 테두리를 얻으며 짧은 일치 행은 옅어진다. 채택 행의 next-hop 셀(R2) 이 펄스로 한 번 빛난다.
- 후: 카드가 R1 에서 R2 방향 링크 위로 떠나는 순간, TTL 칸이 64→63 으로 한 칸 카운트다운한다. R1 의 표는 다시 박스 안으로 접히고 안개가 R1 위에 다시 깔린다. 같은 운동이 R2 도착 시 R2 의 표 펼침으로, 그다음은 R3 의 표 펼침으로 차례로 반복된다. 호스트 B 에 도착한 순간 카드의 TTL 은 61, 헤더의 src·dst 는 처음 그대로다.
- 강조 장치: 표의 커튼 펼침·비트 비교의 좌→우 흐름·일치 길이의 차오름·짧은 일치 행의 페이드·next-hop 셀의 펄스·TTL 칸의 카운트다운 운동·접힌 표 위의 안개 레이어·도착한 라우터 박스 둘레의 밝은 동심원 강조.

### 4. 다른 대상과의 구별점

- **L2 스위칭 (이더넷 스위치 + MAC 학습)**: 같은 봉투 운동을 그리지만 결정의 단위가 다르다. 스위치는 MAC 주소 표를 보고, 모르는 MAC 이면 모든 포트로 같은 프레임을 한꺼번에 뿌리는 플러딩이 정상 동작이다. 시각적으로는 한 노드에서 N 가닥의 가지가 한꺼번에 뻗는 부채꼴 운동이 발화한다. IP 라우팅은 결정의 단위가 prefix 이며, 어떤 라우터도 패킷을 N 가닥으로 복사해 뿌리지 않는다. 한 hop 마다 단 한 줄의 화살표만 next-hop 으로 뻗는다 — "한 점 → N 가닥 동시" 와 "한 점 → 한 가닥, hop 마다 다시 한 가닥" 의 차이가 운동의 모양 자체로 즉시 구별된다. 또한 스위치의 표는 트래픽을 보며 학습으로 채워지고 시간 단위 만료가 있지만, 본 시각화의 라우팅 테이블은 "주어진 것" 으로 정적 톤이다.
- **회선교환 (전화망)**: 송신 직전에 출발지에서 목적지까지 한 줄의 전용 회로가 미리 예약되고, 그 회로 위로만 음성·데이터가 흐르는 모델. 시각적으로는 발신 직전에 호스트 A 부터 호스트 B 까지의 한 경로가 굵은 띠로 한꺼번에 빛나며 예약 사건이 발화하고, 그 후 모든 데이터는 그 띠 위만을 단방향이 아닌 양방향으로 흐른다. IP 라우팅은 어떤 시점에도 출발지에서 목적지까지의 길이 미리 그어지지 않는다 — 길은 매 hop 마다 그 라우터의 결정으로 사후에 형성되며, 같은 출발-도착의 두 패킷이 다른 라우터의 결정에 따라 다른 길로 갈 수도 있다. "미리 깔린 한 줄" 과 "사후에 형성되는 N 개의 결정" 의 차이가 시작 직전 굵은 띠 vs 매 hop 펼쳐지는 표로 즉시 구별된다.
- **우편물 분류 (모든 라우터가 같은 글로벌 지도를 안다는 오해)**: 흔한 비유는 "라우터 = 우체국, 우체국은 세계 지도를 갖고 있어 직접 가는 길을 안다" 인데, 이 비유는 IP 라우팅의 핵심을 거짓으로 만든다. 실제로 어느 라우터도 전역 토폴로지를 알지 못한다. 본 시각화는 R1 이 결정할 때 R2·R3·R4 의 표 영역이 어둡게 가려지는 시야 한계 운동으로 그 사실을 못 박아, "라우터는 세계 지도를 본다" 가 아니라 "라우터는 자기 표만 보고 다음 한 걸음만 정한다" 를 그림 자체로 강제한다. 같은 우편 비유라도 "각 우체국이 다음 우체국까지의 화살표만 안다" 로 정정되는 모양 차이가, 다른 라우터 영역의 안개 레이어로 즉시 보인다.

### 5. 달성해야 할 시각 요소

1. **호스트–라우터–호스트 망형 토폴로지 + 끝점의 위상 차이**
   - 설명: 출발지 호스트와 목적지 호스트가 네트워크의 끝점이고 라우터는 중간 노드라는 사실이 노드의 시각적 위상 차이로 명시되어야 한다. 토폴로지는 직선 사슬 한 줄이 아니라 분기 한 곳을 가진 작은 망형이어서, longest prefix match 의 가치(같은 패킷이 분기 라우터로 가지 않고 더 긴 일치를 가진 본선 라우터로 가는 운동) 가 그림 위에서 구별된다. essence 본질 5(끝점과 중간 노드의 구별), 본질 1(hop-by-hop 의 무대) 을 운반한다.
   - 시각 표현: 좌측 끝에 호스트 A(라운드형 모니터 아이콘, 라벨 "호스트 A · 192.0.2.10"), 우측 끝에 호스트 B(같은 모양·다른 색, 라벨 "호스트 B · 203.0.113.42") 가 놓인다. 그 사이에 R1—R2—R3 의 라우터 사슬이 가로로 늘어서 있으며, R1 에서 위쪽으로 뻗은 분기 링크가 R4 (라벨 "R4 · 198.51.100.0/24 망") 에 닿는다. 라우터 박스는 둥근 모서리 직사각형 + 화살표 마크 한 개, 호스트 박스는 둥근 모니터 아이콘. 링크는 차분한 회색 굵기 1px, 패킷이 흐르는 동안에는 그 링크만 짧게 굵은 색으로 빛난다. 호스트와 라우터 사이에는 위상 색조 차이(호스트 = 부드러운 청록 톤, 라우터 = 차분한 회색 톤) 가 명시되어 끝점과 중간 노드가 즉시 구별된다.

2. **현재 라우터의 표 펼침 + 다른 라우터 표의 안개 — 라우터의 시야 한계**
   - 설명: 라우터는 자기 표만 알고 다른 라우터의 표는 모른다는 사실이 그림 자체의 위상으로 강제되어야 한다. 패킷이 도착한 라우터의 표만 한 시점에 펼쳐지고, 다른 라우터의 표 영역은 안개 레이어로 덮여 가려진다. 학습자가 한 시점에 볼 수 있는 결정 정보는 "지금 라우터의 표 한 장" 뿐이며, "라우터는 전역 토폴로지를 본다" 는 오해가 정면으로 차단된다. essence 본질 2(결정의 지역성), 강화 요소 3(시야 한계) 을 정면으로 운반한다.
   - 시각 표현: 각 라우터 박스 옆 또는 아래에는 라우팅 테이블의 자리(폭·높이가 정해진 펼침 패널 영역) 가 박스 안에 접힌 상태로 미리 마련되어 있다. 접힌 상태에서는 패널 자리에 옅은 회색 안개 레이어 + "표 접힘" 작은 라벨 + 자물쇠 모양의 보조 표지가 표시된다. 패킷이 그 라우터에 도착하는 순간 그 박스의 안개가 200ms 동안 걷히고 패널이 좌→우 커튼 운동(약 250ms) 으로 펼쳐진다. 펼쳐진 패널은 prefix · next-hop · interface 세 컬럼의 표(4~6 행) 로, 행 텍스트는 명료한 단색·1pt 큰 글자. 패킷이 그 라우터를 떠나는 순간 패널이 다시 우→좌 커튼으로 접히며 안개 레이어가 다시 깔린다. 동시에 두 라우터의 표가 펼쳐지는 일은 없다 — 한 시점에 단 한 라우터의 표만 보인다는 약속이 전역 규칙으로 강제된다. 표가 정적·구분된 톤(배경에 옅은 격자, 행 텍스트는 진한 색이지만 표 자체는 잠긴 자물쇠 보조 표지를 항상 동반) 으로 그려져 "이 시각화 안에서 표는 변하지 않는다"(=데이터 평면 베이스) 가 위상으로 운반된다.

3. **longest prefix match 의 비트 비교 운동 — 가장 긴 일치가 이긴다**
   - 설명: 라우터의 결정 순간에 라우팅 테이블의 여러 prefix 행이 동시에 노출되고, 패킷의 목적지 IP 의 비트가 각 행의 prefix 와 한 비트씩 일치 검사되며 일치 길이가 차오른다. 가장 긴 일치 행이 점진적으로 강조되고 짧은 일치 행은 흐려지며, 최종 채택된 한 행이 next-hop 으로 발화한다. "왜 longest 가 이기는가" 가 비트 비교 운동으로 운반된다. essence 본질 3(LPM), 강화 요소 1(LPM 의 운동형 시각) 을 정면으로 운반한다.
   - 시각 표현: 표가 펼쳐지면 표의 prefix 컬럼 위쪽에 패킷의 dst IP 의 비트 표시줄(예: "11001011 00000000 01110001 00101010") 이 한 줄로 떠오른다. 각 prefix 행의 prefix 컬럼 안에는 그 행의 비트 패턴(prefix 길이만큼) 이 같은 정렬로 정렬된다. 비트 비교는 좌→우 방향으로 약 350ms 동안 흐르며, 매 비트마다 일치 칸은 짧은 초록 펄스로, 불일치 첫 칸은 빨간 X 표지로 발화한다. 일치 행의 일치 길이는 "행 옆의 가로 막대 게이지" 로 동시에 차오르고, 비교가 끝났을 때 가장 긴 막대를 가진 행이 굵은 강조 테두리(2px) 를 얻고 그 행의 next-hop 셀이 한 번 펄스로 빛난다. 짧은 일치 행은 막대가 차오르다 멈춘 채로 행 자체가 한 단계 옅어지고, 일치하지 않은 행은 막대가 비어 있으며 가장 옅은 톤으로 잦아든다. 비교 운동이 끝나면 패킷 카드가 채택 행의 next-hop 방향 링크 위로 떠난다. 분기 라우터 R4 가 토폴로지에 있는 이유가 이 운동에서 명시된다 — 짧은 일치 prefix 가 R4 를 가리킨다 해도, 더 긴 일치를 가진 행이 R2 를 가리키면 패킷은 R2 로 간다.

4. **패킷 헤더 카드 + TTL 카운트다운 — 정체성 보존과 수명의 사건성**
   - 설명: 패킷이 색·번호로 고유한 정체성을 가지고, 모든 hop 을 가로지르는 동안 그 정체성이 유지된다. 카드 위에 src·dst·TTL 세 필드가 항상 노출되어 매 hop 에서 src·dst 는 그대로이고 TTL 만 한 칸 줄어드는 비대칭이 운동으로 운반된다. TTL 이 0 에 닿으면 패킷이 정지·폐기되는 실패 사건도 시각으로 발화한다. essence 본질 1(hop 운동), 본질 4(TTL 카운트다운과 폐기), 강화 요소 2(TTL 게이지의 사건성), 강화 요소 4(헤더 펼침과 정체성 보존) 를 운반한다.
   - 시각 표현: 패킷은 너비 약 110px·높이 약 36px 의 둥근 모서리 카드. 카드 안에는 세 줄의 헤더 표시: "src: 192.0.2.10", "dst: 203.0.113.42", "TTL: 64". TTL 칸은 카드 우측에 배치된 폭 28px 의 작은 게이지로, 숫자 + 가로 막대(꽉 찬 64 부터 0 까지의 비율) 의 두 표현이 동시에 보인다. 카드 색은 발신 호스트 색을 그대로 이어받는다(호스트 A 발 = 부드러운 청록). 카드는 링크 위를 약 400ms 의 곡선 운동으로 미끄러지고, 라우터에 도착하면 라우터 박스 옆 정해진 자리에 안착해 LPM 비교가 끝날 때까지 머문다. 카드가 라우터를 떠나는 순간 TTL 칸의 숫자가 1 감소(예: 64→63) 하며 게이지 막대가 한 칸 줄어드는 200ms 의 미소 운동이 발화하고, 임계(TTL ≤ 4) 근방에서는 게이지가 주황으로 색이 바뀌며 카드 둘레에 약한 맥동이 추가된다. TTL 이 0 에 닿는 순간 카드가 회색으로 즉시 페이드되고 X 표지가 카드 위에 발화하며 정지 — 다음 링크로 떠나지 않는다. 캡션에 "TTL 이 0 이다 — 패킷이 폐기되었다" 가 잠시 뜬다. 짧은 시연 시퀀스로 정상 도착(TTL 충분) 과 폐기(처음 TTL 을 작게 설정) 두 시나리오가 모두 보존된다.

5. **표 행과 토폴로지 링크의 시각적 동치 — 채택 행이 가리키는 그 길로 카드가 떠난다**
   - 설명: 라우팅 테이블의 채택 행이 단순한 텍스트가 아니라 토폴로지 위의 한 링크와 동치라는 사실이, 채택 행에서 next-hop 라우터까지 이어지는 짧은 시선 화살표로 운반된다. 표 → 링크 → 카드의 운동 흐름이 한 사건의 한 동선으로 보존된다. essence 본질 2·3(결정의 지역성과 LPM 결과), 강화 요소 4(헤더 펼침과 동시화) 를 보강한다.
   - 시각 표현: LPM 비교가 끝나 채택 행이 굵은 테두리를 얻는 순간, 그 행의 next-hop 셀(예: "203.0.113.0/24 → 10.0.0.2 / R2") 에서 짧은 곡선 화살표 한 줄이 토폴로지 위 R2 박스로 약 250ms 동안 그어진다. 그 화살표가 안착한 직후 카드가 표 자리에서 떠나 같은 R1→R2 링크 위로 미끄러진다 — 이 두 운동이 시간상으로 이어져 "표의 한 행 = 토폴로지 위의 한 링크" 가 시각으로 동치화된다. 화살표는 카드가 다음 링크로 떠나는 순간 페이드아웃해 사라지고, 표가 접히면 흔적이 모두 사라진다. 분기 라우터 R4 로 가는 행이 짧은 일치라면 화살표가 그 행에서 발화하지 않으므로, R4 는 한 hop 도 받지 않는다 — LPM 의 결과가 토폴로지 위에서 가지 않는 길의 부재로 보존된다.

### 6. 시각 구성

- **중심 시각**: 화면 중앙의 토폴로지 그래프 — 호스트 A, 라우터 R1·R2·R3·R4, 호스트 B 의 6 노드 + 5 링크. R1 에서 위쪽으로 R4 분기 한 줄. 라우터 박스는 둥근 모서리 직사각형 + 화살표 마크, 호스트 박스는 라운드 모니터 아이콘. 링크는 점선이 아닌 실선 1px 회색, 패킷이 흐르는 링크만 굵은 색으로 짧게 빛난다. 토폴로지 위에는 한 시점에 단 한 라우터의 표만 펼쳐져 있고, 나머지 라우터 영역은 옅은 안개로 가려져 있다.
- **보조 요소**:
  - 라우터 옆 펼침 표 패널: 도착한 라우터의 prefix · next-hop · interface 세 컬럼 표. 표 위쪽에 패킷 dst 의 비트 표시줄 + 각 행 옆의 일치 길이 게이지.
  - 패킷 헤더 카드 패널: 토폴로지 위에서 카드가 링크를 미끄러지거나 라우터 옆에 안착하는 형태로 떠 있다. 카드 안에는 src·dst·TTL 세 줄.
  - 좌측 호스트 A 옆의 발신 큐: 다음에 발신될 패킷 카드 1~2 개가 호스트 박스 안쪽에 작은 색 카드 줄로 대기.
  - 우측 호스트 B 옆의 수신 누적: 도착한 카드들이 호스트 박스 옆에 색 사슬로 누적되어 그 호스트가 받은 패킷의 연대기를 이룬다.
  - 우상단 보조 정보 영역: 현재 사건의 짧은 한 줄 ("R1 에서 LPM 비교 중 — 가장 긴 일치 행이 R2 를 가리킨다"), 다음 사건 진행 안내.
  - 상단 캡션 영역: 짧은 개념 문장 한 줄 + 현재 강조 메시지.
  - 하단 컨트롤 영역: 발신·한 hop·자동 시연·일시정지·속도·초기 TTL 슬라이더·초기화.
- **요소 간 관계**: 한 hop 의 운동은 (라우터 도착 → 안개 걷힘 + 표 펼침 커튼 → dst 비트 표시줄 떠오름 → 좌→우 비트 비교 + 일치 길이 게이지 차오름 → 채택 행 굵은 테두리 + next-hop 셀 펄스 → 채택 행 → next-hop 라우터 시선 화살표 → 카드가 표 자리 떠남 + TTL 칸 카운트다운 → 표 접힘 + 안개 다시 깔림 → 카드가 다음 링크 위 미끄러짐) 의 9 단 시퀀스로 1:1 대응된다. 라우터의 시야 한계는 안개의 위상으로, LPM 의 비교는 비트 표시줄과 게이지의 운동으로, 결정의 결과는 시선 화살표와 카드의 떠남으로, 수명의 운동은 TTL 칸의 카운트다운으로 각각 운반된다. 다섯 시각 요소가 한 시퀀스 안에서 동시 운동한다.

### 8. 진행과 인터랙션

메시지 시퀀스형. 한 패킷의 운동이 시간축 위에 hop 단위로 펼쳐진다. 시간 진행은 사건 단위로 끊겨, 학습자가 각 hop 의 결정을 차분히 읽을 여지가 있다.

- **참여자**:
  - 호스트 A (출발지, 192.0.2.10): 좌측 끝점. 발신 시점에 패킷 카드를 만들어 첫 링크로 떠나보낸다. 라우팅 테이블이 없다(또는 단일 default 라우트만 가진다 — 본 시각화에서는 단순화하여 호스트의 표는 표시하지 않고 곧장 R1 으로 향한다).
  - 라우터 R1, R2, R3 (본선), R4 (분기): 각자 고유의 라우팅 테이블을 가진다. 한 시점에 단 한 라우터의 표만 펼쳐진다.
  - 호스트 B (목적지, 203.0.113.42): 우측 끝점. 패킷 도착 시 수신 누적에 카드 한 개가 추가된다.
  - (없는 참여자) 라우팅 프로토콜 (RIP·OSPF·BGP), 컨트롤 평면, 다른 호스트들 — 베이스에서는 등장하지 않는다.

- **메시지 흐름 (한 패킷의 9 단 시퀀스)**:
  1. **발신 (t=0)**: 호스트 A 의 발신 큐에서 새 카드(src=192.0.2.10, dst=203.0.113.42, TTL=64) 가 만들어져 호스트 A 박스 옆에 떠오른다. A→R1 링크 위를 약 400ms 의 곡선 운동으로 미끄러진다. 그 링크만 짧게 굵은 색으로 빛난다.
  2. **R1 도착**: 카드가 R1 박스 옆 정해진 자리에 안착한다. R1 박스 둘레에 밝은 동심원 강조가 한 번 퍼지고, R1 위의 안개 레이어가 200ms 동안 페이드 아웃하며 걷힌다.
  3. **R1 표 펼침**: R1 옆 패널이 좌→우 커튼 운동(약 250ms) 으로 펼쳐진다. prefix · next-hop · interface 세 컬럼의 표 4 행이 차례로 등장하고, 표 상단에 패킷 dst 의 비트 표시줄이 떠오른다.
  4. **LPM 비트 비교 (약 350ms)**: 비트 표시줄을 기준으로 각 행의 prefix 와 좌→우 비트 단위 일치 검사가 흐른다. 일치 칸은 짧은 초록 펄스, 불일치 첫 칸은 빨간 X 표지. 각 행 옆의 일치 길이 게이지가 동시에 차오른다. 일치하지 않은 행(default 행 제외) 의 게이지는 비어 있고 그 행은 가장 옅은 톤으로 잦아든다.
  5. **채택 행 강조 + next-hop 펄스**: 가장 긴 게이지를 가진 행이 굵은 강조 테두리(2px) 를 얻고, 그 행의 next-hop 셀(R2) 이 한 번 펄스로 빛난다. 짧은 일치 행(R4 를 가리키는 행) 은 막대가 차오르다 멈춘 채로 한 단계 옅어진다.
  6. **시선 화살표 발화**: 채택 행의 next-hop 셀에서 토폴로지 위 R2 박스로 짧은 곡선 화살표가 약 250ms 동안 그어진다. 화살표 옆에 잠시 "→ R2 (eth0)" 라벨이 뜬다.
  7. **카드 떠남 + TTL 카운트다운**: 카드가 R1 옆 안착 자리에서 떠나 R1→R2 링크 위로 곡선 운동(약 400ms) 을 시작한다. 떠나는 순간 카드의 TTL 칸 숫자가 64→63 으로 카운트다운하며 게이지가 한 칸 줄어드는 200ms 의 미소 운동이 발화한다.
  8. **R1 표 접힘 + 안개 복귀**: 카드가 떠난 직후 R1 패널이 우→좌 커튼 운동(약 250ms) 으로 다시 접히고, R1 위에 안개 레이어가 다시 깔린다. 시선 화살표는 페이드 아웃해 사라진다.
  9. **다음 hop 으로**: 카드가 R2 에 도착하면 같은 9 단 시퀀스가 R2 의 표를 펼치며 반복된다. R2→R3, R3→호스트 B 도 같은 방식. 호스트 B 도착 시 카드가 호스트 B 박스 옆 수신 누적에 안착하고, 캡션에 "패킷이 목적지에 도달했다 — TTL 61 남음" 이 잠시 뜬다.

- **TTL 폐기 시나리오 (실패 사건)**:
  - 사용자가 [초기 TTL] 슬라이더를 작은 값(예: 2) 으로 설정해 [발신] 을 누르면, 같은 9 단 시퀀스가 진행되다가 R2 또는 R3 에서 TTL 이 0 에 닿는 순간 발화한다. 카드가 그 라우터 옆 안착 자리에서 회색으로 즉시 페이드되고 X 표지가 카드 위에 발화하며 정지한다 — 다음 링크로 떠나지 않는다. 캡션에 "TTL 이 0 이다 — 패킷이 폐기되었다" 가 잠시 뜬다. 호스트 B 의 수신 누적에는 아무것도 추가되지 않는다.

- **제어 지점**:
  - **[발신] 버튼**: 호스트 A 가 새 카드를 만들어 9 단 시퀀스를 시작. 자동 시연이 꺼져 있어도 한 패킷의 전체 운동이 자동 진행된다.
  - **[한 hop] 버튼**: 다음 hop 의 9 단 시퀀스 중 한 단계만 진행한 뒤 정지. 학습자가 각 단계를 차분히 읽을 수 있다.
  - **[자동 시연] 버튼**: 일정 간격으로 미리 준비된 발신 시퀀스(서로 다른 dst 를 가진 패킷 3~4 개) 가 차례로 흘러나간다. 짧은 일치만 가진 dst 의 패킷은 R4 분기로 가는 운동도 시연된다.
  - **[⏸] 버튼**: 진행 중인 운동을 한 단계의 끝에서 정지.
  - **속도 슬라이더**: Slow / Normal / Fast 세 구간. 한 단계의 운동 길이가 두 배 / 기본 / 절반.
  - **[초기 TTL] 슬라이더 또는 드롭다운**: 발신 시 카드의 초기 TTL 을 정한다. 64 / 32 / 8 / 4 / 2 의 다섯 단계. 작은 값을 고르면 도중 폐기 시나리오가 발화한다.
  - **[초기화]**: 모든 카드·시선 화살표·표 펼침·수신 누적이 페이드 아웃하며 사라지고, 모든 라우터에 안개가 다시 깔린 t0 상태로 복귀.

- 모든 운동 길이: 링크 위 카드 미끄러짐 400ms, 표 펼침/접힘 250ms, 비트 비교 350ms, 일치 게이지 차오름 350ms(비교와 동시), 시선 화살표 발화 250ms, TTL 카운트다운 200ms, 안개 페이드 200ms. 한 hop 의 9 단 시퀀스 총 약 1.7 초(Normal). Fast 모드에서는 절반, Slow 모드에서는 두 배.

### 9. 초기 상태와 파라미터

- **초기 상태 (t = 0)**: 토폴로지가 정해진 자리에 그려져 있다. 호스트 A·B 와 라우터 R1·R2·R3·R4 의 박스 라벨, 링크 5 줄. 모든 라우터 박스 위에 안개 레이어가 깔려 있고 자물쇠 보조 표지가 표시. 어느 라우팅 테이블도 펼쳐져 있지 않다. 호스트 A 의 발신 큐에 패킷 카드 한 개가 대기(src=192.0.2.10, dst=203.0.113.42, TTL=64). 호스트 B 의 수신 누적은 비어 있다. 우상단 보조 정보 영역에 "발신 대기 중" 표시. 초기 TTL 슬라이더 64 위치, 속도 슬라이더 Normal.
- **권장 시연 시퀀스 (자동 시연 기본 시나리오, 약 4 발신)**:
  1. 발신 1: dst=203.0.113.42 (호스트 B). R1 에서 LPM 결과 R2 채택, R2 → R3 → 호스트 B 도착, TTL 64 → 61.
  2. 발신 2: dst=198.51.100.7 (R4 망 안의 가상 호스트). R1 에서 LPM 결과 R4 채택. 카드가 R1→R4 링크로 진행되어 R4 도착, R4 의 표 펼침. R4 의 표에서 직접 연결망(directly connected) 으로 도달 → 분기 끝에서 정지(또는 R4 옆에 가상 수신 누적 한 개). LPM 의 가치 시연.
  3. 발신 3: dst=8.8.8.8 (default 행이 채택되는 외부 IP). R1 에서 default 행만 매치되어 R3 채택. R3 → 외부(시각화 밖) 화살표만 그려지고 카드가 R3 의 외부 인터페이스에서 페이드(외부로 나갔다는 의미).
  4. 발신 4: dst=203.0.113.42, 초기 TTL=2. R1 떠날 때 TTL 1, R2 에 도착해 다음 hop 떠날 때 TTL 0 직전 — R2 에서 0 에 닿아 폐기. TTL 사건 시연.
- **파라미터**:
  - 노드 수: 호스트 2 + 라우터 4 (기준 폭). 확장 폭에서도 동일.
  - 링크 수: 5 (A—R1, R1—R2, R2—R3, R3—B, R1—R4 분기).
  - R1 의 라우팅 테이블 (예시):
    - 203.0.113.0/24 → 10.0.0.2 (R2) / eth0
    - 203.0.0.0/16 → 10.0.1.2 (R4) / eth1 (짧은 일치, LPM 으로 패배)
    - 198.51.100.0/24 → 10.0.1.2 (R4) / eth1
    - 0.0.0.0/0 (default) → 10.0.2.2 (R3) / eth2
  - R2·R3·R4 의 표는 각자 4~5 행, 본선과 직접 연결망과 default 한 행씩.
  - 초기 TTL 기본값: 64. 슬라이더 단계: 64 / 32 / 8 / 4 / 2.
  - TTL 임계 색 변화: TTL ≤ 4 에서 게이지 색이 청록 → 주황으로 전환.
  - 패킷 카드 색: 호스트 A 발 = 부드러운 청록, 자동 시연의 다음 카드들은 6 색 팔레트 순환(같은 호스트에서 발신해도 카드별로 정체성 색이 다름).
  - 비트 표시줄 폰트: 8pt 모노스페이스, 32 비트 표시 (점 단위 8 비트씩 간격).
  - 안개 레이어 투명도: 50%, 도착 시 200ms 동안 0% 로 페이드.
  - 시선 화살표 색: 채택 행과 같은 강조 색(진한 청록), 굵기 1.5px.
  - 한 hop 시퀀스 총 길이: Slow 3.4 초 / Normal 1.7 초 / Fast 0.85 초.
  - 자동 시연 발신 간격: Normal 기준 한 패킷의 끝과 다음 발신 사이 약 800ms.

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> IP 라우팅은 출발지 호스트가 만든 한 패킷이 인터넷 위 여러 라우터를 거쳐 목적지 호스트에 도달하기까지의 hop-by-hop 분산 결정 과정이다. 각 라우터는 자기만의 라우팅 테이블만 보고 패킷의 목적지 IP 와 가장 긴 일치 prefix 를 골라 next-hop 한 걸음만 결정한다. 패킷은 매 hop 마다 TTL 이 한 칸 줄고 0 이 되면 폐기되며, 어느 라우터도 전역 토폴로지를 알지 못한다 — 길은 각자의 표가 만든 결정의 연속으로 사후에 형성된다.

**학습자 안내 (보조 캡션 회전 문구)**

- "이 라우터는 자기 표만 본다 — 다른 라우터의 표는 가려져 있다."
- "가장 긴 일치가 이긴다 — 짧은 일치는 진다."
- "한 hop 마다 TTL 이 한 칸 줄어든다 — 0 이 되면 사라진다."
- "다음 hop 만 정한다 — 그 다음은 다음 라우터의 일이다."
- "출발지와 목적지는 호스트 — 라우터는 중간에 있을 뿐이다."
- "표가 변하지 않는 동안의 한 패킷 운동 — 표가 어떻게 만들어지는가는 다른 이야기다."

**사건별 짧은 메시지**

- 발신: "호스트 A 가 패킷을 만들었다 — 첫 hop 으로 떠난다."
- 라우터 도착 + 표 펼침: "패킷이 R1 에 도착했다 — 표를 펼친다."
- LPM 비교: "여러 prefix 가 일치한다 — 가장 긴 일치를 찾는다."
- 채택: "203.0.113.0/24 가 가장 긴 일치 — next-hop 은 R2 다."
- 카드 떠남 + TTL 카운트다운: "한 hop 떠났다 — TTL 이 한 칸 줄었다."
- 표 접힘: "R1 을 떠났다 — 다른 라우터의 결정은 알 수 없다."
- 분기 라우터로의 짧은 일치 패배: "203.0.0.0/16 도 일치하지만 더 길게 일치하는 행이 있다 — R4 로 가지 않는다."
- 도착: "패킷이 목적지에 도달했다 — TTL N 남음."
- TTL 폐기: "TTL 이 0 이다 — 패킷이 폐기되었다."
- default 채택: "구체 일치 없음 — default 행이 채택되어 외부로 나간다."

# 스택 시각화 기획서

### 0. 기본 정보

- 대상 이름: 스택 (stack)
- 도메인: 자료구조
- type: dataStructure
- 진행 모델: 입력 반응형
- 본질 한 문장: 스택은 모든 변화가 단 하나의 꼭대기 자리에서만 일어나도록 강제해, 마지막에 들어온 원소가 가장 먼저 나오도록 입력 순서를 거꾸로 뒤집는 자료구조다.

### 1. 시그니처 행동

새 원소는 언제나 더미의 꼭대기에만 얹히고, 떼어낼 때도 오직 그 꼭대기 한 자리만 사라진다.

### 2. 시각적 은유

표준 메타포인 "수직으로 쌓인 박스 더미" 를 채택한다. 화면 중앙에는 바닥선 위에 직사각형 셀들이 아래에서 위로 차곡차곡 쌓인 더미가 놓이고, 가장 위 셀의 윗변 바로 위에는 "꼭대기" 라는 라벨과 굵은 화살표가 항상 떠 있다. 더미 옆 빈 공간에는 박스가 떨어지고 떼어지는 동선의 시작점·끝점이 옅은 점선으로 암시되어, 진입과 퇴출이 모두 같은 윗변 한 점에서만 일어남을 위치만으로 강제한다. 박스 한 개에는 색과 번호가 함께 부여되어 어떤 박스가 어디서 왔는지 끝까지 추적된다.

### 3. 결정적 순간

세 박스 1, 2, 3 을 차례로 쌓은 직후, 떼어내기를 누르면 가장 마지막에 올린 3 이 위로 떠올라 사라지고, 다시 누르면 2 가, 또 누르면 1 이 사라진다. 이 짧은 시퀀스에서 학습자는 입력 트랙에 1, 2, 3 순서로 박스가 흘렀고, 출력 트랙에는 3, 2, 1 순서로 같은 박스가 다시 등장하는 것을 한 화면에서 본다.

- 전: 더미가 비어 있고 꼭대기 라벨이 바닥선 위 빈 자리를 가리킨다. 입력 트랙은 왼쪽에서 박스 1, 2, 3 이 대기 중. 출력 트랙은 비어 있다.
- 중: 1 → 2 → 3 이 차례로 위로 쌓이는 동안 꼭대기 라벨이 박스를 따라 위로 올라가고, 아래쪽 박스 1 의 명도와 채도가 점차 흐려지며 잠긴 느낌을 받는다.
- 후: 떼어내기를 세 번 누르면 3, 2, 1 순서로 박스가 더미에서 위로 솟아 출력 트랙으로 미끄러지듯 빠져나간다. 출력 트랙의 결과 시퀀스가 입력 트랙의 거울상이 되는 그림이 한 시점에 한 화면에 동시 보존된다.
- 강조 장치: 꼭대기 화살표의 굵기·박스 색의 정체성 유지·아래쪽 박스의 흐려짐·입력↔출력 트랙의 좌우 대칭 정렬.

### 4. 다른 대상과의 구별점

- 큐: 큐는 가로 통로의 한쪽 끝에서 들어와 반대쪽 끝으로 나간다. 진입과 퇴출의 위치가 다른 두 점이다. 스택은 한 점에서만 일어나며, 시각적으로 더미가 위로 자랄 뿐 옆으로 흐르지 않는다. 같은 입력 1, 2, 3 을 흘렸을 때 큐의 출력은 1, 2, 3 이고 스택의 출력은 3, 2, 1 이라는 순서 차이가 곧 정책의 차이다.
- 덱: 덱은 양쪽 끝 모두에서 진입과 퇴출이 가능하다. 화면에 두 개의 꼭대기 라벨이 위·아래(또는 좌·우) 양쪽에 동시에 떠 있다. 스택은 라벨이 단 하나, 항상 윗변 한 자리에만 머문다.
- 일반 배열: 배열은 임의의 인덱스에 직접 접근하는 화살표가 어디든 꽂힐 수 있다. 스택은 꼭대기 외 어떤 자리에도 화살표가 향하지 않으며, 아래쪽 셀들은 흐려진 채로 잠겨 있다. "어디든 만질 수 있다" 와 "한 자리만 만질 수 있다" 의 차이가 화살표의 자유도로 드러난다.

### 5. 달성해야 할 시각 요소

1. **꼭대기 표지 — 만질 수 있는 단 하나의 자리**
   - 설명: 어느 시점에든 학습자가 즉시 "지금 변화가 일어나는 자리" 를 식별할 수 있어야 한다. 비어 있을 때조차 그 자리가 정의되어 있다.
   - 시각 표현: 더미 윗변 바로 위에 굵은 화살표 + "꼭대기" 라벨이 항상 떠 있다. 박스가 늘면 라벨이 위로, 줄면 아래로 부드럽게 따라 이동한다. 빈 더미일 때는 바닥선 위 빈 자리를 가리키며 라벨 텍스트가 옅은 회색으로 바뀌고 "비어 있음" 이라는 보조 글자가 작게 붙는다.

2. **잠긴 아래쪽 — 깊이에 따른 시각적 약화**
   - 설명: 꼭대기에서 멀어질수록 박스가 묻혀 있다는 사실이 즉각 느껴진다. 표준 레퍼런스가 모두 동등한 색으로 그려 평면적이었던 부분을 정면으로 강화한다.
   - 시각 표현: 박스의 명도와 채도를 깊이에 따라 단계적으로 낮춘다. 꼭대기 박스는 100% 선명도, 한 단 아래는 70%, 그 아래는 50%, 그 아래는 35% 식으로 점차 흐려진다. 아래쪽 박스 테두리는 점선이 되고, 박스 위로 옅은 회색 안개 레이어가 깔려 "잠금" 의 느낌을 만든다.

3. **두 시간축의 직교 운동 — 입력 트랙과 출력 트랙**
   - 설명: 외부의 입력 시간이 가로로 흐르는 동안, 그 한 단계가 세로 더미의 한 변화로 동시에 동기화된다. 정적 다이어그램이 아니라 두 축이 함께 움직이는 운동.
   - 시각 표현: 더미 왼쪽에 가로 입력 트랙(왼→오), 오른쪽에 가로 출력 트랙(왼→오)을 둔다. 쌓기 시 입력 트랙의 가장 오른쪽 박스가 위로 곡선을 그리며 떠올라 더미 꼭대기에 안착한다. 떼기 시 더미 꼭대기 박스가 위로 솟았다가 곡선을 그리며 출력 트랙의 가장 왼쪽 자리로 미끄러진다. 두 트랙은 더미 바닥선과 같은 높이에서 마주 본다.

4. **순서 역전의 색·번호 추적**
   - 설명: 들어간 박스 각각이 색과 번호로 정체성을 유지하고, 그 정체성이 출력 트랙에서 입력의 거꾸로 순서로 다시 등장한다는 사실이 한 화면 안에서 추적 가능해야 한다.
   - 시각 표현: 박스마다 입력 시점에 부여된 고유 색과 번호가 더미 안에서도, 출력 트랙에서도 그대로 유지된다. 출력이 끝난 후에도 출력 트랙의 박스들은 한동안 잔상으로 남아, 입력 트랙의 1·2·3 색 시퀀스와 출력 트랙의 3·2·1 색 시퀀스가 좌우 거울처럼 마주 보는 정지 화면이 만들어진다.

5. **보기와 떼기의 구별 — peek 와 pop**
   - 설명: 같은 꼭대기를 향한 두 사건이 시각 어휘 자체로 달라야 한다. "본다" 는 형상이 변하지 않고, "떼어낸다" 는 형상이 줄어든다.
   - 시각 표현: 보기를 누르면 꼭대기 박스 둘레에 잠시 동심원 강조가 퍼지고 꼭대기 박스의 색만 한 번 깜빡일 뿐, 더미 높이는 그대로다. 떼기를 누르면 꼭대기 박스가 위로 솟아 출력 트랙으로 빠져나가고 더미 높이가 한 칸 줄어든다.

6. **빈 상태와 거부 반응 — 언더플로/오버플로의 사건성**
   - 설명: 빈 스택에서 떼기 시도, 가득 찬 스택에 쌓기 시도가 작은 텍스트가 아니라 더미 자체의 시각적 거부로 표현되어 "예외가 일어났다" 가 즉시 인지된다.
   - 시각 표현: 빈 더미에서 떼기 버튼이 눌리면 바닥선 위 빈 자리에 짧은 좌우 흔들림이 일고, 꼭대기 라벨이 빨갛게 깜빡인 뒤 원래 색으로 돌아온다. 가득 찬 더미에 쌓기를 시도하면 더미 윗변 위 한 박스 높이가 빨간 점선으로 잠시 그어지고, 입력 트랙의 박스가 더미에 닿지 못한 채 제자리로 튕겨 돌아간다.

### 6. 시각 구성

- **중심 시각**: 화면 중앙의 수직 박스 더미. 바닥선·박스 셀·꼭대기 라벨·꼭대기 화살표로 구성된다. 더미는 바닥에서 위로 자란다.
- **보조 요소**:
  - 좌측 입력 트랙: 다음에 쌓일 박스들이 왼→오 방향으로 대기. 가장 오른쪽 박스가 다음 차례.
  - 우측 출력 트랙: 떼어진 박스들이 왼→오로 누적. 가장 왼쪽 박스가 가장 먼저 떼어진 것.
  - 상단 캡션 영역: 짧은 개념 문장 한 줄과 현재 강조 메시지.
  - 하단 컨트롤 영역: 입력값 박스, 쌓기·떼기·보기·초기화 네 개의 버튼.
- **요소 간 관계**: 입력 트랙 → 더미 꼭대기 → 출력 트랙으로 박스가 곡선 운동으로 이어진다. 꼭대기 라벨은 항상 더미 윗변에 고정되며 더미 높이를 따라 수직으로만 이동한다. 입력·출력 두 트랙은 더미를 사이에 두고 좌우 대칭으로 정렬되어, 입력 시퀀스의 색 배열과 출력 시퀀스의 색 배열이 거울상이 되는 시각이 한눈에 보인다.

### 7. 레이아웃

기준 폭 680px, 확장 폭 880px. 높이 약 460px.

```
┌──────────────────────────────────────────────────────────┐
│  상단 캡션 — 개념 문장 + 현재 사건 메시지            (40)│
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌─입력 트랙─┐    ┌── 꼭대기 라벨 ──┐    ┌─출력 트랙─┐  │
│  │ ▣ ▣ ▣    │    │      ↓          │    │           │  │
│  │          │    │   ┌────┐        │    │           │  │
│  │ (대기)   │    │   │ 3  │ ◀선명  │    │  ▣        │  │ 320
│  │          │    │   ├────┤        │    │  (누적)   │  │
│  │          │    │   │ 2  │ ◀중간  │    │           │  │
│  │          │    │   ├────┤        │    │           │  │
│  │          │    │   │ 1  │ ◀흐림  │    │           │  │
│  │          │    │   └────┘바닥선  │    │           │  │
│  │ 180px    │    │   320px         │    │ 180px     │  │
│  └──────────┘    └─────────────────┘    └───────────┘  │
│                                                          │
├──────────────────────────────────────────────────────────┤
│  [값 입력 ▣] [쌓기] [떼기] [보기] [초기화]          (60)│
└──────────────────────────────────────────────────────────┘
```

- 더미 영역은 중앙에 폭 320px 로 고정. 박스 한 개의 가로 84px, 세로 36px. 최대 높이 8칸까지 시각적으로 안전.
- 입력·출력 트랙은 각각 폭 180px. 가로로 표시 가능한 박스는 5개까지, 그 이상은 가장 오래된 항목이 옅게 페이드되며 가장자리 너머로 사라진다.
- 확장 폭 880px 일 때는 입출력 트랙이 각각 260px 로 늘어나 7개까지 표시한다. 더미 영역은 그대로.
- 상단 캡션 40px, 하단 컨트롤 60px, 본체 320px 로 총 460px.

### 8. 진행과 인터랙션

입력 반응형. 사용자의 한 번의 입력이 화면의 한 번의 변화로 1:1 대응된다.

- **값 입력 + 쌓기 버튼**: 입력 박스에 쓴 값을 색·번호와 함께 새 박스로 만들어 입력 트랙 가장 오른쪽에 즉시 추가하고, 곧바로 곡선 운동으로 더미 꼭대기에 안착시킨다. 더미 높이가 한 칸 늘고 꼭대기 라벨이 위로 따라 올라간다. 깊이가 늘면서 그보다 아래 박스들의 명도가 한 단계씩 더 흐려진다.
- **떼기 버튼**: 더미 꼭대기 박스가 위로 솟아오른 뒤 곡선 운동으로 출력 트랙 가장 왼쪽에 안착한다. 출력 트랙의 기존 박스들은 오른쪽으로 한 칸씩 밀린다. 더미 높이가 한 칸 줄고 꼭대기 라벨이 아래로 내려간다. 아래 박스들의 명도가 한 단계씩 회복된다.
- **보기 버튼**: 더미 꼭대기 박스 둘레에 동심원 강조가 한 번 퍼지고 꼭대기 박스 색이 깜빡인다. 더미 형상은 변하지 않는다. 캡션에 "꼭대기 값을 보았다 — 더미는 그대로다" 가 잠시 뜬다.
- **초기화 버튼**: 더미·입력 트랙·출력 트랙의 모든 박스가 동시에 페이드아웃하며 사라지고, 빈 더미와 빈 트랙으로 돌아간다. 꼭대기 라벨이 빈 자리를 가리키는 회색 상태로 복귀.
- **빈 더미에서 떼기**: 짧은 좌우 흔들림 + 꼭대기 라벨 빨간 깜빡임. 캡션에 "떼어낼 박스가 없다" 가 잠시 뜬다.
- **가득 찬 더미에 쌓기**: 더미 윗변 위에 빨간 점선 한 칸이 잠시 그어지고, 입력 박스가 더미 근처까지 갔다가 제자리로 튕겨 돌아간다. 캡션에 "더 쌓을 자리가 없다" 가 잠시 뜬다.
- 모든 운동은 약 350~500ms. 학습자가 사건 하나를 시선으로 따라가기에 충분한 길이.

### 9. 초기 상태와 파라미터

- **초기 상태**: 빈 더미. 입력 트랙·출력 트랙 모두 비어 있다. 꼭대기 라벨이 바닥선 위 빈 자리를 회색으로 가리키며 "비어 있음" 보조 글자가 붙는다. 입력 박스는 비어 있고, 떼기·보기 버튼은 비활성(반투명) 상태.
- **권장 시연 시퀀스**: 처음 진입 시 1초 간격으로 1, 2, 3 박스가 차례로 자동 쌓인 뒤 멈춘다. 학습자는 그 상태에서 떼기 버튼을 눌러 3, 2, 1 의 역순 출력을 직접 체감한다.
- **파라미터**:
  - 더미 최대 높이: 8칸 (오버플로 거부 시연 가능).
  - 입력/출력 트랙 표시 한도: 기준 폭 5개, 확장 폭 7개.
  - 박스 색 팔레트: 6색 순환 (입력 순서대로 순환 부여).
  - 깊이별 명도 단계: 100% → 80% → 60% → 45% → 35% (5단 이후는 35% 고정).
  - 운동 지속 시간: 쌓기/떼기 400ms, 보기 250ms, 거부 반응 300ms.

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> 스택은 가장 최근에 들어온 원소를 가장 먼저 꺼내도록 강제하는 자료구조다. 모든 변화는 단 하나의 꼭대기에서만 일어나며, 쌓기는 새 원소를 그 위에 얹고 떼기는 그 위 원소를 떼어낸다. 한 점 제약 덕분에 입력 순서가 자연스럽게 뒤집혀 출력된다.

**학습자 안내 (보조 캡션 회전 문구)**

- "맨 위만 만진다 — 다른 자리는 잠겨 있다."
- "마지막에 들어온 것이 가장 먼저 나온다."
- "쌓는 순서가 곧 뒤집히는 순서다."
- "꼭대기 라벨이 어디를 가리키는지 보라 — 그 자리만 변한다."

**사건별 짧은 메시지**

- 쌓기: "꼭대기 위에 새 박스를 얹었다."
- 떼기: "꼭대기의 박스를 떼어냈다."
- 보기: "꼭대기 값을 보았다 — 더미는 그대로다."
- 빈 더미 떼기 시도: "떼어낼 박스가 없다."
- 가득 찬 더미 쌓기 시도: "더 쌓을 자리가 없다."

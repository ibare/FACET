## 1. 대상 정보
- 이름: 배열 (array)
- 영문 식별자: array
- 도메인: datastructure
- 상위 분류: 컴퓨터 과학 기초 자료구조 / 선형 자료구조

## 2. type
dataStructure

선택 근거: 배열은 알고리즘이나 프로토콜이 아니라, 동일 타입의 원소를 연속된 메모리에 저장하는 자료 구조 그 자체다. 정의되는 것은 인덱스 접근 / 갱신 / 삽입 / 삭제 같은 연산의 의미와 비용이며, 이는 dataStructure 범주의 정의와 일치한다.

## 3. 진행 모델
입력 반응형

선택 근거: 배열은 시간 축을 따라 자율적으로 진행하지 않는다. 학습자가 발생시키는 연산 (read, write, insert, delete, resize) 에 반응해 상태가 변한다. 메시지 시퀀스 / 정적 / 시간 진행형 모두 핵심 학습 모델과 어긋나며, 입력 반응형이 가장 정확하다.

## 4. 핵심 학습 질문
"배열은 인덱스 기반 임의 접근을 어떻게 O(1) 로 보장하며, 삽입 / 삭제 / 크기 변경에서 어떤 비용과 메모리 레이아웃 변화가 발생하는가?"

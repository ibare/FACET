## 2. 레퍼런스 요약

### R1. 3Blue1Brown — Linear transformations and matrices (Essence of Linear Algebra, Ch. 3)

- 출처: 3blue1brown.com / Grant Sanderson (사실상 선형변환 시각화의 사실상 표준)
- URL: https://www.3blue1brown.com/lessons/linear-transformations/ , https://www.youtube.com/watch?v=kYB8IZa5AuE
- 한 줄 정체성: "행렬은 두 기저 벡터의 도착지" — i-hat·j-hat 의 운동을 추적해 행렬 두 열의 정체를 시각으로 박아 넣는 기준 영상.
- 시각 객체: 무한 격자(연한 청색) + 기저 벡터 i-hat (빨강·녹색 계열) / j-hat (다른 계열) + 정적 배경 격자(과거 위치 잔상). 우측 상단에 행렬 셀이 떠 있고 그 셀이 곧 두 기저 벡터의 좌표.
- 시간 표현: 항등→대상으로의 부드러운 보간 (수 초 동안 격자가 휘어지며 휨). 이산 단계 아닌 연속 운동.
- 메타포: "공간을 잡고 늘리고 회전시킨다." 평면이 고무판처럼 휘는 직접 비유.
- 강조: (1) 직선은 직선 유지 + 원점 고정 = 선형성의 시각 정의 (2) 두 기저 벡터의 도착지가 곧 행렬의 두 열.
- 인상 디테일: 격자가 휘는 동안에도 정적 배경 격자가 남아 있어 "어디에서 어디로" 가 동시에 보임. 행렬 셀의 숫자가 i-hat / j-hat 색과 같은 색으로 칠해져 시각·기호 동기.
- 약점: 영상이라 임의 행렬을 즉시 입력해 보지 못한다. 보는 사람의 행렬 ↔ 격자 즉시 매핑 능력은 별도 도구가 필요.

### R2. Nicky Case — THE MAGNIFICENT 2D MATRIX

- 출처: ncase.me (Nicky Case, "Explorable Explanations" 의 대표 작가)
- URL: https://ncase.me/matrix/
- 한 줄 정체성: 2×2 행렬의 네 셀을 직접 슬라이더로 잡고 흔들면 단위 정사각형과 두 기저 벡터가 즉시 따라 움직이는 입력 반응형 토이.
- 시각 객체: 한가운데 격자 + 단위 정사각형(반투명 면) + i-hat·j-hat 두 색 화살표 + 좌측 행렬 셀(드래그 가능). 셀 색이 그대로 화살표 색.
- 시간 표현: 보간 없음. 슬라이더 = 결과 — 입력 반응형의 가장 순수한 형태.
- 메타포: "행렬은 두 화살표를 잡는 손잡이." 추상 행렬을 만질 수 있는 물건으로 환원.
- 강조: 행렬 셀 ↔ 기저 벡터 ↔ 단위 정사각형의 3중 동기. 셀 한 칸을 만지면 정확히 한 화살표 한 좌표만 변한다.
- 인상 디테일: 캐주얼한 카피("Behold the beast! The magnificent 2D matrix!") 와 The Unlicense 공개 — 학교·블로그 자유 재사용. 회전·스케일·전단·반사 프리셋 버튼.
- 약점: 합성 (M2·M1) 의 분해 시각이 약하다. determinant 의 부호·면적 표시가 분명히 드러나지 않는다. 비가환성 비교 화면이 없음.

### R3. Immersive Linear Algebra — Chapter 6: The Matrix

- 출처: J. Strom·K. Astrom·T. Akenine-Moller (스웨덴 룬드 대학 그래픽스 그룹의 첫 인터랙티브 선형대수 책)
- URL: https://immersivemath.com/ila/ch06_matrices/ch06.html
- 한 줄 정체성: 슬라이더로 회전각·전단계수를 잡으면 단위 정사각형이 변형되는 인터랙티브 일러스트가 본문 곳곳에 박힌 교과서.
- 시각 객체: 정사각형/이미지(고양이 사진 등) + 격자 + 두 기저 벡터 + 슬라이더 패널 + 행렬 셀 표시.
- 시간 표현: 슬라이더 즉시 반영(입력 반응형). 일부 그림은 변환 전/후를 좌우로 나란히 두어 비교.
- 메타포: "이미지를 행렬이 늘이고 비튼다" — 사진을 직접 변형해 시각적 친밀감.
- 강조: 회전·스케일·전단의 4종이 같은 행렬 골격으로 표현됨. 역행렬은 반대 방향 변환 (전단의 역은 음의 전단) 이라는 시각 사실.
- 인상 디테일: 본문이 수식 → 인터랙티브 그림 → 다시 수식 순서로 짜여 있어, 슬라이더를 흔들어 보고 나면 수식이 자연스럽게 읽힘. 사진을 행렬로 변형해 보는 경험은 OpenGL 텍스처 매핑과 직결.
- 약점: 합성·비가환성·determinant 의 인터랙션은 다른 챕터로 분리. 하나의 화면에서 두 행렬을 동시 곱해 보는 패널은 없음.

### R4. T. Kern — 2 Dimensional Matrix Transformations

- 출처: trkern.github.io (개인 인터랙티브 데모)
- URL: https://trkern.github.io/lineartransform.html
- 한 줄 정체성: 단일 페이지에 "원본 격자 / 변환 격자 / 단위 평행사변형 / eigenvector 축 / Morph 버튼" 까지 한꺼번에 토글하는 모든 것을 켤 수 있는 시각화.
- 시각 객체: 격자 두 겹(원본 회색 + 변환 진한색) + 빨강 e1 / 파랑 e2 화살표 + 그 두 벡터가 만드는 단위 평행사변형 면 + 임의의 이미지 점들 + eigenvector 축 (해당하면).
- 시간 표현: 행렬 셀을 입력하면 즉시 반영. "Morph" 버튼을 누르면 항등→대상 보간 운동 재생.
- 메타포: "원본 격자는 잔상, 새 격자는 결과" — 두 시점을 동시에 보는 영화 트릭.
- 강조: 단위 평행사변형의 면적 = |det| 라는 사실이 토글 한 번으로 드러남. eigenvector 축은 변환 후에도 자기 자신 방향에 머무르는 직선으로 시각화.
- 인상 디테일: 토글이 너무 많아 처음엔 어수선해도, 끄고 켜며 자기만의 그림을 만들 수 있는 "조립식 시각화" 의 미덕이 있다.
- 약점: UI 가 학술적이고 라벨 폰트가 작다. 합성을 두 행렬 곱으로 분해해 보여주는 모드는 없음.

### R5. GeoGebra — Matrix Transformations (by Emma) / Linear Transformations / Shear

- 출처: geogebra.org (전 세계 수학 교사·학생이 쓰는 동적 기하 플랫폼)
- URL: https://www.geogebra.org/m/sqG26hQj , https://www.geogebra.org/m/pDU4peV5 , https://www.geogebra.org/m/z5EGXkXm
- 한 줄 정체성: 행렬 셀을 슬라이더 또는 점(A·B) 으로 잡으면 단위 정사각형/도형이 즉시 변형되는 교실용 공식 애플릿 모음.
- 시각 객체: 격자 + 단위 정사각형 + 변환 전 도형(점선) / 변환 후 도형(실선) + 행렬 셀 텍스트.
- 시간 표현: 입력 반응형. 슬라이더 = 결과. GeoGebra 특성상 점을 끌면 도형도 실시간으로 따라옴.
- 메타포: "두 점이 두 열" — 점 A·B 가 행렬 첫째·둘째 열을 의미하므로 점을 끄는 행위가 곧 행렬 편집.
- 강조: 회전·확대·반사·전단을 같은 인터페이스에서 만들 수 있다. 도형(삼각형·집·텍스트) 을 변환해 일상감을 부여.
- 인상 디테일: 점 A·B 의 위치가 곧 행렬 두 열이라는 매핑이 직관적이다. 학교 수업 표준이라 한국어로도 같은 인터페이스를 그대로 쓸 수 있음.
- 약점: 모듈마다 따로 만들어져 있어 합성/비가환성/determinant 부호를 한 화면에서 비교하지 못한다. 전·후 격자를 함께 깔지 않는 애플릿이 많음.

### R6. Setosa — Eigenvectors and Eigenvalues, Explained Visually

- 출처: setosa.io (Victor Powell·Lewis Lehe, "Explained Visually" 시리즈)
- URL: https://setosa.io/ev/eigenvectors-and-eigenvalues/
- 한 줄 정체성: 행렬 곱 Av 를 점 v 가 새 점 Av 로 이동하는 운동으로 그리고, 같은 직선 위에 머무르는 v 가 eigenvector 임을 한 줄(0,v,Av 의 직선) 로 보여주는 시각.
- 시각 객체: 평면 + 점 v(드래그 가능) + Av 점 + (0,0)·v·Av 를 잇는 직선 + 행렬 셀 a1·a2.
- 시간 표현: v 를 끌면 Av 가 즉시 따라옴 (입력 반응형). 시퀀스 모드는 vk = A^k v 의 궤적을 점선으로 남김.
- 메타포: "행렬은 점을 끌고 가는 자기장." Eigenvector 는 자기장 방향과 평행한 점.
- 강조: Av = λv 를 그래프로 — "원점·v·Av 가 한 직선이면 v 는 eigenvector." 복소 eigenvalue 면 궤적이 나선임을 직접 보여줌.
- 인상 디테일: 시퀀스 모드의 나선 궤적은 회전+스케일 합성의 본질을 한 그림으로 압축한다.
- 약점: 격자 전체보다 단일 점 중심. 본 facet 의 "기저 벡터 도착지 = 행렬 두 열" 동기는 약하게 처리됨.

### R7. LearnOpenGL — Transformations

- 출처: learnopengl.com (Joey de Vries, OpenGL 학습 표준 사이트)
- URL: https://learnopengl.com/Getting-started/Transformations
- 한 줄 정체성: 행렬을 "셰이더로 보내는 변환의 단일 도구" 로 자리매김하고, 회전·스케일·이동을 4×4 행렬 한 장으로 합성하는 그래픽스 관행을 도식·코드로 풀어쓴 표준 챕터.
- 시각 객체: 점·벡터의 행렬 곱 다이어그램 + 단위 행렬 + 회전·스케일·이동 행렬의 도식 그림 + 큐브 텍스처 변환 스크린샷.
- 시간 표현: 정적 다이어그램 + 코드 한 줄 변경 → 화면 큐브가 회전/축소되는 결과 스크린샷의 전·후 비교.
- 메타포: "행렬 = 셰이더 uniform 한 슬롯에 끼워 넣는 변환 도구."
- 강조: (1) 동차좌표 (homogeneous coordinates, w 성분 포함 4D) 가 이동까지 행렬 하나로 표현하기 위해 필요하다는 사실 (2) 합성 시 적용 순서 (스케일 → 회전 → 이동) 의 중요성 — 행렬 곱은 비가환.
- 인상 디테일: glm 코드 (`glm::translate * glm::rotate * glm::scale`) 와 식의 1:1 대응. 2D 학습자가 그래픽스 코드로 넘어갈 때의 다리 역할.
- 약점: 정적 도식 위주라 격자가 휘는 운동이 없다. 학습자는 별도 인터랙티브 도구로 직관을 보강해야 함.

### R8. MDN — Canvas Transformations + CSS `matrix()` / SVG `transform`

- 출처: developer.mozilla.org (웹 표준 표준 문서)
- URL: https://developer.mozilla.org/en-US/docs/Web/API/Canvas_API/Tutorial/Transformations , https://developer.mozilla.org/en-US/docs/Web/CSS/transform-function/matrix , https://developer.mozilla.org/en-US/docs/Web/SVG/Reference/Attribute/transform
- 한 줄 정체성: 같은 2D 어파인 행렬 (a,b,c,d,e,f) 이 Canvas / CSS / SVG 세 표준에서 같은 의미로 쓰임을 보여주는 표준 레퍼런스.
- 시각 객체: 행렬 도식 [a c e ; b d f ; 0 0 1] + 이를 그대로 매핑한 그림(원→타원, 정사각형→평행사변형) + 짧은 코드와 결과 캔버스 출력.
- 시간 표현: 정적 그림 위주. 일부 예제는 인터랙션을 위한 코드 스니펫 포함 (사용자가 다시 실행).
- 메타포: "행렬 한 줄이 곧 한 변환 함수." `matrix()` 가 `scale()`/`rotate()`/`skew()`/`translate()` 의 일반화임을 강조.
- 강조: 동차좌표에서 마지막 행이 (0,0,1) 로 고정되어 있다는 어파인의 시각적 사실. 6개 파라미터의 의미 (수평 스케일, 수직 전단, 수평 전단, 수직 스케일, 수평 이동, 수직 이동) 를 표로 매핑.
- 인상 디테일: CSS `matrix(a,b,c,d,e,f)` 를 직접 디버깅하다 행 우선/열 우선 혼동을 일으키기 쉬운 함정을 명시. Canvas / CSS / SVG 의 미세 차이를 한 페이지로 비교 가능.
- 약점: 사용자가 셀을 흔들어 격자가 휘는 직접 인터랙션은 없음. 시각보다 API 문서 톤.

### R9. 공돌이의 수학정리노트 — 행렬과 선형변환

- 출처: angeloyeo.github.io (한국어 수학·신호처리 학습 자료의 사실상 표준 블로그)
- URL: https://angeloyeo.github.io/2019/07/15/Matrix_as_Linear_Transformation.html
- 한 줄 정체성: "선형변환 = 격자가 변환 후에도 직선이고 간격이 균등" 이라는 한국어 슬로건을 정착시킨 글. 인터랙티브 슬라이드와 정적 그림이 섞여 있음.
- 시각 객체: 정적 격자 그림 + 변환 전(파랑·회색) / 후(빨강) 격자 + 기저 벡터 화살표 + 슬라이드 형식 인터랙티브 패널(임베드).
- 시간 표현: 슬라이드 한 컷 = 한 변환. 일부는 GIF / Flash 형 패널.
- 메타포: "고무판 위의 격자." 격자의 직선성과 등간격이 보존되는지 눈으로 확인.
- 강조: (1) 행렬 한 곱은 곧 좌표계 변환 (2) 두 기저 벡터의 도착지를 통해 행렬 두 열의 의미를 설명. 같은 저자의 "행렬식의 기하학적 의미" 글과 짝을 이뤄 |det| = 면적 변화율을 잇는다.
- 인상 디테일: 한국어 학습자가 가장 먼저 만나는 "선형성의 두 조건 (직선 보존 + 원점 고정)" 의 한국어 어휘 — `격자 / 기저 / 변환 / 직선의 형태 / 균등 간격` 이라는 표준 용어 세트.
- 약점: 슬라이드 임베드 의존이라 모바일·다크모드에서 깨지기 쉽다. 합성·비가환성을 같은 화면에서 시각으로 비교하는 부분은 없다.

### R10. 공돌이의 수학정리노트 — 행렬식의 기하학적 의미

- 출처: angeloyeo.github.io
- URL: https://angeloyeo.github.io/2019/08/06/determinant.html
- 한 줄 정체성: "determinant 는 단위 정사각형이 변환 후 갖는 면적 (부호 포함)" 이라는 사실을 한국어 격자 시각으로 직접 보여주는 글.
- 시각 객체: 단위 정사각형 (변환 전: 한 변 1) + 변환 후 평행사변형 (면적 = |det|) + 두 기저 벡터 도착지 + 부호가 음수일 때 좌우 반전 표시.
- 시간 표현: 정적 그림 위주. 부호 음수 → 양수 전환 시 반사 사건을 좌우 뒤집힌 사진으로 표현.
- 메타포: "정사각형이 평행사변형으로." 면적과 방향성 (orientation) 의 직접 비유.
- 강조: determinant 의 부호 = 변환의 좌우 반사 여부. 0 = 평행사변형 무너져 직선/점.
- 인상 디테일: 음의 determinant 를 "거울에 비친 손바닥" 으로 비유 — 평면이 뒤집혔다는 시각 사건이 직관적.
- 약점: 면적 변화율을 슬라이더로 흔들어 보는 인터랙션은 없음. 본 facet 의 "행렬 셀 ↔ 두 열" 동기와는 결이 다르므로 보조 자료로 봐야 함.

## 3. 공통 요소

거의 모든 레퍼런스가 공유하는 시각 언어:

1. **격자 (grid)** — 평면 전체가 변환 대상임을 보여주는 가장 강한 시각 단위. 정적 배경 격자 위에 변환된 격자를 겹치는 표현이 표준 (R1, R4, R5, R7, R9).
2. **두 기저 벡터 i-hat·j-hat 의 색 분리** — 빨강·파랑 (혹은 빨강·녹색) 두 색으로 구분하고, 두 벡터의 도착지가 곧 행렬 두 열임을 강조. 행렬 셀 색을 같은 색으로 칠해 동기 (R1, R2, R3, R4).
3. **단위 정사각형 / 단위 평행사변형** — i-hat 과 j-hat 이 만드는 면적 1 도형이 변환 후 평행사변형으로. 면적이 곧 |det| (R2, R3, R4, R5, R10).
4. **행렬 셀과 시각의 동기** — 화면 한쪽에 행렬 [[a,b],[c,d]] 가 떠 있고, 셀을 만지면 정확히 그 한 좌표만 즉시 변한다. 입력 반응형의 핵심 (R1, R2, R3, R4, R5, R6).
5. **변환 전·후 동시 표시** — 항등 격자(잔상) 와 변환 격자(결과) 를 같은 화면에 겹치거나 점선/실선으로 분리 (R1, R4, R5, R9).
6. **선형성의 시각 조건** — "직선은 직선 유지 + 격자 간격 균등 + 원점 고정." 이 세 가지가 그림 캡션에 거의 매번 등장 (R1, R3, R9).
7. **표준 4종 변환 — 회전·스케일·전단·반사** — 모듈마다 분리된 데모로 제공되거나 같은 페이지의 프리셋 버튼으로 묶임 (R2, R3, R5, R7, R8).
8. **항등→대상 보간 vs 즉시 매핑의 양분** — 영상·교과서 컷은 부드러운 보간 (R1, R4 의 Morph), 토이·슬라이더 도구는 즉시 매핑 (R2, R3, R5, R6).
9. **이미지/도형의 변형 응용** — 단순 격자에 더해 사진 / 집 / 텍스트를 변형해 일상감 부여 (R3, R5, R7).

학습자에게 형성된 기대:
- 행렬은 두 열이고, 각 열은 변환된 기저 벡터다.
- 격자 전체가 같이 휘며, 직선은 직선으로 남고 원점은 고정이다.
- 단위 정사각형의 면적 = |det| 이고, 부호가 음수면 좌우가 뒤집힌다.

## 4. 공백

기존 레퍼런스에서 약하거나 비어 있는 영역:

1. **합성 (M2·M1) 의 시각적 분해** — "먼저 M1 으로 격자를 휘고, 그 위에 M2 를 다시 적용" 하는 두 단계 운동을 한 화면에서 분해해 보여주는 인터랙션이 부족하다. R7 이 코드와 식으로만 설명, R3 도 챕터를 분리. 한 화면에서 두 행렬을 동시 편집하며 합성 결과를 보는 도구가 드물다.
2. **비가환성 시각 (M1·M2 ≠ M2·M1)** — 같은 두 행렬을 순서를 바꾼 두 화면을 좌·우로 나란히 두어 다른 결과 격자를 한 컷에 보이는 비교 시각이 거의 없다. 텍스트로만 강조됨.
3. **행렬 셀 변경 → 격자 즉시 반영의 직접 동기** — R2/R3/R5 가 일부 구현했으나 셀 한 칸을 잡고 흔들면 정확히 그 한 좌표(i-hat 의 x 또는 y) 만 변한다는 미시적 동기를 시각 사건으로 강조하지는 않는다 (보통 한꺼번에 결과 격자만 바뀜).
4. **determinant 의 부호와 면적의 동시 표시** — R10 이 정적 그림으로만 처리. 슬라이더를 흔드는 동안 |det| 숫자가 함께 변하고, 음수 진입 순간 단위 평행사변형이 시각적으로 뒤집히는 사건성이 부족.
5. **2×2 vs 3×3 동차좌표의 차이** — R7/R8 이 동차좌표를 텍스트로 설명. 2D 행렬의 표현력 한계 (이동 표현 불가) → 3×3 으로 확장하는 시각적 동기가 약하다.
6. **어파인 vs 투영의 경계** — MDN (R8) 이 어파인 (마지막 행 = (0,0,1)) 이 무엇인지 설명하지만, 마지막 행을 흔들면 비어파인 (투영) 이 되는 사실의 시각 사건은 다루지 않는다.
7. **단일 점·도형의 운동과 격자 전체의 운동 동시 추적** — 대부분 도구가 단일 점 (R6) 이거나 격자 전체 (R1, R4) 중 하나에 치우친다. 격자 + 사용자 지정 점 한두 개의 궤적을 함께 그리는 도구는 드물다.
8. **프리셋 4종 (회전·스케일·전단·반사) 토글로 행렬 셀 패턴 학습** — R2/R5 가 프리셋 버튼을 제공하나, 버튼을 누르는 순간 셀 네 칸이 어떤 모양으로 채워지는지 (회전 = `[cosθ -sinθ; sinθ cosθ]`, 전단 = `[1 k; 0 1]`) 의 패턴을 시각적으로 외우게 하는 학습 강조는 약하다.
9. **한국어 인터랙티브 자료의 부족** — R9/R10 이 정적 그림 + 일부 임베드 슬라이드 수준. 한국어 학습자가 직접 셀을 흔들어 보는 도구는 사실상 영문 R2/R3/R5 를 빌려 써야 한다.

## 5. 대상 개념 요약

> 2D 행렬 변환은 평면을 평면으로 보내는 선형 사상이며, 그 정체는 두 기저 벡터 i-hat·j-hat 의 도착지로 완전히 결정된다. 2×2 행렬의 첫 열은 i-hat 이 어디로 가는지, 둘째 열은 j-hat 이 어디로 가는지를 그대로 적은 좌표이고, 평면의 모든 점은 자신을 i-hat·j-hat 의 선형 결합으로 적은 뒤 그 결합을 도착지에서 반복하는 것만으로 따라간다. 변환은 직선을 직선으로 보내고 원점을 고정시키며, 단위 정사각형의 면적 변화는 행렬식의 절댓값과 같고 부호는 좌우 반사 여부를 알려 준다. 합성은 행렬 곱이고 순서가 바뀌면 결과도 바뀐다 (비가환).

학습자가 시각화와 함께 읽을 짧은 캡션:
- "두 열은 두 기저 벡터의 도착지." (행렬 ↔ 시각의 핵심 동기)
- "직선은 직선으로, 원점은 그 자리에." (선형성의 시각 조건)
- "단위 정사각형의 새 면적이 곧 |det|." (determinant 의 기하학적 의미)
- "부호가 음수면 평면이 뒤집힌다." (반사의 시각 사건)
- "쌓는 순서를 바꾸면 결과가 바뀐다." (합성의 비가환성)

용어 풀어쓰기:
- i-hat / j-hat → "x 축·y 축 한 칸짜리 단위 화살표"
- 기저 벡터 → "평면을 짜는 두 기준 벡터"
- 행렬의 열 → "변환된 기저 벡터의 좌표"
- determinant → "단위 정사각형이 변환 후 갖는 면적 (부호 포함)"
- 합성 → "변환을 차례로 적용하는 것 = 행렬 곱"
- 비가환 → "순서를 바꾸면 결과가 달라진다"
- 어파인 → "이동까지 포함한 변환 (3×3 동차좌표 필요)"

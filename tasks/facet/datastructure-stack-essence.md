# 스택 핵심 추출

> 입력 — 분류: type=dataStructure, 진행 모델=입력 반응형, 핵심 학습 질문="LIFO 정책 아래 push/pop 이 어떻게 가장 최근 원소를 우선 반환하도록 순서를 보존하는가". 레퍼런스 10건 (VisuAlgo, USFCA Galles, Liang, DSA Visualizer, CSanim, Python Tutor, Heee 블로그, 위키/Velog, Runestone 괄호, Undo/Redo) 의 공통 시각 언어와 8개 공백을 종합.

## 1. 반드시 다뤄야 할 본질

학습자가 빠뜨리면 시각화가 LIFO 를 거짓으로 보여주게 되는 것.

1. **한 점 제약 — 꼭대기에서만 변화가 일어난다.** 진입과 퇴출이 모두 같은 한 자리에서 발생한다는 사실이 위치만으로 강제되어야 한다. 다른 자리는 시각적으로 만질 수 없는 것처럼 느껴져야 한다.
2. **꼭대기(top) 의 항상적 표시.** 어느 시점에 보든 "지금 만질 수 있는 단 하나의 자리" 가 즉시 식별된다. 비어 있을 때조차 그 자리가 어디인지 시각적으로 비어 있는 채로 남아 있어야 한다.
3. **마지막에 들어온 것이 가장 먼저 나온다 — 입력 순서와 출력 순서의 역전.** 들어간 차례와 나오는 차례가 거꾸로 매핑된다는 사실이 단일 push/pop 한 번이 아니라 일련의 시퀀스에서 시각적으로 드러나야 한다.
4. **잠긴 아래쪽 — 위가 비워질 때까지 아래는 접근 불가.** 더미 아래쪽 원소들이 "묻혀 있다 / 잠겨 있다" 는 사실이 위쪽 원소와 시각적으로 구분되어 보여야 한다. 같은 색·같은 명도로 그리면 본질이 사라진다.
5. **빈 상태와 첫 push, 마지막 pop 의 경계.** 비어 있음 → 하나 → 여럿 → 다시 비어 있음 으로 돌아가는 전이가 명시적인 사건으로 보여야 한다. 빈 스택은 "없음" 이 아니라 "하나의 정의된 상태" 다.

## 2. 강화해야 할 요소

표준 레퍼런스가 약하게 다루는 것을 이 시각화의 차별점으로 끌어올린다.

1. **잊혀짐의 가시화 — 깊이에 따른 시각적 약화.** 꼭대기에서 멀어질수록 명도/채도/선명도가 떨어져 "묻혀 있음" 의 압박감이 즉각 보인다. 표준 레퍼런스는 모든 셀을 동등하게 그려 잠금 상태가 평면적이다.
2. **두 시간축의 동시 운동 — 외부 입력(가로) 과 스택 내부 상태(세로) 의 직교.** 입력 시퀀스가 가로로 흐르는 동안 그 한 단계가 세로 더미의 한 변화로 바로 동기화되어 보인다. 정적 다이어그램이 아니라 두 축이 함께 움직인다.
3. **순서 역전의 명시적 매핑.** 들어간 원소 각각이 색·번호 같은 정체성을 유지하며, 출력 시 그 정체성이 입력의 거꾸로 순서로 다시 등장한다는 사실이 한 화면 안에서 추적 가능하다. 입력 트랙과 출력 트랙이 거울처럼 마주 본다.
4. **peek 와 pop 의 시각적 구별.** "본다" 는 꼭대기를 강조하되 더미 형상은 변하지 않고, "떼어낸다" 는 형상 자체가 줄어든다. 같은 버튼이 다른 결과를 내는 두 사건이 시각 어휘 자체로 구분된다.
5. **언더플로/오버플로의 사건성.** 빈 스택에서 떼어내기, 가득 찬 스택에 쌓기 시도가 작은 텍스트가 아니라 더미 자체의 시각적 거부 반응(흔들림·정지·경계 강조)으로 표현되어 "예외가 일어났다" 가 즉시 인지된다.

## 3. 공백 활용 후보

레퍼런스의 공백 중 흥미로우나 베이스에는 채택하지 않는 것. 미래 창의 사이클의 출발점.

1. **여러 비유의 통합 토글 화면.** 같은 LIFO 동작을 접시 더미·책 무더기·콜스택 프레임·되돌리기 히스토리·여닫는 괄호 다섯 메타포로 한 화면에서 토글하며 "모두 같은 구조" 를 체감하게 하는 메타 시각화.
2. **큐와의 동시 운동 비교.** 같은 입력 시퀀스를 스택과 큐에 동시에 흘려보내, FIFO 였다면 어떻게 달랐을지 반사실로 보여주는 운동형 비교. 정책의 차이가 출력 순서의 차이로 즉시 드러난다.
3. **메모리/콜스택과의 연결.** push 가 곧 스택 포인터 이동이라는 시스템 관점, 함수 호출이 자료구조 push 와 동형이라는 사실을 자료구조 학습 시각에 통합.
4. **시간 양방향 — 가역성으로서의 LIFO.** Python Tutor 식 step back. pop 한 원소가 push 의 정확한 역으로 되돌려지는 운동을 통해 "스택은 시간을 되감을 수 있는 자료구조" 라는 통찰을 시각화.
5. **응용과의 외부 트랙 동기화.** 괄호 매칭·재귀·되돌리기처럼 "외부 사건 한 단계 = 스택 한 변화" 의 1:1 대응을 외부 트랙(문자열·코드·사용자 행동) 위에 얹어 보여주는 응용 모드.

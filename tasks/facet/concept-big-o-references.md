## 2. 레퍼런스 요약

### R1. Sam Rose, "Big O" — samwho.dev
- URL: https://samwho.dev/big-o/
- 한 줄 정체성: 인터랙티브 카드 애니메이션으로 O(1)/O(log n)/O(n)/O(n^2) 의 "한 단계씩 진행되는 감각" 을 직접 만지게 하는 비주얼 에세이.
- 시각 객체: 작은 카드/박스 그리드, 단계별로 색이 바뀌는 셀, 입력 슬라이더, "측정" 버튼이 누른 만큼 막대가 채워지는 측정 패널.
- 시간 표현: 알고리즘이 한 단계 진행할 때마다 셀이 점등 — 클럭 시간이 아니라 "터치 횟수" 가 시간이다.
- 메타포: 카드 뒤집기 게임. 이진 검색은 "1~100 중 숫자 맞히기" 형태로 추측 횟수가 7회로 수렴함을 보여준다.
- 강조: 입력이 커질수록 동작 횟수의 차이가 시각적으로 폭발하는 순간을 슬라이더로 직접 관찰.
- 인상적 디테일: 측정 막대의 단위가 1e9 까지 올라가는 점 — n=1000 의 O(n^2) 가 사람의 인내를 넘는다는 것을 즉각 체감시킨다.
- 약점: 후반에 결국 표준 line chart 비교로 회귀한다. 카드 메타포가 O(n^2) 까지만 잘 작동.

### R2. Eric Drowell, "Big-O Cheat Sheet" — bigocheatsheet.com
- URL: https://www.bigocheatsheet.com/

- 약점: "왜" 가 사라지고 결과만 남는다 — 학습자에게는 외워야 할 표가 되어버림.

## 3. 공통 요소

거의 모든 레퍼런스가 공유하는 시각 언어와 학습자에게 형성된 기대 패턴.

1. **겹친 곡선 그래프 (line chart)** — x=n, y=연산수, 7~8개 곡선이 동일 평면에서 비교되는 형태. (R2/R3/R4/R11/일부 R1)
2. **색-코딩된 비교 표** — Excellent/Good/Bad 같은 정성 라벨이나 녹/노/빨 색띠로 클래스를 줄 세움. (R2/R11)
3. **이진 검색 = 전화번호부 / 사전** 메타포 — O(log n) 의 표준 비유. (R1/R8/R11)
4. **정렬 경주** — 같은 데이터 위에서 알고리즘이 동시에 달리는 시각화. (R1/R9/R10)
5. **n 슬라이더** — 학습자가 입력 크기를 직접 키우며 격차를 키워보는 인터랙션. (R1/R3/R4)
6. **"항의 무시"** 텍스트 설명 — 상수항·계수가 사라지는 정의를 글로 풀어쓰는 패턴. (R2/R11)
7. **연산 횟수 카운터** — 화면 한 구석에서 비교/스왑/방문 수가 누적되는 표시. (R1/R10)

학습자의 기대치: "Big-O 라고 하면 곡선 그래프 한 장이 뜨고, 가장 위쪽 곡선이 가장 나쁘다" 라는 즉각적 연상이 거의 보편적이다. 표준 시각 언어가 이미 학습자의 머리에 박혀 있으므로, B타입은 이것을 "다시 한 번 보여주는 것" 이 아니라 "흔들어서 새로 각인시키는 것" 이어야 한다.

## 4. 공백

표준 표현이 놓치고 있는 영역, 다른 분야에서 차용 가능한 메타포.

### 4.1 시간 표현의 다른 가능성
- **달력/물리적 시간 환산** — n=10^6 에서 O(n) 은 1초, O(n^2) 은 11.5일, O(2^n) 은 우주 나이의 10^200,000 배. 결과를 추상 그래프가 아니라 "시계/달력/은하의 나이" 로 변환하는 표현은 거의 없다.
- **공간 메타포** — 칸의 수가 아니라 "쌓이는 부피"(R7 체스판/쌀) 로 본다. 디지털 화면의 한계를 이용해 "스크롤로 끝까지 못 가는 막대" 표현 가능.
- **연료 소진** — 같은 작업에 같은 에너지를 주고, 누가 먼저 연료를 다 쓰고 멈추는지 보여주는 메타포. (현 레퍼런스에 없음)

### 4.2 다른 분야에서 차용 가능한 메타포
- **복리 이자/전염병 확산** — O(2^n) 의 직관. 학습자에게 친숙한 비-CS 영역의 격차 감각.
- **마라톤/우주여행** — O(log n) 은 "엘리베이터", O(n) 은 "도보", O(n^2) 은 "기어가기", O(2^n) 은 "광년 단위 이동". 거리/이동수단의 도식.
- **요리 시간** — 같은 손님 수에 대해 "1인분 끓이기" vs "전원 1:1 인터뷰 후 메뉴 결정" — n^2 의 친숙한 환원.
- **음악의 박자/감속** — 같은 멜로디(작업)를 다른 박자(클래스) 로 연주. O(n^2) 은 점차 박자가 느려지다 정지하는 곡.
- **유리병 속 모래** — 한 알 = 한 단계. O(log n) 은 한 줌, O(n!) 은 사하라.

### 4.3 차원 변환 가능성
- **2D 곡선 → 1D 진행 막대** — 입력을 키우는 동안 "어디까지 갔는가" 의 막대가 클래스마다 다른 속도로 차오르는 방식. 곡선의 "기울기" 가 막대의 "잔여 거리" 로 변환.
- **2D 곡선 → 입자/별** — 클래스마다 별 하나가 출발해 같은 결승선까지 달리는 별자리 표현. 학습자가 보는 것은 "선" 이 아니라 "위치".
- **2D 곡선 → 부피/적층** — 체스판형 그리드 위에 클래스별 쌀더미가 쌓이는 부피 비교. R7 의 디지털 확장.

### 4.4 인터랙션의 미개척 영역
- **n 을 키울 때 "어떤 클래스가 먼저 포기하는지" 컷오프 알림** — 화면이 감당 못 하는 시점에 클래스를 떨어뜨림.
- **"같은 시간 예산" 모드** — 1초/1분/1시간을 정해 두고 각 클래스가 처리할 수 있는 n 의 한계를 역으로 보여주는 인터랙션. (대부분의 레퍼런스는 n 을 고정하고 시간을 보지만, 시간을 고정하고 n 을 보는 표현은 희소.)
- **클래스 간 "격차의 음향화"** — 시각이 아닌 청각 차원의 격차 표현.
- **로컬 비교 vs 점근 비교의 토글** — 작은 n 에서는 O(n^2) 가 O(n log n) 보다 빠를 수도 있음을 직접 보여주는 토글. 분류 보고서 학습 질문 4 와 직결되지만 거의 모든 레퍼런스가 이 부분을 무시한다.

## 5. 후속 작업 가이드 (B타입)

분류 보고서가 B타입을 선택했으므로 본 가이드는 "흥미로운 공백" 과 "절대 지킬 본질" 을 제시한다.

### 5.1 가장 흥미로운 공백 (적극 시도)
1. **"같은 시간 예산, 다른 도달점"** — 시간을 가로축으로 두지 말고, 시간을 자원으로 고정한 뒤 각 클래스가 처리한 n 을 막대/별의 위치로 환산하라. 이는 곡선 그래프의 회피이자 학습 질문 1·2 에 직격이다.
2. **물리적 부피·거리 메타포** — R7(체스판 쌀)·R5(우주 나이) 의 정신을 잇되, 인터랙티브로 "쌓이는 부피" 또는 "별의 도달 거리" 로 만든다. 학습자가 슬라이더를 밀면 O(n!) 의 별이 화면 밖으로 즉시 사라지는 정도의 비현실 체감.
3. **클래스 간 컷오프** — 화면 안에서 살아남는 클래스만 남기고 나머지는 시각적으로 "포기" 시키는 장치. 격차를 정량 곡선이 아니라 "사라짐" 으로 표현.
4. **"점근 vs 작은 n" 토글** — 작은 n 에서 O(n^2) 가 O(n log n) 을 이기는 영역을 의도적으로 보여 주고, n 을 키우며 "운명이 바뀌는 순간" 을 강조. 점근 정의의 핵심.
5. **카드 한 장 = 한 단계** 의 R1 카드 메타포는 부분 채택 가치가 있음. 단, R1 이 후반에 곡선으로 회귀한 약점을 피해 끝까지 메타포를 유지.

### 5.2 절대 빼면 안 될 본질
- **격차가 폭발하는 순간** 의 체감 — 수가 아니라 시각 변화로.
- **상수·저차항 무시** 의 정의 — 메타포 한 장이라도 좋다. (예: "같은 클래스끼리는 결국 같은 별자리" 의 이미지)
- **클래스 카탈로그** — O(1), O(log n), O(n), O(n log n), O(n^2), O(2^n), O(n!) 의 7개는 학습자 기대치이므로 빼지 말 것.
- **n 을 키우는 인터랙션** — B타입이라도 학습자의 손이 입력을 통제할 수 있어야 한다.
- **점근 상한** 이라는 정의 자체 — 시각의 어떤 구석에든 "결국 위 = 더 나쁨" 의 단조성이 보존되어야 학문적 정합성을 잃지 않음.

### 5.3 회피 권고
- 7개 곡선이 한 평면에 동시에 그려지는 표준 라인 차트.
- "Excellent/Bad/Horrible" 색 밴드만으로 끝나는 표.
- O(log n) 은 전화번호부, O(n) 은 한 장씩 넘기기 식의 "완전히 닳은 비유" — 차용해도 좋되, 새 메타포의 보조로만.

## 6. 대상 개념 요약 (후속 데모가 사용)

Big-O 표기법은 알고리즘이 입력 크기 n 이 커질 때 실행 시간이나 자원 사용이 어떻게 자라는지를 함수의 상한으로 묶어서 부르는 점근 표기법이다. 우리는 1초·1분 같은 절대 시간이 아니라, "같은 작업을 두 배 큰 입력에 시켰을 때 몇 배가 더 걸리는가" 의 성질만을 본다. O(1) 은 입력이 아무리 커져도 시간이 거의 그대로이고, O(log n) 은 매우 천천히 자라며, O(n) 은 입력에 비례하고, O(n log n) 은 그보다 약간 더 빨리, O(n^2) 부터는 입력 두 배에 시간 네 배로 폭발하기 시작하며, O(2^n) 과 O(n!) 은 사실상 풀 수 없는 영역으로 들어간다.

학습자가 시각화 옆에서 함께 읽을 짧은 설명:
- "Big-O 는 시계가 아니라 운명을 말한다. 입력이 커질 때 너의 알고리즘이 어디로 가는지를 본다."
- "상수와 작은 항은 무시한다 — 멀리서 보면 큰 흐름만 남는다."
- "같은 클래스 안에 있다는 것은 결국 같은 부류라는 뜻이다."

전문 용어 풀어쓰기:
- 점근(asymptotic): "입력이 충분히 큰 곳까지 갔을 때" 의 모습.
- 상한(upper bound): "이보다 더 나빠지지는 않는다" 라는 약속.
- 성장률(growth rate): 입력이 두 배가 될 때 시간이 몇 배가 되는가.
- O(log n): 입력이 두 배가 되어도 한 단계만 더 든다 — "절반 가르기" 의 운명.
- O(n^2): 입력이 두 배가 되면 시간이 네 배 — "모두가 모두를 만나야 한다" 의 운명.
- O(2^n): 입력이 한 칸 늘 때마다 시간이 두 배 — "체스판 위 쌀 한 톨이 다음 칸에 두 톨이 되는" 운명.

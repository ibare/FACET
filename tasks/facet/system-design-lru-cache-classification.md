## 1. 대상 정보
- 이름: LRU 캐시 (LRU Cache, Least Recently Used Cache)
- 영역: 시스템 설계 / 캐싱 정책
- 도메인: system-design

## 2. type
dataStructure

- 근거: LRU 캐시는 해시 맵과 이중 연결 리스트를 결합한 자료구조이며, `get` / `put` 두 연산을 외부에서 호출받아 내부 상태(키-값 매핑과 사용 순서)를 유지한다. 단일 알고리즘의 시간 진행이 아니라, 외부 입력에 따라 상태가 갱신되는 자료구조의 정체성이 우선한다.

## 3. 진행 모델
입력 반응형

- 근거: 사용자가 발행하는 `get(key)` / `put(key, value)` 호출 시퀀스에 반응하여 내부 구조가 변형된다. 자율적인 시간 진행이나 메시지 교환 시퀀스가 아니라, 외부 명령에 대한 반응으로 상태 전이가 일어난다.

## 4. 핵심 학습 질문
용량 제한이 있는 캐시에서 "가장 최근에 사용된 순서" 라는 추상 개념이 실제로 어떤 자료구조 조합으로 표현되며, `get` / `put` 호출이 그 순서를 어떻게 갱신하고 용량 초과 시 어떤 항목이 축출되는가?

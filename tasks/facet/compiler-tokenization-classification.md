## 1. 대상 정보
- 주제: 컴파일러 어휘 분석 단계의 토큰화(tokenization, lexing/scanning)
- 범위: 원시 소스 코드 문자열을 입력받아 의미 있는 토큰 시퀀스로 끊어내는 단계
- 도메인: `compiler` (기존 전례 유지)

## 2. type
- 결정: `transformation`
- 근거: 토큰화는 "문자 스트림 → 토큰 스트림"이라는 입력→출력 변환을 수행하는 단계다. 자료구조(dataStructure)도 아니고 외부와의 상호작용 프로토콜도 아니며, 단일 알고리즘이라기보다 컴파일 파이프라인의 한 변환 단계로 보는 것이 자연스럽다.

## 3. 진행 모델
- 결정: `입력 반응형`
- 근거: 토큰화는 입력 문자 스트림을 좌→우로 한 글자씩 소비하며 상태 전이를 수행한다. 시계 기반 진행이 아니라 "다음 입력 문자"라는 이벤트에 반응해 한 토큰씩 만들어 내보내는 모델이며, 유한 상태 기계의 동작 방식과 일치한다.

## 4. 핵심 학습 질문
- 질문: "원시 소스 코드 문자 스트림은 어떤 규칙과 상태 전이를 거쳐 의미 있는 토큰 시퀀스로 잘려 나오는가?"
- 근거: 토큰화의 본질은 "어떤 경계에서 끊고, 어떤 의미 라벨을 붙이는가"이며, 이를 한 문장으로 압축한 형태.

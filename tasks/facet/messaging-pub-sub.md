# Pub/Sub 시각화 기획서

### 0. 기본 정보

- 대상 이름: Pub/Sub (publish/subscribe)
- 도메인: messaging
- type: systemBehavior
- 진행 모델: 메시지 시퀀스형 (구독자 동적 합류·이탈은 입력 반응형 사건으로 함께 다룸)
- 본질 한 문장: Pub/Sub 은 발행자와 구독자가 서로의 신원을 모른 채 가운데 자리한 broker 의 토픽 라벨만 매개로, 한 발행이 모든 구독자에게 사본으로 펼쳐지는 다대다 비동기 메시징을 성립시키는 시스템 행동이다.

### 1. 시그니처 행동

발행자가 토픽에 한 번 던진 메시지가 broker 안에서 사본으로 갈라져, 그 토픽을 신청한 모든 구독자에게 — 오직 그들에게만 — 시간차를 두고 도착한다.

### 2. 시각적 은유

표준 메타포인 "좌·중·우 삼각 배치 + broker 가 1급 박스" 를 채택하고, 그 위에 UML 시퀀스 다이어그램의 "세로 시간축 + 라이프라인" 을 결합한다. 화면 위쪽에는 좌측 발행자 박스 두세 개, 가운데 broker 박스(라벨 "토픽: events"), 우측 구독자 박스 서너 개가 가로로 늘어선다. 각 박스 아래로 점선 라이프라인이 화면 하단까지 뻗어 시간을 1급 시각 차원으로 끌어올린다. 시간은 위에서 아래로 흐르며, 발행과 전달은 모두 라이프라인 사이를 가로지르는 짧은 가로 화살표로 그려진다. 좌측 발행자 영역과 우측 구독자 영역 사이는 broker 라이프라인을 중심으로 **불투명에 가까운 옅은 분리 띠**가 깔려 있어, 어떤 화살표도 그 띠를 건너뛰어 직접 좌→우로 이어지지 못한다. 모든 화살표는 반드시 broker 라이프라인 위의 한 점에서 한 번 끊어지고 다시 시작된다. 메시지 자체는 색·번호 라벨을 가진 작은 칩으로 화살표 위를 미끄러지며, 한 발행이 broker 에서 사본으로 갈라질 때도 칩의 색·번호는 그대로 유지되어 같은 메시지의 사본임이 추적된다.

### 3. 결정적 순간

발행자 P1 이 broker 의 토픽 "events" 에 메시지 #7 을 던지는 한 컷이다. broker 라이프라인 위 한 점에 도착한 칩 #7 이 그 자리에서 같은 색·같은 번호의 사본 N 개로 갈라지고, 우측에 늘어선 구독자 S1, S2, S3 의 라이프라인을 향해 부채꼴 화살표 N 가닥이 뻗는다. 각 화살표가 도착하는 시각은 라이프라인 위의 다른 세로 위치 — 즉 다른 시점 — 이고, 학습자는 "한 번의 발행 → N 개의 사본 → 서로 다른 도착 시각" 이라는 세 사실을 한 화면에서 동시에 본다. 그리고 그 사이 어디에도 P1 에서 S1, S2, S3 으로 직접 향하는 화살표는 한 줄도 없다.

- 전: broker 라이프라인 위에 토픽 라벨 "events" 가 떠 있고, 그 라이프라인 위 일정 간격으로 S1, S2, S3 이 구독선을 그어 "events" 에 연결한 작은 매듭이 찍혀 있다. 발행자 영역과 구독자 영역 사이에는 옅은 분리 띠가 깔려 있다. 발행 직전이라 메시지 칩은 아직 어디에도 없다.
- 중: P1 라이프라인의 한 시점에서 칩 #7 (P1 색) 이 솟아 오른쪽으로 화살표를 그으며 broker 라이프라인 위의 한 점에 도착한다. 그 점에서 칩이 사본 세 개로 갈라지며 broker 박스 안에 잠시 머무는 정지 구간(짧은 세로 체류) 을 가진 뒤, 라이프라인 위 서로 다른 세 시점에서 각각 S1, S2, S3 을 향해 가로 화살표로 출발한다. 화살표의 길이와 출발 시점이 약간씩 다르다.
- 후: 세 사본이 S1, S2, S3 의 라이프라인 위 서로 다른 세 점에 차례로 안착하고, 도착한 자리마다 짧은 펄스가 한 번 퍼진다. 같은 색·같은 번호의 칩 세 개가 시간축 위에 비스듬한 사선으로 배열되어 "한 발행, 다른 수신 시각" 이 한 시점의 정지 화면에 보존된다. 발행자 영역에서 구독자 영역으로 직접 건너뛴 화살표는 0 개다.
- 강조 장치: broker 라이프라인 위 분기점에서의 갈라짐 펄스·각 도착 지점의 짧은 펄스·세 사본의 색과 번호 동일성·발행자/구독자 두 영역 사이의 분리 띠·broker 박스 내부의 짧은 체류 구간이 시간축 위에 보이는 정지 길이.

### 4. 다른 대상과의 구별점

- Observer 패턴: Observer 는 Subject 가 Observer 목록을 직접 알고 차례로 update() 를 호출하는 직접 호출 모델이다. 시퀀스 도식 위에서 Subject 라이프라인에서 Observer 라이프라인으로 화살표가 직접 이어지고, 가운데에 broker 박스가 없다. Pub/Sub 은 발행자 라이프라인에서 구독자 라이프라인으로 향하는 직접 화살표가 한 줄도 없으며, 모든 화살표가 가운데 broker 라이프라인 위의 한 점에서 끊어졌다 다시 시작한다. "서로를 안다 vs 토픽 라벨만 안다" 의 차이가 가운데 박스의 유무로 강제 구별된다.
- 메시지 큐 (Point-to-Point, 라운드로빈): 메시지 큐는 같은 가운데 박스 모양을 가지지만, 한 메시지가 단 한 명의 컨슈머에게만 도달한다. broker 위 분기점에서 화살표가 갈라지지 않고 N 명 중 한 줄로만 뻗으며, 그다음 메시지는 다음 컨슈머로 라운드로빈 분배된다. Pub/Sub 은 broker 위 분기점에서 한 번에 N 가닥의 화살표가 부채꼴로 펼쳐지고 모든 구독자에게 같은 색·같은 번호의 사본이 도달한다. "한 명에게만" 과 "모두에게 사본" 의 차이가 broker 라이프라인 위 분기점의 가닥 수로 즉시 구별된다.
- 단일 Publisher → 다중 Subscriber 의 단순 팬아웃 도식: 발행자가 한 명뿐인 도식은 Pub/Sub 의 절반만 그린다. 본 시각화는 좌측에 발행자 두세 명을 함께 두고, 서로 다른 색의 칩이 같은 broker 라이프라인 위로 섞여 들어가 다시 N 가닥으로 갈라지는 다대다 운동을 정면으로 그려, "발행자도 N, 구독자도 N" 임이 색의 다양성으로 즉시 보인다.

### 5. 달성해야 할 시각 요소

1. **broker 라이프라인 위의 분리 띠 — 직접 화살표의 부재**
   - 설명: 발행자와 구독자 사이에는 어떤 직접 화살표도 없다. 모든 흐름은 가운데 broker 라이프라인 위의 한 점을 반드시 통과하며, 그 사실이 위치만으로 강제된다. (essence 본질 1 — broker 가 사이를 막는다 / 강화 2 — 서로를 모른다의 시각적 강제)
   - 시각 표현: 발행자 영역(좌측)과 구독자 영역(우측) 사이를 broker 라이프라인을 중심선으로 하는 옅은 분리 띠로 채운다. 분리 띠는 좌·우 서로 다른 배경 톤(왼쪽은 차분한 청록 톤, 오른쪽은 차분한 황토 톤) 으로 정의되어 두 세계가 다른 영역임을 명시한다. 발행자→broker 화살표는 분리 띠의 좌측 경계에서 broker 라이프라인 위의 한 점까지 그어지고, 그 점에서 한 번 **끊어진 뒤** 같은 또는 다른 세로 위치에서 새 화살표로 다시 시작해 분리 띠의 우측 경계를 건너 구독자 라이프라인에 닿는다. 끊어짐 지점에는 broker 박스의 작은 사본 정지 구간(세로 체류) 이 항상 존재해, "broker 를 거쳐갔다" 가 시간 위의 정지 길이로 보인다.

2. **세로 시간축 위의 팬아웃 — 한 발행, N 사본, 다른 수신 시각**
   - 설명: 한 번의 발행이 broker 의 한 점에서 같은 색·같은 번호의 사본 N 개로 갈라져 모든 구독자에게 도달한다. 도달 시각은 서로 다르며, 그 차이가 시간축 위의 사선 배열로 보존된다. (essence 본질 2 — 한 발행 → N 사본 / 본질 3 — 시간 디커플링 / 강화 1 — 두 시간축의 직교 배치 / 강화 3 — 메시지 정체성 보존)
   - 시각 표현: 발행자 라이프라인에서 출발한 칩이 broker 라이프라인 위의 한 점에 도착하는 순간, 그 점에서 짧은 갈라짐 펄스가 퍼지며 같은 색·같은 번호 라벨의 사본 N 개가 부채꼴로 솟는다. 사본들은 broker 박스 안에서 각자 약간씩 다른 세로 길이의 정지 구간을 가진 뒤(랜덤한 짧은 지연), 라이프라인 위 서로 다른 세 시점에서 각각 구독자 라이프라인을 향해 가로 화살표로 출발한다. 도착한 자리마다 짧은 펄스가 퍼지고 칩이 안착한다. 같은 발행에서 비롯된 모든 사본은 색·번호가 동일하며, 도착 후에도 잔상으로 남아 시간축 위에 비스듬한 사선이 한 컷의 정지 화면으로 보존된다. 다중 발행자가 섞어 쏠 때는 발행자별로 색이 달라(P1=청록, P2=주황, P3=보라) 출처가 색으로 구별되고, 같은 발행에서 갈라진 사본들은 모두 그 발행자의 색을 그대로 이어받는다.

3. **토픽 라벨이 모든 연결의 매개 — 신원 없이 라벨만**
   - 설명: 발행자도 구독자도 상대의 신원이 아니라 토픽 이름만 안다. 모든 연결이 토픽 라벨을 거쳐서만 성립한다는 사실이 박스와 화살표의 라벨링으로 강제된다. (essence 본질 5 — 토픽 라벨이 라우팅을 정한다)
   - 시각 표현: broker 박스 안에는 항상 토픽 라벨 "events" 가 굵게 표시된다. broker 라이프라인 위의 발행자→broker 도착 점에는 "→ events" 라벨이 짧게 붙고, broker→구독자 출발 점에는 "events →" 라벨이 짧게 붙는다. 구독 매듭(구독선이 broker 라이프라인에 연결되는 작은 원 모양 마커) 옆에도 토픽 라벨 "events" 가 작게 표시된다. 발행자 박스에는 구독자 신원이 어디에도 없고, 구독자 박스에도 발행자 신원이 어디에도 없다. 박스 라벨은 P1, P2 / S1, S2, S3 같이 익명적이며 그들 사이에 직접 라벨로 연결된 줄은 한 줄도 없다.

4. **구독자의 동적 합류·이탈 — 시간축 위의 구독선 사건**
   - 설명: 시뮬레이션 도중 새 구독자가 우측 영역에 등장해 broker 의 토픽으로 구독선을 그으며 운동에 합류하고, 기존 구독자가 구독선을 끊고 사라진다. 합류 이전에 발행된 메시지는 새 구독자에게 가지 않는다는 사실이 시간축 위에서 자연스럽게 드러난다. (강화 4 — 동적 합류·이탈)
   - 시각 표현: 새 구독자 박스가 우측 영역의 빈자리에 페이드인되며 등장하고, 그 박스에서 broker 라이프라인을 향해 짧은 구독선이 그어져 토픽 "events" 위의 한 점에 매듭으로 안착한다. 매듭 옆에 "subscribe" 라벨이 잠시 떴다 사라진다. 그 매듭 시점 이후 broker 에서 펼쳐지는 모든 팬아웃 화살표 부채꼴에 새 구독자를 향한 가닥 한 줄이 추가된다. 매듭 이전 시점에서 그어진 화살표 부채꼴에는 그 가닥이 없으며, 새 구독자의 라이프라인 위 매듭 이전 구간은 빈 채로 남는다. 이탈은 반대로 작동: 기존 구독자의 라이프라인 위 한 시점에서 구독선이 끊어지는 매듭("unsubscribe" 라벨) 이 찍히고, 그 시점 이후의 팬아웃 부채꼴에서 그 가닥이 사라진다. 이탈 이후 도착했어야 할 메시지는 그 구독자에게 가지 않는다.

5. **다대다 + 토픽 격리 보조 — 베이스의 정면, 격리는 보조**
   - 설명: 베이스는 단일 토픽 위에서 발행자 M 명이 메시지를 섞어 쏘고 구독자 N 명이 모두 받는 다대다 운동을 정면으로 그린다. 보조 시각으로 우상단에 두 번째 토픽 채널을 짧게 두어, 한 메시지가 자기 토픽의 구독자에게만 가고 다른 토픽 구독자에게는 가지 않는 격리를 한 컷의 모양 차이로 운반한다. (essence 본질 4 — 다대다 / 강화 5 — 다중 publisher × 다중 subscriber 의 다대다, 토픽 격리는 보조)
   - 시각 표현: 본 시각의 좌측 발행자 박스는 두세 개(P1, P2, P3)이며 각자 고유 색(청록·주황·보라)을 가진다. 우측 구독자 박스는 서너 개(S1, S2, S3, S4)로 늘어서고, 모두 같은 broker 라이프라인 위 토픽 "events" 에 구독선이 매듭으로 안착해 있다. 시간축 위에서 P1, P2, P3 가 차례로 또는 거의 동시에 발행하면, broker 라이프라인 위에 서로 다른 색의 칩이 줄지어 도착하고 각자 사본 N 개로 갈라져 모든 구독자에게 도달한다. 화면 우상단의 작은 보조 영역에는 두 번째 broker 라이프라인(토픽 "alerts") 이 짧게 그려진다. S2 만 두 토픽 모두를 구독한 매듭 두 개를 가지고 있고, 본 시각에서 alerts 토픽으로 발행된 메시지는 broker 의 alerts 분기점에서만 갈라져 S2 에게만 도달한다. events 토픽 구독자들의 라이프라인 위 그 시점에는 화살표가 도달하지 않는 빈 자리가 명시된다. 보조 영역에는 "토픽 events 만 듣는 자리에는 alerts 메시지가 오지 않는다" 한 줄 캡션이 붙는다.

### 6. 시각 구성

- **중심 시각**: 화면 상단에 가로로 늘어선 박스 라인업(좌: P1, P2, P3 / 중: broker[토픽: events] / 우: S1, S2, S3, S4)과 그 아래로 화면 하단까지 뻗는 점선 라이프라인 일곱 줄. broker 라이프라인을 중심선으로 좌·우에 옅은 분리 띠가 깔린다. 시간은 위에서 아래로 흐른다. 메시지는 라이프라인 사이를 가로지르는 짧은 가로 화살표 위의 색·번호 칩으로 표현되며, broker 라이프라인 위에는 분기점마다 짧은 갈라짐 펄스와 broker 박스의 작은 정지 구간이 찍힌다.
- **보조 요소**:
  - 우상단 보조 토픽 채널: 두 번째 broker 라이프라인(토픽 "alerts") 와 그에 연결된 구독자 한 명(S2의 추가 매듭). 본 시각의 약 1/4 크기로 축소되어 격리만 짧게 운반한다.
  - 좌측 발행자 박스 옆의 발행 큐: 다음에 발행될 메시지 칩 두세 개가 발행자 박스 안쪽에 작은 색 칩 줄로 대기.
  - 우측 구독자 박스 옆의 수신 누적: 도착한 칩들이 구독자 박스 옆에 색 사슬로 누적되어, 그 구독자가 시뮬레이션 시작 이후 받은 메시지의 연대기를 이룬다.
  - 시간축 좌측 가장자리의 시간 눈금: 위→아래로 t1, t2, t3 ... 의 옅은 가로 보조선이 broker 박스 분리 띠를 가로질러 화면 폭 전체에 깔린다. 발행/수신 사건은 이 보조선 위 또는 사이에서 발화한다.
  - 상단 캡션 영역: 짧은 개념 문장 한 줄 + 현재 사건 메시지.
  - 하단 컨트롤 영역: 발행 버튼(발행자 선택 토글 P1/P2/P3 + 토픽 토글 events/alerts), subscribe / unsubscribe 버튼(대상 구독자 선택), 자동 시연·일시정지·속도 슬라이더·초기화.
- **요소 간 관계**: 모든 발행 사건은 (좌측 발행자 라이프라인의 한 점) → (broker 라이프라인 위의 한 점에서 한 번 끊어짐 + 짧은 정지 구간) → (broker 라이프라인 위의 같거나 다른 시점에서 N 가닥의 부채꼴 화살표로 다시 시작) → (우측 N 명 구독자 라이프라인 위의 서로 다른 N 시점) 의 4 단계 운동으로 1:1 대응된다. 분리 띠는 시각의 모든 직접 좌→우 단축 경로를 차단하고, 토픽 라벨은 분기점마다 라우팅의 매개임을 명시한다. 보조 토픽 채널은 본 시각과 같은 시간축을 공유하지만 별도 라이프라인 구획에서 동시에 펼쳐져, 같은 시각에 어떤 메시지는 events 로 어떤 메시지는 alerts 로 가는 격리 운동이 한 화면에 보존된다.

### 8. 진행과 인터랙션

메시지 시퀀스형. 시간축이 위→아래로 진행하는 동안 참여자(P1·P2·P3, broker, S1·S2·S3·S4) 사이를 오가는 메시지가 라이프라인 사이의 가로 화살표로 한 사건씩 발화된다. 자동 시연이 시간축을 자동으로 흘려 메시지 시퀀스를 펼치는 것이 기본이며, 사용자는 동적 합류·이탈을 입력 반응형 사건으로 끼워 넣어 시퀀스의 모양을 바꿀 수 있다.

- **참여자**:
  - 발행자 P1, P2, P3: 좌측 영역. 각자 고유 색. 시간축 위 정해진 발행 시점들에 자기 색의 메시지 칩을 broker 의 토픽 "events" 로 던진다.
  - broker(토픽 events): 가운데. 모든 화살표가 통과하는 1 급 라이프라인. 발행자→broker 도착 점에서 사본 N 으로 갈라져 broker→구독자 가닥들로 다시 출발시킨다.
  - 구독자 S1, S2, S3, S4: 우측 영역. 시뮬레이션 시작 시 모두 토픽 events 에 구독 매듭이 찍혀 있다. S2 는 보조 alerts 토픽도 함께 구독한 매듭을 가진다.

- **메시지 흐름 (한 발행 사건의 시퀀스)**:
  1. P1 라이프라인의 한 시점 (t2) 에서 칩 #7 (P1 색, 번호 7) 이 솟아 broker 라이프라인을 향해 가로 화살표로 출발한다.
  2. 칩 #7 이 broker 라이프라인 위 한 점 (t2 직후) 에 도착한다. 도착 점에 짧은 갈라짐 펄스가 퍼지고 broker 박스가 한 칸 굵어진다. 도착 화살표 머리 옆에 "→ events" 라벨이 잠시 뜬다.
  3. broker 박스 안에서 칩 #7 이 사본 4 개(S1·S2·S3·S4 용)로 갈라진다. 사본들은 broker 라이프라인 위 t2 직후 ~ t3 사이의 짧은 정지 구간(세로 체류)을 가진다.
  4. 사본 4 개가 broker 라이프라인 위 약간씩 다른 시점(t2+δ1, t2+δ2, t2+δ3, t2+δ4)에서 차례로 우측 구독자 라이프라인을 향해 가로 화살표로 출발한다. 각 화살표 시작점 옆에 "events →" 라벨이 잠시 뜬다.
  5. 사본 4 개가 S1, S2, S3, S4 의 라이프라인 위 서로 다른 시점에 도달한다. 도달 자리마다 짧은 펄스가 퍼지고 각 구독자 박스 옆 수신 누적에 칩 한 개가 추가된다.

- **다중 발행자의 섞임**:
  - P1, P2, P3 가 각자 다른 시점에 발행하면 broker 라이프라인 위에는 청록 칩, 주황 칩, 보라 칩이 시간 순서대로 도착한다. 각각 자기 분기점에서 사본 4 개로 갈라져 모든 구독자에게 도달하므로, 시간축 끝까지 진행되면 각 구독자의 수신 누적에는 청록·주황·보라 칩이 같은 순서로 한 줄씩 쌓여 있다 — "모든 구독자가 같은 발행 흐름을 같은 순서로 받는다" 가 색의 동일한 연쇄로 보존된다.

- **subscribe (동적 합류)**:
  - 사용자가 [구독 S?] 드롭다운에서 신규 구독자(예: S5) 또는 기존 미구독 구독자를 선택해 [구독] 을 누르면, 우측 영역의 빈자리(또는 해당 박스 옆)에 박스가 페이드인되고 박스에서 broker 라이프라인을 향해 짧은 구독선이 그어진다. 구독선이 broker 라이프라인 위의 현재 시간 점에 매듭으로 안착하고, 매듭 옆에 "subscribe events" 라벨이 잠시 뜬다.
  - 매듭 시점 이후의 모든 broker 분기점 부채꼴에 그 구독자를 향한 가닥이 추가된다. 매듭 이전 시점의 부채꼴에는 그 가닥이 없으며, 새 구독자의 라이프라인 위 매듭 이전 구간은 빈 채로 남는다 — "합류 이전 메시지는 받지 않는다" 가 빈 라이프라인 구간으로 보존된다.

- **unsubscribe (이탈)**:
  - 사용자가 [구독 S?] 드롭다운에서 기존 구독자(예: S3)를 선택해 [구독해지] 를 누르면, 그 구독자 박스에서 broker 로 향한 구독선의 라이프라인 위 현재 시간 점에 끊어짐 매듭("unsubscribe events" 라벨)이 찍힌다. 매듭 이후의 모든 broker 분기점 부채꼴에서 그 구독자를 향한 가닥이 빠진다 — 그 시점 이후 도착했어야 할 메시지가 그 구독자에게 가지 않는 사실이 가닥의 부재로 보존된다.

- **자동 시연 / 일시정지 / 속도 슬라이더**:
  - [자동 시연] 을 누르면 시간축이 위→아래로 자동 흐르고 미리 준비된 발행 시퀀스(P1 발행 → P2 발행 → S5 합류 → P3 발행 → S3 이탈 → P1 발행 → ...) 가 차례로 펼쳐진다. [⏸] 로 어떤 시점에든 멈추고, 속도 슬라이더(Slow / Normal / Fast) 로 한 사건의 운동 길이를 조절한다. 멈춘 시점에는 그 시점까지의 모든 칩과 화살표가 정지 화면으로 보존되어, 학습자는 사선 배열·부채꼴·구독선 매듭을 차분히 읽을 수 있다.

- **보조 토픽 채널 (alerts) 의 격리 시연**:
  - 자동 시연 중 일정 시점에 "alerts 토픽으로의 발행" 사건이 보조 영역에서 한 번 발화한다. 이 사건은 보조 영역의 두 번째 broker 라이프라인 위에서만 분기되어 S2 에게만 도달한다. 본 시각의 events 라이프라인에는 그 시점에 분기점이 찍히지 않으며, 다른 구독자들의 라이프라인 위 그 시점에는 화살표가 도달하지 않는 빈 자리가 명시된다. 캡션에 "토픽 events 만 듣는 자리에는 alerts 메시지가 오지 않는다" 가 잠시 뜬다.

- **수동 발행 (입력 반응형 보조)**:
  - 자동 시연이 멈춘 상태에서 [발행자 P?][토픽 ?][발행] 으로 한 사건을 직접 발화시킬 수 있다. 시간축이 한 칸 진행하며 그 발행 사건이 본 시각 또는 보조 영역에서 펼쳐진다.

- **초기화**: 모든 칩·화살표·매듭·수신 누적이 페이드아웃하며 사라지고, 빈 라이프라인 일곱 줄과 초기 구독 매듭(S1·S2·S3·S4 가 events 에, S2 가 alerts 에) 만 남는 t0 상태로 복귀.

- 모든 운동 길이: 발행자→broker 화살표 350ms, broker 분기점 갈라짐 펄스 200ms, broker 정지 구간(시간차 시각화) 200~600ms, broker→구독자 화살표 350~500ms, subscribe / unsubscribe 매듭 발화 300ms. Fast 모드에서는 절반, Slow 모드에서는 두 배.

### 9. 초기 상태와 파라미터

- **초기 상태 (t0)**: 박스 라인업이 정해진 자리에 그려져 있고, P1·P2·P3 의 라이프라인은 비어 있다. broker 박스에는 토픽 라벨 "events" 가 굵게 표시. broker 라이프라인 위에는 시뮬레이션 시작 시점에서 S1·S2·S3·S4 의 구독선 매듭이 모두 찍혀 있다(events 토픽). 우상단 보조 영역에는 두 번째 broker 라이프라인(alerts) 위 S2 의 매듭 하나가 찍혀 있다. 분리 띠가 발행자 영역과 구독자 영역 사이에 옅게 깔리고, 시간축 좌측 가장자리에는 t0 만 표시. 모든 수신 누적은 비어 있다.
- **권장 시연 시퀀스 (자동 시연 기본 시나리오, 약 8 사건)**:
  1. t1: P1 이 events 에 메시지 #1 (P1 색) 발행 → S1·S2·S3·S4 모두 수신 (도착 시각은 다름).
  2. t2: P2 가 events 에 메시지 #2 (P2 색) 발행 → 모두 수신.
  3. t3: 보조 영역에서 P1 이 alerts 에 메시지 #A1 발행 → S2 만 수신, 본 시각 events 라이프라인은 그대로.
  4. t4: S5 (신규) 가 events 에 subscribe → 우측 영역에 박스 페이드인, 매듭 안착.
  5. t5: P3 가 events 에 메시지 #3 (P3 색) 발행 → S1·S2·S3·S4·S5 모두 수신.
  6. t6: S3 가 events 에서 unsubscribe → 매듭 끊어짐.
  7. t7: P1 이 events 에 메시지 #4 발행 → S1·S2·S4·S5 수신, S3 의 라이프라인 위 그 시점에는 화살표 도달 없음.
  8. t8: 자동 시연 정지. 정지 화면에서 학습자는 각 구독자의 수신 누적과 사선 부채꼴들을 읽는다.
- **파라미터**:
  - 발행자 수: 3 (기준 폭). 색 팔레트는 청록·주황·보라.
  - 구독자 수: 4 (기준 폭, 자동 시연 후 S5 합류로 5 까지). 확장 폭에서는 초기 5.
  - 토픽: 본 시각 events 1 개 + 보조 alerts 1 개. 깊은 토픽 라우팅·와일드카드는 별도 사이클.
  - 시간축 보조선 간격: 60px 마다 t1~t6 (기준 폭). 한 발행 사건의 broker 통과 길이는 약 1.5 t 단위.
  - broker 정지 구간 (시간 디커플링 시각화) 길이: 200~600ms. 같은 발행에서 갈라진 사본들은 약간 다른 길이.
  - 메시지 칩 크기: 14px × 12px. 번호 라벨 흰 글자 8pt.
  - subscribe / unsubscribe 매듭 모양: 작은 원(직경 6px), 안착/끊어짐 시 짧은 펄스.
  - 자동 시연 사건 간격: Slow 1500ms / Normal 900ms / Fast 500ms.

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> Pub/Sub 은 메시지를 보내는 쪽과 받는 쪽이 서로의 신원을 모른 채로, 가운데 자리한 broker 의 토픽 라벨만 매개로 다대다 비동기 통신이 성립하도록 만든 메시징 패턴이다. 발행자가 토픽에 메시지를 한 번 던지면 broker 가 그 토픽을 신청한 모든 구독자에게 메시지의 사본을 한 번씩 펼쳐 보낸다. 발행 시각과 수신 시각은 서로 다르며, 발행자와 구독자는 서로의 가용성에 영향받지 않는다. 단순한 약속 위에서 동적 토폴로지·확장성·이벤트 기반 아키텍처가 자란다.

**학습자 안내 (보조 캡션 회전 문구)**

- "발행자는 토픽에만 던진다 — 누가 받는지 모른다."
- "구독자는 토픽만 신청한다 — 누가 보냈는지 모른다."
- "broker 가 사이에서 사본을 뿌린다 — 모든 화살표가 그 자리를 지난다."
- "한 발행, 여러 사본, 다른 도착 시각."
- "합류 이전의 메시지는 받지 않는다."
- "토픽이 다르면 같은 broker 안에서도 만나지 않는다."

**사건별 짧은 메시지**

- 발행: "토픽 events 에 메시지를 던졌다."
- broker 분기점 도달: "broker 가 받았다 — 사본을 펼친다."
- 사본 도달: "구독자에게 사본이 도착했다."
- subscribe: "새 구독자가 토픽 events 에 합류했다."
- unsubscribe: "구독자가 토픽 events 에서 빠져나갔다."
- 합류 이전 메시지: "이 메시지는 합류 이전이라 새 구독자에게는 가지 않는다."
- 이탈 이후 메시지: "이 시점 이후 그 구독자에게는 가지 않는다."
- 토픽 격리: "토픽 events 만 듣는 자리에는 alerts 메시지가 오지 않는다."

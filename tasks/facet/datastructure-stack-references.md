### R1. VisuAlgo — Linked List / Stack / Queue / Deque

- 출처: visualgo.net (싱가포르 NUS Steven Halim 그룹, 사실상 표준 시각화 도구)
- URL: https://visualgo.net/en/list
- 한 줄 정체성: 스택을 "보호된 단일 연결 리스트" 로 재해석한 표준 인터랙티브 도구. 노드 박스 + 화살표 + 머리 라벨로 LIFO 를 표현.
- 시각 객체: 직사각형 노드(값 표시) + 다음 노드를 가리키는 화살표 + head/tail 텍스트 라벨. 위→아래 수직 배치.
- 시간 표현: 단계 진행/자동 재생/속도 조절. push 는 head 위에 새 박스를 떨어뜨리며 화살표가 즉시 연결. pop 은 head 박스가 페이드아웃하며 다음 노드가 head 가 됨.
- 메타포: "head 만 만질 수 있는 리스트". 물리적 더미보다는 데이터 구조 추상화에 가까움.
- 강조: O(1) 연산 보장, head 이외 접근 금지(보호) 라는 ADT 계약.
- 인상적 디테일: 같은 화면에서 큐/덱과 토글하며 비교 가능. 최대 7개 제한으로 시각적 과부하 방지.
- 약점: 노드 + 화살표 표현이라 "쌓인다"는 물리적 직관이 약하다. 학생이 "왜 head 가 위인가" 를 별도로 학습해야 함.

### R2. USFCA Galles — Stack (Array Implementation) / Stack (Linked List Implementation)

- 출처: cs.usfca.edu/~galles/visualization (DSA 시각화의 고전. 거의 모든 한국 대학 강의 노트가 인용)
- URL: https://www.cs.usfca.edu/~galles/visualization/StackArray.html , https://www.cs.usfca.edu/~galles/visualization/StackLL.html
- 한 줄 정체성: 배열 구현 vs 연결 리스트 구현을 별도 페이지로 분리해 "동일 ADT, 다른 메모리 배치"를 직관화한 클래식 도구.
- 시각 객체: 배열판은 가로 셀 + top 인덱스 화살표. 연결 리스트판은 세로 노드 박스 + next 포인터 + top 라벨.
- 시간 표현: Push/Pop 버튼 클릭 시 즉시 애니메이션. 이산적 단계 — 한 클릭 = 한 연산.
- 메타포: 배열판은 "고정 칸을 채우는 카운터", 연결판은 "노드를 머리에 매달기".
- 강조: top 포인터의 이동(배열은 인덱스 증감, 연결은 노드 머리 교체).
- 인상적 디테일: 두 구현을 분리해 학생이 추상 vs 메모리 레이아웃을 구분할 수 있게 함. CLEAR 버튼으로 초기화.
- 약점: UI 가 90년대 자바 애플릿 스타일. 색감/모션이 평면적이고 underflow/overflow 메시지가 작은 텍스트뿐.

### R5. CSanim — Queues vs. Stacks: A brief visual explanation

- 출처: csanim.com (애니메이션 기반 CS 튜토리얼)
- URL: https://csanim.com/tutorials/queues-vs-stacks-brief-visual-explanation
- 한 줄 정체성: 큐와 스택을 한 화면에서 대비시키는 비교형 시각화. 진입/퇴출 방향이 다른 두 컨테이너를 좌우로 놓고 같은 입력 시퀀스를 흘려보냄.
- 시각 객체: 가로 큐(앞=좌, 뒤=우) vs 세로 스택(top=상). 같은 데이터 항목이 두 구조에서 어떻게 다르게 흐르는지를 동시에 보여줌.
- 시간 표현: 시퀀스 다이어그램(상태 스냅샷 나열). 정적이지만 좌→우 진행으로 시간성 부여.

- 강조: 방향성이 곧 정책(LIFO/FIFO) 임을 위치만으로 전달.
- 인상적 디테일: 두 구조에서 같은 입력 [1,2,3,1] 의 결과 시퀀스가 명확히 다르게 펼쳐지는 시각.
- 약점: 정적 다이어그램이라 운동감이 부족. 인터랙션 없음.

## 3. 공통 요소

거의 모든 레퍼런스가 공유하는 시각 언어:

1. **수직 박스 더미** — 직사각형 셀을 위→아래(또는 아래→위) 로 쌓는다. 각 셀은 값 하나를 담는다. 가장 흔한 표현. (R1, R2-LL, R3, R4, R6, R8, R9, R10)
2. **top 포인터/라벨** — 가장 위 셀을 화살표·테두리·색상으로 강조. "여기만 만진다" 는 LIFO 의 시각적 약속. (전 레퍼런스)
3. **push/pop 의 위쪽 한 점** — 진입과 퇴출이 모두 같은 위치에서 일어난다는 사실을 화살표 두 개가 한 점에 모이는 그림으로 표현. (R1, R2, R3, R8, R10)
4. **수평 입력 트랙과의 분리** — 외부 시간(입력 시퀀스) 은 가로, 내부 시간(스택 상태) 은 세로로 두어 두 시간축을 직교 배치. (R5, R6, R9)
5. **버튼 기반 입력 반응형 인터랙션** — Push 입력 + Pop 버튼 + (선택) Peek 버튼. 한 클릭 = 한 연산이라는 이산적 시간 진행. (R1, R2, R3, R4)
6. **빈 스택의 명시적 상태** — 비어 있을 때 회색/플레이스홀더로 표시하거나 "isEmpty" 라벨을 노출. (R1, R4)
7. **두 구현(배열 vs 연결 리스트) 의 분리 또는 토글** — 동일 ADT 가 다른 메모리 모양을 가질 수 있음을 명시. (R2, R7)
8. **응용 시 외부 트랙과의 동기화** — 괄호 매칭, 콜스택, 되돌리기 모두 "외부 사건 한 단계 → 스택 한 변화" 의 1:1 대응을 보여줌. (R6, R9, R10)

학습자에게 형성된 기대:
- 스택은 위로 자란다(혹은 아래로 자란다는 변형). 어느 쪽이든 "한쪽 끝 한 점" 에서만 변화한다.
- top 은 항상 가장 최근에 들어온 원소다.
- 입력 순서를 거꾸로 뒤집어야 할 때 스택이 등장한다.

## 4. 공백

기존 레퍼런스에서 약하거나 비어있는 영역:

1. **"잊혀짐(LIFO 의 그림자)" 의 시각화** — 스택 아래쪽 원소들은 위가 비워지기 전까지 절대 접근 불가능하다는 사실이 시각적으로 약하게 표현된다. 대부분 같은 색·같은 명도로 그려, "묻혀 있다" 는 압박감이 없다. 잔상/투명도/심도 효과로 강조한 사례 부재.
2. **두 시간축의 동시 강조** — 외부 입력 시간(가로) 과 스택 내부 시간(세로) 을 한 화면에서 운동으로 동기화한 인터랙티브 사례가 드물다. R5/R9 가 정적으로 시도했을 뿐, 운동으로 두 축의 직교를 보여준 도구는 거의 없다.
3. **큐와의 직접 시각 대비** — 같은 입력 시퀀스를 두 자료구조에 동시에 흘려보내는 운동형 비교가 거의 없다(R5 가 정적으로만). "FIFO 면 어땠을까" 라는 반사실적 시각이 LIFO 의 본질을 더 선명하게 만들 여지.
4. **메모리 할당과의 연결** — 콜스택(R6) 외에 "스택 메모리" 자체를 자료구조 학습 시각에 통합한 사례 없음. push 가 곧 SP 감소(또는 증가) 라는 시스템 관점은 별도 학습으로 분리됨.
5. **여러 비유의 통합 모델** — 접시·책·콜스택·되돌리기·괄호 가 모두 같은 LIFO 라는 사실을 한 화면에서 토글하며 보여주는 자료가 없다. 각 비유가 각자의 페이지에서 따로 다뤄짐.
6. **오버플로/언더플로의 정서적 표현** — 빈 스택에서 pop 시도, 가득 찬 스택에 push 시도가 작은 에러 텍스트로 처리될 뿐. "예외" 가 일어났다는 시각적 사건성이 약하다(흔들림, 빨간 깜빡임 등 부재).
7. **"순서 보존" 의 핵심 학습 질문 직접 타격** — 분류 보고서의 핵심 질문은 "어떻게 가장 최근 원소를 우선 반환하도록 순서가 보존되는가" 인데, 대부분의 레퍼런스는 push/pop 의 동작은 보여주지만 "순서가 어떤 규칙으로 보존되는지" 자체를 메타적으로 시각화하지 않는다. 입력 시퀀스의 색을 그대로 가져와 출력 시퀀스로 거꾸로 흘려 보내는 운동(역전 메타포) 이 명시적으로 표현된 사례가 적다.
8. **peek 의 의미** — pop 과 시각적으로 어떻게 다른가가 약하게 처리됨. R3 만 별도 버튼. "본다 vs 가져간다" 의 차이가 시각 사건으로 잘 안 그려진다.

## 5. 대상 개념 요약

> 스택은 가장 최근에 들어온 원소를 가장 먼저 꺼내도록 강제하는 자료구조다(LIFO). 모든 변화는 "꼭대기(top)" 라는 한 점에서만 일어나며, push 는 새 원소를 그 위에 쌓고 pop 은 그 위 원소를 떼어낸다. 이 한 점 제약 덕분에 입력 순서가 자연스럽게 뒤집혀 출력되며, 함수 호출의 복귀, 되돌리기, 괄호 매칭처럼 "가장 최근의 사건을 먼저 끝내야 하는" 모든 상황의 보편 도구가 된다.

학습자가 시각화와 함께 읽을 짧은 캡션:
- "맨 위만 만진다." (한 점 제약)
- "마지막에 들어온 게 가장 먼저 나온다." (LIFO)
- "쌓는 순서가 곧 뒤집히는 순서다." (순서 역전)
- "아래쪽은 위가 비워질 때까지 잠겨 있다." (접근 불가)

용어 풀어쓰기:
- top → "꼭대기. 지금 만질 수 있는 단 하나의 자리"
- push → "꼭대기 위에 새 원소를 쌓는다"
- pop → "꼭대기의 원소를 떼어낸다"
- peek → "꼭대기를 보지만 떼어내지 않는다"
- LIFO → "마지막에 들어온 것이 가장 먼저 나간다"

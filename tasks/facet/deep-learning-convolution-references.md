# 딥러닝 합성곱 연산 — 시각화 레퍼런스 분석

조사 대상: 딥러닝 CNN 의 합성곱(convolution) 연산. 입력 텐서(이미지)와 커널의 슬라이딩 윈도우 곱·합, 스트라이드, 패딩, 다중 채널/필터, receptive field, 가중치 공유.

## 2. 레퍼런스 요약

### CNN Explainer (Polo Club)
- 시각 객체: 색상 입혀진 픽셀 격자(입력 RGB), 3×3 커널 사각형, 화살표로 연결된 중간/출력 피처맵, 우측 수식 패널.
- 시간 표현: 호버 인터랙션 — 출력 픽셀 위에 마우스를 올리면 해당 receptive field 가 입력에서 강조되며 곱셈식이 수식 패널에 표시.
- 메타포: "내려다보는 X-ray". 활성값을 색 강도로 직접 노출.
- 강조: 다중 입력 채널의 커널이 각각 따로 슬라이드 → 합산된다는 사실, 그리고 dot-product 의 수식적 본질.
- 인상적 디테일: Convolutional Elastic Explanation View — 각 입력 채널마다 작은 커널이 동시에 슬라이드. 수식이 픽셀과 라이브 동기화.
- 약점: 화면이 빽빽해 정보 밀도가 매우 높음. 스트라이드/패딩 자체를 변주해 보여 주지는 않음(고정 아키텍처).

### Stanford CS231n 강의 노트
- 시각 객체: 정적 다이어그램 — 입력 볼륨(W×H×D 직육면체), 필터 볼륨, 출력 볼륨이 좌→우로 나열.
- 시간 표현: 정적 + 일부 GIF. "이 슬라이스가 이 슬라이스로 갑니다" 형태의 단계별 다이어그램.
- 메타포: 3D 직육면체 스택. "depth column", "depth slice" 어휘를 그림으로 정의.
- 강조: 차원 산술(W2 = (W1 - F + 2P)/S + 1), 가중치 공유의 의미.
- 약점: 직관보다 표준화된 정의가 우선. 인터랙티브 요소 적음.

### vdumoulin/conv_arithmetic
- 시각 객체: 파란 격자(입력) 위로 그림자 영역(현재 receptive field)이 슬라이드, 위쪽에 시안 격자(출력)가 점차 채워짐.
- 시간 표현: 짧은 루프 GIF — 한 출력 픽셀이 채워질 때 그림자가 한 칸 이동.
- 메타포: "도장 찍기" — 커널이 입력 위를 지나가며 출력에 한 점씩 도장을 남김.
- 강조: 패딩/스트라이드/dilation/transposed conv 의 기하학적 차이. 차원 산술의 시각적 일치성.
- 인상적 디테일: 16 종 변형이 동일한 시각 언어로 정렬되어 비교 가능.
- 약점: 단일 채널, 스칼라 0/1 추상. 값/곱셈은 보여 주지 않음 — 순수히 "어디서 어디로" 만 보여 줌.

### Distill — Computing Receptive Fields
- 시각 객체: 여러 레이어 위에 색이 다른 사각형이 누적, 깊은 층의 한 뉴런이 입력 어느 영역까지 닿는지를 색칠.
- 시간 표현: 정적 + 슬라이더(레이어 깊이를 바꾸면 RF 박스가 커짐).
- 메타포: "빛 원뿔(light cone)" — 깊어질수록 입력에서 끌어오는 영역이 넓어짐.
- 강조: 가중 contribution(중심 픽셀이 더 큰 영향, 가우시안 분포).
- 약점: 합성곱의 수치 연산 자체보다는 "정보가 어디에서 오는가" 라는 거시 관점에 한정.

### Distill — Feature Visualization
- 시각 객체: 합성된(합성곱이 학습한) 환각 이미지들의 격자, 채널/뉴런을 클릭해서 활성화 패턴 비교.
- 시간 표현: 정적이지만 그리드 인터랙션(클릭→큰 이미지로 확대).
- 메타포: "신경의 꿈". 학습된 필터가 무엇을 좋아하는지 역으로 보여 줌.
- 강조: 합성곱이 학습한 "의미" 의 결과물. 연산 메커니즘이 아니라 결과물 미학.
- 약점: 합성곱 메커니즘 자체와는 거리가 있음. 전문가 청자 가정.

### 3Blue1Brown — But what is a convolution?
- 시각 객체: 두 수열이 만나 곱해지는 "두 슬라이딩 함수"; 한쪽이 뒤집혀 다른 쪽 위로 미끄러지는 애니메이션. 이미지 예시는 흐리기/샤프닝 커널 데모.
- 시간 표현: 부드러운 manim 애니메이션. 슬라이드 중간 단계마다 곱·합이 그래프로 누적.
- 메타포: "두 함수가 미끄러지며 만나는 면적". 확률(주사위 합)·다항식·이미지를 한 메타포로 통합.
- 강조: 교환법칙, FFT 가속, 확률·이미지·신호 사이의 통일성.
- 인상적 디테일: 동일 메타포로 1D → 2D → FFT 까지 끌고 가는 일관성.
- 약점: CNN 의 "다중 채널/필터/스트라이드/배치" 같은 딥러닝 전용 디테일은 없음.

### Setosa.io — Image Kernels
- 시각 객체: 한 장의 흑백 이미지 위로 3×3 커널이 슬라이드. 픽셀이 강조되며 9 개의 곱 + 합 식이 옆에 풀어쓰여짐.
- 시간 표현: 마우스 호버에 따라 임의 위치로 이동. 사용자가 시간을 통제.
- 메타포: "Photoshop 필터의 내부". 학습보다는 신호 처리 관점.
- 강조: 동일 데이터에 다른 커널을 갈아 끼우면 출력이 어떻게 달라지는지(엣지/블러/샤픈).
- 인상적 디테일: 출력 픽셀과 입력 9 개를 연결하는 9 개 라인(파이프) 시각화.
- 약점: 단일 채널, 단일 커널, 학습 맥락 없음.

### Deeplizard Interactive Demo
- 시각 객체: 입력 격자 + 커널 + 출력 격자를 나란히 배치. 출력 픽셀 호버 시 입력의 receptive field 와 커널 셀이 동시에 강조.
- 시간 표현: 호버. 자동 재생 없음.
- 메타포: 간단한 표/스프레드시트.
- 강조: 입력↔출력 셀의 1대다 매핑.
- 약점: 디자인이 평범. 채널/필터 개념은 강의 영상에 분리.

### Multi-Channel Convolutions in MS Excel (Thom Lane)
- 시각 객체: Excel 셀로 구성된 입력 RGB 채널 3 장, 커널 3 장, 곱셈 결과 3 장, 합산된 출력 1 장.
- 시간 표현: 정적 스크린샷 시퀀스(스크롤이 곧 시간).
- 메타포: 익숙한 스프레드시트 — "이건 결국 표 계산". 마법을 평범하게 분해.
- 강조: 다중 입력 채널이 어떻게 단일 출력 채널로 줄어드는지(채널 합).
- 인상적 디테일: 수식이 보이는 친밀한 도구로 환원.
- 약점: 정적, 수동 스크롤. 큰 텐서로 확장 불가.

### Adam Harley — 3D CNN Visualization
- 시각 객체: 사용자가 마우스로 그린 28×28 숫자 → 합성곱 레이어들이 3D 카드 더미로 쌓임. 각 활성값이 작은 정육면체로 표현.
- 시간 표현: 사용자가 그릴 때 실시간 forward pass. 카드 더미가 즉시 갱신.
- 메타포: "건물 단면" — 위에서 아래로 정보가 변환되어 흐름.
- 강조: 채널/필터의 다수성, 그리고 깊이가 깊어질수록 추상화되는 활성.
- 인상적 디테일: 카드 위에 마우스를 올리면 그 뉴런으로 들어오는 모든 가중치 라인이 빛처럼 그어짐 — 가중치 공유와 receptive field 가 한 화면에서.
- 약점: 한 번에 보여 주는 정보가 너무 많아 "한 출력 픽셀이 어떻게 계산되었나" 미시 단계는 흐려짐.

### ezyang Convolution Visualizer
- 시각 객체: 입력/커널/출력 격자 3 장. 출력 셀에 호버하면 기여한 입력 셀과 커널 셀이 색으로 매칭. 입력 셀에 호버하면 영향 받은 출력 셀들이 매칭(역방향).
- 시간 표현: 양방향 호버. 자동 재생 없음.
- 메타포: "스프레드시트 + 형광펜". 양쪽으로 추적 가능한 양방향 트레이스.
- 강조: 한 입력 픽셀이 여러 출력에 영향(수용 영역 중복), 한 출력 픽셀이 여러 입력에서 모임.
- 인상적 디테일: 스트라이드/패딩/dilation 컨트롤이 모두 즉시 반영.
- 약점: 값이 모두 0/1 또는 인덱스 — 실제 곱·합 수식은 부차적.

### TensorSpace.js
- 시각 객체: 3D 공간에 레이어가 카드처럼 부유. 카드를 클릭하면 채널이 펼쳐지고, 입력 이미지가 차례로 변환되어 보임.
- 시간 표현: 사용자가 입력을 넣으면 forward 흐름 애니메이션이 좌→우로 흐름.
- 메타포: "공장 컨베이어 벨트" 또는 "유리 빌딩 단면 투어".
- 강조: 모델 전체 구조와 중간 활성. 한 합성곱 연산보다는 "파이프라인 어디쯤" 을 보여 줌.
- 약점: 한 합성곱의 곱·합 미시 단계는 보여 주지 않음. 스트라이드/패딩 변주 어려움.

### Convolution Solver & Visualizer (ybouane)
- 시각 객체: 입력/커널/출력 사각형 + 파라미터 입력 폼(stride, padding, dilation).
- 시간 표현: 자동 재생되는 슬라이딩 윈도우 애니메이션.
- 메타포: "계산기 + GIF 생성기".
- 강조: 출력 크기 산식과 시각적 검증.
- 약점: 단일 채널, 가치 표현 없음.

## 3. 공통 요소

레퍼런스를 넘어 거의 모두가 공유하는 시각 언어 — 학습자가 "합성곱 시각화" 라는 말을 들을 때 자동으로 떠올리는 기대다.

공통 시각 객체
- **2D 격자**: 입력은 정사각 격자, 출력도 정사각 격자. 픽셀은 색 또는 숫자.
- **반투명 사각형 커널**: 입력 위에 슬라이드하는 작은 사각형. 보통 더 진한 테두리/색.
- **그림자/하이라이트**: 현재 커버되는 입력 영역과, 그것이 산출하는 출력 셀에 동일 색.
- **출력 격자가 점진적으로 채워짐**: 빈 격자 → 한 칸씩 채워지는 진행 표현.

공통 시간 표현
- **이산 스텝**: 슬라이딩이 연속이 아니라 격자 칸 단위 점프(스트라이드 만큼).
- **루프**: 끝까지 가면 처음으로 돌아감(GIF 의 천성).
- **호버 = 시간**: 자동 재생 대신 사용자가 출력 픽셀을 가리킬 때만 그 순간이 켜짐.
- **forward 일방향**: 위 → 아래 또는 좌 → 우. 역방향(backprop) 는 거의 다루지 않음.

공통 강조 방식
- **색 매칭**: 입력의 패치 색 = 출력 셀 색 = 커널 색. 같은 색이 곧 "동일한 한 번의 연산".
- **숫자 노출**: 픽셀에 정수 0~9 또는 0/1. 실수와 큰 텐서는 회피.
- **수식 사이드 패널**: "(1×0)+(2×1)+..." 같이 곱셈을 풀어쓴 식이 한쪽에 동기화.

공통 정보 밀도
- 한 화면에 입력/커널/출력 3 장을 나란히. 깊이는 보통 1 채널, 다중 채널은 별도 챕터.
- 텐서 크기는 작게(7×7, 28×28). 실제 224×224 RGB 는 거의 보여 주지 않음.
- 활성값은 색 강도(grayscale 또는 heatmap) 또는 작은 수자.

공통 레이아웃
- 좌측 입력 — 가운데 커널/연산 — 우측 출력. 또는 위 입력 — 아래 출력.
- 다층 시각화는 좌 → 우 파이프라인.

학습자에게 형성된 기대 패턴
- "합성곱 = 작은 사각형이 큰 사각형 위를 돌아다닌다."
- "출력은 입력보다 작거나 같다(패딩 없으면 작다)."
- "한 번에 한 칸 — 도장 찍기."
- "RGB 이면 커널이 3 장 겹쳐 있다."
- "깊이가 늘면 채널이 늘어난다(블록이 두꺼워진다)."

## 4. 공백

레퍼런스 평균이 다루지 않는, 또는 거의 다루지 않는 측면.

차용 가능한 다른 분야의 메타포
- **직조/베틀**: 커널 = 북(shuttle), 입력 = 날실, 출력 = 짜인 천. 가중치 공유가 "같은 패턴이 반복된다" 로 자연 설명됨.
- **도장과 인주**: 커널이 도장, 출력이 종이. 같은 도장이 여러 곳에 찍힌다(가중치 공유의 직접적 은유).
- **체/필터/체질(sieving)**: 입력을 한 쪽에서 흘려 보내면 커널이 채로 작용해 특정 패턴만 통과. dilated conv 가 "구멍 큰 체".
- **음식 조리법**: 커널 = 레시피(예: "위 + 가운데×2 + 아래"), 입력 = 재료, 출력 = 한 입(serving). 한 솥에 여러 레시피(필터) 적용.
- **현미경 스캔**: 커널 = 시야창. 한 점씩 옮기며 신호 강도 기록. receptive field 가 자연스러움.
- **라디오 튜닝/공명**: 학습된 커널이 특정 주파수에 공명하는 안테나. "이 채널은 가로 엣지에 공명한다" 식.

시간 표현의 다른 가능성
- **병렬 시간**: 한 출력 칸의 곱셈 9 개가 *동시에* 발생하는 "한 박자 = 여러 곱셈" 표현. 현재는 거의 모두 직렬 슬라이드.
- **역방향 재생**: 출력 픽셀에서 시작해 입력으로 거슬러 가는 backward 시간. 학습 직관에 결정적.
- **여러 필터의 동시 시간**: 8 개 필터가 동시에 다른 출력 채널을 채워 가는 폴리포니. 보통 한 필터씩 차례로.
- **시간 압축/느림**: 픽셀 단위 슬라이드를 빠르게(전체) 또는 매우 천천히(한 곱셈 단위로 분해) 토글.
- **체크포인트**: 슬라이드의 임의 시점에서 멈추고 부분합·중간 상태를 검사할 수 있는 "스크럽 가능한 타임라인".

차원 변환의 가능성
- **다채널을 다채널로**: 거의 모든 도구가 RGB 입력을 단일 출력으로 줄임. 입력 C_in 이 출력 C_out 으로 *늘어나는* 차원 확장(필터가 여러 개)을 동시 시각화한 사례는 드묾.
- **3D 입체로의 들어 올림**: 2D 격자 대신 입력 큐브(C×H×W) 를 직접 큐브로 보여 주고 커널을 3D 슬랩으로 표현. CS231n 이 정적으로만 함.
- **벡터 필드/광장 vs 격자**: 픽셀 격자 외 비격자(point cloud, graph)에서의 합성곱.
- **차원 1로의 환원**: 합성곱을 GEMM(im2col) 로 펼쳐 보이는 경로. 격자 슬라이드와 행렬 곱 사이의 동치를 *동일 화면* 에서 토글로 비교한 사례 거의 없음.

정보 밀도의 다른 접근
- **"오케스트라" 줌 아웃**: 64 개 필터가 동시에 도는 거시 화면 → 한 필터로 줌 인 → 한 픽셀 곱셈으로 줌 인. 줌 레벨이 곧 추상화 레벨.
- **희소 표현**: 모든 픽셀을 그리지 않고, 활성도 상위 N % 만 강조. 학습된 모델의 sparsity 를 직접 활용.
- **수치 대신 의미**: 픽셀 자리에 "엣지 감지기/색 감지기/모서리 감지기" 같은 학습된 의미 라벨을 직접 표시.
- **누적 contribution**: 한 출력에 몇 개의 곱이 더해졌는지를 셀에 점/획으로 누적해 *한 컷에* 전체 과정을 압축.

인터랙션의 미개척 영역
- **사용자가 커널을 직접 그리기**: 사용자가 마우스로 3×3 셀에 값을 찍어 즉시 출력 변화 확인. Setosa 가 일부 시도.
- **"왜 이 픽셀이 활성?" 의 인터랙티브 역추적**: 활성 큰 출력에서 클릭 → 입력 영역과 커널을 동시에 강조하며 *원인 분해*.
- **스트라이드/패딩의 슬라이더 토글**: ezyang 정도만 본격 시도. 대부분 정적.
- **다른 입력으로 같은 커널 시험**: 학습된 커널을 사용자 이미지에 즉시 적용해 "이 커널이 좋아하는 패턴" 을 직관 파악.
- **가중치 공유의 인터랙티브 증명**: 한 위치에서 커널 값을 바꾸면 *모든 출력 위치* 가 동시에 바뀜을 라이브로 보여 주기. 거의 시도 없음.
- **소리/햅틱 차용**: 곱셈 한 번마다 "딸깍" 소리 또는 박자 표시. CNN 시각화 영역에서는 거의 시도 없음.

본질적이지만 평균이 잘 다루지 못하는 개념들
- **가중치 공유(weight sharing)** 자체의 시각화. 거의 모든 도구가 "한 칸 슬라이드" 를 보여 주지만 *그 칸의 가중치가 동일하다* 는 사실은 암묵.
- **배치 차원**: 입력이 32 장 동시에 들어가는 batched conv 는 사실상 어디에도 없음.
- **bias 항**: 출력 = (가중합) + bias. bias 는 거의 누락.
- **활성 함수와의 연결**: conv 다음 ReLU. 합성곱 단독으로 그치고 비선형 결합은 다른 챕터로 분리.

## 5. 후속 작업 가이드

### 공통 강화 에이전트에게

가장 가치 있는 공통 요소(반드시 보존)
- **2D 격자 + 슬라이딩 사각형**: 학습자의 자동 연상. 이걸 빼면 "이건 합성곱 시각화가 아니다" 라는 위화감을 만든다.
- **색 매칭 트레이스**: 한 출력 셀 ↔ 그 receptive field ↔ 사용된 커널 셀이 같은 색. 이 한 가지 규칙이 "어디서 어디로" 를 한 번에 가르친다.
- **수식 사이드 패널**: 곱·합의 풀어쓴 식. 시각만 있으면 추상이 안 잡히고, 수식만 있으면 직관이 안 잡힌다. 양손이 필요.
- **이산 스텝**: 부드러운 보간이 아니라 격자 칸 단위 점프. 스트라이드 의미가 시간에 새겨져야 한다.
- **호버로 시간 통제**: 자동 재생만 있으면 학습자가 따라가지 못한다. 호버/클릭으로 멈출 수 있어야 한다.

다듬을 여지
- **양방향 트레이스**(ezyang)를 기본으로 격상: 출력에서 입력으로, 입력에서 출력으로 모두 추적.
- **다중 채널을 일급 시민으로**: RGB 가 부속 챕터가 아니라 첫 화면의 일부. 3 장 채널이 동시에 슬라이드해 합쳐지는 모습을 처음부터 보여 줘야 한다.
- **bias 와 activation 가시화**: 풀어쓴 식의 마지막 줄에 + b, 그리고 ReLU(·) 한 단계 더.
- **스트라이드/패딩을 변수로**: 정해진 스트라이드 1 이 아니라, 사용자가 슬라이더로 바꾸며 출력 크기 변화를 즉시 보는 것이 vdumoulin 의 진짜 가치. 정적 GIF 16 종을 인터랙티브 1 개로.

### 새로운 방식 에이전트에게

가장 흥미로운 공백(시도 우선순위)
1. **가중치 공유의 라이브 증명** — 사용자가 커널 한 셀의 값을 바꾸는 순간, 출력 격자의 *모든 칸* 이 동시에 갱신된다. 가중치 공유라는 추상적 사실을 인터랙션으로 *증명*. 평균이 누락한 본질.
2. **스케일 줌(오케스트라 → 솔로)** — C_out 개 필터가 동시에 도는 거시 → 한 필터로 줌 → 한 픽셀 곱셈으로 줌. 줌 레벨이 추상화 레벨이며, 한 도구로 거시·미시를 통합한다.
3. **forward/backward 토글** — 출력 픽셀에서 시작해 입력으로 거슬러 가는 시간. 학습 직관(어떤 입력이 이 활성을 만들었나) 을 *합성곱 도구 안에서* 다루는 사례가 거의 없다.
4. **격자 슬라이드 ↔ im2col 행렬곱 토글** — 동일 연산의 두 표현. "이건 도장 찍기다 / 이건 사실 행렬 곱이다" 를 한 화면에서 등치시키는 시각화는 드묾.

시도해 볼 만한 메타포
- **직조의 북(shuttle)**: 가중치 공유와 받침의 일관성이 자연스러움.
- **도장**: 가장 직접적이고, 학습자가 "왜 같은 결과가 여러 곳에" 를 묻지 않게 됨.
- **튜닝된 안테나/공명기**: 학습된 필터가 *특정 패턴을 좋아한다* 는 의미적 사실을 메커니즘과 한 화면에 둘 수 있음.
- **현미경 스캔**: 한 점 한 점 측정 — 시간이 자연스럽고, dwell time 으로 dilation/stride 가 표현 가능.

절대 빼면 안 될 본질적 개념
- **슬라이딩**: 같은 연산이 위치를 바꿔 가며 반복된다.
- **곱·합(dot product)**: 한 출력 = 입력 패치와 커널의 원소별 곱의 합.
- **가중치 공유**: 슬라이드해도 커널 값은 *같다*. 위치마다 바뀌지 않는다.
- **다중 채널 합산**: C_in 채널 각각의 곱·합을 *더해서* 한 스칼라.
- **다중 필터 → 다중 출력 채널**: 필터 하나당 출력 채널 하나.
- **스트라이드 / 패딩 / 출력 크기 산식**: (W - F + 2P)/S + 1 이라는 기하.
- **receptive field**: 한 출력 한 점이 입력의 어느 영역을 본다.

이 일곱 가지를 모두 담는 새 메타포가 있다면 그것이 평균을 넘는 후보다. 하나라도 *누락*하면 합성곱이 아니라 합성곱 비슷한 무언가가 된다. 메타포 후보 중 직조와 도장이 본질 보존도가 가장 높다(슬라이딩/공유/곱·합이 직접 대응됨). 안테나는 "학습된 필터의 의미" 까지 끌어와 본질을 *확장*하는 보너스가 있지만 곱·합의 셈은 약하므로 도장/직조와 결합해야 한다.

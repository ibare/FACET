# 컴파일러/언어 분야 유한오토마타 — 시각화 레퍼런스 분석

조사 대상: 컴파일러와 언어 분야의 유한오토마타(Finite Automata). DFA / NFA 가 정규언어를 인식하는 메커니즘, 렉서/스캐너에서 입력 문자열을 한 글자씩 소비하며 상태 사이를 전이하다가 수락/거부에 이르는 과정. NFA 의 동시 다중 상태와 ε-전이, 부분집합 구성으로 DFA 가 만들어지는 맥락도 부수적으로 포함.

## 2. 레퍼런스 요약

### JFLAP
- 시각 객체: 라벨된 원형 노드(상태) + 화살표(전이) + 이중원(accept). 사용자가 캔버스에 직접 노드/엣지를 그리는 그래프 에디터.
- 시간 표현: "Step" / "Step with Closure" 버튼 — 입력 문자열을 한 글자씩 소비, 현재 활성 상태가 하이라이트로 점등. NFA 는 *여러* 노드가 동시에 점등.
- 메타포: 손으로 그린 회로도. 학생이 직접 만드는 도구라는 정체성.
- 강조: 입력 테이프 위 커서가 한 칸씩 진행, 상태 박스 안에 "현재 상태 집합" 이 명시. ε-전이 강조 보기, 비결정성 강조 보기 토글.
- 인상적 디테일: NFA 시뮬레이션 시 화면이 분기되어 *여러 분기 트리* 가 펼쳐지는 옵션. accept 가 가능한 분기는 녹색.
- 약점: 자바 스윙 UI, 시각적 매력이 낮음. 그래프 자동 레이아웃이 거칠어 학생이 직접 정리해야 함.

### Automata Tutor
- 시각 객체: 깔끔한 원/화살표 그래프 + 채점 결과 패널. 학습자가 그린 오토마타와 정답을 비교하는 reverse-feedback 형식.
- 시간 표현: 정적 검증 위주. 단계별 시뮬레이션은 부수적.
- 메타포: "숙제 자동 채점기".
- 강조: 학습자의 오토마타가 어떤 입력에서 정답과 다르게 수락/거부하는지의 *반례* 를 자동 생성해 보여 줌.
- 약점: 한 입력의 시뮬레이션을 따라가는 시각화는 약함.

### jFAST
- 시각 객체: JFLAP 보다 단순한 그래프 에디터. 노드/엣지 + 한 줄 입력창.
- 시간 표현: 단계 진행 버튼으로 한 심볼씩 소비.
- 메타포: 미니멀 칠판.
- 강조: 입문 학습자 대상으로 화면 군더더기를 줄임.
- 약점: 다중 상태 NFA 의 동시성 표현은 약함.

### AutomataVerse
- 시각 객체: 모던 웹 그래프 (drag-and-drop) + 토글로 DFA / NFA / PDA / 튜링.
- 시간 표현: 입력 문자열을 넣으면 한 글자씩 자동 재생, 현재 노드가 펄스로 점등.
- 메타포: "scratch 같은 현대 웹 도구".
- 강조: 다중 오토마타 모델을 같은 인터페이스로 비교.
- 약점: 한 입력의 *왜 이 경로가 선택되는가* 의미적 설명은 약함.

### ivanzuzak FSM Simulator (noam)
- 시각 객체: Graphviz 기반 정적 도식 + 입력 박스 + "Run / Step" 버튼.
- 시간 표현: 한 심볼 단위 진행. 현재 상태가 색으로 강조.
- 메타포: 디버거의 step-into.
- 강조: ε-NFA 까지 명시적으로 다루며, regex → NFA → DFA 변환을 한 화면에서 단계별로.
- 인상적 디테일: 변환 결과 자동 레이아웃 + 동등성 테스트.
- 약점: 다이어그램이 정적 SVG 이고 인터랙션 폭이 좁음.

### Stanford CS103 Subset Construction Guide
- 시각 객체: 손으로 그린 듯한 다이어그램 + 표(부분집합 → 새 DFA 상태 매핑).
- 시간 표현: 종이 만화 — 한 페이지가 한 단계.
- 메타포: 칠판 강의록.
- 강조: NFA 의 *상태 집합* 이 어떻게 한 DFA 상태로 통합되는지의 표. 시각과 산술의 일치성.
- 약점: 정적. 입력 시뮬레이션 시각화는 부재.

### Regexper
- 시각 객체: 좌→우 흐름의 철도 다이어그램. 분기는 갈라지는 선로, 반복은 루프, 옵션은 우회로.
- 시간 표현: 정적. 사용자의 시선이 곧 시간.
- 메타포: "철도/지하철 노선". 한 문자열 = 한 열차가 좌→우로 달림.
- 강조: 정규식의 *구조* (분기/반복/그룹).
- 인상적 디테일: 시각적으로 가장 우아한 정규식 표현. 비전공자도 읽음.
- 약점: 입력 문자열을 따라 *어떻게 매칭되는가* 의 동적 시뮬레이션은 없음. 오토마타 자체는 보여 주지 않고 "그래프적 동치물" 만 제공.

### Debuggex
- 시각 객체: 노드 + 엣지로 구성된 *진짜 NFA 모양* 의 그래프. Regexper 의 깔끔함과 오토마타의 구조를 절충.
- 시간 표현: 입력 문자열을 넣으면 매칭된 부분이 그래프 위에 색으로 트레이스.
- 메타포: 회로 + 형광펜.
- 강조: 백트래킹 위치. 매칭이 실패한 분기도 시각적으로 표시.
- 인상적 디테일: 한 입력 = 그래프 위의 *경로*. 정규식이 곧 NFA 라는 사실을 시각적으로 등치.
- 약점: 한 글자씩 소비하는 step-by-step 진행은 약함. 결과만 보여 주는 편.

### regex101
- 시각 객체: 정규식 / 입력 / 매칭 결과 / 토큰 설명 4-패널. 그래프 다이어그램은 없음, 대신 *언어적* 설명.
- 시간 표현: 매칭 디버거 — Step / Skip Steps 으로 백트래킹 단계를 *글자 + 토큰* 단위로 진행.
- 메타포: 디버거 + 수업 노트.
- 강조: 한 글자가 어느 토큰과 매칭되었는지의 자세한 추적. 시간이 *역방향* (백트래킹) 으로 흐를 수 있음.
- 인상적 디테일: 매 스텝마다 매칭/실패/소비/이동을 텍스트와 색으로 동시에. 백트래킹의 *되감기* 연출.
- 약점: 그래프 시각화 없음 — "엔진 내부의 시간" 만 보여 줌. NFA 의 다중 분기는 텍스트 트리로만.

### RegExr
- 시각 객체: 정규식 / 입력 / 토큰 설명. regex101 과 유사하지만 더 캐주얼.
- 시간 표현: 매칭 결과의 정적 하이라이트.
- 메타포: 입문자용 워크벤치.
- 약점: 디버깅의 단계별 진행은 regex101 만큼 정밀하지 않음.

### Russ Cox — Thompson NFA
- 시각 객체: 작은 노드/엣지 다이어그램(논문체) + ASCII 표 + 코드.
- 시간 표현: 텍스트로 서술된 lockstep — "한 입력 글자마다 모든 활성 상태를 동시에 한 칸씩 진행".
- 메타포: "병렬 실행 부대(parallel troops)". 다중 상태가 한 박자에 한 걸음.
- 강조: NFA 가 backtracking 으로 구현될 수도 있고 lockstep simulation 으로 구현될 수도 있다는 *근본적 분기*. 후자가 선형 시간.
- 인상적 디테일: 병렬 부대 비유는 동시성 시각화의 좋은 출발점이지만 글로만 제시됨.
- 약점: 시각화는 본문이 아니라 독자의 상상 속.

### nagendrajamadagni/Lexer
- 시각 객체: lex 명세를 입력하면 NFA / DFA / Minimal DFA 그래프를 별도 창에 띄우는 데스크톱 도구.
- 시간 표현: 정적 그래프 + 토큰화 결과.
- 메타포: gcc 의 보이지 않는 내부를 *해부*해서 보여 줌.
- 강조: "정규식 → NFA → DFA → 최소화" 의 컴파일 파이프라인이 *4 개의 그래프* 로 동시에.
- 약점: 입력 한 줄을 따라가는 토큰화 시뮬레이션은 부수적.

### McGill CS520 / 강승식 강의노트 (교과서적 자료)
- 시각 객체: 정적 다이어그램 — 노드/엣지/입력 테이프. 표(전이 테이블) 와 그림이 짝.
- 시간 표현: 페이지 단위 점프.
- 메타포: 칠판/책.
- 강조: DFA 의 전이 함수를 *표* 로 보여 주는 전통 — 시각(그림) 과 산술(표) 의 이중성. 한국어 자료에서 특히 "δ(q, a) = q'" 표가 강조됨.
- 약점: 인터랙션 없음. 한 입력의 시간 흐름은 학생이 머릿속에서 따라가야 함.

### joeylemon/nfa-to-dfa, aydinschwa/Regex-Engine, 27t Regex Visualizer
- 시각 객체: 웹 기반 모던 그래프 + 변환/시뮬레이션 단계별 애니메이션.
- 시간 표현: 부분집합 구성을 한 단계씩 진행, 새 DFA 상태가 차례로 추가됨.
- 메타포: 인디 해커톤 톤의 "직접 만든 가시화 장난감".
- 강조: 변환 자체의 *과정* 을 시각화 (정적 그래프가 아니라 만들어지는 것).
- 약점: 디자인 일관성/완성도는 학술 도구보다 떨어지지만 인터랙션 아이디어가 신선.

### swaminathanj — FSM Application: Lexical Analysis
- 시각 객체: 식별자 / 정수 / 키워드 등을 인식하는 작은 DFA 들이 *한 페이지 안에 늘어선* 그림 + 입력 스트림 + 토큰 출력 패널.
- 시간 표현: 입력 스트림이 좌→우로 흐르며 어떤 DFA 가 "이긴 max-munch" 가 되는지 표시.
- 메타포: "여러 검문소가 줄지어 서서 가장 길게 잡는 검문소가 토큰을 가져간다".
- 강조: 렉서가 *여러 작은 오토마타의 합집합* 이라는 사실. 토큰 인식과 입력 소비가 한 화면에.
- 인상적 디테일: 한 입력 스트림에서 *여러 토큰* 이 차례로 잘려 나가는 lexer 다운 시각.
- 약점: 정적 페이지, 인터랙션 부재.

## 3. 공통 요소

레퍼런스를 가로질러 학습자가 "오토마타 시각화" 라는 말에서 자동으로 떠올리는 시각 언어.

공통 시각 객체
- **원형 노드 = 상태**: 라벨은 q0, q1, A, B 등 짧은 식별자.
- **이중원 = 수락 상태**: 거의 모든 도구가 이 약속을 지킴 (예외: 정규식 다이어그램).
- **시작 상태 표시**: 화살표가 어디서도 오지 않고 들어옴, 또는 "start" 라벨.
- **라벨된 화살표 = 전이**: 입력 심볼이 화살표 위에 적힘. 다중 심볼은 콤마/슬래시로 묶음.
- **자기 루프**: 같은 상태로 돌아오는 화살표 — DFA 에서 가장 흔한 패턴 ("문자 더 읽어도 같은 상태").
- **ε 라벨**: ε 또는 λ 로 표기된 빈 전이 — NFA 의 시그니처.
- **입력 테이프 / 입력 문자열 + 커서**: 하단 또는 상단에 문자열, 현재 소비 위치를 ▲ 또는 색 강조로 표시.
- **수락/거부 결과 표시**: 끝에 도달했을 때 녹색 체크 / 빨간 X 또는 "ACCEPT / REJECT" 텍스트.

공통 시간 표현
- **이산 스텝**: 한 입력 심볼 = 한 박자. 연속 애니메이션이 아니라 글자 단위 점프.
- **현재 상태 펄스**: 활성 상태가 색/굵기로 강조. 다음 심볼을 소비하면 다른 상태로 *점프*.
- **입력 커서의 단조 진행**: 좌→우 한 칸씩. 거의 모두 forward only (regex101 의 백트래킹 디버거가 예외).
- **Step / Run 토글**: 사용자가 한 박자씩 직접 누르거나 자동 재생.

공통 메타포
- **그래프 = 미로/도시 지도**: 노드는 장소, 화살표는 길, 입력 = 따라갈 방향 지시서.
- **상태 = 기억**: "지금까지 무엇을 봤는가" 의 응축.
- **수락 상태 = 목적지**: 이중원 = 보물 위치.
- **NFA = 동시에 여러 길**: 한 위치에서 갈 수 있는 길이 여러 개라 분기.

공통 강조 방식
- **색**: 활성 상태 한 색, 비활성 회색, 수락 가능 분기 녹색, 막다른 분기 빨강.
- **번호된 노드**: q0, q1, ... 으로 *식별*에 집중. 의미적 라벨 ("정수 첫 글자 본 뒤") 은 드묾.
- **표 + 그래프 이중**: 전이 함수 δ 를 표로도 동시에 보여 주는 전통적 패턴 (특히 한국어 강의자료).

공통 정보 밀도
- 상태 5~10 개, 알파벳 2~3 글자(보통 a, b 또는 0, 1) 의 작은 예제.
- 실제 렉서가 다루는 큰 DFA (수십~수백 상태) 는 거의 보여 주지 않음.
- 입력 문자열 길이는 5~15 글자.

공통 레이아웃
- 그래프가 화면의 80%, 입력/결과 패널이 하단/우측에 붙음.
- 좌→우 흐름이 기본. 입력 테이프도 좌→우.

학습자에게 형성된 기대 패턴
- "원이 여러 개 있고 화살표가 그 사이를 잇는다."
- "이중원에서 끝나면 받아들여진다."
- "한 글자에 한 칸 — 도장이 아니라 *말 이동*."
- "DFA 는 갈 길이 한 개, NFA 는 여러 개."
- "ε 화살표는 글자 안 먹고 그냥 가는 비밀 통로."

## 4. 공백

레퍼런스 평균이 다루지 못하거나 거의 시도하지 않는 측면.

차용 가능한 다른 분야 메타포
- **지하철 노선도 + 달리는 열차**: 상태 = 역, 전이 = 노선, 입력 = 차표(목적지 지시서). 노선도가 곧 정적인 그래프이고, 한 입력 문자열 = 한 열차의 주행. NFA 는 한 출발지에서 여러 열차가 동시에 다른 노선으로 출발 — *시각적으로 자연스러운 분기*.
- **보드게임 말 이동**: 노드 = 칸, 입력 글자 = 주사위. DFA 는 결정적이라 주사위가 한 면이고, NFA 는 *복제 말* 이 갈래마다 한 개씩 놓임. 수락 = 골인 칸.
- **공항 게이트 안내**: 입력 = 탑승권 시퀀스, 상태 = 현재 위치한 게이트, 전이 = "다음 게이트로". 막다른 길은 거부.
- **강 흐름 / 갈래**: 입력 글자 한 개 = 물 한 양동이. 강이 갈라지는 곳에서 NFA 는 *모든 갈래* 로 동시에 흐름. DFA 는 갈래마다 한 길만 정해짐.
- **음악 악보 진행 / 단계 시퀀서**: 입력 글자 = 박자, 상태 = 현재 마디. 수락 상태에서 끝나면 곡 완성.
- **전화 자동 응답 트리(IVR)**: 상태 = 메뉴, 입력 = 누른 숫자, 잘못된 시퀀스 = "올바르지 않은 입력". 친근한 메타포.
- **퀘스트 스토리 / 상태 기반 게임 진행**: 상태 = 진행도, 입력 = 플레이어 행동. 거부 = bad ending, 수락 = good ending.

시간 표현의 다른 가능성
- **잔상(trail)**: 지나온 경로를 *흔적* 으로 누적. 학습자가 "이 입력은 q0→q1→q1→q2 를 따라왔다" 를 한 컷에 봄. 거의 모든 도구가 *현재* 만 강조하고 *경로* 는 안 보여 줌.
- **입력 테이프와 그래프의 결합 시간**: 한 글자가 *소비되며 그래프의 한 화살표를 따라간다* — 글자가 화살표를 따라 그래프 안으로 빨려들어가는 애니메이션. 보통 입력 테이프와 그래프가 분리된 두 패널로 그려져 시간이 분리됨.
- **NFA 분기의 폴리포니**: 다중 상태가 동시에 여러 색으로 점등하며 *각 분기가 끝까지 살아남는지* 한 컷에 표시. JFLAP 이 부분 시도, 대부분 도구는 한 분기씩 차례대로.
- **백트래킹의 되감기**: regex101 정도만 시도. NFA backtrack 구현이 *시간을 거꾸로 감는다* 는 사실을 시각적으로 보여 주는 사례 거의 없음.
- **Thompson NFA lockstep 의 행진**: 모든 활성 상태가 한 박자에 한 걸음. "한 시각 = 상태 집합" 을 한 화면에 행진식으로 보여 주는 시도가 본문 시각화에는 거의 없음.
- **타임라인 스크럽**: 입력 문자열의 임의 위치에 즉시 점프. 거의 모든 도구가 처음부터 끝까지 forward only.

차원 변환의 가능성
- **NFA 의 "동시 여러 상태" 를 명시적으로**: 한 NFA 노드에 *여러 토큰* (각 토큰 = 살아 있는 분기) 을 얹어 쌓아 가는 시각. NFA → DFA 부분집합 구성이 *그 토큰 더미가 곧 새 DFA 상태* 라는 시각적 등치로 한 화면에. 거의 시도 없음.
- **그래프 + 표 + 코드 동기**: 같은 입력이 그래프(노드 점등)에서, 전이표(δ 셀 강조)에서, 코드(switch case) 에서 동시에 진행. 한국어 강의자료가 표를 보여 주지만 동시 동기화는 없음.
- **NFA 와 DFA 를 같은 화면에서**: 같은 입력이 NFA 위에서는 *분기 트리* 로, DFA 위에서는 *한 점의 단조 이동* 으로 동시에 진행되는 비교. nagendrajamadagni 가 정적으로 시도, 시뮬레이션 동시 진행은 없음.
- **렉서 맥락의 압축**: "이 정규식은 이 NFA 이고, 이 입력은 이 토큰이며, 이 토큰은 코드의 이 부분에서 사용된다" 의 4-층 동기. swaminathanj 가 정적으로 시도. 인터랙티브 동시 시각화 사례는 사실상 없음.

DFA vs NFA 의 시각적 구별
- 거의 모든 도구가 *같은 그래프 형식* 으로 둘 다 그림. 차이는 "한 노드에서 같은 글자에 화살표가 여러 개 있느냐" 정도. 학습자가 "결정성이란 결국 *시간 흐름의 일관성*" 이라는 본질을 짚기 어렵다.
- DFA 의 시간이 단조 이동인 반면 NFA 의 시간이 *분기되는 다발* 이라는 사실을 *시간 자체로* 표현한 시도는 거의 없음.

컴파일러 렉서로서의 맥락
- 대부분 도구가 학술적 토이 알파벳(a/b, 0/1) 만 다룸. *프로그래밍 언어의 토큰* (식별자, 정수, 키워드, 문자열 리터럴) 을 인식하는 *현실적인* DFA 를 보여 주는 도구는 swaminathanj 와 일부 강의자료뿐.
- 렉서가 *여러 토큰 종류 DFA 의 합* 이라는 사실, max-munch 규칙, 토큰 분리 후 다음 토큰을 다시 처음 상태에서 시작한다는 사실을 시각화한 도구는 매우 드묾.
- 토큰 시퀀스 = 입력 스트림이 *여러 조각* 으로 잘리는 현상을 그래프 위에서 동시에 보여 주는 사례 거의 없음.

인터랙션의 미개척 영역
- **사용자가 입력을 라이브로 타이핑**: 글자를 칠 때마다 즉시 그래프가 한 칸씩 이동. regex101 이 매칭에서 시도, 그래프 시각화 도구에서는 드묾.
- **상태 클릭 → 의미 표시**: "이 상태는 *마지막 글자가 a 였음을 기억* 한다" 같은 의미 라벨. q0/q1 라벨만 있고 의미는 학생이 추론.
- **거부 사례 자동 생성**: Automata Tutor 가 일부. 학습자가 그린 오토마타가 어떤 입력에서 깨지는지 즉시 보여 주는 인터랙션은 가치 큼.
- **속도/일시정지/되감기**: 미디어 플레이어 같은 시간 컨트롤. 대부분 Step / Run 만.
- **동시 입력 비교**: 두 개 이상의 입력을 같은 그래프 위에 다른 색으로 동시에 흘려 보내 어디서 갈라지는지 비교. 거의 시도 없음.
- **NFA 분기 가지치기**: 살아있는 분기/죽은 분기를 학습자가 클릭으로 죽이고 살리며 *결정성을 손으로 빼앗아 보는* 인터랙션.

본질적이지만 평균이 잘 다루지 못하는 개념들
- **상태가 "기억" 이라는 사실** — 노드 라벨이 q0/q1 이라 기억의 *내용* 을 시각화하지 못함.
- **트랩(dead) 상태** — 거부로 가는 흡수 상태. 그래프에 그려지긴 하지만 *이 상태에 빠지면 더 이상 희망 없음* 의 시각적 위계가 없음.
- **알파벳 보집합 처리** — DFA 가 모든 입력 심볼에 대해 정의되어야 한다는 사실, 그리고 빠진 전이는 암묵적 trap 이라는 사실.
- **수락의 양면성** — 수락 상태에서 *끝났을 때만* 수락. 수락 상태를 지나가도 거기서 끝나지 않으면 거부. 시간 종료라는 *외부 신호* 의 중요성을 강조한 사례 적음.
- **부분집합 구성에서 ε-closure** — 보이지 않는 ε 통로로 미리 따라가는 *전 처리*. 정적 표로만 보여지고 동적으로 "ε 거품이 부푼다" 시각화는 드묾.

## 5. 후속 작업 가이드

### 공통 강화 에이전트에게

가장 가치 있는 공통 요소(반드시 보존)
- **원형 노드 + 라벨된 화살표 + 이중원**: 학습자의 자동 연상. 이걸 빼면 "이건 오토마타 시각화가 아니다" 라는 위화감.
- **입력 테이프 + 커서**: 시간을 *외부* 에 두고 그래프와 동기화. 입력 = 시간이라는 약속.
- **현재 상태 펄스**: 한 박자에 어느 노드가 살아 있는지. NFA 라면 여러 노드가 동시에.
- **이산 스텝(Step) 컨트롤**: 자동 재생만 있으면 학습자가 따라잡지 못한다. 한 박자씩 멈출 수 있어야.
- **수락/거부 표시**: 입력 끝에서의 결과. 이중원 강조 + 텍스트 결과 둘 다.

다듬을 여지
- **의미적 상태 라벨**: q0 대신 "아직 아무것도 못 봤음", "정수 첫 자리 봤음", "소수점 봤음". 클릭 시 토글로 q0 ↔ 의미 라벨. 학습자가 *왜 상태가 분리되어야 하는가* 를 즉시 이해.
- **표/그래프 동기**: 한국어 교과서 전통의 δ 표를 그래프 옆에 두고 *현재 사용 중인 셀* 을 같이 강조. 시각과 산술의 이중 트레이스.
- **경로 잔상**: 지나온 경로를 흐린 색으로 누적해 한 컷에 전체 궤적. 입력이 끝났을 때 학습자가 "어디를 지나왔는지" 한 번에 본다.
- **NFA 의 동시성을 일급으로**: 다중 상태가 동시에 점등할 뿐 아니라 *각각이 분기 트리의 한 가지로 살아 있음* 을 명시. JFLAP 의 분기 트리 보기가 좋은 출발.
- **Step + Run + Scrub**: 미디어 플레이어 같은 시간 컨트롤. 임의 위치 점프, 속도 조절, 되감기.

### 새로운 방식 에이전트에게

가장 흥미로운 공백(시도 우선순위)
1. **NFA 동시 분기를 "복제 말 / 동시 출발 열차" 로** — 평균이 잘 못 다루는 비결정성을 시간으로 직접 표현. 한 입력 글자에 노드 위의 토큰이 *복제되어 모든 가능한 다음 상태로 동시에 이동*. 부분집합 구성이 *살아 있는 토큰 집합 = 새 DFA 상태* 로 자연 등치되는 보너스.
2. **렉서 4-층 동시 시각화** — 정규식 / NFA / DFA / 토큰화된 출력이 한 화면에서 동시에 한 입력을 처리. 입력 한 글자가 정규식의 한 토큰을 점등시키고, NFA 의 한 분기를 따라가고, DFA 의 한 노드로 점프하고, 출력 토큰의 한 글자를 채운다. 평균이 *각각 분리된 도구* 로만 다루는 4 층의 동기화가 본질적으로 비어 있음.
3. **상태 = 기억의 직접 시각화** — 각 상태에 들어 있는 *기억의 내용* (이미 본 prefix 의 어떤 측면) 을 라벨 또는 작은 아이콘으로. 클릭 시 의미 텍스트, 호버 시 "이 상태로 오는 모든 입력 prefix" 의 예시 모음.
4. **DFA vs NFA 시간 비교** — 같은 정규식의 NFA / DFA 가 위아래로 놓이고, 같은 입력이 *동시에* 진행. NFA 는 분기 다발이 부풀고 줄어드는 호흡, DFA 는 한 점이 단조 이동. 결정성의 본질이 *시간의 모양 차이* 로 즉각 보임.
5. **에러/trap 의 위계** — 거부 상태와 trap 상태에 시각적 무게 차이를 두고, *희망이 사라지는 순간* 을 입력 진행 도중에 명시. 평균이 거부를 *끝에서야* 보여 주는데, 사실 입력 중간에 이미 결정될 수 있다는 사실.

시도해 볼 만한 메타포
- **지하철 노선도 + 열차**: 가장 본질 보존도가 높음. 노드 = 역, 화살표 = 노선, 입력 = 차표 시퀀스, 수락 = 종착역. NFA 는 한 출발에서 여러 열차가 동시 출발.
- **보드게임 말 이동**: 결정성/비결정성을 *말의 개수* 로 표현. DFA 는 말 한 개가 정확히 한 칸씩, NFA 는 분기마다 말이 복제. 부분집합 구성 = "복제 말 묶음 = 한 새 칸" 의 자연스러운 합치.
- **강 갈래**: 입력이 물처럼 흘러가며 NFA 갈래에서 자연스럽게 분기.
- **퀘스트 진행**: 상태 = 스토리 진행도, 입력 = 플레이어 선택, 수락 = 굿엔딩, 거부 = 배드엔딩. 학습자가 *상태가 기억* 이라는 본질을 자연스럽게 받아들이는 구조.
- **자판기 / 전화 ARS**: 입력 시퀀스가 정확해야만 결과가 나오는 일상 사례. 거부의 일상성.

절대 빼면 안 될 본질적 개념
- **상태(state)**: 현재까지의 입력을 응축한 기억의 단위. 유한 개.
- **전이(transition)**: 입력 글자를 소비하며 한 상태에서 다른 상태로 이동.
- **입력 소비(consume)**: 글자가 한 번 소비되면 되돌릴 수 없음(엔진의 backtracking 은 외부 트릭).
- **시작 상태**: 시간 0 의 위치.
- **수락 상태**: 입력 종료 시 여기 있어야 받아들여짐. 지나치는 것만으로는 부족.
- **거부**: 수락 상태가 아닌 채로 끝나거나, 정의되지 않은 전이를 만남.
- **결정성 vs 비결정성**: 한 상태 + 한 글자에 갈 곳이 정확히 한 곳(DFA) vs 0~여러 곳(NFA + ε-전이).
- **(부수적) 부분집합 구성**: NFA 의 *살아 있는 상태 집합* 이 곧 DFA 의 *한 상태*.
- **(부수적) 컴파일러 렉서 맥락**: 정규식이 토큰 종류를 정의 → 각각이 NFA → 합쳐서 DFA → 입력 스트림에서 토큰 잘라냄.

이 일곱 가지(혹은 부수 개념까지 아홉)를 모두 담는 새 메타포가 있다면 평균을 넘는 후보다. 메타포 후보 중 *지하철 노선도 + 달리는 열차* 가 본질 보존도가 가장 높다 — 노드/엣지/시간/분기/종착이 모두 자연 대응되며, *NFA 의 다중 동시 출발* 도 시각적으로 이상하지 않다. *보드게임 말 이동* 은 부분집합 구성을 *말 묶음 = 새 칸* 으로 직관 변환한다는 보너스를 제공한다.

## 6. 대상 개념 요약

(후속 데모가 시각화 옆 또는 안에 표시할 수 있도록 짧은 설명을 마련했다.)

### 한 단락 요약 (4 문장)

유한오토마타는 입력 문자열을 *한 글자씩* 읽으면서 *유한 개의 상태* 사이를 옮겨 다니다가, 입력이 끝났을 때 어디에 있는지로 *받아들일지 거절할지* 를 결정하는 단순한 인식기다. 갈 길이 항상 한 곳뿐이면 결정적 유한오토마타(DFA), 같은 글자에 갈 곳이 여러 곳일 수 있으면 비결정적 유한오토마타(NFA) 다. 두 모델은 같은 부류의 언어(정규언어)를 인식하지만, NFA 는 *동시에 여러 상태에 살아 있을 수 있어* 구조를 간결하게 적기 좋고, DFA 는 항상 한 상태만 활성이라 컴퓨터로 빠르게 돌리기 좋다. 컴파일러의 렉서/스캐너는 이 메커니즘으로 소스 코드 글자 스트림에서 식별자/숫자/키워드/연산자 같은 토큰을 잘라낸다.

### 한 줄 캐치프레이즈

"한 글자씩 먹으며 상태 사이를 옮겨 다니다, 다 먹었을 때 받아들일지 거절할지를 결정하는 작은 기계."

### 시각화 옆에 띄울 짧은 안내문 (3 줄)

- 노드는 *상태*. 지금까지 본 글자들의 요약된 기억.
- 화살표는 *전이*. 글자 하나를 먹으며 한 상태에서 다른 상태로.
- 이중원에 *멈춰 끝나면* 받아들임, 그 외에는 거절.

### 전문 용어 풀어쓰기 (시각화 안에 함께 표시할 후보)

- **DFA(결정적 유한오토마타)** = "갈 길이 항상 한 곳" 기계.
- **NFA(비결정적 유한오토마타)** = "갈 길이 여러 곳일 수 있는" 기계.
- **상태(state)** = 지금까지 본 글자를 응축한 *기억 한 칸*.
- **전이(transition)** = "이 글자를 보면 저쪽으로 간다" 라는 약속된 길.
- **입력 소비(consume)** = 글자 하나를 *읽고 지운다*. 되돌릴 수 없다.
- **수락(accept)** = 입력이 끝났을 때 *이중원 위에 있음*.
- **거부(reject)** = 수락 상태가 아닌 채 끝남, 또는 갈 길이 없는 곳에서 막힘.
- **ε-전이(엡실론 전이)** = 글자를 *안 먹고* 그냥 옮겨 가는 비밀 통로(NFA 만).
- **부분집합 구성(subset construction)** = NFA 의 "동시에 살아 있는 상태 집합" 을 묶어 새 DFA 의 한 상태로 본다는 변환 트릭.
- **렉서/스캐너(lexer/scanner)** = 컴파일러의 첫 단계. 소스 코드 글자 스트림을 *토큰* 으로 잘라낸다. 내부에 큰 DFA 가 있다.
- **토큰(token)** = 식별자/숫자/키워드처럼 *언어 차원의 한 단어*.
- **max-munch 규칙** = 가능한 한 *가장 긴* 토큰을 잡는다는 렉서의 규칙.

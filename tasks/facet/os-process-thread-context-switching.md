# 컨텍스트 스위칭 시각화 기획서

### 0. 기본 정보

- 대상 이름: 컨텍스트 스위칭 (context switching)
- 도메인: 운영체제 / 프로세스와 스레드
- type: systemBehavior
- 진행 모델: 시간 진행형
- 본질 한 문장: 컨텍스트 스위칭은 단 하나뿐인 CPU 라는 무대 위에서 한 실행 흐름의 상태 일습을 자기 보관소로 떠내고 다른 흐름의 상태 일습을 그 자리에 되돌려 넣어, 두 흐름이 각자 멈춘 지점부터 정확히 이어 실행되도록 시간을 다시 잇는 사건이다.

### 1. 시그니처 행동

진행 중인 흐름이 외부 트리거에 끊기면, 그 흐름의 레지스터 묶음이 자기 보관소로 떠내져 잠기고, 다른 흐름의 보관소에서 같은 자리로 다른 묶음이 되돌려져, 같은 무대 위에서 주인이 바뀐다.

### 2. 시각적 은유

표준 메타포인 "단일 CPU 무대 위에서의 주자 교대" 를 정교화해 채택한다. 화면 한가운데에는 직사각형의 CPU 무대가 놓이고, 무대 안에는 레지스터 슬롯 여덟 칸이 격자로 자리한다. 무대 좌우에는 두 실행 흐름 A·B 의 보관소(PCB/TCB 박스) 가 마주 보고 서 있고, 각 보관소 안에도 같은 형태의 레지스터 슬롯 격자가 비어 있거나 채워진 채 보관된다. 무대 아래에는 가로 시간 띠가 한 줄 깔려, 어느 시점에 누가 무대를 점유했는지가 색으로 채워진다. 무대 위쪽에는 트리거 표식이 내려오는 천장 영역이 있어, 외부 사건이 흐름 안이 아니라 바깥에서 들어옴이 위치만으로 강제된다. 흐름 A 와 B 는 시종일관 두 가지 색으로 구별되고, 빈 시간 구간은 회색으로만 채워진다.

### 3. 결정적 순간

흐름 A 가 점유 중인 무대 천장 위에서 트리거 표식이 내려와 A 가 멈춘 직후, 무대의 레지스터 슬롯 여덟 칸이 한 묶음으로 들어 올려져 좌측 보관소로 떠내지고, 무대가 잠시 회색으로 비어 있고 시간 띠 또한 회색으로 진행하다가, 우측 보관소에서 B 의 묶음이 들려 올라와 무대 슬롯에 차례로 정렬되어 안착하면 시간 띠 색이 B 색으로 바뀌며 다시 진행한다. 이 세 박자(저장 → 무대 비움 → 복원) 가 한 호흡으로 이어지는 한 컷이 결정적이다.

- 전: 흐름 A 의 색이 무대를 채우고 시간 띠도 같은 색으로 진행 중. 좌측 보관소는 비어 있고 우측 보관소에는 B 의 묶음이 잠긴 채 보관되어 있다. 천장은 비어 있다.
- 중: 천장에서 트리거 표식이 무대로 내려오는 순간 A 의 색이 한 단계 흐려지며 무대의 슬롯 묶음이 한 덩어리로 떠올라 좌측 보관소로 미끄러져 들어간다. 이어 무대는 텅 비고 시간 띠에 회색 구간이 자라기 시작한다. 곧이어 우측 보관소의 묶음이 들려 올라와 무대 슬롯 칸 하나하나에 정렬되며 자리잡는다.
- 후: 무대가 B 의 색으로 채워지고 시간 띠도 B 색으로 진행. 좌측 보관소에는 A 의 묶음이 잠긴 채 보관되어 있고 우측 보관소는 비어 있다. 시간 띠에는 회색 구간이 한 칸 자국으로 남는다.
- 강조 장치: 천장에서 내려오는 외래 표식·묶음을 들어 올리는 동작과 정렬해 내려놓는 동작의 무게 차이·무대가 비어 있을 때 시간 띠의 회색이 자라는 폭·두 보관소의 좌우 대칭 정렬·시간 띠 위 색 띠와 회색 자국의 누적.

### 4. 다른 대상과의 구별점

- 스케줄링 알고리즘 자체와의 구별: 스케줄러는 "다음에 누구를 무대에 올릴지" 를 정하는 정책이고, 컨텍스트 스위칭은 그 결정이 내려진 직후의 한 사건이다. 시각화에서 스케줄러 자체는 보이지 않고, 트리거가 도착해 무대 위 묶음이 이동하는 운동만이 사건으로 그려진다. 정책의 비교(간트 차트의 누적)가 아니라 한 번의 교대(저장-비움-복원)가 시각의 단위다.
- 스레드 스위치 대 프로세스 스위치: 같은 프로세스 안의 두 스레드 사이를 옮길 때는 보관소가 얇은 박스로 그려지고 들어 올려지는 묶음의 두께도 얇다. 서로 다른 프로세스 사이를 옮길 때는 보관소가 두꺼운 박스로 부풀고 묶음에 추가 층(주소 공간 표지)이 함께 얹혀 들려 옮겨지며, 시간 띠의 회색 구간도 더 길다. 토글 한 번으로 두 모드의 부피·운동 길이 차이가 같은 무대 위에서 직접 비교된다.
- 동기화·IPC 와의 구별: 뮤텍스/세마포어/메시지 패싱은 두 흐름 사이에 메시지나 신호를 주고받는 사건이다. 시각적으로 두 흐름의 띠 사이를 잇는 화살표가 그려진다. 컨텍스트 스위칭은 두 흐름 사이에 아무것도 오가지 않으며, 같은 무대(CPU 자원) 의 점유자만이 바뀐다. 화살표는 흐름끼리가 아니라 무대와 보관소 사이를 오갈 뿐이다.

### 5. 달성해야 할 시각 요소

1. **단일 무대의 단일 점유 — 한 시점의 한 색**
   - 설명: 어느 시점에든 무대를 채우는 색은 하나뿐이다. 두 흐름이 동시에 무대 위에 있는 시각이 단 한 순간도 발생해서는 안 된다. 무대가 비어 있는 짧은 회색 구간만이 두 색 사이에 허용된다.
   - 시각 표현: CPU 무대 사각형 안의 슬롯 격자가 통째로 한 흐름의 색으로 칠해진다. 트리거 직후 색이 한 단계 흐려지며 묶음이 떠오르고, 무대가 비는 동안에는 슬롯 격자가 모두 회색의 빈 칸으로 그려진다. 시간 띠도 같은 규칙으로 한 시점 한 색만을 채운다.

2. **외부 보관소의 자기 자리 — 좌우 두 박스**
   - 설명: 각 흐름은 자기만의 보관소를 흐름 본체 바깥에 가지고, 그 안에는 무대와 같은 형태의 슬롯 격자가 들어 있어 묶음이 들고 나는 단위가 된다. 보관소가 흐름 안에 있으면 "꺼냈다 넣는다" 의 운동이 사라진다.
   - 시각 표현: 무대 좌우에 흐름 색의 외곽선이 가는 두 직사각형 박스. 박스 머리에 흐름 라벨(A / B), 박스 안에 무대와 동일한 슬롯 격자. 점유 중인 흐름의 보관소 슬롯은 비어 있고(묶음이 무대에 있으므로), 대기 중인 흐름의 보관소 슬롯은 그 흐름 색으로 잠겨 채워져 있다.

3. **저장과 복원의 비대칭 운동 — 다른 무게의 두 동작**
   - 설명: 저장은 무대의 묶음을 그대로 떠내 보관소에 떨어뜨리는 일방적 캡처, 복원은 보관소의 묶음을 무대 슬롯의 정해진 자리에 정확히 정렬해 내려놓는 운동. 두 운동의 시각 어휘가 거울이 아니어야 한다.
   - 시각 표현: 저장 — 무대의 슬롯 묶음 전체가 한 덩어리로 위로 떠올랐다가 보관소 위로 짧게 미끄러져 떨어진다. 운동선은 굵고 빠르며, 도착 시 보관소 슬롯이 한 번에 색으로 채워진다. 복원 — 보관소의 묶음이 위로 들어 올려진 뒤 무대 위에서 잠시 정지하고, 슬롯 칸 하나하나에 차례로 정렬되며 안착한다. 운동선은 가늘고 단계적이며, 마지막 슬롯이 자리잡는 순간 무대 색이 켜진다.

4. **사이의 빈 시간 — 회색이 자라는 폭**
   - 설명: 저장이 끝난 시점부터 복원이 시작되기까지 두 흐름이 모두 정지해 있는 짧은 회색 구간이 시간 띠 위에 자리를 차지한다. 그 폭이 곧 오버헤드의 양감이다.
   - 시각 표현: 시간 띠는 무대 아래 가로로 깔린 한 줄 띠. 무대 점유 시점은 흐름 색으로 채워지고, 저장 시작부터 복원 끝까지의 구간은 회색 사선 패턴으로 채워진다. 회색 구간의 가로 폭은 운동의 실시간 길이와 정확히 일치해, 빠르게 진행되면 좁고 느리게 진행되면 넓다. 누적된 회색 자국이 시간 띠 위에 그대로 남아 학습자가 한 화면에서 누적 오버헤드를 본다.

5. **세 박자로 펼쳐진 한 사건 — 점이 아니라 운동**
   - 설명: 저장 → 무대 비움 → 복원이 한 사건 안에서 세 단계로 식별되어, 스위치가 한 점이 아니라 내부 구조를 가진 운동임이 보여야 한다.
   - 시각 표현: 한 번의 스위치는 세 구간으로 시간 띠를 차지한다. 1구간 — 저장 운동(이전 흐름 색이 흐려진 상태로 묶음이 보관소로 이동). 2구간 — 빈 무대(회색 사선). 3구간 — 복원 운동(다음 흐름 색이 슬롯에 단계적으로 채워짐). 세 구간 사이에는 각각 가는 세로 구획선이 그어져 박자감이 살아난다.

6. **외부 트리거의 외래성 — 천장에서 내려온다**
   - 설명: 트리거는 흐름 자신이 결정하지 않는다. 흐름 바깥에서 들어오는 표식이 시각적으로 외래여야 한다.
   - 시각 표현: 무대 위쪽 약 60px 의 천장 영역에서 트리거 표식이 내려온다. 표식은 종류에 따라 모양·색이 구별된다. 타이머 — 노란 별표, 시스템 호출 — 파란 점선 화살표, I/O 완료 — 초록 동그라미, 인터럽트 — 빨간 번개. 표식은 무대 윗변에 닿는 순간 짧게 펄스하고 사라지며, 그 직후 저장 운동이 개시된다.

7. **컨텍스트 양의 차이 — 보관소 부피의 토글**
   - 설명: 같은 프로세스 안의 두 스레드 스위치는 묶음이 얇고, 서로 다른 프로세스 사이의 스위치는 묶음이 두껍다. 박스 부피와 묶음 두께로 차이가 직접 보여야 한다.
   - 시각 표현: 토글이 "스레드" 일 때 보관소 박스 폭이 좁고 슬롯 격자가 한 층(레지스터). 토글이 "프로세스" 일 때 박스 폭이 1.5 배로 부풀고 슬롯 격자 위에 추가 층(주소 공간 표지·페이지 테이블 표지) 이 얹힌다. 묶음 운동도 그만큼 더 두껍게 떠오르며, 시간 띠의 회색 구간이 자동으로 더 길어진다.

### 6. 시각 구성

- **중심 시각**: 화면 한가운데의 CPU 무대 사각형 + 무대 아래 가로 시간 띠. 무대 안의 슬롯 격자가 흐름의 색으로 채워지거나 비워지며, 시간 띠가 색·회색의 누적으로 사건의 역사를 보존한다.
- **보조 요소**:
  - 좌측 보관소(흐름 A): 흐름 A 색의 외곽 박스 + 슬롯 격자 + 라벨.
  - 우측 보관소(흐름 B): 흐름 B 색의 외곽 박스 + 슬롯 격자 + 라벨.
  - 상단 천장 영역: 트리거 표식이 내려오는 자리. 표식 종류 범례 한 줄.
  - 상단 캡션: 짧은 개념 문장 + 현재 사건 메시지.
  - 하단 컨트롤: 재생/일시정지 / 속도 슬라이더 / 트리거 종류 선택 / 프로세스·스레드 토글 / 단일 스위치 단계 진행 버튼.
- **요소 간 관계**: 묶음 운동의 화살표는 항상 무대와 보관소 사이만을 오간다. 두 보관소 사이를 직접 잇는 화살표는 그려지지 않는다(그 운동은 무대를 거쳐서만 일어남). 트리거 표식은 천장에서 무대로만 내려오고, 무대에서 흐름으로 다시 나가지 않는다. 시간 띠는 무대의 현재 점유 상태를 매 프레임 받아 색을 한 칸씩 누적해 그린다.

- CPU 무대 폭 300px, 슬롯 격자 4×2 (8 칸, 칸당 약 60×40px). 좌우 보관소는 각 폭 150px, 같은 4×2 격자.
- 천장 영역 60px — 트리거 표식이 위에서 무대 윗변까지 약 40px 거리를 내려오는 운동 공간.
- 시간 띠 40px — 가로 폭 전체. 한 칸은 약 8px, 누적 80 칸까지 시각화. 80 칸을 넘으면 가장 오래된 칸부터 좌측으로 페이드 아웃.
- 확장 폭 880px 일 때는 무대 그대로, 좌우 보관소가 각 250px 로 부풀어 프로세스 모드의 부피 차이가 더 강하게 드러난다. 시간 띠도 가로로 길어져 누적 100 칸까지 보존.
- 상단 캡션 40px, 천장 60px, 본체 260px, 시간 띠 40px, 컨트롤 60px = 총 460px.

### 8. 진행과 인터랙션

시간 진행형. 시간이 자동으로 흐르며 사건이 펼쳐지고, 학습자는 가벼운 입력으로 흐름을 조절한다.

**기본 진행 단계 (한 번의 스위치 사건)**

1. **흐름 A 실행 중** — CPU 무대가 A 색으로 채워져 있다. 시간 띠가 매 프레임 A 색으로 한 칸씩 자란다. 좌측 보관소는 비어 있고 우측 보관소는 B 색으로 잠긴 채 보관 중. 천장은 비어 있다.
2. **트리거 도착** — 천장 위에서 선택된 종류의 트리거 표식이 내려와 무대 윗변에 닿는다. 표식이 짧게 펄스하고 사라지며, 무대의 A 색이 한 단계 흐려진다. 시간 띠의 진행이 멈춘다.
3. **저장** — 무대의 슬롯 묶음이 한 덩어리로 떠올라 좌측 보관소로 굵은 곡선을 그리며 미끄러진다. 도착 시 좌측 보관소 슬롯이 한 번에 A 색으로 채워진다. 무대 슬롯은 빈 회색이 된다. 시간 띠에 회색 사선 칸이 자라기 시작한다.
4. **무대 비움** — 무대가 텅 빈 상태로 짧게 정지. 회색 사선이 시간 띠 위에 누적된다. 학습자에게 "지금은 누구도 진전하지 않는다" 가 폭으로 보인다.
5. **복원** — 우측 보관소의 묶음이 들어 올려져 무대 위에서 정지한 뒤, 슬롯 칸 하나하나에 차례로 정렬되며 안착한다. 마지막 슬롯이 자리잡는 순간 무대가 B 색으로 켜진다. 우측 보관소 슬롯은 비워진다.
6. **흐름 B 실행 시작** — 시간 띠가 B 색으로 다시 자라기 시작한다. 회색 구간 자국은 그대로 누적되어 한 화면에 보존된다.

**제어 지점 (학습자 입력)**

- **재생 / 일시정지**: 시간 흐름을 멈추거나 다시 흘린다. 일시정지 중에도 무대·보관소·시간 띠의 현재 상태는 유지된다.
- **속도 슬라이더**: 0.25× ~ 4× 사이로 시간 흐름의 빠르기를 조절한다. 슬라이더가 빠를수록 운동·시간 띠 진행이 빨라지고 회색 구간 폭은 절대 시간 그대로 짧아 보인다.
- **트리거 종류 선택**: 다음 트리거를 타이머·시스템 호출·I/O 완료·인터럽트 중 무엇으로 할지 고른다. 천장에서 내려오는 표식의 모양·색이 그에 맞춰 바뀐다.
- **모드 토글: 스레드 / 프로세스**: 보관소 박스 부피와 묶음 두께가 즉시 달라진다. 프로세스 모드에서는 묶음 위에 주소 공간 표지가 얹혀 들려 옮겨지고, 회색 구간이 자동으로 약 1.6 배 길어진다.
- **단계 진행 버튼**: 일시정지 상태에서 한 번 누르면 세 박자(저장 → 무대 비움 → 복원) 중 다음 한 박자만 진행한다. 학습자는 한 사건의 내부 단계를 천천히 짚어 본다.
- 모든 운동은 1× 속도 기준 약 600~900ms. 한 사건 전체는 약 2초 안에 끝난다.

### 9. 초기 상태와 파라미터

- **초기 상태**: 흐름 A 가 무대를 점유한 상태. 좌측 보관소(A) 비어 있음, 우측 보관소(B) 가 B 색으로 잠긴 채 보관됨. 천장 비어 있음. 시간 띠는 빈 채 시작. 모드 토글은 "스레드" 기본. 트리거 종류는 "타이머" 기본. 속도는 1× 기본. 재생 상태로 시작해 약 1.5 초 후 첫 트리거가 자동 도착하도록 미리 예약된다.
- **권장 시연 시퀀스**: 자동 진행으로 A → B → A → B 순으로 네 번의 스위치를 보여 준다. 두 번째 스위치 직전에 학습자가 모드 토글을 "프로세스" 로 바꾸면 보관소 부피가 부풀고 회색 구간이 더 길어지는 것을 직접 본다. 세 번째 스위치 전에 트리거 종류를 "I/O 완료" 로 바꾸면 천장에서 내려오는 표식이 초록 동그라미로 바뀐다.
- **파라미터**:
  - 슬롯 격자: 무대 4×2 (8 칸), 보관소 동일.
  - 흐름 색: A — 청록, B — 주황. 회색 — 중간 명도 회색의 사선 패턴.
  - 트리거 표식: 타이머(노란 별표), 시스템 호출(파란 점선 화살표), I/O 완료(초록 동그라미), 인터럽트(빨간 번개).
  - 시간 띠 칸 수: 기준 폭 80 칸, 확장 폭 100 칸.
  - 운동 지속 시간(1× 기준): 저장 700ms, 무대 비움 400ms, 복원 800ms.
  - 모드별 회색 구간 배율: 스레드 1.0×, 프로세스 1.6×.
  - 속도 범위: 0.25× ~ 4×.

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> 컨텍스트 스위칭은 단 하나의 CPU 무대 위에서 한 실행 흐름의 상태 일습을 자기 보관소로 떠내고, 다른 흐름의 상태 일습을 그 자리에 되돌려 넣는 사건이다. 외부 트리거에서 시작해 저장·빈 무대·복원의 세 박자를 거치며, 그동안 어떤 흐름도 진전하지 않는 짧은 빈 시간이 비용으로 남는다. 같은 사건이 같은 프로세스 안의 스레드 사이에서는 얇게, 서로 다른 프로세스 사이에서는 두껍게 일어난다.

**학습자 안내 (보조 캡션 회전 문구)**

- "무대 위의 색은 한 시점에 단 하나다 — 둘이 동시에 일하지 않는다."
- "트리거는 위에서 내려온다 — 흐름이 스스로 멈추지 않는다."
- "저장과 복원은 거울이 아니다 — 떠내는 운동과 정렬해 내려놓는 운동의 무게가 다르다."
- "시간 띠의 회색이 곧 오버헤드의 폭이다."
- "보관소가 두꺼워지면 빈 시간도 길어진다 — 프로세스 스위치의 비용."

**사건별 짧은 메시지**

- 트리거 도착: "트리거 도착 — 흐름이 멈춘다."
- 저장 시작: "현재 상태를 보관소로 떠낸다."
- 무대 비움: "무대가 비어 있다 — 누구도 진전하지 않는다."
- 복원 시작: "다음 흐름의 상태를 무대로 되돌린다."
- 재개: "이어서 시작 — 멈춘 자리에서 정확히 재개된다."
- 모드 변경(스레드 → 프로세스): "보관소가 두꺼워졌다 — 옮길 묶음이 늘어난다."
- 트리거 변경: "다음 트리거 종류가 바뀌었다."

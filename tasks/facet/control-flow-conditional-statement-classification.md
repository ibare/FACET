## 1. 대상 정보
- 대상: 프로그래밍 기초 제어 흐름의 "조건문" (if / else if / else)
- 도메인: control-flow
- 근거: 입력 조건의 참/거짓 평가에 따라 실행 경로가 분기되는 언어 차원의 제어 구문이며, 자료구조나 시스템 행위가 아닌 코드 흐름의 일반 개념에 해당.

## 2. type
- 결정: concept
- 근거: 특정 알고리즘의 단계적 절차나 상호작용 프로토콜이 아니라, "조건에 따라 실행 경로가 갈라진다"는 추상 개념을 학습 대상으로 하기 때문. 자료구조도 아니고, 입력→출력 변환 자체가 본질도 아님.

## 3. 진행 모델
- 결정: 입력 반응형
- 근거: 시간 축이 아니라 주어진 조건값(입력)에 반응하여 어느 분기로 흐를지가 결정됨. 메시지 시퀀스나 정적 구조보다 "입력에 대한 즉시 분기 반응"이 본질.

## 4. 핵심 학습 질문
- 조건의 참/거짓에 따라 실행 경로가 어떻게 갈라지고, 어떤 분기 단 하나만 실행되는가?

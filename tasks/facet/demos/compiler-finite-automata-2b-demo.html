이 시각화의 영감

  • Russ Cox — Thompson NFA: "병렬 부대(parallel troops)" 비유. 글로만 있던 동시성을 시각 언어로.
  • Stanford CS103 Subset Construction Guide: "한 페이지가 한 단계" 의 종이 만화 흐름 — 곡 전체를 한 컷에.
  • regex101: 시간 가역성 / 스크럽 감각 — 입력 위치를 자유롭게 점프.

의식적으로 회피한 표준 레퍼런스

  • JFLAP / Automata Tutor / AutomataVerse / jFAST: 원-화살표-이중원 그래프 컨벤션을 메인 시각으로 두지 않음.
  • Debuggex / Regexper: 좌→우 노드 그래프 / 철도 다이어그램의 "노선" 메타포를 차용하지 않음.
  • 일반적인 Step 버튼 패러다임: "현재만 점등" 이 아니라 곡 전체를 한 컷에 보이고 슬라이더로 시야만 이동.
유한오토마타란. 입력 글자를 한 개씩 읽으면서 유한 개의 상태 사이를 옮겨 다니다가, 입력이 끝났을 때 어디에 있는지로 받아들일지 거절할지를 결정하는 단순한 인식기다. 갈 길이 항상 한 곳뿐이면 결정적(DFA), 같은 글자에 갈 곳이 여러 곳일 수 있으면 비결정적(NFA). 두 모델은 같은 부류(정규언어)를 인식하지만, NFA 는 동시에 여러 상태에 살아 있을 수 있다. 컴파일러의 렉서/스캐너가 소스 코드 글자 스트림을 토큰으로 잘라내는 그 메커니즘이다.

"오토마타의 한 입력은 한 곡이다. 갈래가 늘어나는 비결정성은 합창의 화음이 부푸는 것."

곡: 정규식 (a|b)*abb 의 NFA 가 입력 aabb 를 부른다 박자 0 / 4

보이스(가로줄) = 상태(지금까지의 기억).   박자 안의 음표 = 그 시각에 살아 있는 상태 — NFA 는 한 박자에 여러 음이 동시에 울리는 합창.   슬러(잇는 곡선) = 전이.   마지막 박자의 ★ 보이스에 음표가 살아 있어야 수락. 막다른 분기는 𝄽 쉼표로 침묵.

곡 아래 좁은 한 줄이 DFA 축약 보표다. 매 박자에 단성 음표 하나가 떠 있고, 그 라벨이 위쪽 NFA 합창의 그 박자에 살아 있는 보이스 집합과 일치한다. 이 시각적 일치가 곧 부분집합 구성의 의미다.

## 1. 반드시 다뤄야 할 본질

1. **두 자료구조의 결합과 동일 노드 공유** — hash map 과 doubly linked list 가 화면에 별개 영역으로 동시 존재하며, hash 의 값과 list 의 노드가 같은 객체임이 화살표/연결선으로 항상 보여야 한다. get/put 어떤 호출에서도 두 영역이 동기화된 상태로 갱신되는 모습이 끊기지 않아야 한다.
2. **"최근성" 의 리스트 위치 매핑** — 한쪽 끝이 MRU, 반대쪽 끝이 LRU 라는 공간 규약이 라벨/축으로 명시되고, 호출이 거듭될수록 노드들이 이 축 위에서 재배열되는 양상이 시각적으로 드러나야 한다. 어느 끝이 MRU 인지는 화면 내내 일관된다.
3. **promotion (접근 시 노드 이동)** — get 또는 기존 키에 대한 put 이 발생하면, 해당 노드가 list 에서 unlink 되어 MRU 끝으로 재삽입되는 두 단계 변형이 보여야 한다. get 이 단순 조회가 아니라 list 를 회전시키는 쓰기성 행위라는 점이 시각적으로 강조되어야 한다.
4. **capacity 도달과 eviction** — 용량 한계가 화면 내 시각적 경계 (점선/카운터/슬롯 수) 로 상시 보이고, 새 키를 put 해서 초과가 발생할 때 LRU 끝 노드가 list 와 hash 양쪽에서 함께 사라지는 결정적 순간이 분리된 프레임으로 드러나야 한다.
5. **호출 단위 스냅샷 시퀀스** — 외부 입력 (`get(k)` / `put(k,v)`) 한 번이 한 단위의 상태 전이를 일으키며, 직전 상태 → 현재 상태의 변화가 호출 인덱스와 함께 추적 가능해야 한다. 자율 시간 흐름이 아니라 입력 반응형이라는 정체성이 표현된다.

## 2. 강화해야 할 요소

1. **추상 → 공간 좌표 변환의 명시화** — "마지막 접근 시각" 이라는 추상 시간 축이 "list 위치" 라는 공간 인덱스로 어떻게 환원되는지를, 호출 시각 라벨과 위치 라벨을 한 화면에 병기해 보여준다. 레퍼런스가 "최근성" 어휘로만 처리하던 부분을 시각적 변환 자체로 만든다.
2. **한 노드의 두 좌표 표현** — 동일 노드가 hash 영역에서는 "key 좌표" 로, list 영역에서는 "순서 좌표" 로 동시에 호명된다는 invariant 를, 두 영역을 잇는 항시적 연결선과 노드 강조로 입체화한다. 두 자료구조의 결합 동기를 직관으로 끌어올린다.
3. **get 의 부수효과 강조** — get 호출 시 노드 이동 단계 (조회 → unlink → MRU 재삽입) 가 분해되어 보여, "읽기인데 쓰기" 라는 비직관적 성격이 한눈에 잡혀야 한다. NeetCode 외에는 옅게 다루는 지점을 베이스 본질로 끌어올린다.
4. **eviction 임계의 단계 분리** — capacity-1 (여유 1칸), capacity (꽉 참), capacity+1 시도 (초과/축출) 의 세 상태가 시각적으로 분리되어, "꽉 차서 누가 떨어진다" 의 결정적 프레임을 별도 단계로 배치한다.
5. **호출 시퀀스 트레이스의 가시화** — 지금까지의 get/put 호출 이력이 화면 한쪽에 누적되어, 현재 list 배열이 어떤 호출 history 의 결과인지 역추적할 수 있어야 한다. 학습자가 "왜 이 노드가 LRU 끝에 있는가" 를 자기 시뮬레이션 없이 검증 가능하게 한다.

## 3. 공백 활용 후보

- **반사실 (counterfactual) 정책 비교** — 같은 호출 시퀀스에서 "LRU 였다면 / FIFO 였다면 / LFU 였다면 누가 빠졌을까" 를 나란히 보여주는 평행 트랙. 정책 선택의 의미를 동적으로 체감하게 한다.
- **메타포-자료구조 이중 표기** — 도서관/책상 비유 레이어와 hash+list 도식 레이어가 동시에 떠 있어, 같은 호출이 양쪽 표상에서 어떻게 해석되는지 매핑되는 구성.
- **실패 모드 시각화** — "get 에서 promotion 누락", "hash 만 갱신하고 list 미갱신" 같은 잘못된 구현이 호출 몇 회 후 어떻게 invariant 를 깨는지 보여주는 모드. 올바른 구현의 필요성을 부정형으로 학습.
- **temporal locality 의 hit/miss 패턴 가시화** — 호출 시퀀스의 지역성 분포에 따라 hit ratio 가 어떻게 변하는지 누적 막대/히트맵으로 보여주는 메타 레이어.

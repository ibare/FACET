# BFS (너비 우선 탐색) 시각화 기획

### 0. 기본 정보

- **대상 이름**: BFS (Breadth-First Search, 너비 우선 탐색)
- **도메인/카테고리**: 컴퓨터 과학 기초 / 그래프 알고리즘 / 그래프 순회
- **대상의 본질**: 출발 정점으로부터 **같은 거리에 있는 정점들을 한 덩어리로 묶어 바깥쪽으로 한 층씩 동시에 확장**하며 도달 가능한 모든 정점을 빠짐없이 찾는 탐색.

### 1. 시그니처 행동

출발점에서 거리가 정확히 `k` 인 모든 정점이 하나의 **레이어**로 묶여 동시에 발견되고, 한 레이어가 완전히 소진되어야 비로소 거리 `k+1` 레이어가 깨어난다. 이 동시성은 큐(FIFO) 가 "먼저 들어온 정점의 이웃을 먼저 꺼내 본다"라는 단 하나의 규칙으로 자연스럽게 만들어진다.

### 2. 시각적 은유

- **잔잔한 수면에 돌을 떨어뜨렸을 때 번지는 동심 파문** — 자연 현상. 파의 선두가 항상 원형을 이루고, 안쪽 파문은 지나간 자리로 남아 옅어진다. BFS 의 "같은 거리는 같은 시점에 도달"을 가장 정확히 드러낸다.
- **낮은 지대부터 차오르는 범람** — 자연 현상. 물은 높이가 같은 지점에 동시에 닿는다. 노드가 공간상 불규칙하게 놓였을 때 "거리 등고선"으로 차오르는 느낌을 표현한다.
- **공항 보안검색 대기열** — 사회적 행동. 먼저 줄 선 사람이 먼저 통과한다. 처리 순서의 *공정성*(FIFO)을 알려주는 보조 은유로, 큐 패널의 감정적 근거가 된다.

### 3. 결정적 순간

**한 레이어가 동시에 점등되는 "플래시" 순간.**

- **전**: 현재 레이어(거리 `k`)의 정점들이 밝게 빛나며 처리 대기 중. 바깥쪽 정점들은 모두 회색(미발견).
- **중**: 현재 레이어에서 동시에 뻗어 나간 간선들을 따라 파면이 한 프레임에 몰려들며, 다음 레이어(거리 `k+1`)의 모든 정점이 **동시에 한 번의 섬광**으로 점등된다. 큐 패널 뒤쪽으로 이 새 정점들이 한 묶음으로 입장한다.
- **후**: 새 레이어는 "활성(선두)" 색으로, 직전 레이어는 "지나간" 색으로 한 단계 식는다. 거리 라벨이 새로 점등된 정점들에 `k+1` 로 동시에 찍힌다.

시각적 강조: 섬광은 **한 tick 안에 끝나는 짧은 확산 glow + 레이어 공통 ring pulse**. "하나씩 켜지는" 애니메이션은 절대 사용하지 않는다 — 이것이 BFS 를 BFS 답게 만드는 장치다.

### 4. 다른 대상과의 구별점

- **vs DFS (깊이 우선 탐색)**: DFS 는 하나의 가느다란 "탐침"이 한 경로를 따라 깊숙이 파고들었다가 막히면 되돌아 나오는 지렁이 같은 움직임이다. 화면 어딘가 한 곳만 계속 밝고, 스택 되돌림으로 왔던 길이 깜빡인다. BFS 는 탐침이 없다. 파면 전체가 한꺼번에 밖으로 밀려 나가고, 한 번 지나간 곳을 **절대 다시 밟지 않는다**. DFS 는 "선", BFS 는 "링".
- **vs Dijkstra (가중 그래프 최단 경로)**: 비가중 그래프에서 Dijkstra 는 BFS 와 동치이므로, 반드시 **가중 그래프**를 나란히 놓고 비교한다. Dijkstra 의 파면은 간선 가중치에 따라 일그러진 불규칙 윤곽(가까운 노드가 나중에 확정되기도 함)이지만, BFS 의 파면은 항상 완벽한 "그래프 거리 등고선"으로 매끈하다. BFS 에는 우선순위 큐의 "재평가 점멸"(거리 갱신 시 색이 흔들림) 이 없다.
- **vs 그리드 Flood Fill / BFS on Grid**: 격자 BFS 는 사각 파문이지만, 일반 그래프 BFS 는 **위상적 거리** 파문이라 화면상 원형이 아닐 수 있다. 이를 분명히 하기 위해 "거리 동심원 가이드"를 배경에 깔아 그래프 거리와 기하 거리가 다르다는 사실을 드러낸다.

### 5. 달성해야 할 시각 요소

- **이름**: 동심 파면
  - **설명**: 같은 BFS 거리에 있는 정점들이 동일한 "고리(ring)" 에 속한 것처럼 보인다. 거리가 커질수록 바깥쪽 고리.
  - **구현 힌트**: 레이아웃 단계에서 BFS 거리로 방사상(concentric) 좌표 계산(출발점을 중심으로 거리 = 반지름, 같은 거리 내에서 각도 균등 분배). 배경에 옅은 점선 원을 거리 단계마다 깔아 "등고선" 을 명시. 정점 색은 HSL 의 hue 혹은 lightness 를 거리에 따라 단조롭게 변화(예: 선두는 따뜻한 색, 거리 증가 시 차가운 색).

- **이름**: 동시 점등 섬광
  - **설명**: 새 레이어가 발견되는 순간, 해당 레이어의 모든 정점이 **같은 프레임**에 번쩍인다.
  - **구현 힌트**: algorithm 에서 "한 레이어 전체를 큐에서 꺼내 그 이웃을 한 번에 discover" 하는 phase 를 한 이벤트로 묶어 emit. projector 는 이 이벤트를 받아 해당 정점 집합에 동시에 `pulse` 애니메이션(반지름 1.0 → 1.4 → 1.0, 120ms) 과 밝기 스파이크를 적용. 하나씩 순차 트윈 금지.

- **이름**: FIFO 큐의 몸통
  - **설명**: 큐에 들어오고 나가는 순서가 곧 파면 확장의 이유임을 보여준다. 항상 **뒤로 들어가고 앞에서 나간다**.
  - **구현 힌트**: 화면 하단 또는 우측에 가로 긴 "트랙" 형태의 큐 패널. enqueue 는 우측에서 슬라이드 인, dequeue 는 좌측에서 슬라이드 아웃. 큐 요소의 색은 그 정점이 속한 레이어 색과 동일하게 유지해, 큐 안에서도 "레이어 묶음"이 이어지는 블록으로 보이게 한다. 스택(DFS) 과 반대임을 분명히 하기 위해 좌→우 흐름 방향을 화살표로 표시.

- **이름**: 확정 거리 라벨
  - **설명**: 정점이 발견되는 순간 `dist = k` 숫자 라벨이 박혀, BFS 가 단순 방문 체크가 아닌 **최단 거리 계산**이라는 점을 드러낸다.
  - **구현 힌트**: 정점 원 위/옆에 작은 배지. 발견 섬광과 동시에 0% → 100% 로 페이드 인. 이후에는 절대 바뀌지 않음(이 점이 Dijkstra 와의 차이). 같은 레이어 라벨들은 동일한 숫자라 "3, 3, 3, 3" 같은 반복이 나란히 보여야 한다.

- **이름**: 비가역 꼬리 (뒤돌아보지 않음)
  - **설명**: 지나간 레이어는 차분히 식어가기만 할 뿐, BFS 가 그 위를 다시 밟는 장면은 **절대** 나오지 않는다.
  - **구현 힌트**: "방문 완료" 상태는 채도를 낮추고 불투명도 70% 로 고정. 현재 처리 중인 간선만 선두 색으로 빛나며, 간선 강조는 항상 **안→밖** 방향의 흐름 입자(수 개의 점이 간선을 따라 흘러나가는 트레일) 로만 표현. 반대 방향 하이라이트는 렌더러에 존재하지 않도록 한다.

### 6. 시각 구성

- **중심 시각**: 그래프 정점/간선 캔버스. 정점은 출발점을 중심으로 한 **동심 방사형 레이아웃**으로 배치되고, 현재 파면에 해당하는 고리가 가장 뜨겁게 빛난다. 간선은 평소엔 옅은 선으로만 존재하다가, 파면이 지나갈 때 선두 색으로 흐름 입자를 뿜고 즉시 식는다.
- **보조 요소들**:
  - **큐 트랙 패널**: 화면 하단의 가로 띠. FIFO 흐름(우측 입장, 좌측 퇴장) 을 실시간으로 보여준다. 각 요소는 정점의 레이어 색을 그대로 입는다.
  - **거리 레이어 인덱스(카운터)**: 현재 처리 중 거리 `k` 와 다음 거리 `k+1` 를 표시하는 미니 숫자 디스플레이. 레이어가 전환될 때 `k` 가 딸깍 증가하며 0.1s 짧게 강조.
  - **출발점 배지**: 출발 정점은 항상 작은 "돌" 아이콘 + 중심 표식으로 식별. 파문의 발원지임을 상기시킨다.
  - **배경 등고선**: 거리 단계마다 옅은 점선 원. 위상적 거리가 기하적 거리로 번역된 것임을 조용히 알린다.
- **요소 간 관계**: 큐 패널의 "좌측 퇴장"(dequeue) 은 캔버스의 "해당 정점 처리 시작" 과 동기화되고, "우측 입장"(enqueue) 은 캔버스의 섬광(신규 레이어 발견) 과 동기화된다. 큐의 한 블록이 통째로 화면에서 동시에 처리되어 다음 레이어 블록이 통째로 큐에 들어오는 **묶음 리듬**이 사용자에게 보여야 한다.

### 7. 레이아웃

- 상단 ~70%: 그래프 캔버스. 출발점은 캔버스 중앙보다 약간 좌측. 레이어가 바깥으로 퍼질 공간을 우측으로 더 많이 확보한다.
- 우상단 코너: 거리 레이어 인덱스(`k = 2 → 3`) 미니 패널.
- 하단 ~20%: 가로 FIFO 큐 트랙. 좌측에 "OUT ▶", 우측에 "◀ IN" 라벨.
- 우측(선택): 코드 패널(파일 `algorithm.ts` 의 핵심 루프) — 현재 실행 라인을 BFS 의 "레이어 꺼내기 / 이웃 보기 / enqueue" 단계와 연동.
- 캔버스 배경: 출발점을 중심으로 한 동심 점선 원(등고선). 파문이 지나간 안쪽은 아주 옅은 음영으로 채워지며, 바깥쪽은 투명.

### 8. 진행과 이벤트

주요 진행 단계:

1. **초기**: 출발점만 선두 색으로 점등(거리 0), 큐에 출발점 한 개. 등고선 외곽은 전부 회색.
2. **레이어 처리 시작**: 현재 거리 `k` 의 첫 정점을 큐에서 dequeue — 큐 좌측에서 슬라이드 아웃, 캔버스에서 해당 정점이 "처리 중" 하이라이트(외곽 링) 로 바뀐다.
3. **이웃 탐사**: 해당 정점의 간선들을 따라 흐름 입자가 **안→밖** 으로 동시에 뻗어 나감. 아직 방문하지 않은 이웃은 **대기 집합**에 모인다(한 tick 내에서).
4. **레이어 완료(결정적 순간)**: 현재 거리 `k` 의 모든 정점이 소진되면, 그 동안 모인 대기 집합 — 즉 거리 `k+1` 레이어 전체 — 가 **한 프레임에 동시 섬광**으로 점등, 거리 라벨 `k+1` 이 일제히 박히고, 같은 순간 큐 우측으로 한 블록이 통째로 슬라이드 인.
5. **식음**: 직전 레이어는 "방문 완료" 저채도로 가라앉고, 등고선 음영이 해당 반지름까지 채워진다.
6. **반복**: 2~5 를 큐가 빌 때까지 반복.
7. **종료**: 큐가 비는 순간 캔버스 전체가 거리별 색 스펙트럼의 "지도" 로 고정되어 정적인 스냅샷으로 남는다. 이 최종 프레임만 봐도 "출발점에서 각 노드까지의 최단 거리" 가 색으로 읽혀야 한다.

사용자 제어 지점:

- **재생/일시정지** (공통).
- **단계 전진**: 세 해상도 — (a) tick 단위(정점 한 개 dequeue), (b) **레이어 단위**(다음 섬광까지 한 번에), (c) 처음부터 끝까지 스냅샷.
- **속도 슬라이더**.
- **출발점 재선택**: 정점 클릭으로 출발점 변경, 전체 재생.

레이어 단위 스텝이 BFS 를 가장 잘 보여주므로 기본값으로 삼는다.

### 9. 초기 상태와 파라미터

- **초기 데이터**: 정점 수 10~14 개 내외의 연결된 무방향 단순 그래프. 각 레이어에 2~4 개 정점이 들어가 "동시 점등" 이 뚜렷이 보이도록 의도적으로 너비가 있는 구조로 고른다(한 줄로 길게 이어진 그래프는 BFS/DFS 구별이 약하므로 피한다).

  예시 인접 리스트(거리 0~3 레이어가 뚜렷한 구조):
  ```
  A : [B, C, D]
  B : [A, E, F]
  C : [A, F, G]
  D : [A, G, H]
  E : [B, I]
  F : [B, C, I, J]
  G : [C, D, J, K]
  H : [D, K]
  I : [E, F, L]
  J : [F, G, L]
  K : [G, H, L]
  L : [I, J, K]
  ```
  출발점 = `A` 일 때 레이어: {A} → {B,C,D} → {E,F,G,H} → {I,J,K} → {L}.

- **파라미터**:
  - `source`: 출발 정점(기본 `A`, 클릭으로 변경).
  - `graph preset`: "너비가 넓은 그래프(기본)" / "깊고 좁은 그래프(DFS 비교용)" / "가중치 포함(Dijkstra 비교용, BFS 는 가중치 무시)".
  - `step mode`: `tick` / `layer`(기본) / `full`.

## 부록: 기획 메모 (메타 서술)

- **수렴 방지 포인트**: BFS 를 AI 로 만들게 두면 "노드에 번호 0,1,2 붙이고 화살표 따라 색칠" 로 수렴한다. 본 기획은 그 시각을 버리고 **(1) 레이어의 동시성 = 한 프레임 섬광**, **(2) 거리의 공간화 = 동심 레이아웃**, **(3) FIFO 의 블록 리듬 = 큐 패널과 캔버스의 동기화** 세 축으로 BFS 를 재정의했다.
- **Dijkstra/DFS 와의 대비**를 시각 장치(링 vs 선, 매끈한 등고선 vs 일그러진 윤곽) 레벨까지 내린 이유: FACET 의 비교 학습 맥락에서 같은 도메인의 다른 facet 과 나란히 놓일 때 반드시 구별되어야 하기 때문.
- **구현 단계로 넘길 판단 사항(의도적으로 열어 둠)**: (a) 방사형 레이아웃을 레이어별 각도 자동 분배로 할지 수작업 좌표로 둘지, (b) 흐름 입자의 구현을 canvas particle / SVG animate / css keyframe 중 어느 것으로 할지, (c) "레이어 묶음 이벤트" 를 algorithm 계층에서 네이티브로 emit 할지 projector 에서 버퍼링해 합성할지 — 이 세 가지는 `rules/concerns/` 의 4-layer 경계와 이벤트 어휘 규칙을 따라 구현 기획에서 결정.

## 1. 시그니처 행동

**한 글자를 먹을 때마다 그래프 위 활성 토큰이 라벨된 화살표 한 개를 따라 다음 상태로 점프하고, 같은 박자에 전이표의 한 셀이 점등하며, 지나온 자취가 옅은 색으로 누적된다.**

DFA 라면 토큰 한 개가 한 화살표를 따라 가고, NFA 라면 *토큰이 복제되어 여러 화살표로 동시에* 흩어진다 — 같은 박자, 같은 시각 언어 안에서.

## 2. 시각적 은유

보고서의 공통 메타포(**그래프 = 미로/도시 지도**, **상태 = 기억의 한 칸**, **수락 상태 = 목적지**, **NFA = 동시에 여러 길**) 를 채택하고 정교화한다.

핵심 은유는 **"라벨된 도시 지도 위를 달리는 발광 토큰"**:

- 노드는 **이름이 붙은 정거장**. 단순한 q0/q1 이 아니라 클릭/토글로 *의미 라벨*("정수 첫 자리 봤음", "소수점 봤음") 이 드러나는 *기억의 응축*.
- 화살표는 **글자 라벨이 박힌 길**. 자기 루프는 같은 정거장으로 돌아오는 짧은 우회로.
- 입력 테이프는 **머리 위에 떠 있는 차표**. 한 글자가 소비되면 토큰을 따라 *그래프 안으로 빨려 들어가* 화살표를 점등한다.
- 토큰은 **현재 위치를 가리키는 발광점**. NFA 에서는 한 글자에 토큰이 *복제되어* 여러 정거장을 동시에 점유.
- 자취(trail)는 **토큰이 지나온 길 위에 옅게 남는 빛 잔상**. 입력이 끝나면 그래프에 *전체 궤적* 이 한 컷으로 남는다.

이 은유는 보고서가 정리한 "원이 여러 개 있고 화살표가 그 사이를 잇는다 / 한 글자에 한 칸 / 이중원에서 끝나면 받아들여진다" 는 학습자 자동 연상을 그대로 유지한다.

## 3. 결정적 순간

표준 시각 언어 안에서 다음 다섯 순간을 강조한다.

1. **입력 글자 소비 (매 박자)** — 입력 테이프의 헤드가 한 칸 전진하고, 그 글자가 시각적으로 그래프 안으로 *빨려 들어가* 해당 화살표를 0.2~0.3초 동안 굵게 발광시킨다. 화살표 위 라벨이 같이 펄스. 동시에 전이표의 (현재 상태, 글자) 셀이 같은 색으로 동기 점등.
2. **상태 점프** — 토큰이 화살표 끝의 다음 노드로 이동. 도착 노드는 짧게 펄스, 출발 노드는 활성을 잃고 *경로 잔상* 이 옅게 남음.
3. **NFA 분기 — 토큰 복제** — 한 글자에 갈 곳이 여러 개일 때 토큰이 *둘 이상으로 갈라지며* 동시 점등. 죽은 분기는 회색으로 페이드, 살아 있는 분기는 채도가 유지된다.
4. **막힘 (정의되지 않은 전이 / trap 진입)** — 갈 길이 없는 글자가 들어오면 토큰이 *깜빡이며 사라지고*, 입력 테이프의 그 글자가 빨간색으로 잠긴다. trap 상태는 처음부터 어두운 회색 톤으로 그려져 *희망이 없음* 의 시각적 무게가 미리 부여된다.
5. **입력 종료 — 수락/거부 판정** — 입력 테이프 헤드가 끝에 도달하는 순간, 토큰이 위치한 노드가 *이중원이라면* 녹색 광휘 + "ACCEPT", 아니라면 빨간 외곽 + "REJECT". 보고서가 강조하는 "수락의 양면성(지나가는 것이 아니라 *거기서 끝나야* 수락)" 을 종료 신호의 시각 효과로 명시한다.

## 4. 다른 대상과의 구별점

### vs 푸시다운 오토마타 (PDA)
- PDA 는 **스택 패널이 그래프 옆에 따로** 존재해 push/pop 이 시각적으로 동반된다. 유한오토마타 시각화에는 *스택이 없다* — 보조 메모리가 0 이라는 사실 자체가 시각적 약속.
- 화살표 라벨이 PDA 는 `a, X / YX` 같은 3-튜플인 반면 유한오토마타는 단일 글자(또는 ε).

### vs 튜링 기계
- 튜링 기계는 입력 테이프가 **양방향 + 쓰기 가능** — 헤드가 좌우로 움직이고 셀에 새 글자가 덮어 써진다. 유한오토마타는 헤드가 *단조 우진행 + 읽기 전용*.
- 튜링 기계 시각화는 테이프가 주역, 그래프는 보조. 유한오토마타는 그래프가 주역, 테이프는 보조.

### vs 정규식 철도 다이어그램 (Regexper)
- 철도 다이어그램은 **노드 = 글자, 엣지 = 인접 가능성** — 이중원 / 라벨된 전이 / 자기 루프가 *없다*. 유한오토마타는 *그래프가 닫혀 있고 자기 루프가 흔하다*.
- 철도는 정적, 유한오토마타는 한 글자 = 한 박자의 이산 진행.

### vs 일반 상태 머신 다이어그램 (UML statechart)
- UML 은 **계층 상태 / 가드 / 진입-진출 액션** 이 풍부. 유한오토마타는 *계층 없는 평탄한 상태 + 라벨된 입력 글자만 있는 전이*.
- 유한오토마타의 화살표 라벨은 *알파벳 한 글자* 여야 한다 — 시각적으로 라벨 길이가 *짧고 균일*.

이 차이는 그래프 외곽의 패널 구성(스택 없음 / 테이프 단방향 / 자기 루프 흔함 / 라벨 균일) 으로 즉시 드러난다.

## 5. 달성해야 할 시각 요소

### 요소 1 — 의미 라벨 토글 (Semantic State Labels)
- **설명**: 각 노드는 기본적으로 q0/q1 식별자를 보이지만, 우상단의 "의미 라벨" 토글을 켜면 *이 상태로 오는 모든 prefix 의 의미 요약*("아직 아무것도 못 봤음", "정수 첫 자리 봤음", "소수점 봤음", "지수부 시작 글자 e 봤음") 으로 라벨이 바뀐다. 호버 시 그 상태로 오는 입력 prefix 예시 3개가 떠오른다.
- **구현 힌트**: `state.id` + `state.semanticLabel`(Optional) 을 facet 데이터에 둔다. View 가 토글 상태를 받아 라벨을 스왑. NodeView 의 라벨 길이가 길어질 수 있으므로 노드 크기는 라벨 폭에 맞춰 가로로 늘어나는 *capsule shape*.

### 요소 2 — 전이표 동기 트레이스 (Synced δ-Table)
- **설명**: 그래프 옆/아래에 **δ 전이표** (행 = 상태, 열 = 알파벳, 셀 = 다음 상태) 가 항상 떠 있다. 한 박자에 사용된 셀은 그래프의 화살표와 *같은 색* 으로 동시 점등. 셀에서 화살표로 가는 *옅은 연결선* 이 한 박자만 잠깐 보였다 사라진다.
- **구현 힌트**: 표는 grid layout. 활성 셀의 `data-active` attribute 가 그래프의 활성 transition 과 같은 ID 를 공유. 펄스 효과는 동일한 keyframe 을 같은 박자에 발사.

### 요소 3 — 경로 잔상 (Trail)
- **설명**: 토큰이 지나간 노드와 화살표는 *옅은 발광 잔상* 으로 누적된다. 잔상 진하기 = 그 길을 *얼마나 자주* 지나갔는가. 자기 루프를 N 번 돌면 그 루프가 N 단계로 진해진다. 입력이 끝났을 때 학습자는 그래프 한 컷에서 *전체 궤적* 을 본다.
- **구현 힌트**: 각 transition 에 `visitCount` 누적. SVG stroke-opacity 를 `min(0.15 + visitCount * 0.15, 0.8)` 로. 토큰 이동 시 짧은 펄스는 별개 레이어.

### 요소 4 — NFA 분기 트리 일급화 (Branch Tree as First-Class)
- **설명**: NFA 모드에서 그래프 우측에 **분기 트리 미니 패널** 이 동시에 자란다. 박자마다 살아 있는 토큰 집합 = 트리의 그 시점 노드 집합. 죽은 분기는 가지가 회색으로 시들고, 수락 가능 분기에 도달한 가지는 녹색으로 빛난다. 그래프 위의 토큰 색과 트리 노드 색이 *짝지어* 있어 어느 토큰이 어느 가지인지 즉시 보인다.
- **구현 힌트**: `BranchTreeView` 를 별도 view 로. NFA Projector 가 각 박자에 `tree.addLevel(activeStateSet)` 을 호출. 임베드 폭이 좁을 때(< 700px) 는 트리가 그래프 *아래로* 접힌다.

### 요소 5 — 미디어 플레이어 시간 컨트롤 (Step / Run / Scrub)
- **설명**: 하단에 입력 테이프와 **타임라인 스크럽 바** 가 결합. ▶/⏸/⏮/⏭ 와 더불어 *입력 위치를 클릭/드래그* 해 그 시점으로 즉시 점프. 속도 0.5x / 1x / 2x. 되감기 시 토큰과 잔상이 함께 *역방향으로* 흐른다.
- **구현 힌트**: 알고리즘은 *결정성 기준*으로 매 박자 스냅샷을 배열에 누적. Scrub 은 단순히 인덱스 점프 + 그 인덱스까지의 transition 을 다시 적분해 잔상 재구성.

## 6. 시각 구성

### 중심 시각
**라벨된 그래프** — 원형 노드 + 라벨된 화살표 + 이중원. 5~8개 노드의 작은 예제 (예: "정수 또는 부동소수 토큰을 인식하는 DFA"). 그래프는 임베드 영역의 상단 약 60% 를 차지.

### 보조 요소들
- **입력 테이프** — 그래프 바로 위, 가로 한 줄. 글자별 셀, 헤드 ▼ 가 현재 위치를 가리킴. 소비된 글자는 옅은 회색 + 잔상 색, 미소비는 진하게.
- **δ 전이표** — 그래프 우측(또는 좁을 때 아래). 행 = 상태, 열 = 알파벳, 셀 = 다음 상태 라벨. 한 박자마다 사용 셀이 그래프와 같은 색으로 점등.
- **분기 트리 (NFA 모드 한정)** — 그래프 우측 또는 아래. 박자별 살아 있는 상태 집합이 한 레벨씩 자라남.
- **결과 패널** — 우상단 작은 배지. "ACCEPT / REJECT" + 입력 끝 위치까지의 토큰 위치 요약.
- **컨트롤 바** — 하단. ▶ / ⏸ / ⏮ / ⏭ + 속도 + 스크럽 슬라이더 + DFA/NFA 토글 + "의미 라벨 보기" 토글.

### 요소 간 관계
- 입력 테이프 → 그래프: 헤드가 가리키는 글자가 *그래프 안으로 빨려들어가* 화살표를 점등.
- 그래프 → 전이표: 활성 화살표의 (출발 상태, 글자) 가 표의 셀과 같은 색으로 페어.
- 그래프 → 분기 트리: 살아 있는 토큰 집합이 트리의 현재 레벨 노드 집합과 일대일.
- 컨트롤 → 모든 요소: 한 박자가 진행되면 모든 패널이 *같은 박자에* 점등 / 점프 / 자라남.

**임베드 폭(640~720px) 안에 모두 들어간다**: 표준 720px 기준 그래프 영역 폭 약 460px + 우측 패널(전이표 또는 분기 트리) 폭 약 240px. 좁아지면 우측 패널이 그래프 *아래로* 세로 적층.

### NFA 모드 활성 시 (분기 트리 등장)
전이표 영역이 분기 트리로 *교체*된다 (탭 토글). 두 패널을 동시에 띄우지 않는다 — 보고서가 경고한 "좌우 분할 4~6개" 에 빠지지 않기 위함.

### 모바일 / 좁은 임베드 (< 640px)
세로 적층:
1. 입력 테이프
2. 그래프
3. 전이표 또는 분기 트리 (탭으로 하나만)
4. 컨트롤
5. 안내문

좌우 분할이 깨지지 않도록 우측 패널은 항상 *접을 수 있게* 설계.

## 8. 진행과 이벤트

### 단계 0 — 초기 정지 상태
- 그래프가 정적 표시. 시작 상태 q0 가 *대기 펄스* (느린 호흡).
- 입력 테이프에 예시 문자열이 채워져 있고 헤드는 첫 글자 직전.
- 전이표는 모든 셀이 회색.
- 결과 패널: "READY".

### 단계 1 — 첫 글자 소비 (▶ 또는 ⏭)
- 입력 헤드 전진 → 그 글자가 그래프 안으로 빨려들어가는 0.25초 모션.
- 출발 노드 q0 의 해당 화살표가 굵게 발광 + 라벨 펄스.
- 동시에 전이표의 (q0, 글자) 셀이 같은 색으로 점등.
- 도착 노드가 펄스 → 활성 인계.
- 출발 노드와 화살표에 *옅은 잔상* 이 남음.

### 단계 2~N — 반복
- 매 박자마다 단계 1 의 시각 변화가 같은 색 코드로 반복.
- 자기 루프를 돌 때는 토큰이 *작은 원호로 빠져나갔다 돌아오는* 모션 — 단순 점프와 시각적으로 구분.
- NFA 모드라면 한 박자에 토큰이 분기되어 복제. 분기 트리 한 레벨이 자람.

### 단계 종료 — 수락/거부
- 입력 헤드가 끝에 도달.
- 토큰 위치가 이중원이면: 노드가 녹색 광휘 + 결과 패널 "ACCEPT".
- 아니면: 노드가 빨간 외곽 + 결과 패널 "REJECT — 수락 상태가 아님".
- 도중에 막혔다면: 토큰이 깜빡이며 사라진 시점의 입력 테이프 글자가 빨간색으로 잠기고 결과 "REJECT — 막다른 길".

### 사용자 제어 지점
- ▶ / ⏸: 자동 재생 토글. 기본 1x = 박자당 600ms.
- ⏮ / ⏭: 한 박자 뒤/앞.
- 스크럽 슬라이더: 임의 박자로 점프. 잔상은 그 시점까지로 재구성.
- 속도 0.5x / 1x / 2x.
- "의미 라벨" 토글: 노드 라벨을 q0/q1 ↔ 의미 텍스트로 스왑.
- "NFA / DFA 모드" 토글: 같은 정규식의 두 오토마타를 전환. 전환 시 그래프가 부드럽게 morph (가능하면).
- 입력 문자열 편집: 텍스트 박스에서 직접 입력하면 즉시 처음으로 리셋.

## 9. 초기 상태와 파라미터

### 초기 데모 예제
부동소수 리터럴 인식 DFA — 컴파일러 렉서 맥락이 자연스럽게 따라온다.

- **알파벳**: 숫자(d), 점(.), 지수 표시(e), 부호(±)
- **상태 (5개)**:
  - q0 = "아직 아무것도 못 봤음" (시작)
  - q1 = "정수부 봤음"
  - q2 = "소수점 봤음, 소수부 미시작"
  - q3 = "소수부 봤음" (수락)
  - q4 = "지수부 시작 글자 e 봤음"
  - q5 = "지수부 숫자 봤음" (수락)
- **수락 상태**: q3, q5 (이중원)
- **초기 입력 문자열**: `3.14e5` (수락) — 학습자가 한 박자씩 따라가면 q0→q1→q1→q2→q3→q4→q5 로 끝.
- 추가 예시 입력 (드롭다운): `3.14` (수락), `3..14` (거부, 막힘), `e5` (거부, 시작부터 막힘).

### 색 팔레트 (테마 토큰 사용 가정)
- 활성 토큰: accent-primary (파랑/보라 계열)
- 잔상: accent-primary 의 alpha 0.2~0.5
- 수락 광휘: success-green
- 거부/막힘: error-red
- trap / dead 상태: neutral-dim (어두운 회색, 처음부터)
- 의미 라벨 토글 OFF: monospace q0/q1
- 의미 라벨 토글 ON: 부드러운 sans, 라벨 길이에 따라 노드 폭이 늘어남

### 파라미터
- `initialMode`: "DFA" | "NFA" — 기본 "DFA"
- `showSemanticLabels`: boolean — 기본 false
- `playbackSpeed`: 0.5 | 1 | 2 — 기본 1
- `autoplayOnMount`: boolean — 기본 false (학습자가 ▶ 를 직접 누르도록)
- `trailDecay`: number — 잔상 감쇠. 기본 0 (감쇠 없음, 누적만)

## 10. 레퍼런스 대비 차별점

### 공통 요소를 어떻게 더 잘 다루는가
- **원형 노드 + 라벨된 화살표 + 이중원**: 표준 도구가 모두 갖지만, 이 기획은 **노드 라벨 자체가 토글 가능한 의미 표상**이다. JFLAP / ivanzuzak / Debuggex 어느 도구도 q0 ↔ "정수 첫 자리 봤음" 같은 의미 라벨 토글을 일급으로 두지 않는다.
- **현재 상태 펄스**: 표준 도구는 *현재만* 강조. 이 기획은 **경로 잔상 누적**으로 *과거 전체 궤적* 을 한 컷에 보존. 입력이 끝났을 때 학습자는 "내가 어디를 지나왔는가" 를 즉시 본다.
- **이산 스텝**: 표준 도구는 Step / Run 토글만. 이 기획은 **미디어 플레이어식 스크럽 + 속도 + 되감기**로 시간을 자유롭게 다룰 수 있게 한다.
- **수락/거부 표시**: 표준 도구는 끝에서만. 이 기획은 **막힘 시점의 입력 글자를 빨간색으로 잠그는 시각** 으로 *입력 도중에 이미 결정될 수 있다* 는 본질을 직접 표현 (보고서가 지적한 "수락의 양면성" 공백을 채움).
- **그래프 + 표 이중성**: 한국어 강의노트의 δ 표 전통을 *동기 트레이스*로 격상. 정적 표가 아니라 한 박자에 한 셀이 그래프와 같은 색으로 맞불.

### 표준이 놓치는 어떤 측면을 추가하는가
- **NFA 분기 트리 일급화**: JFLAP 이 옵션으로 가진 분기 트리를 *항상 같이 자라는 일급 패널* 로 만든다. 토큰 색 ↔ 트리 노드 색의 페어링.
- **trap 상태의 시각적 위계**: 평균 도구가 trap 을 일반 노드와 동등하게 그리는 반면, 이 기획은 처음부터 어두운 회색 톤으로 그려 *희망이 없음* 의 무게를 시각적으로 부여.
- **소비 글자가 그래프 안으로 빨려들어가는 모션**: 표준 도구는 입력 테이프와 그래프가 분리된 두 패널이라 시간이 분리된다. 이 기획은 *글자 자체가 화살표를 점등시키는* 결합 모션으로 시간을 통합.
- **컴파일러 렉서 맥락의 자연스러운 연결**: 초기 예제가 *부동소수 토큰 인식 DFA* — 학습자가 즉시 "이건 컴파일러가 실제로 하는 일" 임을 안다. 평균은 a/b, 0/1 의 학술 토이.

### 학습자가 "표준 도구의 그 시각화보다 낫다" 고 느낄 부분
- 표준 도구를 본 학습자가 "어, 이거 한 글자 먹을 때마다 *어디를 지나왔는지* 한 번에 보이네" 라고 느끼는 경로 잔상.
- "q0 이 뭔지 헷갈렸는데 라벨 토글 누르니까 *기억이 무엇인지* 즉시 보이네" — 의미 라벨.
- "전이표가 그래프랑 같은 박자에 점등하니까 표 보는 법을 따로 안 배워도 되네" — 동기 트레이스.
- "되감기로 다시 보고 싶은 박자만 콕 집을 수 있네" — 스크럽.

## 11. 글 흐름 안에서의 위치

### 위치
**개념 도입부 직후 ~ 원리 설명의 핵심**. 글이 정규언어 / 렉서 / 토큰의 동기를 깐 후, "그래서 유한오토마타가 무엇이며 어떻게 글자를 먹는가" 를 처음 시각으로 보여 주는 자리.

이 시각화 *이후* 에 본문은 (a) 부분집합 구성으로 NFA→DFA, (b) 정규식 ↔ NFA 의 톰슨 구성, (c) 렉서의 max-munch 같은 응용으로 자연스럽게 확장한다.

### 시각화 안 / 옆에 포함될 짧은 설명 텍스트 (3줄)

> - **노드는 *상태***. 지금까지 본 글자들의 요약된 기억.
> - **화살표는 *전이***. 글자 하나를 먹으며 한 상태에서 다른 상태로.
> - **이중원에 *멈춰 끝나면* 받아들임**, 그 외에는 거절.

(보고서의 "시각화 옆에 띄울 짧은 안내문" 을 그대로 채택.)

### 시각화 안에 작은 글씨로 함께 띄울 용어 (호버 / 토글)
- DFA = "갈 길이 항상 한 곳" 기계.
- NFA = "갈 길이 여러 곳일 수 있는" 기계.
- ε-전이 = 글자를 안 먹고 그냥 옮겨 가는 비밀 통로 (NFA 만).
- 수락 = 입력 끝에서 이중원 위에 있음.

### 한 줄 캐치프레이즈 (시각화 헤더에)

> *"한 글자씩 먹으며 상태 사이를 옮겨 다니다, 다 먹었을 때 받아들일지 거절할지를 결정하는 작은 기계."*

## 자가 점검

- **시그니처 행동의 변별성**: "한 글자에 토큰이 화살표를 따라 점프 + 전이표 동기 + 잔상 누적" 은 PDA(스택), 튜링(테이프 쓰기), 철도 다이어그램(정적), UML statechart(계층) 어디와도 혼동되지 않음. ✓
- **표준 대비 차별점**: 의미 라벨 토글, 경로 잔상, 전이표 동기, 분기 트리 일급화, 미디어 플레이어 시간 컨트롤 — 보고서가 정리한 어느 표준 도구도 한 화면에 *동시에* 갖지 않음. ✓
- **임베드 폭 640~720px 안의 안전한 레이아웃**: 그래프 460 + 우측 패널 240, 좁아지면 세로 적층. 좌우 4분할 없음. ✓
- **개념을 모르는 학습자도 따라올 수 있는 짧은 설명**: 시각화 하단 3줄 안내 + 노드/화살표/이중원의 용어 호버. ✓
- **초심자 이해 가능성**: 초기 예제가 학술 a/b 가 아닌 *부동소수 토큰* — 컴파일러 맥락이 즉시 잡힘. 자동 재생 OFF + Step 컨트롤로 학습자가 박자를 직접 통제. ✓

## 부록: 기획 메모

### 보고서의 "공백" 중 이 기획이 채택한 것
- 잔상(trail) 누적 — 채택 (요소 3).
- 입력 테이프와 그래프의 결합 시간 — 채택 (글자 빨려들어가는 모션).
- NFA 분기 폴리포니 — 채택 (요소 4).
- 타임라인 스크럽 — 채택 (요소 5).
- 상태 = 기억의 직접 시각화 — 부분 채택 (의미 라벨 토글).
- 그래프 + 표 동기 — 채택 (요소 2).

### 이 기획이 의도적으로 *남긴* 영역 (2B 새로운 방식이 가져갈 수 있는)
- "복제 말 / 동시 출발 열차" 같은 강한 새 메타포 — 이 기획은 표준 그래프 메타포를 강화한다. 메타포 교체는 2B 의 영역.
- 4-층 동시 시각화 (정규식 / NFA / DFA / 토큰화 출력) — 이 기획은 *한 오토마타* 의 디테일에 집중. 4-층은 임베드 폭 안에서 깨질 위험 + 표준 강화 범위 초과.
- 백트래킹 되감기를 *시간이 거꾸로 감기는* 별개 시각 메커니즘으로 — 이 기획은 일반 스크럽의 부수 효과 정도.
- DFA 와 NFA 를 *동시* 진행하며 시간의 모양 차이 비교 — 이 기획은 토글로 단일 표시.

### 기술적 주의
- 잔상 누적은 SVG 의 stroke 를 누적 그릴 경우 박자 수가 많을 때 성능 부담. WebGL 또는 Canvas 레이어 분리 검토. 다만 임베드 폭 + 작은 예제(10~20 박자) 라면 SVG 로 충분.
- 의미 라벨 토글 시 노드 폭 변화는 그래프 자동 레이아웃을 흔들 수 있음. 의미 라벨이 길어도 *원래 노드 위치를 유지*하고 라벨이 노드 옆/아래로 흘러나오는 방식이 안전.
- 전이표가 알파벳이 클 때(ASCII 128) 임베드 폭을 깨뜨림. 데모는 알파벳 4~6개 짜리로 한정.
- 분기 트리 깊이는 입력 길이에 비례. 박자 10 이하의 예제로 한정해 트리가 임베드 폭 안에 들어오게.

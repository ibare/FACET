# RSA 핵심 추출

## 1. 반드시 다뤄야 할 본질

학습자가 빠뜨리면 시각화가 RSA 를 거짓으로 보여주게 되는 것.

1. **두 소수로부터의 출생 — 키 짝은 합성수 n 에서 태어난다.** 두 소수 p, q 가 먼저 자리에 등장하고, 그것이 곱해져 모듈러스 n 을 낳고, 그 위에서 공개 지수 e 와 비밀 지수 d 가 차례로 결정되는 출생의 순서가 시간축 위에 보여야 한다. 키 짝이 갑자기 두 박스로 등장하면 "왜 이 둘이 짝인가" 의 근거가 사라진다.
2. **공개·비밀의 분포 — 누가 무엇을 가지는가.** (n,e) 는 모두에게 열린 영역에, (p,q,d) 는 한 사람의 닫힌 영역에 배치되어, 같은 부모(소수 두 개) 에서 났음에도 둘이 다른 권한 영역으로 나뉘는 사실이 한 화면에 동시에 보여야 한다. 두 키가 같은 공간에 그려지면 "공개" 라는 행위 자체가 시각에서 사라진다.
3. **한 방향으로만 굴러가는 잠금·풀림 사건.** 평문이 공개키로 잠기면 암호문이 되고, 같은 사건은 오직 비밀키로만 풀려 평문으로 돌아오며, 잠그는 데 쓴 그 공개키로는 절대 풀리지 않는다는 비대칭이 두 다른 운동으로 보여야 한다. 두 변환이 거울처럼 대칭으로 보이면 RSA 가 단순 가역 함수처럼 거짓 표상된다.
4. **인수분해의 무거움 — n 만 보고는 d 에 닿지 못한다.** 외부 관찰자에게는 공개된 n 이 보이지만, 그것을 다시 p 와 q 로 쪼개는 길이 사실상 막혀 있고, 따라서 d 에도 닿지 못한다는 막힘이 시각의 사건으로 그려져야 한다. "어렵다" 가 텍스트 주석으로만 처리되면 일방향성의 본질이 빠진다.
5. **되돌아옴의 짝 — 같은 메시지가 한 짝의 두 키를 거쳐 자기 자신으로 귀환.** 평문 m 이 공개키로 한 번, 비밀키로 한 번 — 정확히 두 번의 변환을 거쳐 원래 m 으로 돌아온다는 닫힌 운동이 한 흐름으로 추적되어야 한다. 두 변환이 분리된 도식으로 흩어지면 "같은 짝" 의 의미가 무너진다.

## 2. 강화해야 할 요소

표준 레퍼런스가 약하게 다루는 것을 이 시각화의 차별점으로 끌어올린다.

1. **두 키의 시각적 비대칭 — 거울이 아니라 다른 형상.** 표준 자료는 공개키와 비밀키를 좌우 대칭의 같은 모양으로 그려 둘이 "다른 일을 한다" 는 사실을 잃는다. 이 시각화에서는 두 키가 형태·운동·접근 권한이 모두 달라, 모양만 봐도 "이쪽은 잠그는 손, 저쪽은 푸는 손" 이라는 역할 비대칭이 즉시 인지된다.
2. **키 생성의 운동성 — 정적 표가 아닌 출생 시퀀스.** p, q → n → φ(n) → e → d 가 정적 칸 다섯 개로 동시에 나타나는 것이 아니라, 시간축 위에 한 단계씩 차례로 태어나는 운동으로 보인다. 각 객체의 등장이 이전 객체들에 의존한다는 인과 사슬이 시간차 그 자체로 강제된다.
3. **곱셈의 가벼움과 인수분해의 무거움 — 두 화살표의 무게 차이.** p·q → n 의 화살표는 가볍고 빠르게, n → (p,q) 의 화살표는 무겁고 막히는 운동으로 그려져, 같은 두 점을 잇는 두 방향이 형식적 역방향이 아니라 비용이 비대칭인 두 사건임이 운동의 무게로 즉시 보인다.
4. **잠금에 쓴 키와 풀림에 쓴 키가 다르다는 한 프레임의 사건성.** 평문 m → 암호문 c → 평문 m 의 흐름 안에서, 첫 번째 변환과 두 번째 변환에 사용되는 키가 명백히 다른 객체임이 같은 프레임에 동시에 보여야 한다. 두 줄 수식으로만 분리되어 있으면 "한 짝의 두 다른 사건" 이라는 핵심이 흩어진다.
5. **권한 영역의 시각적 분리 — 공개 마당과 비밀 방.** 누구나 접근 가능한 공개 영역과 한 사람만 접근 가능한 비밀 영역이 화면 위에서 다른 자리로 분리되고, 어떤 객체가 어느 영역에 사는가가 위치만으로 즉시 식별된다. 객체의 가시성이 데이터 라벨이 아니라 공간 분포로 강제된다.

## 3. 공백 활용 후보

레퍼런스의 공백 중 흥미로우나 베이스에는 채택하지 않는 것. 미래 창의 사이클의 출발점.

1. **디지털 서명 모드 — 짝을 거꾸로 쓰면 인증이 된다.** 같은 키 짝을 비밀키로 먼저 잠그고 공개키로 풀면 메시지의 출처가 증명되는 대칭 사건. 암호화 모드와 서명 모드가 같은 짝의 두 다른 운동임이 한 화면에서 토글로 비교되는 모드.
2. **OAEP/PSS 패딩의 자리.** 평문이 암호화 전에 패딩되어 들어가고 복호화 후에 벗겨지는 별도 사건이 시각의 한 단계로 추가되어, "교과서 RSA" 와 "실제 RSA" 의 차이가 운동으로 드러나는 모드.
3. **키 길이별 비용의 호흡.** 작은 키와 큰 키에서 잠금·풀림 운동의 길이가 다르고, 인수분해 시도 화살표의 막힘 정도가 더 깊어지는 비교 모드. 비대칭의 강도가 키 길이라는 정책 변수에 어떻게 반응하는지가 운동 비율로 보인다.
4. **측면 채널 — 시간·전력의 누설 사건.** 비밀키 연산이 외부에서 관찰 가능한 부수적 신호(연산 시간·전력 패턴) 로 새어 나가는 사건이 별도 트랙으로 그려져, 수학적 비대칭이 무너지지 않아도 구현이 새어 나갈 수 있다는 사실의 시각화.
5. **왜 되돌아오는가의 시각적 증명 — 오일러 정리의 회전.** m^(ed) ≡ m mod n 이 성립하는 이유가 모듈러 시계 위의 회전 운동으로 보여, 수식 증명에 위탁되던 정확성이 그림의 사건으로 진입하는 모드.
6. **인수분해 공격자의 시점 — 외부 관찰자의 시도와 좌절.** 공개된 n 만 들고 p, q 를 찾으려는 공격자의 시도 시퀀스가 별도 시점으로 추가되어, 같은 시각화가 "정당한 사용자의 운동" 과 "공격자의 막힌 운동" 두 시점에서 동시에 굴러가는 비교 모드.
7. **commutative 자물쇠 — 이중 자물쇠로 키 교환을 분리.** Wikibooks 의 이중자물쇠 비유를 별도 트랙으로 분리해, RSA 직관과 키 교환(Diffie-Hellman) 직관이 같은 메타포 안에서 다른 사건임을 학습자가 직접 발견하는 비교 모드.

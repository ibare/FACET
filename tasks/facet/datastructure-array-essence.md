# 배열 시각화 핵심 추출

> 입력: datastructure-array-classification.md (type=dataStructure, 진행 모델=입력 반응형) / datastructure-array-references.md (R1~R10, 공통 요소 8종 + 공백 10종). 베이스는 표준 강화 방향이며, 창의 메타포는 별도 사이클로 미룬다.

## 1. 반드시 다뤄야 할 본질

베이스 시각화에서 빠지면 "배열" 이라는 명제 자체가 거짓이 되는 핵에 해당하는 항목들이다.

1. **연속된 가로 셀 띠와 0-기반 인덱스 라벨** — 같은 너비의 직사각형 칸이 빈틈 없이 좌→우로 늘어서 있고, 각 칸 위/아래에 0..N-1 의 자리 번호가 항상 함께 보인다. "옆자리끼리 붙어 있다 + 번호로 호명한다" 는 contiguity + indexing 이 한눈에 자명해야 한다.
2. **인덱스 → 셀 직접 점프 (O(1) 임의 접근)** — get(i) / write(i, v) 시 화살표·포인터가 중간 셀들을 건드리지 않고 곧장 i 번 칸으로 이동하고 그 칸의 색이 변한다. 임의 접근이 "옆 칸을 거치지 않는 한 걸음" 이라는 사실이 운동으로 보여야 한다.
3. **삽입/삭제의 시프트 파동 (O(N))** — insert(i, v) / remove(i) 시 i 이후의 모든 칸이 도미노처럼 한 자리씩 우/좌로 밀려나는 운동이 보인다. "가운데를 끼우면 옆 사람이 다 밀려난다" 가 시간 위의 사건으로 발생해야 한다.
4. **size 와 capacity 의 색 분리, 그리고 resize 사건** — 점유 칸은 채색, 예비 칸은 회색으로 같은 띠 위에 동시 표시된다. capacity 가 가득 차 push 가 일어나면 두 배 길이의 새 띠로 모든 항목이 통째로 옮겨가고 헌 띠가 사라진다. 평소 빠른 push 와 가끔 한 번의 비싼 이사가 같은 화면에서 대비되어야 한다.
5. **입력 반응형 시간축** — 자율 진행 없음. 학습자가 누른 연산 (read / write / insert / delete / push / resize) 한 번이 셀 상태 변화 한 묶음과 1:1 로 대응한다. 단계 / 재생 / 되감기 컨트롤로 한 연산 = 한 사건이라는 약속이 유지되어야 한다.

## 2. 강화해야 할 요소

레퍼런스가 약하게 다루는 영역 중, 베이스에 포함해 표준을 끌어올릴 수 있는 것들이다 (references 공백 1·2·3·5·7 활용).

1. **인덱스 점프의 산술적 근거를 셀 위에 노출** — get(i) 시 "base + i × stride" 또는 "시작 + i" 라는 오프셋 계산을 셀 띠 아래/위에 짧은 보조 라벨로 동적으로 표시한다. 학생이 O(1) 을 마법이 아니라 "주소 산술 한 번" 으로 보게 한다 (공백 2 정면 대응).
2. **임의 접근 vs 순차 검색의 운동 대비** — get(i) 의 직접 점프와 search(v) 의 셀-한-칸씩 스캔이 같은 화면·같은 띠 위에서 같은 배율로 비교된다. 한쪽은 한 걸음, 다른 쪽은 N 걸음이라는 사실이 운동의 길이 차로 즉시 체감되어야 한다 (공백 7 + R4 의 강의 퍼포먼스를 도구화).
3. **시프트 파동의 누적 비용 가시화** — insert/remove 한 번마다 밀려난 셀 수를 보조 막대 / 카운터로 누적해 표시한다. "한 번은 N 걸음, 여러 번이면 누적 N²" 이 셀 운동 옆에서 정량으로 같이 자란다 (공백 3 정면 대응).
4. **resize 사건의 단계 운동화** — 더블링이 단일 컷이 아니라 (1) 새 띠 등장 → (2) 항목들의 묶음 복사 운동 → (3) 헌 띠 소멸 의 다단계로 풀린다. capacity 가 두 배로 늘어나는 그 순간이 "이사 운동" 으로 시간을 차지하며, amortized 의 직관 (드물게 한 번 비싸다) 이 길이로 보인다 (공백 4 정면 대응).
5. **외부 입력 시간 × 내부 셀 시간의 동시 강조** — 학습자의 연산 호출 이력 (입력 반응형 모델의 외부 시간) 과 셀 띠 위의 색·위치 변화 (내부 시간) 가 한 화면에서 직교 보조 트랙으로 같이 흐른다. 어떤 입력이 어떤 셀 사건을 낳는지 매핑이 한눈에 보여야 한다 (공백 5 정면 대응 — 입력 반응형 모델의 본질).

## 3. 공백 활용 후보

흥미롭지만 베이스 (표준 강화) 에는 채택하지 않는다. 미래 창의 사이클의 출발점이다.

- **연속 메모리의 물리성 강조** (공백 1) — 셀 사이 균등 간격을 버리고 칸들을 바이트 단위로 물리적으로 붙여 그리고, 띠 옆에 다른 변수의 영역이 시작되는 압박감을 시각화한다. 시스템적 메타포 사이클에서 살릴 가치.
- **빈 셀 vs 미할당 영역의 명시적 분리** (공백 6) — 회색 셀이 "할당됐지만 비었다" 인지 "아예 없다" 인지를 두 가지 시각 토큰으로 구분해 그린다. 메모리 의미를 깊게 가는 창의 변형용.
- **압축 규칙의 정서적 사건화** (공백 8) — "가운데에 빈 칸이 못 생긴다" 는 사실을 셀들이 자석처럼 저절로 모이는 운동으로 의인화. 정서적 메타포 사이클용.
- **다차원과 1차원의 펼침 다리** (공백 9) — 1D 배열 띠를 2D 격자로 접고 다시 펴는 토글 운동으로 row-major / column-major 의 stride 를 한 화면에서 잇는다. 다차원 확장 사이클용.
- **메타포 통합 토글** (공백 10) — 같은 배열 상태 위에 사물함 / 메모리 격자 / 스프레드시트 / NumPy 블록 비유를 토글로 갈아끼운다. "같은 인덱스 + 연속" 사실을 비유로 회전시키는 메타-시각화 사이클용.

# 컨텍스트 스위칭 분류

## 1. 대상 정보

- 이름: 컨텍스트 스위칭 (Context Switching)
- 분야: 운영체제 / 프로세스와 스레드
- 핵심 사건: 현재 실행 흐름의 컨텍스트(레지스터, 프로그램 카운터 등) 저장 → 다른 실행 흐름의 컨텍스트 복원 → 실행 재개
- 범위: 한 실행 흐름에서 다른 실행 흐름으로의 "저장-전환-복원" 사건 그 자체
- 범위 밖: 스케줄링 알고리즘 선택 정책, 동기화 원시(뮤텍스/세마포어), 메모리 관리 / 가상 주소 변환

## 2. type

systemBehavior

## 3. 진행 모델

시간 진행형

## 4. 핵심 학습 질문

CPU 가 한 실행 흐름의 상태를 어떻게 저장하고 다른 실행 흐름의 상태를 복원해야, 두 흐름이 각자 멈춘 지점부터 정확히 이어서 실행될 수 있는가?

### R1. CP-Algorithms — Z-function

- 출처: cp-algorithms.com (커뮤니티 표준 레퍼런스, e-maxx.ru 번역본)
- URL: https://cp-algorithms.com/string/z-function.html
- 한 줄 정체성: 경쟁 프로그래밍계의 사실상 표준 텍스트북. Z-함수의 정의/구현/응용을 가장 정제된 형태로 서술.
- 시각 객체: 거의 없음. `[l, r)` 구간을 텍스트로만 묘사. 예시는 `"aaaaa" → [0,4,3,2,1]` 같은 수치 나열.
- 시간 표현: 단계 없음. 결과 배열만 보여줌.
- 메타포: "rightmost segment match" — 가장 오른쪽 매치 구간을 유지한다는 개념적 표현.
- 강조: O(n) 증명, l/r 갱신 조건의 정확성.
- 인상적 디테일: 코드 한 페이지에 응용 6가지(매칭/주기/접두사 카운트 등)를 결합. 시각화는 약함.
- 약점: 그림이 거의 없어 초심자가 l/r 의 의미를 시각적으로 잡기 어렵다.

### R2. Carl Kingsford CMU 강의노트 — Z-algorithm (Gusfield Ch.2)

- 출처: CMU 15-451/Bioinfo lecture
- URL: https://www.cs.cmu.edu/~ckingsf/bioinfo-lectures/zalg.pdf
- 한 줄 정체성: 학계 표준. Gusfield 책 2장을 그대로 옮긴 강의 노트로, 케이스 1/2a/2b 분기를 정식 도식으로 보여준다.
- 시각 객체: 문자열 가로 트랙 + 위에 곡선 화살표(매칭된 prefix ↔ Z-box 영역) + l, r, k=i-l 라벨.
- 시간 표현: 정적. 케이스별로 그림이 따로 그려짐.
- 메타포: "거울(mirror) 위치 k" — Z-box 안의 i 위치를 prefix 의 k 위치로 비추어 본다.
- 강조: 케이스 2 의 `Z[k] < r-i+1`, `Z[k] > r-i+1`, `Z[k] = r-i+1` 세 분기.
- 인상적 디테일: 같은 색의 호(arc) 두 개로 "이 두 영역이 같다"는 invariant 를 직관적으로 전달.
- 약점: 정적 PDF. 단계가 진행되는 느낌이 없어, 학습자가 머릿속에서 시간축을 따로 상상해야 한다.

## 3. 공통 요소

거의 모든 레퍼런스가 공유하는 시각 언어:

1. **문자 가로 트랙** — 문자열을 셀(또는 모노스페이스 글자) 한 줄로 펼친다. 위쪽에 인덱스 i 라벨.
2. **Z 배열 동행 표시** — 문자 트랙 바로 아래에 Z[i] 값을 같은 폭의 셀로 정렬. 막대그래프나 숫자 셀 두 형태.
3. **현재 위치 i 의 강조** — 색칠된 셀, 화살표, 또는 굵은 테두리.
4. **l, r 의 박스/라벨** — Z-box 를 사각형 테두리 또는 위/아래 라벨로 표시.
5. **두 영역의 동일성** — prefix `S[0..r-l]` 와 Z-box `S[l..r]` 를 같은 색/패턴으로 칠해 "이 둘은 매치"라는 invariant 를 시각화.
6. **케이스 분기 그림** — `i > r` (재출발) vs `i ≤ r` (재활용) 두 다이어그램.
7. **단계 진행 인터랙션** — Next 버튼 / 자동 재생 슬라이더.
8. **코드 동기화** — 의사코드 또는 실제 코드의 현재 라인 하이라이트(상위 도구에서).

학습자에게 형성된 기대:
- 좌→우로 시간이 흐른다는 가로 진행 메타포.
- "박스가 미리 계산한 영역" 이라는 의미가 박스 형태로 시각화된다.
- 거울 위치 k = i - l 의 재활용을 같은 색으로 묶어 보여준다.
- r 이 단조 증가한다는 사실이 시각적으로 드러나야 한다(박스가 절대 줄지 않는다).

## 4. 공백

기존 레퍼런스에서 약하거나 비어있는 영역:

1. **k = i - l 거울 비추기 애니메이션** — 대부분 정적 그림. i 위치를 prefix 의 k 위치로 "비춘다"는 동작이 운동(motion)으로 표현된 사례가 거의 없다. R3 정도가 케이스 분기 색만 보여줄 뿐.
2. **Z[k] vs r - i + 1 의 비교 분기** — Case 2a/2b/2c 의 미세 분기를 시각적으로 구분한 자료가 매우 드물다(R2 강의 PDF 만 정적으로). 인터랙티브 도구는 거의 모두 Case 1/2 두 단계로만 압축.
3. **r 의 단조성 시각화** — r 이 절대 줄지 않음을 시각적으로 강조한 자료가 없다. 박스가 "오른쪽으로만 이동/확장"하는 모션을 명시적으로 보여주는 사례 부재.
4. **건너뛴 비교 vs 실제 비교의 색 구분** — 비교 건너뛰기를 시각적으로 표현한 사례가 R3 외에는 거의 없다. "이 부분은 비교 안 했어요"라는 절약 효과를 색/투명도로 강조하면 O(n) 의 직관이 살아난다.
5. **누적 비교 카운터** — R4 가 일부 보여주지만 표준화되지 않음. Naive 와의 비교에서 O(n) 우월성을 수치로 체감시키는 메트릭이 거의 없다.
6. **패턴 매칭 응용 단계** — Z 배열 자체 시각화에 집중하다 보니 `P$T` 결합 후 매칭 결과를 같은 화면에서 이어 보여주는 사례가 적다.
7. **Z-box 의 "기억 장치" 메타포 시각화** — Z-box 가 "지금까지 가장 오른쪽까지 도달한 매치"라는 메모리 역할을 시각적 단서(예: 박스의 잔상, 이동 궤적)로 표현한 사례 없음.

## 5. 후속 작업 가이드

전략 = A타입(공통 강화) 이므로 공백을 채우기보다 **공통 요소를 가장 잘 다듬는 방향**.

### 반드시 채택할 공통 요소 (베이스라인)

- 문자 가로 트랙 + Z 배열 동행 막대/셀 (R3, R5, R8 표준).
- l, r, i 포인터 라벨과 Z-box 사각형 (R2, R3 의 강점).
- prefix 영역과 Z-box 영역의 동일 색상 매핑(R2, R6 의 invariant 표현).
- 단계 진행 버튼 + 자동 재생.
- 코드 라인 동기화 하이라이트(R5).

### 다듬을 여지 (공백 중 채울 만한 것)

1. **k = i - l 거울선** — i 가 Z-box 안에 들어왔을 때 prefix 의 k 위치로 곡선 호(arc) 또는 점선을 그어 "이 두 위치가 거울"임을 운동으로 표현. R2 의 정적 호를 애니메이션화.
2. **건너뛴 비교 영역의 시각 절약 표시** — Z[k] 길이만큼 "이미 같음을 안다"는 영역을 다른 색/투명도로 미리 칠한 뒤, 그 너머만 실제 비교(예: 빨간 깜빡임)로 표현. O(n) 의 직관이 살아난다.
3. **r 의 단조 증가 강조** — Z-box 가 줄어들지 않는다는 사실을 우측 r 에 잔상(트레일) 또는 "r 최댓값" 라벨로 명시. r 이 갱신될 때마다 박스 우측 경계가 우측으로만 이동.
4. **Case 2a/2b/2c 미세 분기** — Z[k] 와 `r - i + 1` 의 대소를 작은 비교 위젯으로 보여주고, 결과에 따라 박스가 어떻게 바뀌는지 분기 애니메이션.
5. **누적 비교 카운터** — 화면 한 켠에 "총 비교 횟수" 카운터를 두고 Naive 의 예상치(n*m)와 대비.
6. **패턴 매칭 응용 모드** — `P$T` 결합 모드를 토글로 제공해, Z 값이 |P| 와 같은 위치를 매칭 hit 로 표시.

### 절대 빼면 안 될 본질

- l/r 과 i 의 동시 표시.
- Z-box 가 사각형으로 명시될 것(라벨만으로는 부족).
- prefix 와 Z-box 의 색상 매핑.
- 케이스 분기(i > r vs i ≤ r) 의 시각적 구분.

## 6. 대상 개념 요약

> Z-알고리즘은 문자열 S 의 각 위치 i 에서 "S 의 처음과 S[i..] 가 몇 글자나 같은지" 를 한 번의 좌→우 스캔으로 모두 구한다(시간 O(n)). 이때 직전까지 발견한 "전체 prefix 와 일치하는 가장 오른쪽 구간" [l, r] 을 박스 형태로 기억하고, 새 위치 i 가 박스 안이면 거울 위치 k = i - l 의 결과 Z[k] 를 빌려와 비교를 건너뛴다. r 이 절대 줄지 않기 때문에 전체 비교 횟수는 선형이다. 이 Z 배열로 패턴 P 를 텍스트 T 에서 찾을 때는 `P + 구분자 + T` 의 Z 배열에서 |P| 와 같은 값이 나오는 위치를 보면 된다.

학습자가 시각화와 함께 읽을 짧은 캡션:
- "각 위치에서 처음과 얼마나 겹치나?" (Z[i] 정의)
- "박스 [l, r] 은 지금까지 본 가장 멀리 뻗은 매치." (Z-box 의 역할)
- "박스 안이면 거울 위치의 답을 빌려 비교를 건너뛴다." (재활용 원리)
- "박스 오른쪽 끝 r 은 절대 왼쪽으로 가지 않는다 — 그래서 전체가 O(n)." (단조성)

용어 풀어쓰기:
- prefix → 접두사(문자열의 맨 앞부터 일정 길이까지의 조각)
- Z-box → "지금까지 발견한 가장 오른쪽까지 닿는 접두사 매치 구간"
- 거울 위치 k = i - l → "박스 안 i 위치를 박스 시작점 l 기준으로 접두사 쪽에 비춘 위치"
- 단조 증가 → "한 번 늘어난 값은 다시 줄지 않음"

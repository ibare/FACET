# 2D 행렬 변환 핵심 추출

> 입력 — 분류: type=transformation, 진행 모델=입력 반응형, 도메인=graphics, 핵심 학습 질문="2×2 행렬의 두 열이 어떻게 평면의 기저 벡터를 새 위치로 매핑하며, 평면의 모든 점이 그 매핑을 선형 결합으로 따라가는가". 레퍼런스 10건 (3Blue1Brown, Nicky Case Magnificent 2D Matrix, Immersive Linear Algebra, T. Kern, GeoGebra Emma, Setosa Eigen, LearnOpenGL, MDN Canvas/CSS/SVG, 공돌이 선형변환·행렬식) 의 공통 시각 언어와 9개 공백을 종합. 베이스의 행렬 표현은 **2×2 (동차좌표 미사용)** 으로 고정하고, 표준 4종 변환은 **회전·스케일·전단·반사** 로 한정한다 — 이동·어파인·투영·합성·비가환성·고유벡터는 공백 후보로 미룬다. 진행 모델은 **입력 반응형** 이며 보간 애니메이션은 베이스에 두지 않는다 (옵션 부속 가능). 격자 운동만이 아니라 **사용자 보조 점 1개** 의 운동도 함께 추적한다.

## 1. 반드시 다뤄야 할 본질

학습자가 빠뜨리면 시각화가 2D 행렬 변환을 거짓으로 보여주게 되는 것.

1. **두 기저 벡터의 도착지가 곧 행렬의 두 열.** 변환된 i-hat 의 좌표가 행렬 첫째 열, 변환된 j-hat 의 좌표가 둘째 열이라는 사실이 한 화면에서 동시에 보인다. 행렬 셀의 숫자와 두 기저 벡터의 끝점이 같은 색·같은 위치 정보로 묶여, "행렬을 본다 = 두 화살표의 도착지를 본다" 가 즉시 성립해야 한다. 이 동기가 무너지면 2×2 행렬은 그저 네 숫자의 표로 전락한다.
2. **격자가 휘어도 직선은 직선, 원점은 그 자리 — 선형성의 시각 조건.** 변환 후 격자선이 곡선이 되거나 원점이 움직이면 그 변환은 선형이 아니다. 격자가 새 모양으로 바뀌더라도 모든 가로·세로선이 여전히 직선이고 등간격이며, 좌표계의 중심이 한 점에 박혀 있다는 사실이 시각으로 보장되어야 한다. 학습자가 "왜 이런 형태만 가능한가" 의 답을 그림에서 직접 읽을 수 있어야 한다.
3. **평면의 모든 점은 두 기저 벡터의 선형 결합으로 따라간다.** 임의의 점 p = u·i-hat + v·j-hat 이 변환 후에도 같은 (u, v) 계수로 새 기저의 결합 p' = u·i-hat' + v·j-hat' 이 된다는 사실이 보조 점의 운동으로 직접 증명되어야 한다. 격자 전체의 휨이 곧 한 점의 운동을 결정한다는 사실이, 보조 점이 격자와 동기화되어 움직이는 운동으로 운반된다.
4. **단위 정사각형 → 단위 평행사변형, 면적이 곧 |determinant|.** i-hat 과 j-hat 이 만드는 한 변 1 의 정사각형이 변환 후 평행사변형으로 바뀌고, 그 면적이 행렬식의 절댓값과 같다는 사실이 시각으로 즉시 읽혀야 한다. 부호가 음수일 때는 평행사변형이 좌우로 뒤집혀 (반사) 표시되어 "행렬식의 부호 = 방향성의 보존 여부" 가 색·면적의 사건으로 분명해진다. 면적이 0 일 때는 평행사변형이 한 직선으로 무너진다는 시각 사건도 같은 어휘 안에서 자연스럽게 발화한다.
5. **회전·스케일·전단·반사 4종이 모두 같은 2×2 행렬 형식의 특수 사례.** 네 가지 변환이 별개의 도구가 아니라 같은 행렬 골격에서 셀 네 칸이 어떤 패턴으로 채워지느냐의 차이일 뿐이라는 통일성이 보여야 한다. 프리셋 토글이 셀의 네 숫자 패턴 (회전 = `[cosθ -sinθ; sinθ cosθ]`, 스케일 = `[s 0; 0 t]`, 전단 = `[1 k; 0 1]`, 반사 = `[1 0; 0 -1]` 등) 으로 직접 환원되어, 학습자가 "행렬 한 형식이 네 가지 운동을 모두 포섭" 한다는 사실을 시각으로 외운다.

## 2. 강화해야 할 요소

표준 레퍼런스가 약하게 다루는 것을 이 시각화의 차별점으로 끌어올린다. 레퍼런스 보고서의 9개 공백 중 베이스에 적합한 5개를 선별한다.

1. **셀 한 칸 ↔ 한 좌표의 미시 동기.** 행렬의 a / b / c / d 한 칸을 잡고 흔들면, 정확히 i-hat 또는 j-hat 의 한 좌표 (x 또는 y) 만 변하고 나머지는 그대로다. "한 칸 = 한 좌표" 의 1:1 대응이 셀 색·화살표 색의 동기와 함께 시각 사건으로 발화된다. 표준 레퍼런스 (R2/R3/R5) 는 셀을 만지면 결과 격자 전체가 한꺼번에 바뀌는 식이라 이 미시적 매핑이 묻히기 쉽다. 베이스에서는 어느 셀이 어느 화살표의 어느 좌표를 잡고 있는지가 시각으로 항상 표시된다.
2. **변환 전 격자(잔상) 와 변환 후 격자(결과) 의 동시 겹침.** 항등 상태의 옅은 격자가 배경에 항상 깔리고, 그 위에 변환된 진한 격자가 겹쳐 보인다. "어디에서 어디로" 가 한 컷에 동시에 읽히는 시각이 베이스의 기본 어휘다. 보간 애니메이션 없이도 두 격자의 차이만으로 변환의 운동 방향이 즉시 인지된다. R1 의 영상이 잔상으로 만 처리하던 효과를 정적 입력 반응형 안으로 끌어들인다.
3. **보조 점 한 개의 동기 운동 — 격자와 한 점이 함께 따라간다.** 사용자가 평면 위에 한 점을 찍어 두면, 행렬을 흔드는 동안 그 점이 새 좌표 (u, v) 의 새 기저 결합으로 즉시 이동한다. 점의 (u, v) 좌표 라벨이 변환 동안에도 유지되고, 위치만이 새 기저에 맞춰 바뀐다는 사실이 라벨과 함께 보인다. 격자 전체의 휨과 단일 점의 운동이 같은 운동 법칙임이 시각으로 증명되며, R6 의 단일 점 시각과 R1·R4 의 격자 시각이 한 화면에서 합류한다.
4. **표준 4종 프리셋 토글로 셀 패턴 학습.** 회전·스케일·전단·반사 네 버튼이 항상 노출되어, 누르는 순간 네 셀이 해당 패턴으로 채워지고 격자가 즉시 그 변환의 결과로 바뀐다. "각 변환은 셀 어디에 어떤 모양의 숫자가 들어가는가" 의 패턴이 토글 한 번으로 외워지도록, 셀 패턴과 격자 모양이 1:1 로 연결된다. R2/R5 의 프리셋 버튼이 변환 결과만 강조하던 한계를 넘어, 셀 패턴 자체의 학습으로 끌어올린다.
5. **|determinant| 의 면적·부호 동시 사건성.** 단위 평행사변형이 슬라이더를 흔드는 동안 면적이 변하고, 그 면적의 숫자가 도형 옆에 항상 따라다니며 변한다. 부호가 음수로 진입하는 순간 평행사변형이 색·채움 패턴으로 시각적으로 뒤집혀 (예: 색조 반전 / 빗금 방향 반전) "방향이 뒤집혔다" 가 사건으로 발화된다. 면적이 0 으로 무너지는 순간도 같은 어휘 안에서 평행사변형이 직선으로 압축되는 사건으로 보인다. R10 의 정적 그림이 다루지 못한 운동성을 베이스가 흡수한다.

## 3. 공백 활용 후보

레퍼런스의 공백 중 흥미로우나 베이스에는 채택하지 않는 것. 미래 창의 사이클의 출발점.

1. **합성 (M2·M1) 의 단계별 분해 시각.** 두 행렬을 한 화면에서 동시에 편집하며, 격자가 먼저 M1 으로 휘었다가 그 위에 M2 가 다시 적용되는 두 단계 운동을 분해해 보여주는 모드. 합성이 곧 행렬 곱이라는 사실이 두 단계의 시각 사건으로 운반된다.
2. **비가환성 (M1·M2 ≠ M2·M1) 좌우 비교.** 같은 두 행렬을 순서만 바꾸어 좌·우 두 격자에 동시에 적용해 결과의 차이를 한 컷에 보여주는 비교 시각. "쌓는 순서가 결과를 바꾼다" 가 텍스트가 아닌 두 격자의 모양 차이로 증명된다.
3. **항등→대상 보간 애니메이션 모드.** 베이스의 입력 반응형 위에 옵션으로 얹는 부드러운 보간 운동 (R1 / R4 Morph 식). 격자가 휘는 운동의 시간 진행이 학습 효과를 더할지를 별도 사이클에서 검증.
4. **동차좌표 확장 — 이동·어파인.** 2×2 의 표현 한계 (이동 표현 불가) 를 보이고 3×3 동차좌표로 확장하는 시각적 동기. 마지막 행 (0,0,1) 이 어파인의 시각적 사실임을 운동으로 도입.
5. **투영 변환 / 비어파인.** 마지막 행을 흔들면 어파인이 깨지고 격자가 사다리꼴로 휘는 사건. 어파인 vs 투영의 경계가 시각 사건으로 발화되는 별도 사이클.
6. **고유벡터·고유값 시각.** Setosa (R6) 식 — 변환 후에도 같은 직선 위에 머무르는 점이 eigenvector 라는 사실, 복소 고유값 시 나선 궤적. 고유 분해의 직관을 본 facet 의 격자 어휘 위에 얹는 확장.
7. **CSS / Canvas / SVG 변환과의 매핑.** MDN (R8) 의 어파인 (a,b,c,d,e,f) 6 파라미터가 현재 facet 의 셀과 어떻게 대응하는지를 보여주는 웹 그래픽스 다리. 학습한 시각이 실제 웹 코드로 직접 환원되는 응용 모드.
8. **이미지·도형의 변형 응용.** 단순 격자에 더해 사진·집·텍스트를 보조 점 대신 변형 대상으로 두는 응용. R3/R5/R7 의 일상감 부여 어휘를 본 facet 위에 얹는 별도 사이클.

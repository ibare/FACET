# 조건문 (if / else if / else)

### 0. 기본 정보
- 대상 이름: 조건문 (if / else if / else)
- 도메인: control-flow
- type: concept
- 진행 모델: 입력 반응형
- 본질 한 문장: 흐르던 길이 분기점에 도착해 그 자리에서 조건의 참/거짓을 평가하고, 여러 갈래 중 단 하나의 길만 통과한 뒤 다시 한 줄로 합쳐지는 약속.

### 1. 시그니처 행동
입력값이 분기점에 닿는 순간 위에서부터 조건들이 차례로 평가되다가 처음으로 참이 된 한 가지에서 흐름이 확정되고, 나머지 가지들은 어두워진 채 닫힌다.

### 2. 시각적 은유
산업 표준 플로차트의 어휘를 그대로 채택해 정교화한다. 위에서 내려오는 한 줄의 흐름선, 분기점을 알리는 다이아몬드 모양의 평가 마름모, 두세 갈래로 갈라지는 가지선과 가지마다 붙은 참/거짓 라벨, 그리고 갈래가 끝나는 지점에서 다시 한 점으로 모이는 합류 노드. 메타포 자체는 새로 만들지 않고, 가지의 활성/비활성, 평가의 박자, 합류의 봉합을 디테일로 끌어올린다.

### 3. 결정적 순간
입력값이 위에서 흘러내려와 평가 마름모 안으로 빨려 들어가는 한 박자.
- 전: 모든 가지가 균등하게 흐릿한 대기 상태. 마름모 안 조건식은 회색 텍스트로 잠들어 있다.
- 중: 입력값이 마름모에 닿으면 식이 또렷해지고, 한 박자 동안 식이 환하게 빛나며 참 또는 거짓 한 글자로 응결된다.
- 후: 응결된 결과를 따라 한 가지선이 굵고 또렷한 색으로 켜지며 흐름이 그 길로 흘러내려간다. 동시에 나머지 가지는 회색 반투명으로 가라앉고 가지선 위에 빗금 또는 가는 빗장이 그어져 "이번에는 닫힘"이 형태로 드러난다. 켜진 길은 합류 노드에서 다른 가지들이 들어왔어야 할 자리와 만나며 한 줄로 봉합된다.

강조 장치: 평가 박자의 잠깐의 정지(타이밍), 활성 가지의 굵기·채도 상승, 비활성 가지의 빗장과 회색화, 합류 점의 작은 봉합 애니메이션.

### 4. 다른 대상과의 구별점
- 반복문(for/while): 같은 한 가지를 여러 번 되돌아 흐른다. 조건문은 되돌아오지 않고, 한 번의 통과 안에서 갈래가 갈라졌다 합쳐질 뿐이다. 반복문은 순환 화살표가 핵심이고, 조건문은 분기·합류 한 쌍이 핵심이다.
- switch/case: 단일 값과 여러 라벨을 일대다로 직접 매칭한다. 조건문은 마름모 하나에 임의의 식이 들어가고, 위에서부터 차례로 평가된다는 순서감이 형태로 드러난다. switch가 빗 모양 동시 매칭이라면, 조건문은 사슬 모양 순차 시도다.
- 함수의 입출력 분기: 함수는 호출 단위로 들어갔다 나오는 경계가 있다. 조건문은 같은 흐름선 위에서 갈래만 잠깐 갈라졌다 다시 한 줄이 된다. 경계 박스가 없다.
- 결정 트리: 한 번 갈라진 가지는 다시 만나지 않는다. 조건문은 합류 노드가 분명히 존재한다는 점이 결정적 차이다.

### 5. 달성해야 할 시각 요소

1. 평가 마름모 (분기점)
   - 설명: 흐름이 도착해 조건의 참/거짓이 결정되는 단 하나의 지점.
   - 시각 표현: 다이아몬드 도형 안에 조건식 텍스트. 평소엔 차분한 윤곽, 입력이 닿는 박자에 테두리가 환해지고 식 글자가 또렷해진다. 결과가 도출되는 순간 다이아몬드 우측에 작은 결과 칩(참/거짓)이 한 박자 떴다 사라진다.

2. 활성 가지 vs 비활성 가지
   - 설명: 이번 통과에서 단 하나의 가지만 실행되고 나머지는 동작하지 않았음을 한눈에 보이게 한다.
   - 시각 표현: 활성 가지선은 짙은 색에 굵기 2~3px, 흐름 점이 따라 내려가는 선형 애니메이션. 비활성 가지선은 회색·반투명에 빗장 무늬가 겹쳐져 "닫힘"을 능동적으로 표현. 가지에 매달린 실행 블록도 같은 규칙(또렷함/회색)으로 함께 가라앉는다.

3. 위에서부터 첫 매칭의 순서감 (else if 사슬)
   - 설명: 조건이 여러 단계일 때 위 마름모부터 차례로 시도되고, 처음 참이 된 곳에서 멈춘다는 순서가 보여야 한다.
   - 시각 표현: 마름모들을 위에서 아래로 줄지어 배치하고, 평가가 일어날 때 위 마름모부터 한 박자 간격으로 짧게 점등되며 거짓이면 다음 마름모로 작은 화살표가 이어진다. 첫 참이 나타나는 순간 그 마름모만 강하게 빛나고 아래 마름모들은 차례 자체가 박탈된 듯 처음의 흐릿함으로 굳는다.

4. 합류 노드
   - 설명: 어느 가지를 탔든 갈래가 끝나면 한 줄로 다시 모여 다음 흐름으로 이어진다.
   - 시각 표현: 가지들이 모이는 지점에 작은 원형 합류 노드. 활성 가지선이 노드에 닿는 순간 짧은 봉합 애니메이션(원이 잠깐 부풀었다 가라앉음)으로 "한 줄로 합쳐졌다"가 형태가 된다. 노드 아래로는 항상 한 줄의 두꺼운 흐름선이 이어진다.

5. 입력 반응성
   - 설명: 같은 도식·같은 코드라도 입력값이 바뀌면 통과하는 가지가 즉시 다른 가지로 바뀐다.
   - 시각 표현: 도식 상단 또는 좌측에 입력 슬라이더/토글. 사용자가 값을 흔들면 평가 마름모의 식 결과가 다시 응결되고, 활성 가지가 즉시 다른 가지로 옮겨가며 빗장과 강조가 자리를 바꾼다. 전환은 짧은 디졸브로 부드럽게 이어져 "같은 코드, 다른 길"이 한 흐름 안에서 보이게 한다.

### 6. 시각 구성
- 중심 시각: 위에서 아래로 흐르는 한 줄 흐름선과, 그 위에 놓인 평가 마름모를 중심으로 좌우로 갈라지는 두세 가지의 분기 도식. 가지마다 매달린 실행 블록(둥근 모서리 사각형)과, 가지가 다시 모이는 합류 노드.
- 보조 요소: 상단의 입력 컨트롤(작은 슬라이더 또는 두세 개의 토글), 도식 우측 좁은 띠에 자리한 짧은 개념 설명 영역, 하단의 참고 레퍼런스 칩.
- 요소 간 관계: 입력 컨트롤은 평가 마름모의 식과 직접 연결되어, 컨트롤 변경이 마름모의 결과를 다시 응결시키고 활성/비활성 가지 상태를 즉시 갱신한다. 합류 노드는 모든 가지의 종착이며, 그 아래 한 줄의 흐름선이 도식의 마무리를 책임진다.

### 7. 레이아웃
임베드 폭 기본 680px(640~720px 허용), 확장 시 880px. 높이는 약 520~600px.

- 상단 띠 (높이 약 64px): 좌측에 대상 이름과 한 줄 정체성. 우측에 입력 컨트롤(슬라이더 1개 또는 토글 2~3개).
- 중앙 본문 영역 (높이 약 360~420px):
  - 좌측 약 70% 폭: 분기 도식. 위→아래 흐름선, 평가 마름모 1개(else if 사슬 변형 시 2~3개), 좌우로 갈라지는 가지와 매달린 실행 블록, 하단 합류 노드. 흐름선의 좌우 여백을 충분히 두어 가지가 좁아 보이지 않게 한다.
  - 우측 약 30% 폭: 좁은 사이드 띠. 상단 2/3는 짧은 개념 설명(2~4문장), 하단 1/3은 학습자가 무엇을 보아야 하는지 안내.
- 하단 띠 (높이 약 60px): 참고 레퍼런스 칩 2~4개를 가로로 나열. 각 칩은 이름 + 작은 외부 링크 표시.

확장 폭 880px에서는 도식의 좌우 여백과 가지 사이의 간격이 넓어지고, 사이드 띠의 텍스트 행 길이가 자연스러워진다. 좁은 폭에서도 좌우 4~6 패널 분할은 사용하지 않는다.

### 8. 진행과 인터랙션
진행 모델은 입력 반응형이다. 시간 축이 아니라 입력값이 시각의 갱신을 주도한다.

- 입력 → 시각 갱신 매핑:
  - 슬라이더 또는 숫자 토글의 값 변경 → 평가 마름모 안 식이 새로 평가되고, 결과 칩이 한 박자 깜빡인 뒤 활성 가지가 갱신된다.
  - 활성 가지 갱신 시 비활성 가지에는 빗장 무늬와 회색화가 적용되고, 매달린 실행 블록도 같은 규칙으로 함께 변한다.
  - else if 사슬 변형의 경우 입력 변경 시 위 마름모부터 차례로 점등되는 짧은 시퀀스가 반복되고, 첫 참이 나타나는 마름모에서 시퀀스가 멈춘다.
  - 합류 노드는 활성 흐름선이 닿을 때마다 짧은 봉합 애니메이션을 재생한다.
- 추가 인터랙션:
  - 가지선 또는 실행 블록에 마우스를 올리면 그 가지의 조건 라벨(참/거짓 또는 식의 한국어 풀이)이 작은 라벨로 따라 붙는다.
  - 마름모를 누르면 식 텍스트가 잠시 확대되어 읽기 편한 크기로 보인다.
- 자동 재생은 없다. 사용자가 입력을 흔들지 않으면 도식은 직전 상태에 머문다.

### 9. 초기 상태와 파라미터
- 초기 입력값: 평가 결과가 첫 번째 가지(참)로 분기되는 값에서 시작. 학습자가 처음 보았을 때 "참 → 위쪽 가지가 환하게 켜짐 / 아래 가지는 빗장으로 닫힘 / 합류 노드에서 한 줄로 합쳐짐"이 한 화면에 모두 보이게 한다.
- 가지 수: 기본 변형은 두 갈래(if/else). 사이드 띠 또는 컨트롤 영역의 작은 토글로 세 갈래(if/else if/else) 변형으로 전환 가능. 4개 이상은 도식이 좁아지므로 허용하지 않는다.
- 색 파라미터: 활성 가지는 단일 강조색, 비활성 가지는 회색 계열. 결과 칩은 참/거짓 두 가지 색만 사용. 전체 채도는 차분하게 유지하고, 활성 가지의 채도만 한 단계 높여 시선을 끈다.
- 애니메이션 시간: 평가 박자 약 220ms, 가지 전환 디졸브 약 180ms, 합류 봉합 약 160ms. 모든 모션은 짧고 단호하게.
- 입력 컨트롤 범위: 슬라이더의 경우 0~100 정수. 토글의 경우 두세 개의 켜짐/꺼짐 조합으로 식이 평가된다.

### 10. 시각화 안 텍스트
- 개념 설명(우측 사이드 띠 상단): "조건문은 흐르던 코드가 갈림길에 도착했을 때, 지금의 값이 참인지 거짓인지를 보고 갈래 중 단 한 길만 골라 통과하는 약속이에요. 고르지 않은 길은 이번 흐름에서는 닫혀 있어 단 한 줄도 동작하지 않아요. 갈림길이 끝나면 어느 길로 갔든 모두 다시 한 줄로 모여 다음 코드로 이어집니다."
- 학습자 안내(우측 사이드 띠 하단): "위쪽 입력을 흔들어 보세요. 마름모에서 결과가 다시 응결되고, 켜지는 길과 닫히는 길이 바뀌는 모습을 보세요. 빗장이 그어진 가지는 이번에 동작하지 않은 길이에요."

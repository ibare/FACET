# 해시 테이블 핵심 추출

> 입력 — 분류: type=dataStructure, 진행 모델=입력 반응형, 핵심 학습 질문="해시 함수가 키를 슬롯에 분산시키고 충돌이 발생했을 때 어떤 해결 전략이 평균 O(1) 접근을 어떻게 유지하는가". 레퍼런스 10건 (VisuAlgo, USFCA Galles, Liang, AlgoVis.io, Kozma 쿠쿠, UCR 주차장, Chris Laux 블룸, basecs 책장, 배하람 블로그, 위키/Velog) 의 공통 시각 언어와 9개 공백을 종합. 베이스의 충돌 해결 전략은 가장 표준적이며 한국어 학습자의 시각 관습에 정착된 **분리 체이닝 (separate chaining)** 으로 고정한다 — 개방 주소법은 강화 5번에서 메모리 모양 대비로만 짧게 다룬다.

## 1. 반드시 다뤄야 할 본질

학습자가 빠뜨리면 시각화가 해시 테이블을 거짓으로 보여주게 되는 것.

1. **유한한 슬롯 배열 — 이산 인덱스 0..M-1.** 해시 테이블의 정체성은 "가로로 누운 유한한 셀 줄" 이다. 인덱스 라벨이 시각적으로 항상 보이고, 이 줄의 길이가 곧 함수의 출력 공간이라는 사실이 즉시 인지된다. 키가 무한히 다양하더라도 떨어질 자리는 이 줄 안에서만 존재해야 한다.
2. **키 → 인덱스의 결정적 매핑 — 함수가 자리를 정한다.** 어떤 키가 들어올 때 그 키가 "왜 그 자리로 갔는가" 가 임의가 아니라 함수의 출력으로 정해진다는 사실이 보여야 한다. 키가 함수 박스를 통과해 한 인덱스로 떨어지는 운동이 모든 삽입·검색·삭제의 첫 단계여야 한다.
3. **충돌은 필연적 사건 — 두 키가 같은 자리로 떨어진다.** 키 공간이 슬롯 공간보다 크므로 충돌은 예외가 아니라 정상 작동의 일부다. 같은 인덱스에 두 번째 키가 도착하는 순간이 시각적 사건으로 발화되고, 그 자리에서 무엇인가 변화가 시작되어야 한다.
4. **체이닝 — 같은 자리에 사슬을 늘여 모두를 품는다.** 충돌이 일어나면 그 슬롯에서 노드가 아래로 늘어지고, 같은 인덱스의 키들은 같은 사슬 안에서 공존한다. 슬롯 자체는 사슬의 머리 자리이고, 사슬의 길이가 곧 그 자리에 떨어진 키의 수다.
5. **검색의 두 단계 — 한 번의 점프 + 사슬 안의 비교.** 키를 찾을 때 함수가 우선 한 자리로 점프시키고, 그 자리의 사슬을 따라가며 키를 비교한다. 평균적으로 사슬이 짧다면 거의 한 번의 접근이고, 길어지면 여러 번을 비교해야 한다는 두 단계의 운동이 명시적으로 구분되어야 한다.

## 2. 강화해야 할 요소

표준 레퍼런스가 약하게 다루는 것을 이 시각화의 차별점으로 끌어올린다. 레퍼런스 보고서의 9개 공백 중 베이스에 적합한 5개를 선별한다.

1. **해시 함수의 분산 자체의 시각화.** 함수가 블랙박스로 머물지 않고, 키 공간(다양한 키들) 이 슬롯 공간으로 어떻게 흩뿌려지는가 가 한 화면에서 보인다. 같은 함수가 여러 키를 받을 때 각 키가 각자의 인덱스로 던져지는 운동이 누적되어, 분포가 균등한지 편향되었는지를 시각이 직접 운반한다. 함수는 "키를 인덱스로 바꾸는 결정적 규칙" 이라는 사실이 출력의 모양으로 증명된다.
2. **적재율 α 의 실시간 운동 — 표가 차오르는 압박감.** 키가 채워질수록 α=N/M 이 게이지·막대·수치로 실시간 차오른다. 학습자가 키를 더 넣을 때마다 표가 답답해지는 정서적 시각이 함께 발생하고, 임계값 근방에서 색·두께가 변하며 "곧 무언가 일어난다" 는 긴장이 시각으로 운반된다. 정적 곡선 그래프가 아니라 사건성 있는 게이지여야 한다.
3. **리해싱의 운동성 — 모든 키가 새 자리로 이주한다.** 임계를 넘는 순간 표 자체가 더 길게 늘어나고, 기존 사슬이 모두 풀려 모든 키가 새 함수로 다시 던져지는 일제 이주가 한 사건으로 펼쳐진다. 이전 표와 새 표가 잠시 나란히 보이고 키들이 새 인덱스로 점프하는 운동이 "해시는 표 크기에 의존" 한다는 사실을 강제 학습시킨다.
4. **사슬 길이의 누적 분포 — 평균 O(1) 의 시각적 증명.** 단일 삽입 한 번이 아니라 다수 키를 흘려 넣은 뒤의 슬롯별 사슬 길이 분포가 한눈에 보인다. 짧은 사슬이 다수, 긴 사슬이 소수인 모양이 시각으로 형성되어, "한 번이 아니라 전체 분포가 좋다" 는 평균 O(1) 의 약속이 운동의 결과로 증명된다. 한 번의 좋음/나쁨이 아닌 분포의 모양 자체가 학습 대상이다.
5. **체이닝 vs 개방 주소의 메모리 모양 대비 (보조 비교 시각).** 베이스는 체이닝이지만, 한쪽 보조 영역에서 같은 입력 시퀀스가 개방 주소법(선형 탐사) 위에서 어떻게 보일지를 짧게 대비한다 — "아래로 사슬이 늘어지는 모양" vs "옆으로 비집고 전진하는 모양". 두 전략이 같은 충돌을 다른 메모리 형태로 흡수한다는 사실이 시각적 대비 한 컷으로 운반된다. 깊은 비교는 별도 사이클로 미루고, 베이스에서는 모양의 차이만 명시한다.

## 3. 공백 활용 후보

레퍼런스의 공백 중 흥미로우나 베이스에는 채택하지 않는 것. 미래 창의 사이클의 출발점.

1. **LP / QP / DH 의 동시 비교 운동.** 같은 입력 시퀀스를 선형·제곱·이중 해싱 세 트랙 위에 동시에 흘려보내, 같은 시간에 세 분포가 어떻게 갈라지는가 를 한 화면에서 비교. 모드 토글이 아닌 동시 운동.
2. **충돌 클러스터링의 시간 누적.** 개방 주소법에서 1차 클러스터가 시간이 지남에 따라 어떻게 자라고 합쳐지는가를 색의 누적·열지도로 표현하는 운동. "처음에는 흩어졌다가 점점 뭉친다" 의 시간축.
3. **삭제와 tombstone 의 사건성.** 개방 주소법에서 빈 셀과 삭제 흔적의 차이가 검색을 어떻게 보호하는가를 운동으로 보여주는 시각. 묘비 토큰이 검색 경로의 사슬을 끊지 않는 이유가 사건으로 발화.
4. **여러 메타포의 통합 토글 — 책장·주차장·뻐꾸기 둥지·전화번호부.** 같은 해시 동작을 다섯 일상 비유로 한 화면에서 토글하며 "모두 같은 매핑" 을 체감하게 하는 메타 시각화. R5/R6/R8/R10 의 비유들이 흩어진 상태를 통합.
5. **쿠쿠 해싱 / 블룸 필터 / 완전 해시 같은 변종 트랙.** 두 테이블 사이의 변위 사슬, 비트 필드의 공간-속도 트레이드오프, 키 집합이 고정될 때의 충돌 0 보장 — 표준 해시 테이블의 외연을 확장하는 별도 사이클. 베이스의 단일 정체성을 흐리지 않기 위해 분리한다.

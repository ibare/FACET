# 배열 시각화 기획서

### 0. 기본 정보

- 대상 이름: 배열 (array)
- 도메인: 자료구조
- type: dataStructure
- 진행 모델: 입력 반응형
- 본질 한 문장: 배열은 같은 너비의 칸을 옆자리끼리 빈틈 없이 붙여 놓고 0부터 매긴 번호로 호명하게 함으로써, 번호만 알면 한 번에 도달하는 빠름과 가운데를 건드리면 옆 칸이 줄줄이 밀리는 느림을 같은 띠 위에 동시에 새겨 두는 자료구조다.

### 1. 시그니처 행동

번호로 부르면 화살표가 옆 칸을 거치지 않고 곧장 그 칸으로 한 걸음에 도달하지만, 가운데에 한 칸을 끼우거나 빼면 그 뒤의 모든 칸이 도미노처럼 한 자리씩 동시에 밀려난다.

### 2. 시각적 은유

표준 메타포인 "연속된 가로 셀 띠 + 0-기반 인덱스 라벨" 을 채택해 디테일로 강화한다. 화면 가운데에 같은 너비의 직사각형 칸들이 좌→우로 빈틈 없이 붙어 한 줄을 이룬다. 칸 사이의 세로 경계선은 한 픽셀 두께로 인접해 있어 "옆자리끼리 붙어 있다" 는 contiguity 가 형태 그 자체로 보인다. 각 칸 아래에는 0, 1, 2, … 의 자리 번호가 항상 함께 떠 있고, 띠 전체 위에는 호명을 위해 떠다니는 단일 화살표가 한 자리만 가리킨다. 띠는 둘로 색이 갈린다. 채워진 앞쪽 N 칸은 채색 상태, 그 뒤의 예비 칸은 회색 격자 무늬로 같은 띠 위에 함께 놓여 size 와 capacity 가 한 줄에 동시에 보인다.

### 3. 결정적 순간

같은 띠 위에서 두 가지 사건이 같은 배율로 잇달아 일어난다. 먼저 학습자가 인덱스 4 를 호명하면 화살표가 0, 1, 2, 3 칸을 건드리지 않고 곧장 4 번 칸으로 한 걸음에 점프하고 4 번 칸이 즉시 강조색으로 빛난다. 곧이어 인덱스 1 에 새 값을 끼우면, 1 번 칸 자리에 새 칸이 들어서고 2~5 번 자리에 있던 모든 칸이 한 자리씩 우측으로 동시에 밀려나 6 번 자리까지 차오른다.

- 전: 띠 위에 6 칸이 채워져 있고 뒤로 회색 예비 칸이 이어진다. 화살표는 띠 위 어디에도 고정되어 있지 않다.
- 중 (점프): 호명 즉시 화살표가 띠 위쪽에서 4 번 칸 위치로 호 모양 짧은 곡선을 그리며 단일 동작으로 이동하고, 도착한 4 번 칸이 한 박자 빛난다. 띠 아래 보조 라벨에 "시작 + 4" 라는 짧은 산술식이 0.4 초간 떠 있다 사라진다.
- 중 (시프트): 1 번 자리에 새 칸이 솟아오르며 2~5 번 자리의 칸들이 일제히 우측으로 한 자리씩 미끄러진다. 같은 운동이 한 프레임 차이로 칸마다 짝힘처럼 번지며 도미노 파동이 보인다. 띠 옆 작은 카운터에 "밀린 칸 4" 가 즉시 적힌다.
- 후: 띠 길이는 그대로지만 채워진 칸 수가 7 로 늘었고 회색 예비 칸이 한 자리 줄었다. 누적 카운터는 이번 사건의 숫자를 한 줄 막대로 누적해 띠 옆에 짧게 자란다.
- 강조 장치: 점프 화살표의 호 곡선 / 시프트 도미노의 한 프레임 시차 / 띠 아래 산술식 라벨 / 시프트 비용 누적 막대 / 채색 칸과 회색 칸의 색 분리.

### 4. 다른 대상과의 구별점

- 연결 리스트: 연결 리스트는 셀들이 옆자리에 붙어 있지 않다. 화면에 떨어져 놓인 노드들이 화살표로 이어지고, 인덱스 4 를 호명해도 화살표는 0 → 1 → 2 → 3 → 4 의 네 단계를 거쳐 천천히 진행한다. 배열은 같은 호명에서 단 한 번의 호 곡선으로 곧장 도달한다. 또한 연결 리스트는 가운데 삽입에서 옆 칸이 밀리지 않고 화살표 두 개만 다시 묶이지만, 배열은 가운데 삽입이 곧 도미노 파동이다.
- 동적 배열의 resize 사건과 정적 배열: 같은 띠 위에 채색 칸과 회색 칸이 동시에 있다는 점은 같지만, 정적 배열은 회색 영역이 결코 늘어나지 않는다. 동적 배열은 회색 칸이 다 차는 순간 두 배 길이의 새 띠가 아래에 등장하고 모든 칸이 한 묶음으로 새 띠로 이사한 뒤 헌 띠가 사라진다. 평소의 빠른 push 와 가끔의 비싼 이사가 같은 화면에서 길이 차이로 대비된다.
- 해시 테이블: 해시 테이블도 칸과 인덱스가 있지만 호명 전에 키를 해시 함수로 변환하는 한 단계가 끼어든다. 배열은 인덱스 그 자체가 자리이며, 띠 아래 산술식이 "시작 + i" 라는 한 줄 덧셈으로 끝난다. 또한 해시 테이블은 충돌과 재해시 같은 사건이 띠 위에 오는 데 비해, 배열의 띠 위에서 일어나는 사건은 점프·쓰기·시프트·이사뿐이다.

### 5. 달성해야 할 시각 요소

1. **연속된 가로 셀 띠와 인덱스 라벨 — 옆자리끼리 붙어 있고 번호로 호명된다**
   - 설명: 배열의 정체성을 형태 그 자체로 못박는 1차 시각. 같은 너비의 칸이 빈틈 없이 늘어서고, 모든 칸 아래에 0 부터 시작하는 자리 번호가 항상 함께 보인다.
   - 시각 표현: 칸 한 개의 가로 56px, 세로 56px. 칸 사이는 1px 공유 경계선으로 인접 (간격 0). 띠 길이는 capacity 칸 수에 따라 자동 조정. 칸 아래 자리 번호는 12px 모노스페이스, 채색 칸 아래는 진한 회색, 회색 칸 아래는 옅은 회색으로 색이 한 단계 빠진다.

2. **size 와 capacity 의 색 분리, 그리고 이사 사건**
   - 설명: 점유된 앞쪽 N 칸과 그 뒤의 예비 칸이 한 띠 위에 동시에 그려져, 학습자가 "지금 채워진 자리" 와 "미래에 자랄 자리" 를 한눈에 분리해 본다. capacity 가 다 차는 push 가 일어나면 더 큰 새 띠로의 이사가 단계 운동으로 풀린다.
   - 시각 표현: 채색 칸은 facet 토큰 팔레트 중 base 채움색, 예비 칸은 30% 명도의 회색 격자 무늬. 띠 좌측에 "size N / capacity M" 의 짧은 라벨이 항상 붙는다. resize 시: 첫 단계에서 기존 띠 아래 한 칸 거리에 두 배 길이의 빈 회색 띠가 fade-in 되고, 두 번째 단계에서 기존 칸들이 한 묶음 운동으로 새 띠로 이동, 세 번째 단계에서 헌 띠가 fade-out 으로 사라진다. 이사 운동 옆에 "복사된 칸 M" 카운터가 한 번 깜빡인다.

3. **인덱스 → 셀 직접 점프 + 산술 라벨 — O(1) 의 근거를 띠 위에 노출**
   - 설명: get(i) / write(i, v) 호명이 옆 칸을 건드리지 않는 단 한 걸음이라는 사실과, 그 한 걸음의 근거가 "시작 + i" 라는 산술이라는 사실을 같은 운동에 포개 놓는다.
   - 시각 표현: 호명 즉시 띠 윗변 위에 굵은 화살표가 임의 위치에서 i 번 칸 위로 짧은 호 곡선을 그리며 단일 동작으로 이동한다. 곡선 길이는 출발지와 무관하게 200ms 로 일정. 도착한 칸은 한 박자 강조색 (base 채움색의 보색 계열) 으로 빛난다. 띠 아래에는 그 순간 "시작 + i" 라는 짧은 산술식이 i 번 칸 바로 아래에 0.4초 떠 있다 사라진다. 학습자가 산술식 보기를 끄면 라벨만 사라지고 운동은 그대로다.

4. **삽입·삭제의 시프트 도미노 파동 + 비용 누적 막대 — O(N) 의 운동과 정량**
   - 설명: insert(i) / remove(i) 가 i 이후 모든 칸을 한 자리씩 동시에 밀어내는 도미노 운동임을 보이고, 그 비용 (밀린 칸 수) 을 운동 옆에서 정량 막대로 누적한다.
   - 시각 표현: 삽입 시 i 자리에 새 칸이 위에서 솟아 내려오는 동안, i+1 자리부터 N-1 자리까지의 칸들이 한 프레임 차이로 짝힘처럼 우측으로 한 자리씩 미끄러진다. 칸마다 8ms 의 시차로 파동이 번져 도미노 시퀀스가 보인다. 삭제는 같은 운동의 좌측 거울상. 운동이 끝나는 순간 띠 우측 30px 자리에 가로 누적 막대가 자라고, 막대 옆에 "이번 사건 +N, 누적 N합" 의 짧은 숫자 라벨이 붙는다. 막대는 사건마다 한 칸씩 누적되며, 초기화 전까지 사라지지 않는다.

5. **임의 접근 vs 순차 검색의 운동 대비 — 한 걸음과 N 걸음**
   - 설명: 같은 띠 위에서 get(i) 의 직접 점프와 search(v) 의 한 칸씩 스캔이 같은 배율로 잇달아 일어나, 한쪽은 한 걸음 다른 쪽은 N 걸음이라는 사실이 운동의 길이 차이로 즉시 체감된다.
   - 시각 표현: 검색 모드를 누르면 화살표가 띠 위 0 번 칸 위로 가서 한 칸씩 우측으로 똑딱 진행한다. 한 칸당 120ms 로 일정. 일치 칸을 만나면 그 자리에서 강조색으로 빛나고 멈춘다. 직후 학습자가 인덱스 호명을 누르면 같은 띠 위에서 200ms 단일 호 곡선으로 점프가 한 번 일어난다. 두 사건의 운동 시간 (예: 600ms vs 200ms) 이 띠 위 짧은 시간 막대로 좌우 비교 표시된다.

### 6. 시각 구성

- **중심 시각**: 화면 가운데를 가로지르는 단 하나의 셀 띠. 채색 칸 + 회색 예비 칸 + 칸 아래 인덱스 라벨 + 띠 위에 떠다니는 단일 화살표 + 띠 좌측의 size/capacity 라벨로 구성된다. 띠는 좌우로 자라며, resize 시 한 단계 동안만 두 번째 띠가 임시 등장한다.
- **보조 요소**:
  - 띠 우측 누적 막대: 시프트 비용을 사건마다 한 칸씩 쌓는 가로 막대. 폭 60px 의 작은 영역.
  - 띠 아래 산술 라벨 자리: get(i) 호명 시에만 i 번 칸 아래에 잠시 떠오르는 "시작 + i" 텍스트.
  - 상단 캡션 영역: 짧은 개념 문장 한 줄과 현재 사건 메시지.
  - 하단 컨트롤 영역: 인덱스 입력 박스, 값 입력 박스, 호명·쓰기·삽입·삭제·뒤에 추가·검색·초기화 버튼.
- **요소 간 관계**: 모든 사건은 단 하나의 띠 위에서 일어난다. 화살표는 어떤 사건에서도 띠를 벗어나지 않으며, 칸은 좌우로만 미끄러진다. 누적 막대는 띠와 같은 가로축을 공유해 띠의 우측 끝에 잇닿아 자라기 때문에, 띠의 길이 변화 (resize) 와 막대의 길이 변화 (시프트 누적) 가 한 가로선 위에서 함께 보인다. 산술 라벨과 인덱스 라벨은 같은 i 번 칸 아래에 위·아래로 분리되어 충돌하지 않는다.

### 7. 레이아웃

기준 폭 680px, 확장 폭 880px. 높이 약 280px.

```
┌──────────────────────────────────────────────────────────────┐
│  상단 캡션 — 개념 문장 + 현재 사건 메시지                (40)│
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   size 5 / cap 8                                  ↓ 화살표   │
│   ┌──┬──┬──┬──┬──┬──┬──┬──┐                                  │
│   │7 │3 │9 │1 │4 │░░│░░│░░│  ▍▍▍ 누적 막대                 │ 140
│   └──┴──┴──┴──┴──┴──┴──┴──┘                                  │
│    0  1  2  3  4  5  6  7                                    │
│              시작+3                                          │
│                                                              │
│   resize 사건 진행 시: 위 띠 아래에 길이 16 새 띠 fade-in    │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│  [i ▣] [v ▣] [호명] [쓰기] [삽입] [삭제] [추가] [검색] [↺] │
└──────────────────────────────────────────────────────────────┘
```

- 칸 한 개 56×56px. 기준 폭 680px 에서 capacity 최대 10 칸까지 안전 (10×56 + 9 경계선 = 569px, 좌우 여백 포함 680px 안에 들어옴). 그 이상은 칸 너비를 48px 로 한 번 축소.
- 확장 폭 880px 에서는 capacity 최대 14 칸까지 56px 너비로 표시.
- 누적 막대 영역 폭 60px, 띠 우측 끝에서 12px 여백 후 시작.
- resize 임시 새 띠는 기존 띠 아래 24px 거리에 등장. 새 띠가 보이는 동안에만 본체 높이가 한시적으로 200px 로 늘어나고, 이사가 끝나면 다시 140px 로 복귀.
- 상단 캡션 40px, 본체 140px (resize 중 200px), 하단 컨트롤 60px, 칸 하단 인덱스 라벨 영역 20px, 칸 위쪽 화살표 자리 20px 포함 총 280px (resize 중 340px).

### 8. 진행과 인터랙션

입력 반응형. 사용자의 한 번의 입력이 화면의 한 번의 사건과 1:1 로 대응된다.

- **인덱스 + 호명 (read)**: 입력한 인덱스 i 번 칸 위로 화살표가 200ms 호 곡선으로 점프한다. i 번 칸이 한 박자 강조색으로 빛난다. 띠 아래에 "시작 + i" 라벨이 0.4초 뜬다. 띠 길이·칸 수는 변하지 않는다.
- **인덱스 + 값 + 쓰기 (write)**: 같은 점프 운동 후 i 번 칸의 값이 새 값으로 바뀌며 칸 색이 한 박자 더 깊어졌다 돌아온다. 띠 길이·칸 수는 변하지 않는다.
- **인덱스 + 값 + 삽입 (insert)**: i 자리에 새 칸이 위에서 솟아 내려오고 i+1 부터 N-1 까지의 칸이 한 프레임 시차로 우측으로 한 자리씩 미끄러진다. size 가 1 증가, 회색 예비 칸 수가 1 감소. 누적 막대에 (N-i) 만큼 한 칸이 자란다. capacity 가 가득 차 있다면 삽입 전에 resize 사건이 먼저 끼어든다.
- **인덱스 + 삭제 (remove)**: i 칸이 위로 페이드아웃하고 i+1 부터 N-1 까지의 칸이 한 프레임 시차로 좌측으로 한 자리씩 미끄러진다. size 가 1 감소. 누적 막대에 (N-i-1) 만큼 자란다.
- **값 + 뒤에 추가 (push)**: N 자리의 회색 칸이 채색 칸으로 바뀌며 새 값이 들어온다. 시프트 없음, 누적 막대 변화 없음. capacity 가 가득 차 있으면 resize 가 먼저 일어난다.
- **검색 (search)**: 화살표가 0 번 칸부터 한 칸씩 120ms 간격으로 우측으로 진행한다. 일치 칸에서 강조색으로 멈춘다. 끝까지 일치가 없으면 마지막 칸을 지나친 후 빨간 점선으로 한 박자 깜빡이며 "찾지 못함" 캡션이 뜬다.
- **resize 사건 (자동 발화)**: push 또는 insert 시 capacity 가 가득 차면 자동으로 (1) 두 배 길이 새 띠 fade-in 200ms → (2) 모든 칸이 한 묶음 운동으로 새 띠 같은 자리로 이사 400ms → (3) 헌 띠 fade-out 200ms 의 3 단계가 진행된다. 이사 운동 옆에 "복사된 칸 M" 카운터가 한 번 깜빡인다.
- **초기화**: 모든 칸·누적 막대·화살표가 페이드아웃하고 초기 상태로 복귀.
- **잘못된 인덱스 입력**: i 가 음수이거나 size 이상이면 입력 박스 테두리가 빨갛게 깜빡이고 "범위를 벗어난 자리" 캡션이 뜬다. 띠 위 운동은 일어나지 않는다.
- 운동 길이: 점프 200ms, 쓰기 200ms, 삽입·삭제 400ms (도미노 시차 포함), 검색 한 칸 120ms, resize 단계당 200~400ms.

### 9. 초기 상태와 파라미터

- **초기 상태**: capacity 8 의 띠 위에 size 5 의 채색 칸이 채워져 있다. 초기 원소는 [7, 3, 9, 1, 4]. 6, 7, 8 번 자리 (인덱스 5, 6, 7) 는 회색 예비 칸. 화살표는 띠 위에서 어느 칸도 가리키지 않는 상태로 옅게 떠 있다. 누적 막대는 0 칸. 인덱스 입력 박스·값 입력 박스 모두 비어 있다.
- **권장 시연 시퀀스**: 처음 진입 시 (1) 인덱스 3 호명 자동 발화 (점프 + 산술 라벨) → 1 초 멈춤 → (2) 인덱스 1 에 값 5 삽입 자동 발화 (도미노 + 누적 막대) → 1 초 멈춤 → (3) 학습자에게 컨트롤이 넘어간다. 이 두 사건만으로 시그니처 행동이 한 번씩 보인다.
- **파라미터**:
  - 초기 capacity: 8.
  - 초기 size: 5.
  - 초기 원소: [7, 3, 9, 1, 4].
  - resize 배율: 2 배 (capacity 가 가득 차면 16 으로 더블링).
  - 칸 색 팔레트: facet 토큰 팔레트 중 base 채움색 1종 (모든 채색 칸 동일색, 정체성은 칸 안의 숫자가 담당).
  - 강조색: base 채움색의 보색 계열 1종 (호명 도착 칸·검색 일치 칸 공용).
  - 회색 예비 칸 명도: 30%.
  - 누적 막대 한 칸 가로 4px, 한 칸 = 시프트 1회.
  - 운동 지속 시간: 점프/쓰기 200ms, 삽입/삭제 400ms, 검색 한 칸 120ms, resize 800ms (3 단계 합).

### 10. 시각화 안 텍스트

**대상 개념 설명 (상단 캡션 기본 문구)**

> 배열은 같은 종류의 값을 옆자리끼리 빈틈 없이 붙여 놓고 0 부터 매긴 번호로 호명할 수 있게 만든 자료구조다. 번호만 알면 옆 칸을 거치지 않고 한 번에 도달하지만, 가운데에 한 칸을 끼우거나 빼면 그 뒤의 모든 칸이 한 자리씩 밀려난다. 칸이 다 차면 두 배 큰 새 띠로 모든 항목을 한꺼번에 옮긴다.

**학습자 안내 (보조 캡션 회전 문구)**

- "옆자리끼리 붙어 있다 — 그래서 번호만 알면 한 번에 간다."
- "가운데를 건드리면 옆 사람이 다 밀려난다 — 띠 옆 막대가 그 비용을 쌓는다."
- "회색 칸은 미래의 자라날 자리 — 다 차는 순간 두 배 큰 새 띠로 이사한다."
- "검색은 한 칸씩 걷는다 — 호명은 한 걸음에 점프한다. 띠 위 운동의 길이가 곧 비용이다."

**사건별 짧은 메시지**

- 호명: "번호 i 칸으로 곧장 점프 — 시작 + i 한 번이면 도달한다."
- 쓰기: "i 번 칸의 값을 바꾸었다 — 옆 칸은 그대로다."
- 삽입: "i 자리에 끼웠다 — 뒤의 칸이 한 자리씩 밀렸다."
- 삭제: "i 자리를 비웠다 — 뒤의 칸이 한 자리씩 당겨졌다."
- 뒤에 추가: "끝 자리에 얹었다 — 옆 칸이 밀리지 않았다."
- 검색: "처음부터 한 칸씩 살핀다 — 일치 칸에서 멈춘다."
- resize: "칸이 다 찼다 — 두 배 큰 새 띠로 모든 항목을 옮긴다."
- 범위 밖 인덱스: "범위를 벗어난 자리 — 띠 위에서 일어나는 일은 없다."

# 02. DSL

FACET DSL은 마크다운 안에 임베드되도록 설계되었다. 문법은 극도로 얇으며, 두 식별자의 인접만으로 구성된다.

## 문법

### 인라인 표기

마크다운의 인라인 코드(`` ` ``)와 닮은 형태로 한 줄에 임베드된다.

```
{facet:loop facet:bubbleSort}
```

문법 규칙 (BNF 비슷한 표기):

```
facet-expr   = "{" identifier (WS identifier)* "}"
identifier   = "facet:" name
name         = [a-zA-Z][a-zA-Z0-9_-]*
WS           = " "+
```

식별자는 `facet:` 네임스페이스 접두사로 시작한다. 이 접두사가 일반 텍스트와 식별자를 토큰 레벨에서 구분한다.

표현식 안의 모든 토큰은 식별자다. 첫 식별자가 컨테이너, 그 뒤가 본체(들). v1에서는 (컨테이너, 본체) 쌍 한 형태만 지원.

### 블록 표기 (Oon의 코드펜스 패턴 재사용)

긴 표현이나 합성을 위한 블록 형태:

````markdown
```facet
loop
  bubbleSort
```
````

들여쓰기로 중첩 표현. v1에서는 사용 안 함 — 인라인만 지원. 블록 문법은 합성 도입 시 정식화.

## 마크다운 통합

### 인라인 표기의 위치

마크다운 본문 어디에나. 단락 안, 리스트 항목 안, 표 셀 안 등.

```markdown
정렬 알고리즘은 입력 배열을 정렬된 순서로 재배치합니다.
{facet:loop facet:bubbleSort}
이처럼 시각화로 직접 확인할 수 있습니다.
```

호스트 어댑터가 마크다운 파싱 시 `{facet:...}` 패턴을 발견하고, 그 자리에 인터랙티브 영역을 삽입한다.

### 코드펜스와의 충돌 방지

블록 표기에 ` ```facet ` 펜스를 쓰므로, `facet`이라는 언어 식별자가 다른 도구에서 충돌하지 않는지 호스트가 확인. 일반적으로 안전 (Markdown 표준에 등록된 언어 ID 아님).

## 파서

파서는 호스트 독립적 코어 모듈로 제공된다.

### 입력

마크다운 문자열 또는 마크다운의 일부 텍스트 노드.

### 출력

```typescript
type FacetExpr = {
  container: { ns: 'facet', name: string };
  bodies: { ns: 'facet', name: string }[];
  raw: string;          // 원본 표현식 (디버깅·재출력용)
  position: { start: number; end: number };
};
```

v1에서는 `bodies` 배열 길이가 0 또는 1이다. 0이면 컨테이너 단독 모드, 1이면 (컨테이너, 본체) 쌍.

### 파서 동작

1. 마크다운 텍스트에서 `\{facet:[^\}]+\}` 패턴 스캔
2. 각 매치에 대해 토큰화: 공백으로 분리, 각 토큰이 `facet:name` 형태인지 검증
3. 첫 토큰을 컨테이너로, 나머지를 본체들로 분류
4. 카탈로그에서 식별자 존재 확인 (없으면 파싱 에러로 표시 — UI에 빨간 밑줄 등)
5. 위 형태의 `FacetExpr` 객체 반환

### 카탈로그 의존성

파서는 카탈로그의 식별자 존재만 확인하지, 카탈로그의 동작은 호출하지 않는다. 파서는 카탈로그 카테고리를 모른다 — `loop`가 컨테이너인지 본체인지 모른다. 단순히 식별자 위치(첫 번째 / 그 외)로만 분류한다.

런타임이 파서 결과를 받아 카탈로그를 조회하며 카테고리를 검증 (첫 식별자가 컨테이너인지, 두 번째가 본체인지).

## 에러 처리

파서 에러:
- `{facet:` 열림 후 `}` 없음
- 식별자가 `facet:` 접두사 없음
- 빈 표현식 `{facet:}`

런타임 에러 (파서는 통과했지만 카탈로그 검증 실패):
- 첫 식별자가 컨테이너 카탈로그에 없음
- 두 번째 식별자가 본체 카탈로그에 없음
- 본체의 알고리즘이 알고리즘 카탈로그에 없음

호스트 어댑터는 에러를 시각적으로 표시한다 (인라인 빨간 밑줄, 호버 시 메시지 등). 파서/런타임 자체는 throw하지 않고 에러 객체를 반환.

## 예시

```markdown
# 정렬 입문

가장 단순한 정렬 알고리즘부터 시작합니다.
{facet:loop facet:bubbleSort}

위 시뮬레이션의 막대를 다양한 분포로 시도해보세요. "역순"으로 두면
교환이 가장 많이 일어나고, "거의 정렬"로 두면 한두 번의 패스로 끝납니다.

같은 동작을 두 패러다임의 코드로 동시에 볼 수 있습니다.
이게 같은 개념이 어떻게 다르게 표현되는지 보여줍니다.
```

이 마크다운이 Tiptap 편집기에 입력되면, 두 번째 줄의 `{facet:...}` 위치에 v7 데모와 같은 인터랙티브 영역이 렌더링된다. 나머지는 일반 마크다운 텍스트로 표시.
